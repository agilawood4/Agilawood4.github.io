概率与期望

通过阅读本文，您可以大致了解概率与期望在算法竞赛中的有关考点，以及几道简单的例题

期望DP

概率DP：用动态规划的思想去解决和概率，期望等有关的题目

常用技巧：

  1. 求概率，一般使用正推，即通过起始状态到终止状态进行枚举
  2. 求期望，一般使用倒推，即通过终止状态到起始状态进行枚举

蓝桥杯 - 苏苏的01字符串

【期望DP】

思路

本例不同于常规的期望 DP，属于讨论一个线性序列上，因为每个位置填写不同数值，得到的总贡献不同，进而求解总贡献期望的问题，对于这类问题，我们通常需要设计线性的算法，根据当前位置填写的字符进行状态转移

我们定义 f [ i ] 表示长度为 i 的前缀，得到的总贡献的期望值，此外因为贡献和连续的 1 的长度有关，我们还需要额外维护一个 cnt 变量，记录以 i 作为结尾的，当前连续的 1 串的长度，那么根据 i 位置为 0 1 ？三种符号，对应的讨论如下：

  • 0：最简单的情况，不带来新的贡献，f [ i ] = f [ i-1 ]
  • 1：比较简单的情况，获得新的贡献，原本以 i-1 结尾的连续 1 的长度为 cnt，现在以 i 结尾的连续 1 的长度变成 cnt+1，因此 f [ i ] = f [ i-1 ] + (cnt+1)^2^ - cnt^2^，之后再更新 cnt 的值
  • ?：比较复杂的情况，此时我们如果写 0，就不带来新的贡献，如果写 1，则需要更新 f 和 cnt 的数值，所以我们有 1/2 的概率对 f 和 cnt 进行更新，对应则有 f [ i ] = f [ i-1 ] + 1/2×{(cnt+1)^2^ - cnt^2^}，同时 cnt = 1/2×(cnt+1)，而这里为了保证结果是整数，我们的除2操作需要写成乘 inv2，因此涉及到期末逆元，用快速幂求解即可

因为这里对 ? 的处理，我们本身就是同时讨论了 1 和 0 的情况，因此最后得到的 f [ n ] 本身就是期望值，直接输出即可

蓝桥杯 - 鳄鱼水泡大作战

思路

本例和苏苏的01字符串那道题的思路几乎完全相同，我们按照线性DP的方式进行求解，求解的同时额外维护一个当前的命中率即可，对于这类题目，当前的得分不仅和当前命中率有关，还和上一个物品是否命中有关，也就是我们需要知道上一轮的命中率

对于当前第 i 个物品，假设其价值为 b，初始命中率为 a，上一个物品的最终命中率为 a_pre，则我们需要先计算当前物品的最终命中率：

  • 若上一个物品最终未命中（1-a_pre），则当前物品命中率为 a
  • 若上一个物品最终命中（a_pre），则当前物品命中率为 a+0.1

因此当前物品最终的平均命中率为(1-a_pre)(a) + (1_pre)(a+0.1)，值得注意的是，我们需要时刻保证总命中率不得超过 100%，因此最终表达式为min(1, (1-a_pre)(a) + min((1_pre)(a+0.1), 1))

而有了当前的最终命中率，那么当前的得分就在上一轮得分的基础上，类加上 b × a（此时 a 已经被更新过）即可

P1280 - 换教室

【NOIP2016-T3】

思路

首先本例中总节点数很少，只有 300，因此我们可以先用 Floyd 算法求出所有教室两两之间的距离，时间复杂度为 O(V^3^)

接着我们考虑进行动归：设计状态 dp [ i ] [ j ] [ k ] = c 表示前 i 个时间单位中，选择使用了 j 次交换申请，且最后一次使用了申请（k=1）/ 没有使用申请（k=0）所得到的最大收益

这里之所以需要额外增加一个 k 维度进行表示，是因为在本例中，使用申请 ≠ 申请成功，也就是说我们使用了申请，只有 p 的概率是能成功的，还有 (1-p) 的概率是白申请了，无事发生，而我们真正在状态转移时的分类依据是是否申请成功，因此我们就需要额外开一个维度来记录是否请求了申请（换言之，如果申请是有求必应的，我们完全可以当成二维动归，按照最后一个时间点是否发出申请进行分类）

假设我们 i 时间不使用申请，我们需要考虑 i-1 时间是否使用申请，以及 i-1 时间使用申请时，申请是否通过（3 个子状态）；而假设我们 i 时间使用申请，我们需要考虑 i 时间的申请是否通过，i-1 时间是否使用申请，以及如果 i-1 时间使用申请是否能通过（2×3=6 个子状态），如果转移成功就在 d 位置，失败就在 c 位置，根据概率分别进行转移即可

注意 DP 数组的初始化，第一个时间点不管是否发出申请，移动距离都是 0（默认一开始在第一间教室不需要移动）

组合计数

另一类和概率与期望相关的问题即为古典概型，也就是需要计算目标样本数占总体样本数的比例，这一部分常常会和组合计数相结合

在这一章节我们一般会用到期望的线性性质，也就是 E(∑x~i~) = ∑E(x~i~)，假设我们要对各个位置做一些操作以达到某种状态，问操作数的期望，我们可以利用线性性质拆成单点贡献，也就是考虑每一个点要达到他在全局最终状态中的那个状态，所需要的操作次数的期望，再把期望线性累加即可

CF280C

【贡献法】

思路

我们利用期望的线性性质，全局删除次数的期望，就应该等于每个节点被删除次数的的期望的线性相加，因此我们考虑每个节点被删除次数的期望

很显然，每个点要么被删除 1 次，要么被删除 0 次，不可能被删除的次数超过 1 次，我们考虑，如果一上来先把这个节点的父节点或者它到根节点路径上的任意一个祖宗节点 f 先删掉，那么这个点会跟着 f 一起被删掉，删除次数就是 0 次，要想让删除次数是 1 次，就必须保证这个节点在它的祖先节点之前被删除

也就是说，假设从根节点到当前考虑的节点 u 的路径为 f1 f2 f3… fn-1 u，那么我们考虑这些点的删除顺序，只有 u 节点第一个删除的时候，它才会有一次被删除的机会，对于其他的删除顺序，只要先删了某个祖先节点，u 节点就不可能有被删除的机会，假设这条路径上共有 n 个点：

  • u 排在第一个删除的方案数：(n-1)!
  • 总体方案数：n!

因此 u 被删除次数的期望就是1 * (n-1)!/n! + 0 * (1 - (n-1)!/n!)，这里路径上点的数量其实就等价于 u 节点在树上的深度 dep，因此最终单点的期望值贡献为1 * (dep_u-1)!/dep_u! + 0 * (1 - (dep_u-1)!/dep_u!) = 1/dep_u。我们最后只需要对所有深度的倒数求和即可

思路

我们仍然从期望的线性性质出发，考虑每个点所贡献的得分的期望，因为格子上的分只能获取一次，因此每个点的分值贡献要么是 1，要么是 0，假设在这 K 次拿取中，每个点被拿到的概率为 p，因此这个点的分值贡献就是p*1+(1-p)*0 = p，因此我们就是要求这个概率 p

p 的含义是在 K 次拿取中，某个点至少被拿到过一次的概率，这个概率显然不好求，我们正难则反，考虑它的补集 p1，也就是在 K 次拿取中，某个点一次都没有被拿到过的概率，这个 p1 显然可以用乘法原理进行拆分，假设在一次拿取中，某个点没有被拿到的概率为 p2，那么一定有p1 = p2^k，我们就转化为求 p2

这里我们可以再用正难则反转化一次，考虑在一次拿取中，某个点被取到的概率为 p3，我们求解 p3，这就转化成了一个几何概型，也就是在一个二维点阵上选择两个点，构成子矩形，求这个子矩形覆盖目标点的概率：

假设我们选择的点为 A 和 B，那么 A B 的分布情况只有上图所示的三种，即 A 在左上角和 B 在右下角，以及 A 在右上角和 B 在左下角，以及 A 占目标格子，B 占剩下的任意一个格子，我们分别计算这三种情况下 A 和 B 所能覆盖的面积，则总方案数即为 SA1×SB1 + SA2×SB2 + SA3×SB3，而总方案数则为 S×(S-1)/2（这里除以 2 是因为我们不考虑 A 和 B 落点的顺序），因此p3 = 2(SA1×SB1 + SA2×SB2 + SA3×SB3)/(S×(S-1))

最后我们根据 p3 倒推回 p 即可，最后总体期望就是每个格子的 p 值累加之和

【醉汉漫步问题】

在求解本例之前，我们先给出一个引理：在数轴上有顺序的三个点 A B C，其中 B 在中间，A C 在两侧，一个人从 B 出发，由于它喝醉了，每次向左走一个格和向右走一个格的概率都是 1/2，这个人每单位时间会等概率向左或向右走一个格，问这个人走到 B 或 C 点的时间的期望值？

  • 起点：一开始处于起点，所以起点被到达的次数天然+1，也就是 f(1) = 1 + ∑p(v)f(v)
  • 重点：只要到达终点就算结束，也就是终点不会再走向其他节点，也就是说终点不会走任何边，那么我们应该强制设置它的期望值为 0（这样和终点连接的那些边才不会计入终点的贡献），且不会对其它节点的期望值产生贡献

因此我们实际上得到了 n-1 个独立的线性方程，因为不需要考虑终点，我们对这 n-1 个线性方程进行高斯消元，即可得到除终点外的每个节点被经过的期望，再推出每条边被经过的期望
