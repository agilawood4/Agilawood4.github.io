概率与期望

通过阅读本文，您可以大致了解概率与期望在算法竞赛中的有关考点，以及几道简单的例题

概率DP

概率DP：用动态规划的思想去解决和概率，期望等有关的题目

常用技巧：

  1. 求概率，一般使用正推，即通过起始状态到终止状态进行枚举
  2. 求期望，一般使用倒推，即通过终止状态到起始状态进行枚举

思路

对于期望 DP 而言，一般都是把最终的状态作为起点，把最初的状态作为终点，从末态到始态倒推（也就是倒序循环遍历），对于本例而言，最终的状态就是有 s 个子系统和 n 种 bug，因此起点可以设置为 f [ s ] [ n ]，起点的期望值为 0（如果一上来就满足 s 和 n，则再有 0 天即可满足条件），初始的状态就是 0 个子系统和 0 种 bug，因此待求解的终点就是 f [ 0 ] [ 0 ]

接下来我们考虑推导关系，假设我们当前的状态为 f [ i ] [ j ]，表示有 i 个子系统和 j 种 bug，我们考虑按照最后发现的 bug 所属的类别进行分类，大致可以分成以下四类：

  • 该 bug 既属于新系统也属于新 bug，也就是在已有的 i 个系统和 j 种 bug 中又找到了一个 bug，对应的概率为 (i / s)×(j / n)，bug 数和系统数都不变
  • 该 bug 不属于新系统但属于新 bug，也就是在已有的 i 个系统和剩下的 n - j 种 bug 中又找到了一个 bug，对应的概率为 (i / s)×(1 - j / n)，对应 bug 数+1，变为 f [ i ] [ j+1 ]
  • 该 bug 属于新系统但不属于新 bug，也就是在剩下的 s - i 个系统和已有的 j 种 bug 中又找到了一个 bug，对应的概率为 (1 - i / s)×(j / n)，对应系统数+1，变为 f [ i+1 ] [ j ]
  • 该 bug 不属于新系统也不属于新 bug，也就是在剩下的 s - i 个系统和 n - j 种 bug 中又找到了一个 bug，对应的概率为 (1 - i / s)×(1 - j / n)，对应系统数和 bug 数都+1，变为 f [ i+1 ] [ j+1 ]

蓝桥杯 - 苏苏的01字符串

【期望DP】

思路

本例不同于常规的期望 DP，属于讨论一个线性序列上，因为每个位置填写不同数值，得到的总贡献不同，进而求解总贡献期望的问题，对于这类问题，我们通常需要设计线性的算法，根据当前位置填写的字符进行状态转移

我们定义 f [ i ] 表示长度为 i 的前缀，得到的总贡献的期望值，此外因为贡献和连续的 1 的长度有关，我们还需要额外维护一个 cnt 变量，记录以 i 作为结尾的，当前连续的 1 串的长度，那么根据 i 位置为 0 1 ？三种符号，对应的讨论如下：

  • 0：最简单的情况，不带来新的贡献，f [ i ] = f [ i-1 ]
  • 1：比较简单的情况，获得新的贡献，原本以 i-1 结尾的连续 1 的长度为 cnt，现在以 i 结尾的连续 1 的长度变成 cnt+1，因此 f [ i ] = f [ i-1 ] + (cnt+1)^2^ - cnt^2^，之后再更新 cnt 的值
  • ?：比较复杂的情况，此时我们如果写 0，就不带来新的贡献，如果写 1，则需要更新 f 和 cnt 的数值，所以我们有 1/2 的概率对 f 和 cnt 进行更新，对应则有 f [ i ] = f [ i-1 ] + 1/2×{(cnt+1)^2^ - cnt^2^}，同时 cnt = 1/2×(cnt+1)，而这里为了保证结果是整数，我们的除2操作需要写成乘 inv2，因此涉及到期末逆元，用快速幂求解即可

因为这里对 ? 的处理，我们本身就是同时讨论了 1 和 0 的情况，因此最后得到的 f [ n ] 本身就是期望值，直接输出即可

蓝桥杯 - 鳄鱼水泡大作战

思路

本例和苏苏的01字符串那道题的思路几乎完全相同，我们按照线性DP的方式进行求解，求解的同时额外维护一个当前的命中率即可，对于这类题目，当前的得分不仅和当前命中率有关，还和上一个物品是否命中有关，也就是我们需要知道上一轮的命中率

对于当前第 i 个物品，假设其价值为 b，初始命中率为 a，上一个物品的最终命中率为 a_pre，则我们需要先计算当前物品的最终命中率：

  • 若上一个物品最终未命中（1-a_pre），则当前物品命中率为 a
  • 若上一个物品最终命中（a_pre），则当前物品命中率为 a+0.1

因此当前物品最终的平均命中率为(1-a_pre)(a) + (1_pre)(a+0.1)，值得注意的是，我们需要时刻保证总命中率不得超过 100%，因此最终表达式为min(1, (1-a_pre)(a) + min((1_pre)(a+0.1), 1))

而有了当前的最终命中率，那么当前的得分就在上一轮得分的基础上，类加上 b × a（此时 a 已经被更新过）即可
