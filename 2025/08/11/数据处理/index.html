数据处理

通过阅读本文，您可以大致了解 Pytorch 中和数据处理有关的内容，包括数据加载，图像处理等知识

数据集常见组织形式

在目前的学习阶段，我们的模型所进行的一般都是监督学习（数据有标签），而有标签的数据集一般有以下两种组织方式：

  1. 不同类别的数据存放在不同文件夹内，文件夹名称就是对应数据的类别
  2. 所有数据都存放在同一个文件夹（data）内，但是额外准备标签文件夹（label），该文件夹中为每个数据存一个 txt 文件，记录这个数据的类别，其中每一个 data 文件都会有一个与它名称相同的 txt 标记文件

数据的加载

在 Pytorch 中，我们可以通过继承 Dataset 库，实现我们自己的数据集加载类，在继承的过程中，我们需要根据自己加载的数据集特点，重写原 Dataset 库的以下三个方法：

  1. __init__()：自定义数据加载类的初始化方法
  2. __getitem__()：自定义从数据加载类中获取目标数据的方法
  3. __len__()：自定义获取目标数据集大小的方法

我们自定义的数据加载类需要完成的两个基本功能即为：帮助我们获取目标数据集中的元素（通常以列表形式返回），获取目标数据集的大小（待加载的有多少个数据），我们定义的myData类结构一般如下：

在加载自定义数据集时，我们需要把待加载的数据集存在我们电脑本地，因此我们需要给出待加载数据集在电脑中存放的地址路径root_dir，这里可以是相对路径也可以是绝对路径，其中相对路径以 Linux 路径习惯组织，绝对路径以 Windows 路径习惯组织：

其中 Windows 路径中的/在 Python 中是转义字符，需要手动修改成//，而 Linux 路径则不需要修改

此外，我们还需要传入对应数据集的标签label_dir，用于表示在模型训练阶段，我们那当前加载的是什么类别的数据集，因此我们的__init__()方法可以如下设计：

这里我们的数据组织方式是：数据集路径 + 标签 + 具体数据的形式，也就是（1）类型的数据组织形式，其中 root_dir 用于指向数据集地址路径，label_dir 用于表明在当前数据集下到底取哪一个标签分类，调用os.path.join()方法把这两段路径进行拼接，即可得到对应标签类别的数据集的完整地址

这里之所以使用 join() 方法而不是简单的字符串拼接，是因为 join() 方法能够按照路径的方式进行拼接，自动处理兼容 Windows 和 Linux 路径，完成 / 和 \ 的正确处理等功能，得到合法的路径

这样self.path中就存放了我们当前要加载的数据集的正确路径，我们最后调用os.listdir()方法，该方法用于遍历并且返回其入参目录下所有子文件的地址，以列表形式返回，并且存入到self.img_path中，换言之，也就是待加载数据集目录下的所有数据的相对地址，会以列表的形式存入self.img_path这个变量中，供我们后续调用

注意这里存的是相对地址，比如我们当前数据集的路径为：A // B //，每一个数据相对于数据库的路径为 C D … ，我们存的就是单独的 C D …，不会存上级目录

__getitem__()

该方法用于实现从我们已经加载好的数据集中获取某个数据，也就是从上文我们维护的self.img_path中获取数据，因为此时数据以列表的形式进行组织，我们可以通过下标索引的方式，从列表中取出某个位置的数据的地址（注意是相对地址），存入img_name中

接下来我们考虑拼接出这个数据的绝对地址，这个数据完整的地址来源于三部分：数据集本身的地址（self.root_dir），数据集内某个类别的相对地址（self.label_dir），某个具体数据的相对地址（img_name），我们使用os.path.join()把这三者按顺序拼接，即可得到一张图片的完整路径

最后我们通过 PIL 库中的 Image 库提供的Image.open()方法，通过传入 img 的路径，得到对应的 img 对象（这个 img 对象可以通过调用img.show()方法进行显示），返回这个 img 对象和它对应的标签即可

__len__()

该函数用于获取数据集的大小，当前数据集所有数据的相对地址都已经以列表形式加载到self.img_path中，我们只需要获取列表长度，即可得到数据集的大小

我们通过调用add_image()方法向某个 TensorBoard 中加入一张或多张图片，该函数本身支持的图片元素入参需要是 numpy 数组或者 tensor 张量的形式，如果我们的图片本身不是这两种形式，需要进行预先的转换

transforms

torchvision.transforms 是 PyTorch 中用于图像预处理和数据增强的核心模块，该模块提供了一系列可组合的图像变换操作，能够将原始图像转换为适合神经网络训练的格式，使用时需要从 torchvision 中引入 transforms 模块

ToTensor()

该方法用于把 Image 类型或者 Numpy 类型的图像统一转化为 Tensor 类型的图像，在使用时需要先实例化对应的转换器再进行使用

有时候为了方便我们也可以把上述两句代码压缩成一句transforms.ToTensor()(img)，在转化为 tensor 类型后，我们就可以把对应的图片传入 TensorBoard 中进行展示（对应上面我们使用的np.array()方法）

ToPILImage()

该方法用于把 tensor 类型的图像反向转化回 PIL.Imgae 类型的图像，功能与用法和 ToTensor 相类似

Normalize()

该方法用于对图像张量进行标准化（归一化）处理，也是深度学习中预处理的常见操作，也需要先初始化出对应的转换器再进行操作

该方法支持两组参数，每组参数为第一个列表，分别表示归一化以后各个通道的均值（mean）和标准差（std），每个列表的三个值分别表示图像的高度，宽度和通道值

这里均值和标准差均设为 0.5，是一种常见的对称标准化方式，会将输入张量的范围从 [0, 1] 映射到 [-1, 1]，输出的图像仍然是 Tensor 类型，且形状仍然是（C，H，W）和原本形状一致

resize()

该方法用于对图像进行尺寸调整，使用过程中也要先创造一个图像尺寸调整器，该方法可以接受一个或两个参数：

  1. 如果只有一个参数：把图像长边和短边中的短边缩放到该值，长边按比例调整
  2. 如果有两个参数 [a, b]：则把 height 调整为 a，width 调整为 b

向转换器对象中传入的图片要求是 PIL 图像或者 Numpy 数组，返回值类型保持不变，仍然是 PIL 图像或者 Numpy 数组

Compose()

该方法用于生成一个由多个转换器对象组成的集合，可以把多个变换操作（tensor_trans，norm_trans 等）组合成一个顺序执行的管道，传入一个由转换器组成的列表，返回的也是一个转换器对象

接下来我们用 trans_compose 转换器对图像进行处理时，会先对其进行缩放，再转化为张量类型，接收一张 PIL.Image 或 Numpy 类型的图片，返回 Tensor 类型的图片

官方数据集使用

除了自己手动下载数据集并定义Dataset之外，我们也可以直接下载官方现成的数据集，他们都集合再 Pytorch 库torchvision中

数据集加载

我们可以直接从 torchvision 的 datasets 模块中下载我们想要的数据集，在加载过程中一般需要设置下面几个参数：

  1. root：表示下载的数据集在本地存放的路径
  2. train：设置为 True 时表示加载训练集，False 时表示加载测试集
  3. transform：对图片进行预处理的格式，支持一个列表，具体用法与 Compose 相同，传入一个转换器对象构成的列表即可
  4. download：设置为 True 时，如果本地 root 路径下没有数据集，则会自动从官网进行下载

我们以CIFAR10数据集的加载作为示例，该数据集包含 10 个类别（猫，狗，飞机等）的 60000 张彩色图像，每类 6,000 张：

dataloader

上面我们通过自定义或下载的方式，得到了目标的 dataset（数据集），但是此时只支持我们通过下标索引的方式进行访问，而 dataloader 则支持更加灵活的访问方式

Dataloader 实例化

在数据加载过程中，dataloader 一般接受以下几个常见的参数：

  1. dataset：待加载的数据集，只需要传入我们自定义或下载的数据集的实例化对象即可
  2. batch_size：单次加载的数据条数（也就是一次取几个数据一起加载）
  3. shuffle：设置为 True 的时候表示对加载的数据进行乱序，使得每次加载得到的数据顺序都不同，设置为 False 时每次加载的数据顺序是相同的
  4. num_workers：默认为 0，表示只用一个主进程进行加载
  5. drop_last：如果数据总量和 batch_size 不整除的时候，如果设置为 True 则舍弃最后余下的数据，设置为 False 则不舍弃（比如 100 个数据，batch_size=33，那么最后会余下 1 个数据）

值得注意的是，在 Windows 系统下，如果 dataloader 报错，可以尝试强制设置num_workers=0进行修复

这里我们选择一组加载 64 个数据，组间进行打乱，不选择多线程，同时不保留余量数据进行加载，加载结果会返回到一个 DataLoader 的实例化对象中，这里我们命名为 test_loader
