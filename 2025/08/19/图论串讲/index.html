图论串讲

通过阅读本文，您可以大致了解图论算法中的有关知识和简单例题

算法回顾

最短路算法

DIjksral 算法

  • 朴素做法 O(n^2^)，堆优化做法 O(nlogn)，其中 n 是图上节点数，m 为图上边数
  • 对于稀疏图（m << n^2^），使用该算法效果一般很好
  • 如果是稠密图（边很多），有时候堆优化的做法甚至不如朴素做法
  • 无法处理有负边权的图

SPFA 算法（Bellman-Ford 优化算法）

  • 最坏复杂度 O(mn)，其中 n 为图上节点数，m 为图上边数，但在随机图上一般跑不满这个复杂度，可能会跑到 O(km) 和边数线性相关
  • 可以处理负边权，可以用于找负环

Floyd 算法

  • 可以求多源最短路（任意两点之间最短路）
  • 复杂度为 O(n^3^)，n 是图上节点数，时间复杂度较高

最小生成树算法

Kruskal 算法

  • 复杂度为 O(mlogm)，m 为图上边数，对于稀疏图效果较好
  • 需要借助并查集实现

Prim 算法

  • 复杂度为 O(n^2^)，n 为图上节点数，如果是稠密图，可能 Prim 效果更优
  • 算法流程和 DIjksra 算法相类似

P9751

思路

根据到达某个景区时对应的时间 % k 的余数进行分层，因为 k 的值很小，在分层图上跑 DIjksra 算法不会超过 100 层，时间复杂度为 O(nklogn)

此外，本例还有一个额外的限制条件，假设 u 到 v 的路径上有 w 的时间限制，那么到达 u 这座景点的时间不得早于 w，否则就需要在 u 景点停留，会违反题意，因此假设我们当前在 u 节点，将要去往 v 节点的时候，d = dist [ u ] < w，我们考虑在起点位置晚来一段时间（乘坐时间更靠后的车），以使得时间向后平移，从而使得平移后的时间晚于 w

又因为车到达起点的时间都是 k 的整数倍，问题就变成了 d 向后延迟多少个 k 能够 >= w，也就是 d + kx >= w 求解 x 的问题，x = (w-d)~向上取整~ / k，因此我们用 u 更新 v 的新时间应该为 t = d + x + 1，注意这里不能直接修改在 d 本身，因为只是当前这个 w 不满足约束，对于其他可能满足约束的 w，我们直接使用 d 本身即可，或者他们可能对应不同的 x

差分约束模型

差分约束用于解不等式组（n 个变量，m 个不等式），其中每个不等式都形如【xi + wi >= xj】或【xi + wi <= xj】的形式，其中到底是跑最短路还是最长路是我们自己决定的

最短路模型：所有不等式都化成【xi <= xj + wj】的形式，对应从 xj 到 xi 连接边权为 wj 的边，求最短路，负环表示无解

最长路模型：所有不等式都化成【xi >= xj + wj】的形式，对应从 xj 到 xi 连接边权为 wj 的边，求最长路，正环表示无解

解的构造：有时候所有不等式都是相对关系而没有绝对关系，且有的变量可能不参与不等关系的构成，为了解决这两个问题，我们通常设置一个超级源点 S，让它向所有节点连边

  • S = INF，对应不等式【xi <= S】，从 S 向 xi 连接边权为 0 的边，对应最短路模型
  • S = -INF，对应不等式【xi >= S】，从 S 向 xi 连接边权为 0 的边，对应最长路模型

合法解：若存在合法解，则从超级源点跑一遍最短路，每个节点对应的 dist 值即为一组合法解

P2294

把连续几个月的收入看成区间和，显然可以用前缀和作差的形式表示，从而凑出差分约束需要的表达式，比如 l 到 r 月份的收入为 w，则有 sum~r~ - sum~l-1~ = w，这里我们跑最短路模型，因此等式可以拆成两个不等式：

此外，我们还需要建立一个虚拟源点，用来把所有节点联系起来（现有的不等式可能无法表示 0 到 n 的所有节点），这里因为用到前缀和，因此 0 号节点也被用到，我们需要选择 n+1 号节点作为虚拟源点，这里我们强调一个重要的思想，即虚拟源点的值不一定需要确定，也可以取成相对值，因为这里我们选虚拟源点并不是为了引入绝对关系，而仅仅是为了把所有变量联系起来形成完整约束，换言之，这里我们不关心每个变量最终的绝对取值是什么，我们只关心每个变量有没有合法解，因此这里 n+1 节点对应的 dist 取值其实是很自由的

但是需要注意，dist [n+1] 的取值需要保证，n+1 节点的 dist 值能够用来更新其他节点的 dist 值，也就是能够使其余 dist 的值变小，换言之，如果所有 dist 值都设成相同的（比如都是 INF），那么根本 dist [n+1] + w >= dist [u] 必然成立，根本不会跑 SPFA，也就求不出最优解了，因此其实从解题角度讲，我们的 dist [n+1] 其实是越小越好的，最起码要和其他初始化的 dist 形成足够的差距（比如 INF 和 0，或者 2INF 和 INF）

传递闭包

给定一个有 n 个节点的图的邻接矩阵，要求根据邻接矩阵求出其传递闭包

邻接矩阵：

  • g [ i ] [ j ] = 1，i 和 j 节点之间有边
  • g [ i ] [ j ] = 0，i 和 j 节点之间没有边

传递闭包：

  • d [ i ] [ j ] = 1，i 和 j 节点之间可达（包括直接可达与间接可达）
  • d [ i ] [ j ] = 0，i 和 j 节点之间不可达

可以使用 Floyd 算法求解传递闭包，传统 Floyd 算法的复杂度为 O(n^3^)，如果只是求传递闭包则可以进行 bitset 压位优化，把复杂度降低到 O(n^3^ / w)，w 是一个字长（32）

欧拉回路 / 路径

有向图情况

有向图的欧拉回路：给定一张有向图，要求一条路径，从一个点出发，经过每条边恰好一次以后，回到原点

欧拉回路存在的充要条件：整张图是连通图，且每个节点入度 = 出度

有向图的欧拉路径：给定一张有向图，要求一条路径，从一个点出发，经过每条边恰好一次以后，不回到原点

欧拉路径存在的充要条件：整张图是连通图，起点出度 = 入度+1，终点入度 = 出度+1，其他每个节点有出度 = 入度

无向图情况

无向图的欧拉回路：给定一张无向图，要求一条路径，从一个点出发，经过每条边恰好一次以后，回到原点

欧拉回路存在的充要条件：整张图是连通图，每个节点度数为偶数

无向图的欧拉路径：给定一张无向图，要求一条路径，从一个点出发，经过每条边恰好一次以后，不回到原点

欧拉路径存在的充要条件：整张图是连通图，恰好两个点度数为奇数，奇遇点度数均为偶数

我们给出的算法的时间复杂度为严格 O(m+n)，同时适用于有向图和无向图

有向图欧拉路径的求解

【例题：P7771】

思路

我们借助栈结构结合深度优先搜索进行求解，具体求解流程如下：

  • 遍历当前节点的每一条出边，得到这条边指向的终点
  • 删除这条出边防止重复遍历
  • 接着向终点进行搜索
  • 当前节点作为起点所能到达的所有终点都遍历完以后，当前节点压入栈中

最后，我们倒序输出栈中所有节点，得到的就是欧拉路径：

以上是我们求解欧拉路径的主算法流程，这里我们遍历每条出边并不是采用标准的链式前向星遍历法，而是选择边遍历边删边的形式，因为这里是图结构（可能有环路），当前节点 u 搜到目标节点 v1，v2，…以后可能再次回到 u，此时我们需要从上次 u 结束搜索的下一条边开始遍历，也就是我们每遍历完一条边就要删掉他，而不是写成下面的形式：

这种写法当某个节点连接很多出边的时候，每次来到这个点都需要遍历所有的边判断之前是否走过，而来到这个点的平均次数也和这个点连接的边数线性相关，那么这个点遍历的复杂度达到 O(m^2^) 会超时，因此这里给出边遍历边删除边的写法，当我们需要每条边遍历且仅遍历一次的时候，推荐这种写法

最后，在进行主流程求解之前我们还需要进行欧拉路径是否存在的特判和若干预处理：

  • 必须保证整体是弱连通图（本例默认满足）

  • 如果存在某个点的入度和出度相差大于 1，则不存在欧拉路径

  • 如果所有的节点入度都等于出度，则说明是欧拉回路，此时需要额外特判设置起点为 bg=1，如果默认 bg=0 会莫名其妙输出一堆 0；如果存在一个点的入度=出度+1，一个点的出度=入度+1，则存在欧拉路径，设置正确的起点即可

  • 本例还要求按字典序输出，这里我们需要保证链式前向星优先遍历编号小的节点，也就是对于所有的边，我们需要进行排序，排序规则如下：

    • 起点升序排序
    • 起点相同时，按终点降序排序

    这样我们能够保证对于同一个节点的若干出边，编号大的终点对应的边先被 add 到十字链表中，又因为我们采用头插法，会自动把他们挤到后边，因此遍历的时候就能实现先遍历编号小的出边终点
