线性基

通过阅读本章内容，您可以大致了解线性基的概念，以及与之有关的几道简单题目和综合题目

异或空间线性基

我们一般提到的线性基，其实指的都是异或空间线性基，还有一种是向量空间的线性基，这里仅做简单介绍

基本概念

定义

现在有一个由若干个非负数组成的集合 A，我们选择这个集合 A 中的任意几个数字取异或和，所得到的所有可能的非零异或和组成集合 B，我们希望找到一个元素个数尽量少的集合 C，使得集合 B 中的元素能由集合 C 中的元素取异或和得到，这个集合 C 就称为集合 A 的线性基

值得注意的是：集合 C 中的元素没必要都来自集合 A，也可以在集合 A 中不存在，同时我们也不关心得到的每种异或和的个数，只关心不同异或和的总的种类数

几组结论

（1）一堆数字中，对于任意的 a 和 b，如果用 a^b 的结果代替 a 和 b 中的任意一个数字，不会影响非 0 异或和的组成

（2）一堆数字中，对于任意的 a 和 b，如果有 a^b = 0，那么舍弃掉 a 和 b 中的任何一个数字，不会影响非 0 异或和的组成

（3）一堆数字能否异或出 0，在求出异或空间线性基以后，需要被单独标记

求解方法

一堆数字的异或空间线性基，可能不止一组，求解方式有普通消元和高斯消元两种

普通消元：能够得到某一组线性基，进而可以得知线性基大小，异或和个数，异或和中是否有 0，最大异或和等信息

高斯消元：得到标准形式的异或空间线性基，进而可以得知普通消元得到的信息，还可以得到第 k 小的异或和

对于一般题目而言，只需要通过普通消元得到一组线性基即可足够解题，除非需要知道第 k 小的异或和，才会用到高斯消元

复杂度分析：两种方法的时间复杂度都为 O(n × m)，其中 n 为数字个数，m 为最大数字的位数，异或空间线性基的大小为 O(m)，m 为数字的位数

普通消元法

我们线性基的最大个数不会超过原始的一堆数字中最大数字的位数，我们按照任意顺序依次遍历原始的数字，对于当前的数字 num，我们从高到低依次遍历它的每一位：

  1. 如果当前位 i 是 0，直接跳过，遍历下一位
  2. 如果当前位 i 是 1，我们看第 i 个线性基是否被填过数字，若没被填过直接把 num 填入第 i 个线性基，若已经填过数字，则把 num 和第 i 个线性基进行异或，异或得到的新值 num1，继续遍历下一位

如果我们在插入数字的过程中，遍历完数字的所有位，都还没有插入成功，就说明此时这个数字被已有的线性基异或成 0 了（因为只有在某一位为 1 的时候我们才尝试插入，一直没插入成功就等价于这个数字每一位都是 0），此时我们需要设置 zero 标记，说明原始数字可以异或得到 0

附加结论

通过普通消元得到一组线性基以后，我们可以进一步得到如下结论：

  1. 线性基的大小：遍历 0 ~ BIT 范围内的 basis 数组，统计非 0 个数即可
  2. 原始数字所能得到的非 0 异或和的个数：已知线性基大小为 T，则非零异或和个数为2^T
  3. 最大异或和：我们记录初始异或和为 x，从 BIT 到 0 依次倒序遍历 basis 数组的每一位，每次x = max(x, basis[i] ^ x)，即可得到最大异或和
  4. 原数组异或和是否有 0：通过 zero 标记即可知道

第 k 小的异或和

如果我们已经按照高斯消元求出了标准线性基，现在要我们求原数字异或所能得到的第 k 小的异或和，首先假设线性基大小为 T，我们比较 k 和 2^T 的大小，如果 k 超过异或和的总个数，直接返回无解

如果 k 合法，我们先对线性基的下标进行重新规划，假设一共有 T 个线性基，分别存在原数组 a[1] ~ a[T] 这段有效位置中，现在我们令a[T] = basis[0], a[T-1] = basis[1],..., a[1] = basis[T-1]，依次类推，从小到大从 0 到 T-1 按 basis 数组组织

把 k 进行二进制拆分，对应 k 的二进制拆分的哪些数位上有 1，我们就把 basis 数组对应下标上的数字异或起来，得到的结果就是第 k 小的异或和，比如 k = 10 = 1010，我们就把 basis 数组的第 1 个和 第 3 个数字异或起来，就是第 10 小的异或和

P4570

思路

本例每个数据都有 num 和 magic 两个维度，现在要求我们按照 num 求出一组线性基，其中每个数据的 magic 之和最大

我们考虑线性基生成的过程（以普通消元为例），当前遍历到的数字，如果能加入到线性基中，则它一定会被加入，也就是说越靠前的数字，在不未被线性基插入原则的前提下，有越大的机会被插入线性基中

根据此原理，我们可以设计一个简单的贪心，把所有的元素按照 magic 维度从大到小排序，这样我们会优先尝试把那些 magic 值大的元素插入线性基中，就越有可能得到更大的 magic 值累加和

向量空间线性基

向量空间线性基和异或空间线性基的求解原理完全相同，其定义可以对标线性代数中的极大无关组的概念，只不过相比于极大无关组，向量空间线性基中的向量并不一定必须来自原向量的集合

求解方法也分为普通消元和高斯消元，这里我们以普通消元为例进行展示：首先我们遍历每一个向量，对于当前的向量 v，我们从第 0 维开始逐个遍历它的各个维度，对于第 i 维：

  1. v 的第 i 维为 0，直接跳过，接着处理下一维
  2. v 的第 i 维不为 0，我们去看第 i 个线性基是否存在，如果已经存在，我们就用已经存在的第 i 个线性基去消掉当前 v 向量的第 i 维（注意这里可能会产生浮点数，我们只保证第 i 维恰好被消掉即可，对于后面维度的值可以消成浮点数）；如果第 i 个线性基不存在，就直接让 v 作为第 i 个线性基即可，接着处理下一维

这个处理过程和方程组的消元过程高度类似

线性基的综合应用

P4839

【线段树 & 线性基合并】

思路

通过本例，我们可以知道由于线性基的空间复杂度上限进取决于数值位数，因此和线性基有关的很多操作我们都可以暴力完成，对应的时间复杂度可以看做常数时间，常见的操作有插入与合并两种

插入操作：把一个数值插入一组已有的线性基中，时间复杂度 O(m)，m 位=为线性基大小

合并操作：合并两个线性基到一个新的线性基，时间复杂度为 O(m^2^)

如果我们已经会了这两个操作，那么本例就相对简单了，因为本例要求实现的就是（1）向某个线性基中插入数字（2）合并多个线性基对和线性基求最大异或和，因为涉及到多个线性基的合并，我们考虑用线段树进行维护

线段树的每个节点都维护一个线性基，假设节点 ind 维护的区间是 l 到 r，那么 tr [ ind ] 节点上就存 l 到 r 范围内的线性基，我们只需要结合线性基的插入与合并，完成线段树上的单点修改与区间查询操作即可，对于线段树的维护，UP 方法和区间查询都等价于两个线性基的合并

P3292

【树上倍增 & 线性基合并 / 替换】

思路

本例可以暴力树剖+线段树，转化成维护区间上最大异或和，然后用线段树给每个节点都维护一组线性基，通过线性基的暴力合并完成区间查询，在上例中我们已经使用过这种方法，因此本例介绍另一种思路

首先我们要求树上两个节点之间的路径，则要先求出 lca，这里我们选择使用倍增法求 lca，假设我们的路径为 x -> lca -> y，最暴力的做法就是为每个节点都维护一组仅属于他自己的线性基，然后遍历路径，暴力合并线性基，这样做的复杂度就变成了 O(路径长度 × 线性基大小)，显然是不可取的

我们考虑能否用一种前缀和的思想进行优化，假设根节点是 root，x 处我们存从 root 到 x 的路径上，所有节点贡献得到的线性基，lca 处存从 root 到 lca 的路径上，所有节点贡献得到的线性基，我们考虑通过取二者的差集，得到 x 到 lca 路径上的线性基

我们注意到，线性基本身的空间大小是很小的，只取决于最大数字的位数，也就是不会超过 60，遍历线性基的代价可以看成是常数级别，这里我们考虑为每个线性基中的数值都额外记录一个深度值 dep，这个 dep 值的定义和节点的 dep 值一一对应，从根节点向下遍历到各个节点的时候，各个节点的权值在插入线性基的时候，我们同时把这个权值对应的深度也额外记录

这样我们最后只需要遍历 x 中记录的，从 root 到 x 路径上的所有线性基空间，对于得到的每一个数值，我们都看他的 dep 值是否不小于 lca 的 dep 值，如果 dep(val) < dep(lca)，则说明是在 lca 之前插入的数值，从 x 到 lca 的路径上，不应该考虑他，这样我们就在 O(60) 的复杂度内，完成了对线性基的差集工作

因此，我们当然希望对于每个节点而言，它维护的线性基中的数字对应的深度越大越好，因为越大的深度意味着这个数字加入到越晚，在用 lca 的深度去过滤的时候，越不容易被卡掉，所以我们在从父节点推知子节点的线性基时，需要遵循后插入者优先的思想：

  1. 首先子节点无条件继承父节点的整个线性基空间和每个数值对应的深度
  2. 接着考虑插入子节点本身的数值，在插入过程中还是从高到低位遍历，如果 basis 的某一位为空，就直接插入；如果 basis 的某一位有数值，但是这个数值对应的深度小于当前待插入数值的深度，我们就要用当前数值替换掉已有数值（因为我们希望线性基深度越大越好）
  3. 注意这样的替换可能不是一次性的，比如我们用深度为 5 的数值替换出来一个深度为 4 的数值，那么这个数值还有可能替换掉深度为 3 2 1 的数值，因此我们要把换下来的这个数值作为新的待插入数值，先和当前位替换它的那个数值取异或，接着取遍历更低位，考虑去替换其他数值（这一点很容易忽略）

所以这里的插入我们可以看成是一种升级的插入操作，也就是在原有插入失败的情况下，我们还会考虑是否发生替换操作，而不是直接让待插入数值 ^ 线性基已有数值

补充数据范围：N - 50000，M - 1000000，D - 1e18

思路

本例需要考虑有环的情况，因为如果没有环，从起点走到终点的简单路径一定是唯一的，就不需要考虑决策了

我们以下图为例进行分析：

我们先考虑一条简单路径：1->2->5->6->9->10->11，再此基础上，我们可以考虑走第一个环，或者走第二个环，或者两个环都走，因此我们有了一个大致的思路，考虑先找到所有的环，把换上路径的异或和存入一组线性基中，最后令简单路径上的异或和 P 和线性基中所有换上路径的异或和逐个进行异或，取最大值即可

因此我们需要实现在图上找环，并且每找到一个环，我们都需要把这个环的路径异或和存入线性基，因此我们在搜索过程中需要维护一个 path 变量和一个 p 数组：

  1. path：从根节点开始搜索，搜索到当前节点，路径上的总异或和
  2. p [ u ] : 从根节点到 u 节点的简单路径的异或和

通过这两个结构我们即可加工出环上路径的异或和，比如我们当前来到 2 号节点时，p[2] 中存 1->2 的路径异或和，接着 2 号节点继续向下搜索经过 3 和 4，当再次回到 2 号节点的时候，搜索变量 path 的值是 1->2->3->4->2 这条路径上的异或和，和 p[2] 再进行一次异或，即可得到环上 2->3->4->2 这条路径本身的异或和

最后，我们如何判环，这里可以通过全局 vis 数组，假设我们从 u 搜索到 v 节点，v 节点不是 u 的父节点，同时 v 对应的 vis 数组有标记，那就说明 v 节点必然曾经作为环的起点，现在又作为终点被再次遍历到，我们此时就需要存 path ^ w(u->v) ^ p[v] 的值到线性基中（因为此时还在 u 这一层，path 中不包含 u->v 这条边）
