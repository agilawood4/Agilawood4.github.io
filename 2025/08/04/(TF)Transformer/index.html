Transformer

通过阅读本文，您可以大致理解 Transformer 的原理，基本组成与代码实现

基本组成

Transformer 最初用于机器翻译，比如最简单的中文到英文的翻译

我们把内部的 TRM 黑箱进行进一步的细化，分为下面这两部分

  1. Encoders：编码器
  2. Decoders：解码器

其中每个 Encoders 模块和 Decoders 模块都可以由多个子模块组成，以 Encoders 模块为例，它可能由若干个 Encoder 子模块组成，其中每个 Encoder 子模块结构相同，参数可能不同，各个子模块独立训练，Decoder 则同理，而 Encoder 和 Decoder 的架构是完全不同的

Encoder（编码器）

单个的 Encoder 模块可以进一步分成三部分：

  1. 输入部分
  2. 注意力机制
  3. 前馈神经网络

对于编码器而言，它的输入是若干个向量，输出是与输入相同数量，相同维度的向量，只不过输入的每个向量，都代表待解析文本中一个独立的 Token，而输出的每个向量，都可以代表整个待解析文本，只是从不同的角度进行解析，由 Tokeni 输入得到的输出，会以 Tokeni 作为中心词 / 主题 / 侧重点去解析整个文本，不同 Token 对文本解析的侧重点不同，因此输出的向量也不尽相同，但是他们都可以代表整个输入的段落

因此我们可以认为，在编码器中进行了信息聚合，原本的单一向量只代表单个 Token，现在变成了可以表示以这个 Token 作为主要侧重视角考虑下的整个待解析文本，供解码器作为参考

Decoder（解码器）

Decoder 层和 Encoder 结构基本类似，唯一不同的几个地方分别是：

  1. Decoder 层所用到的多头注意力中加入了 Mask 机制，会掩盖当前单词和它后边的单词内容，也就是下图中的（1）层
  2. Decoder 层额外加入了一层交互层，也就是下图中的（2）层

Linear & Softmax

这里值得注意的是，解码器初始输出的也是代表 Token 的向量，在经过Linear层和Softmax层以后，才能得到完整的单词

在全局中我们会维护一个字典，其中记录每一个 Token 和它对应的向量，在 Decoder 层每输出一个向量之后，这个输出值会和字典中的每一个向量计算相似度，也就是 Linear 操作

而 Softmax 操作则是把所有的相似度值进行归一化，得到 (0, 1) 空间内的概率值，通过选择最大概率对应的 Token，确定最后要输出的是哪一个单词（Token）

Mask 机制

这里在 Decoder 层中之所以要对（1）层进行 Masked 遮挡，主要是为了保证训练环境和真实预测环境相一致，我们以下面这个模型为例，在输入 S 的时候，希望模型输出 I，在输入 I 的时候，希望模型输出 LOVE，在输入 LOVE 的时候，希望模型输出 YOU

此时我们希望模型综合之前的输入 S I LOVE 推断出此时应该输出 YOU，而不希望模型参考后面的 YOU 和 NOW 这两个输入，因为在实际预测模型中，模型是得不到 YOU 和 NOW 的参考的，如果我们不对 YOU 和 NOW 做 mask 遮盖操作，模型在训练过程中就会得到过多的提示，从而导致实际应用中模型效果势必不好

输入处理

输入部分又分成 Embedding 和位置嵌入两部分

Embedding 机制

Embedding 用于对我们输入的句子按字分割，其中把每个字对应到高维空间中的一个特征向量

事实上，这里我们所说的按字分割其实并不准确，严格来讲，对于一个句子中拆分出来的每一个字，我们还会根据它的编码（utf8 编码或者 unicode 编码）进行Tokenizaion操作，也就是把字进一步拆分成Token，最终在 Encoder 输入时，一个Token会对应一个向量

如下图所示，我们输入一个长度为 12 个字的句子，其中句子中的每个字都会被转化成一个 512 维的特征向量，其中 512 最原始的 Transformer 架构中使用的向量维度

使用 Embedding 进行 Token 到高维向量的映射时，我们希望满足以下几个条件：

  1. 不同的单词映射到不同的向量
  2. 语义相近的词语映射到的向量距离相近（比如国王，君王，君主等词汇），这里评估两个向量距离可以用欧氏距离，也可以用余弦距离（两个向量之间的夹角大小，一般认为夹角越小，向量距离越近）
  3. 向量之间具有一定的逻辑关系，可以进行运算与推理，比如：国王 - 男人 + 女人 = 女王，那么对应前三个向量计算的结果应该约等于代表 “女王” 的向量

位置编码

一句话的语义不仅取决于组成这句话的文字本身，也受到这些文字之间相对位置关系的影响

相比与 RNN 等天然就具有时序处理关系的网络结构，Transformer 采用并行化处理，也就是一句话中的所有字会一起进行处理，这么做增快了处理速度，但是会忽略单词之间的序列关系 / 先后关系，比如对于 “我爱博文” 而言，“博” 和 “文” 一定要出现在 “我” “爱” 后边，才是这句话完整却原始的语义表述

因此我们需要某种机制来告诉 Transformer 我们输入的句子中，每个字的先后关系，这里我们采用的位置编码规则如下，对于每个字，我们枚举它在句子中的每个位置，2i 表示句子中偶数位置，用 sin 赋值，2i+1 表示句子中奇数位置，用 cos 赋值，同样可以得到一个特征向量：

  1. d~model~ 是我们每个 Token 要转化成的向量维度

在有的模型中，Embedding 过程也被称为 TokenEmbedding 操作，而位置编码过程也被称为 PositionEmbedding 操作

注意力机制

引入

所谓注意力机制，就是我们希望模型多关注重要的内容，在其上投入更多的 Attention，而具体的实现方法就是让模型在参考这些内容的时候，使用较大的权重系数

我们设想这样一个场景：现在我们已经知道了班级中若干同学的身高和体重，现在有一个新同学到来（身高已知），我们希望根据已有的身高 - 体重关系，预估这个新同学的体重？

我们很容易想到，对于和这个人身高差不多的那些人，他们的体重可能对于估计这个人的体重更有参考价值，因此我们在考虑的时候，为不同的身高赋予不同的参考权重，身高越接近 178 的，赋予越大的权重，这里的权重，也就是我们下图中的相关度系数

这只是一个用身高预测体重的模型，有可能我们要参考的信息不止身高这一个维度，可能还有胸围、腿长等多个维度；而我们要预测的信息也不止体重这一个维度，可能还有腰围等多个维度，那么上面这个模型就可以进行如下的拓展

此时我们可以把 [身高，胸围，腿长] 看成是一个向量，此时我们的相似度不再是两个数值之间进行比较，而是基于两个向量之间进行相似度比较（欧氏距离或余弦距离）

这里我们用 Q（Query）表示待查询的问题，K（Key）表示已有样本中，用来和查询问题计算相似度的键，V（Value）表示已有样本中，用于和相似度系数相乘再对位相加，估计查询问题答案的值，Q K V 就是注意力机制计算公式中的核心概念

下面我们再对上述模型进行扩展，Q 此时可以表示一个待回答的问题，K 表示若干已有问题的题面，V 表示这些题面各自对应的答案，现在我们想根据已有问题的答案，预估出新问题的答案，也可以通过计算问题之间相似度的方式，在已有答案中进行参考

以上就是注意力机制中最基本的模型概念

自注意力机制的核心目的是进行信息聚合

我们把原文中第 i 个 Token 对应的向量，和第 j 个 Token 对应的向量，二者之间的相似度记为 wij，而由 wij 所组成的矩阵就称为相似度矩阵

这里我们观察得到，整个 self-Atention 机制中，是不涉及到任何神经网络的，完全就是向量之间的相似度判定与数值计算，我们无法通过调参改善模型的性能，因此我们需要确保初始的 Q K V 都具有良好的性质

这里我们在实际应用中，并不会让 Q K V 完全相同，而是让它们三个都在源文本的 1300 个 Token 的基础上，经过不同的神经网络，先进行一次预处理变换，从而使得某些性质突出，某些性质弱化

这里我们使用的神经网络一般都是只有一层输入层和一层输出层，那么我们需要的参数个数就是 12289 × 12288（对于每个向量 i，它有 12288 个权重系数，加一个常数 b），我们可以把这些参数写成一个 12289 × 12288 的矩阵，也就是神经网络的权重矩阵，用于生成 Q K V 的权重矩阵中参数并不相同，分别记作 W^Q^ W^K^ W^V^

多头注意力机制

这里我们所说的多头注意力机制，实际上指的是多头自注意力机制（Multi-Head Self Attention）

在上面的自注意力机制中，当我们迭代足够多的轮次时，我们得到的 1300 组向量中，每一个向量中都已经包含了较为丰富的全局信息（已经和其他向量进行过多轮聚合），那么我们再进行后续的信息聚合时，就很难再提取出有用的信息，某些相关度系数会因此变小

基于此问题，我们考虑使用在一轮计算中，同时使用多组不同的 W^Q^ W^K^ W^V^ 让它们分别生成侧重方向不同的 Q K V，进行多组相似度计算，最后得到的输出结果再进行整合

拆分过程

这么会引入新的问题：我们需要生成的相似度矩阵的复杂度为 O(N^2^)，N 为 Token 数，当 Token 数较大的时候，生成相似度矩阵与权重矩阵、进行特征比较等操作的复杂度也会平方级别增加，因此我们考虑，在由初始向量生成 Q K V 向量时，进行降维操作，比如原本为 12288 维的向量，我们缩小 96 倍，变成 128 维的向量，但是向量总数不变

对向量进行降维的过程，实际上就是对向量核心特征的总结与提取过程，如果我们只做一组特征提取，势必无法得到全部的特征，因此在这一轮操作中，我们会使用不同的 W^Q^ W^K^ W^V^ 进行多个角度的特征提取，也就是会生成很多组 Q K V 向量（均为 128 维），让这些向量分别按照自注意力机制进行相似度比较，这就是多头注意力机制的核心

在 GPT3 模型中，我们选择了 96 个不同的角度进行特征提取，那么一轮计算中，我们就会得到 96 组注意力矩阵，分别进行加权聚合

拼接过程

这样又会产生新的问题，我们输入的向量为 12288 维，输出的向量维度应保持不变，因此我们需要考虑把拆分后的 128 维向量进行拼接，这里的拼接主要分为两步：

  1. 因为对于输入的 1300 个初始向量而言，他们在这 96 轮计算中的每一轮里，都会被拆成 1300 个 128 维的向量，这些向量和原始向量是一一对应的，因为我们需要把每一轮中对应位置的 128 维向量都取出来，进行首尾拼接，得到 128 × 96 = 12288 维的向量，这样的拼接要进行 1300 次，最终得到 1300 个 12288 维的向量
  2. 这样的首尾拼接只是完成长度上的拼接，要完成信息上的拼接，还需要在拼接后经过一个新的神经网络 w0，才能得到这一轮真正用于输出的 1300 个 12288 维的向量

这里需要区分，相似度计算有多少轮，以及每一轮中选取几个头的概念，只是 GPT3 模型中选取这两个参数都是 96，实际运行中这两个参数没有必然关系

交叉注意力机制

交互注意力机制又称为 Cross Attention，用于完成 Encoder 和 Decoder 两大模块之间的信息交换，实现由 Encoder 层的输出的向量得到 K 和 V 矩阵，与 Decoder 输出向量得到的 Q 矩阵进行交互

这里我们需要注意，Decoder 模块中也是会保存向量的，Decoder 每解析出一个向量，这个向量经过 Linear 与 Softmax 层输出一个具体单词的同时，也会被存入 Decoder 模块本身（Decoder 的 input），这些已经输出过的向量就会作为 Decoder 生成 Q 向量的来源，其生成的 Q 向量，结合由 Encoder 模块给出的 K 和 V 向量，最终实现新文本的生成

区分自注意力和交叉注意力机制，由 Decoder 和 Encoder 内部进行的 Attention 处理称为自注意力机制，Decoder 和 Encoder 层交互进行的称为交叉注意力机制

机制 Query 来源 Key-Value 来源 作用
自注意力 Eecoder 自身 Eecoder 自身 完成全局的信息聚合
交叉注意力 Decoder 自身 Encoder 输出 生成新文本或问题的答案

在交互层中，Encoder 的输出会作为 Decoder 的 Key 和 Value 作为参考

  • Encoder 的最终输出：Encoder 对输入序列（如源语言句子）进行多层自注意力计算后，输出一个上下文感知的表示矩阵​，该矩阵包含源序列的全局语义信息（如单词间的依赖关系）
  • Decoder 的 Cross-Attention 输入：Decoder 在每一层的自注意力后，会通过 Cross-Attention 将 Encoder 的输出作为 Key 和 Value，而 Decoder 自身的隐藏状态作为 Query

假设 “婴儿” 和 “左上” 向量点乘的结果为 0.7，其余三个点乘结果分别是 0.1 0.3 0.2，那么我们把这四个点乘结果依照顺序组成一个新的向量 w = [0.7, 0.1, 0.3, 0.2]，每一个权重和对应的 value 向量相乘，再把结果相加，就得到最后的 Attention Value（在本例中，Attention Value = 0.7V1 + 0.1V2 + 0.3V3 + 0.2V4 最后得到的也是一个向量）

残差模型

残差网络可以用来缓解随着网络深度增加，连乘导致梯度消失的问题，有效提高网络训练的深度

残差的概念：我们原始输入的词向量为 input，经过 Embedding 和位置编码以后得到的向量为 x，x 经过 self-Attention 注意力机制处理后得到的输出为 z，我们把对应的 x 和 z 对位相加，相加后得到新的结果进行输出，这一过程就称为残差处理，也就是下图中 LayerNorm 层所输出的内容

不失一般性，假设我们手上现在有一个若干层的网络，我们把原始输入 x 原封不动地对位相加到最后一层网络的输出结果 F(x) 上，把 F(x) + x 作为最终输出，这一过程就称为残差处理

前馈神经网络

前馈神经网络又称为 MLP（全连接神经网络）

上面我们提到，每一轮 Decoder 的时候，都会分为若干个头分别进行，而完整的一轮 Decoder，实际上是由多头注意力机制（Multi-Head Self Attention）和前馈神经网络（Feed Forward）两部分所组成的

由多头注意力机制输出的结果 z 值，经过残差处理以后作为前馈神经网络的输入，经过 Feed Forword 中两层全连接层以后，加上 z 本身再次做残差处理，即可作为 Encoder 层最终的输出

前馈神经网络内部，会先对输入向量的维度进行扩大，然后再对其维度进行缩小，这样做的目的是先放大所有可能有效的信息，再进行关键特征提取，有利于我们捕获 / 提纯原向量中的重要信息
