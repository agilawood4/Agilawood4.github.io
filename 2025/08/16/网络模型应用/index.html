网络模型应用

通过阅读本文，您可以大致了解基于 nn.Module 的神经网络的实际应用与搭建过程，以及模型训练与测试的完整框架

现有网络模型的修改

vgg16网络

在 Pytorch 中为我们提供了许多和图像识别与分类有关的现成的网络模型，他们全都集成在torchvision.models库中，比较常用的为vgg16网络模型，该模型在加载时接受以下的参数：

  • pretrained：是否要进行预训练，如果设置为 False，相当于我们只加载网络结构（把对应的模块加载出来），但各层的参数都是随机初始化的，也就是得到一个没有任何训练的初始模型；如果设置为 True，相当于我们加载一个别人已经帮我们训练过的模型，模型有一定的性能，我们只需要进行微调即可

增加模型

我们通过调用add_module()方法，向已有的网络结构中加入新的网络层，该函数接受两个参数：

  1. name：新加入网络层的名称
  2. module：新加入网络层的结构（传入 nn 中的某个网络层）

如果我们直接调用add_module()，会在整个网络结构的最后加入新的网络层（不属于任何一个 Sequential），我们也可以指定在某个特定的 Sequential 结构的最后加入这个新的模型

  1. 保存完整模型+参数
  2. 仅保存模型的参数

仅保存参数

这也是官方比较推荐的保存形式，因为保存完整模型需要占用较大的空间，通过仅保存参数的方式可以实现空间减省，我们通过调用torch.save()方法保存模型的参数，该函数接收（1）模型的参数字典state_dict()（2）保存路径，保存后需要通过torch.load()方法进行加载，加载结果为模型参数，需要在本地建立空模型以后，通过load_state_dict()加载到模型中才可以使用

上述两种模型保存和加载方式需要一一对应，不可以混用，值得注意的是，如果是自定义网络模型并且采用方法（1）进行保存时，模型加载的时候需要确保当前文件中有这个网络模型的定义（有定义即可，不需要实例化），否则会报错

  1. 先把优化器在当前 data 步数的梯度清零（zero_grad()）
  2. 另当前损失函数进行反向传播得到参数的梯度（backward()）
  3. 根据参数的梯度调整模型参数（step()）

模型模式选择

当我们搭建好网络以后，我们的模型model可以设置train和eval两种模式，如果是训练模型，我们需要选择train模式，如果是评估或者验证模型，我们需要选择eval模式

基础概念

要想搞清楚 train 和 eval 模式的区别，我们现需要明白设置这两个模式都会影响模型的哪些参数：

  • dropout：
    • 在训练模式下，nn.Dropout(p) 会对输入张量按概率 p 随机将一部分元素置 0，并把未被置零的元素除以 (1-p)，以保持输出的期望不变，这样可以打破神经元间的共适应，防过拟合、提升泛化
    • 而在评估 / 推理时，Dropout 不再置零，等价于恒等映射（也就没有缩放）
  • BatchNorm：指的是 BatchNormalization（批归一化）
    • 作用：减少内部协变量偏移，让每一层见到更“稳定”的分布，从而加速收敛、允许更大学习率
    • 训练 vs 评估：训练：用 batch 统计做归一化，并更新 running_mean/var（指数滑动平均）；‘评估：用 running 统计量做归一化

这里我们详细补充一下批归一化的过程：

（1）激活（activation）：

  • 指层的输出张量。比如：y = Conv2d(x) 的 y，或 y = ReLU(z) 的 y，这些都叫“激活”，BatchNorm 就是对“激活”做统计与变换

（2）running_mean / running_var（运行时均值/方差，滑动统计）：

  • 不是“当前 batch”的统计，而是跨多个 step 累积的指数滑动平均（EMA），在 pytorch 中，如果设置 momentum=m，则每次训练更新：runningmean=(1−m)∗runningmean+m∗μbatchrunning_mean=(1-m)*running_mean+m*\mu_{batch}runningm​ean=(1−m)∗runningm​ean+m∗μbatch​，runningvar=(1−m)∗runningvar+m∗σbatch2running_var=(1-m)*running_var+m*\sigma^2_{batch}runningv​ar=(1−m)∗runningv​ar+m∗σbatch2​
  • 也就是说每一次更新时，全局的均值和方差都由上一轮结束时全局的均值和方差，以及当前轮次的均值和方差加权得到

（3）动量（momentum）：

  • 在 BN 里，“动量”指滑动平均的系数 m（EMA 的平滑系数）
  • 直观：m 越大，当前 batch 的统计占比越高；m 越小，历史统计的“记忆”更重，上一轮的全局统计占比更高

优化器中的动量

刚才我们提到了一个动量（momnetum）的概念，它指的是更新 running_mean 和 running_var 的滑动平均系数，而在优化器中，我们也有动量的概念

在梯度下降过程中，如果梯度方向抖动，则参数的更新会按照之字型震荡，收敛速度较慢，我们把梯度下降类比成小球在山谷里滚动，假设我们给小球一个动量的概念，它就能记住之前的速度，滚动会更加平滑，使小球更快滑向谷底

加上动量的概念之后，我们就可以有效解决梯度下降不收敛的问题，在具体实现上，我们只需要在优化器中给出一个合适的动量系数即可，也就是设置momentum参数，通常我们给出的参数值都是 0.9，这是一个比较好用的经验值

有了动量以后，我们会用累积动量以后的速度更新参数，而不再仅仅使用当前梯度更新

train模式

通过调用model.train()即可在内部设置model.training=True，此时进入训练模式，在该模式下，模型有以下特点：

  • dropout：开启随机丢弃
  • batchnorm：使用当前 batch 的均值 / 方差并更新其running_mean/var
  • 该设置会递归作用到模型的所有子模块上，所有依赖self.training的层或分支都会被设置为训练状态
  • 不会关闭梯度

eval模式

通过调用model.eval()即可在内部设置model.training=False，此时进入评估 / 预测模式，在该模式下，模型有以下特点：

  • dropout：关闭随机丢弃，相当于 p=0
  • batchnorm：使用已经累积的 running 均值 / 方差，并且不再更新这些缓冲区
  • 该设置会递归作用到模型的所有子模块上，所有依赖self.training的层或分支都会被设置为训练 / 评估状态
  • 不会关闭梯度，需要手动设置torch.no_grad()来关闭梯度

训练阶段（每个 epoch 的训练循环前）：model.train()；验证/测试前：model.eval()，并配合 torch.no_grad()（见下一节）；验证结束后记得切回：model.train()（否则后续继续训练时 Dropout/BN 行为不对）

这里需要注意，train 模式和 eval 模式与是否关闭梯度之间没有必然关系，model.train()/eval() 不控制是否计算梯度，是否计算梯度由 autograd 上下文决定：

  • with torch.no_grad(): → 关闭梯度（前向更省内存/更快）
  • with torch.set_grad_enabled(flag): → 显式打开/关闭

损失函数的重要性

在神经网络的学习中，我们往往把重点放在网络层的设计上，但是实际上，损失函数才是决定我们模型最终训练目标的关键

在很多大型模型的设计中，网络层往往很普通（简单的nn.Linear），比如在 VAE 中，fc31 / fc32 就是最普通的 nn.Linear；分类网络里，最后输出类别 logits 的层也是 nn.Linear；回归任务里，最后一层可能也只是一个 nn.Linear(..., 1)

而通过对上述这三个模型的观察，我们发现：

  • 在 VAE 模型中，我们人为规定 fc31 输出要解释为 μ，fc32 输出要解释为 log⁡σ2，然后在 KL 项和重建项里这样使用它们，于是它们的输出逐渐学成了“均值/方差”
  • 在分类任务中，我们人为把 nn.Linear 的输出解释为 logits，然后用 CrossEntropyLoss，于是它们的输出被学成“类别打分”
  • 在 GAN 模型中，判别器的最后一层只是 Linear -> Sigmoid，但我们用对抗损失，让它学会输出“真/假概率”

这里我们提到的 KL项+重建项，交叉熵损失，对抗损失其实都是损失函数，换言之，我们选用的网络层往往大致相同的框架，是通过精巧的损失函数的设计，能够驱动网络通过减小损失的方式，朝着我们设计的那个特定的方向进行训练，也就是说，损失函数才规定了网络输出应该扮演的角色

通道概念补充

在深度学习中，我们经常提到通道的概念，通道数（Channels）指的是数据在某一个维度上分层的数量，而具体来讲，图像的通道数和神经网络的通道数又有所不同，下面我们进行简单的区分

图像的通道

在深度学习中，对于图像而言，通道数一般指的就是颜色或信息的分层

在 Pytorch 中，我们处理的图像的原始形状一般都是三维张量，可以表示为（C，H，W），三个参数分别表示图像的通道数，高度，宽度，其中 C 表示的就是通道数，不同类型的图片的通道数不同：

  1. 灰度图：[1, 高度, 宽度]，1 个通道，只有亮度信息
  2. RGB 彩色图：[3, 高度, 宽度]，3 个通道，分别表示红，绿，蓝
  3. RGBA 图像：[4, 高度, 宽度]，4 个通道，分别表示红，绿，蓝，透明度，常见的 png 格式图像就是 4 通道

网络的通道

在深度学习中，对于神经网络而言，每一层的输入和输出都有通道，但含义不同：

（1）输入层：通道数 = 输入图像的通道数（如 RGB 是 3 通道，灰度图是 1 通道，以此类推）

（2）隐藏层：输入通道数 = 上一层的输出通道数；输出通道数则由卷积核的数量决定（人为设定的超参数）

  • 例如：nn.Conv2d(in_channels=3, out_channels=64, kernel_size=3)输入 3 通道（RGB），输出 64 通道（生成 64 张特征图）

换言之，输入层实现了图像的通道和网络的通道的接洽，进入输入层的通道是图像的原始通道数，而在隐层中，每个通道是一个特征图，代表某种抽象的特征，较浅隐层的通道可能表示边缘、纹理、颜色等特征，而深层隐层的通道，可能表示猫耳，车轮等高级特征

在我们现阶段搭建的网络中，一般认为隐层深度越大，对应通道数越多

通道转化

有时候我们会读取初始通道数各不相同的图片，为了进行统一处理，我们需要把他们的通道数进行归一化，也就是转化为相同的通道数再进行处理，可以通过调用Image.convert()函数实现，该函数接收一个表示图像类型的字符串作为转换目标

常见的和通道数相关的转换模式如下：

模式参数 描述 通道数 典型用途
“L” 灰度（Luminance） 1 MNIST手写数字、黑白处理
“RGB” 红绿蓝三通道 3 大多数彩色图像任务
"RGBA” RGB + 透明度（Alpha） 4 需要透明背景的PNG图像
“CMYK” 印刷四色模式 4 专业印刷领域
“1” 二值化（黑白像素） 1 文字扫描、OCR预处理

此外，为了使得输入适配网络模型，若图像的高度和宽度需要裁剪，可以通过transforms.resize()实现；若图像需要转化为 tensor 格式，可以通过transforms.ToTensor()实现，这里不再赘述

通过查阅 Pytorch 官网中的 Docs 文档，可以学习 Pytorch 中 nn 库内各个网络结构与模块的参数与用法

谷歌 Colab：https://colab.research.google.com/

类似一样 Jupyter Notebook 的在线代码编辑器，可以提供一定时长的免费 GPU 运行环境

paperswithcode：https://huggingface.co/papers/trending

可以查阅当下流行的深度学习算法，以及对应的代表论文和开源代码
