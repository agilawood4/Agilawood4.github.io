网络模型应用

通过阅读本文，您可以大致了解基于 nn.Module 的神经网络的实际应用与搭建过程，以及模型训练与测试的完整框架

现有网络模型的修改

vgg16网络

在 Pytorch 中为我们提供了许多和图像识别与分类有关的现成的网络模型，他们全都集成在torchvision.models库中，比较常用的为vgg16网络模型，该模型在加载时接受以下的参数：

  • pretrained：是否要进行预训练，如果设置为 False，相当于我们只加载网络结构（把对应的模块加载出来），但各层的参数都是随机初始化的，也就是得到一个没有任何训练的初始模型；如果设置为 True，相当于我们加载一个别人已经帮我们训练过的模型，模型有一定的性能，我们只需要进行微调即可

可知该网络主要有两个Sequential结构组成，分别是（features）和（classifier）两部分，最后实现 1000 个类别分类的是 classifier 部分中的（6）位置的线性层，该层把对应的 4096 个特征的输入，线性变换为 1000 个特征的输出，实现了最终的分类，因此我们要想实现 10 类别分类，有以下两种思路：

  1. 再额外增加一个线性层实现（1000）->（10）
  2. 直接修改最后一个线性层把（4096）->（1000）变成（4096）->（10）

增加模型

我们通过调用add_module()方法，向已有的网络结构中加入新的网络层，该函数接受两个参数：

  1. name：新加入网络层的名称
  2. module：新加入网络层的结构（传入 nn 中的某个网络层）

如果我们直接调用add_module()，会在整个网络结构的最后加入新的网络层（不属于任何一个 Sequential），我们也可以指定在某个特定的 Sequential 结构的最后加入这个新的模型

网络模型的保存与读取

当我们训练得到一个网络模型以后，可以将该模型保存在本地，并且在后续其他文件用到它的时候进行调用，常见的保存方式有以下两种：

  1. 保存完整模型+参数
  2. 仅保存模型的参数

完整保存

我们可以通过torch.save()方法保存完整的模型，该函数接收（1）具体模型（2）保存路径这两个参数，对应保存后的模型需要通过torch.load()方法进行加载，加载的返回值就是模型本身：

仅保存参数

这也是官方比较推荐的保存形式，因为保存完整模型需要占用较大的空间，通过仅保存参数的方式可以实现空间减省，我们通过调用torch.save()方法保存模型的参数，该函数接收（1）模型的参数字典state_dict()（2）保存路径，保存后需要通过torch.load()方法进行加载，加载结果为模型参数，需要在本地建立空模型以后，通过load_state_dict()加载到模型中才可以使用

上述两种模型保存和加载方式需要一一对应，不可以混用，值得注意的是，如果是自定义网络模型并且采用方法（1）进行保存时，模型加载的时候需要确保当前文件中有这个网络模型的定义（有定义即可，不需要实例化），否则会报错

我们首先给出几个训练与测试过程中常用变量的定义：

  1. total_train_step：总共训练的步数（每遍历一个 data 算一次训练步数）
  2. total_test_step：总共测试的步数（每遍历一个 epoch 算一次测试步数）
  3. epoch：训练的轮数（和 total_test_step 相对应，包含多个 total_train_step）

在训练+测试过程中，我们也可以使用tensorboard对训练与测试结果进行展示，这里我们的横轴选取为训练 / 测试的步数，纵轴选为每一轮训练与测试的损失值的总和，对于一个正确训练的模型，二者都应该呈现总体下降的趋势

对于每一轮训练中一个 data 对应的单次训练步数，我们一般遵循下面的顺序：

  1. 先把优化器在当前 data 步数的梯度清零（zero_grad()）
  2. 另当前损失函数进行反向传播得到参数的梯度（backward()）
  3. 根据参数的梯度调整模型参数（step()）

有时候为了防止有过多的输出，我们可能每训练 100 轮展示一次输出

GPU训练

有时候为了加快模型的训练速度，我们选择使用 GPU 来代替 CPU 进行训练，常用的 GPU 训练方式有以下两种

GPU张量

我们在基础知识中提到的 Tensor 张量也可以通过调用*.cuda()方法，实现其在 GPU 环境上的运行，同时需要注意的是，CPU 和 GPU 不能混用，比如我们用 CPU 上定义的张量作为 GPU 环境下网络的输入，执行时解释器会报错

使用*.to()切换

我们常用的运行环境有 cpu 和 cuda 两种，其中 cuda 所代表的就是 GPU 环境，我们可以通过torch.device()定义我们想要使用的运行环境

通道概念补充

在深度学习中，我们经常提到通道的概念，通道数（Channels）指的是数据在某一个维度上分层的数量，而具体来讲，图像的通道数和神经网络的通道数又有所不同，下面我们进行简单的区分

图像的通道

在深度学习中，对于图像而言，通道数一般指的就是颜色或信息的分层

在 Pytorch 中，我们处理的图像的原始形状一般都是三维张量，可以表示为（C，H，W），三个参数分别表示图像的通道数，高度，宽度，其中 C 表示的就是通道数，不同类型的图片的通道数不同：

  1. 灰度图：[1, 高度, 宽度]，1 个通道，只有亮度信息
  2. RGB 彩色图：[3, 高度, 宽度]，3 个通道，分别表示红，绿，蓝
  3. RGBA 图像：[4, 高度, 宽度]，4 个通道，分别表示红，绿，蓝，透明度，常见的 png 格式图像就是 4 通道

网络的通道

在深度学习中，对于神经网络而言，每一层的输入和输出都有通道，但含义不同：

（1）输入层：通道数 = 输入图像的通道数（如 RGB 是 3 通道，灰度图是 1 通道，以此类推）

（2）隐藏层：输入通道数 = 上一层的输出通道数；输出通道数则由卷积核的数量决定（人为设定的超参数）

  • 例如：nn.Conv2d(in_channels=3, out_channels=64, kernel_size=3)输入 3 通道（RGB），输出 64 通道（生成 64 张特征图）

换言之，输入层实现了图像的通道和网络的通道的接洽，进入输入层的通道是图像的原始通道数，而在隐层中，每个通道是一个特征图，代表某种抽象的特征，较浅隐层的通道可能表示边缘、纹理、颜色等特征，而深层隐层的通道，可能表示猫耳，车轮等高级特征

在我们现阶段搭建的网络中，一般认为隐层深度越大，对应通道数越多

通道转化

有时候我们会读取初始通道数各不相同的图片，为了进行统一处理，我们需要把他们的通道数进行归一化，也就是转化为相同的通道数再进行处理，可以通过调用Image.convert()函数实现，该函数接收一个表示图像类型的字符串作为转换目标

常见的和通道数相关的转换模式如下：

模式参数 描述 通道数 典型用途
“L” 灰度（Luminance） 1 MNIST手写数字、黑白处理
“RGB” 红绿蓝三通道 3 大多数彩色图像任务
“RGBA” RGB + 透明度（Alpha） 4 需要透明背景的PNG图像
“CMYK” 印刷四色模式 4 专业印刷领域
“1” 二值化（黑白像素） 1 文字扫描、OCR预处理

此外，为了使得输入适配网络模型，若图像的高度和宽度需要裁剪，可以通过transforms.resize()实现；若图像需要转化为 tensor 格式，可以通过transforms.ToTensor()实现，这里不再赘述
