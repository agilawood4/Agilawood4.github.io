线段树扩展

通过阅读本文，您可以大致了解到基础线段树中（1）势能分析，（2）权值线段树的相关内容，加强对线段树原理的理解

您需要了解以下前置知识：

  1. 线段树的作用：快速完成区间修改，单点修改，以及区间中某些统计信息的维护和查询

  2. 需要了解的基本操作：线段树维护的单点修改，区间修改（区间加，区间强制修改），线段树维护区间最大值，区间和，区间异或和

  3. 有一套自己实现的线段树模版（普通线段树即可，不需要动态开点）

权值线段树

权值线段树可以类比权值树状数组，也就是以一个序列中所有数字的数值范围（而不是序列下标区间）作为节点维护的区间范围的线段树，在这类题目中，通常序列中数字本身的数值很大，但是数字个数很少，因此我们通常需要先通过离散化的方式，对数值进行预处理

假设我们有一个线性序列：12 33 4 22 128 5 22 109 22 439 4 12

我们首先对其进行保序离散化得到：3 5 1 4 6 2 4 7 4 8 1 3，离散化之后序列中最大的数值为 8，因此我们权值线段树需要维护的区间最大范围就是 1 到 8（维护数值的区间），在概念上，我们先以 build(1, 1, 8) 方法建立一棵空树，再通过单点修改的方式增加每个位置对应的数值，但是因为初始化的是空树，在实际操作中，这一步往往被省略，只需要留出对应大小的空间即可，我们在逻辑上建立的空树结构如下：

P1168

【区间第 k 小】

思路

本例是区间第 k 小的一个简单应用，如果我们能在 O(logn) 复杂度内求出一个线性序列中第 k 小的数值，即可边动态插入数值，边进行查询，每当插入奇数个数值的时候，就查询一次第 n+1>>2 小的数值即可

假设我们当前已经有了权值线段树，该树维护 tr [ ind ] 表示 ind 节点所维护的数值区间中数字的个数，我们借助这个数组，通过形如 splay 中中序遍历为第 rank 名的数值的方法，进行查找，假设我们当前来到 ind 节点，我们要查找第 k 小的数，第 k 小就等价于，比它小的数字有 k-1 个，不超过它的数字有 k 个：

  1. 如果 tr [ L(ind) ] 的值 >= k，则说明左儿子节点中维护的数值个数不少于 k 个，又因为越靠左的区间维护的数值越小，因此第 k 小的数一定在左儿子对应的区间中，向左查找即可
  2. 如果 tr [ L(ind) ] 的值 < k，则说明左儿子节点中维护的数值个数足 k 个，因此当前全局第 k 小的数一定在右儿子对应的区间中，向右查找即可，注意到了右子树中，需要减掉左子树中的数值个数，作为在右子树中查找的新的 k 值
  3. 如果 l == r 说明查找到目标数值，直接返回 l 或者 r 即可，因为此时维护的区间本身就是数值

LC699

思路

俄罗斯方块的简单模拟，对于当前落下的一个 l 到 r 范围内的方块，在它落下之后，l 到 r 这段范围的真实高度取决于以下两部分：

  1. 这个方块本身的高度 r - l + 1
  2. 这个区间原有高度的最大值：比如 l 到 r 范围中的某个子区间中曾经有高度 h，那么这个新的方块就只能落在 h 之上（这一点遵循俄罗斯方块的机制，只要有一个位置垫着高度，上面的方块也没法落下）

因此我们需要用线段树维护整个区间轴上的两个操作：维护区间高度值，进行区间整体修改和区间最值查询即可

P3740

思路

本例是线段树维护区间信息中一个比较特殊的例子，中间过程中只有修改，只在最后有唯一的一次查询，因为查询的次数很少，因此我们考虑把查询设成 O(n) 暴力的也没关系，反正只会查一次

因此我们很容易想到，维护一个 post 数组作为区间信息，其中 post [ ind ] = i 表示 ind 节点所维护的区间，被贴上了编号为 i 的报纸，这个标记既作为区间本身的信息，又作为懒标记使用，如果我们遇到一个整区间，我们就直接给这个区间打上 post 标记，直到下次遍历到它的子区间的时候，我们再把这个标记下传即可

因为我们不需要时刻保证查询到的信息正确，因此我们不需要设计 UP 函数，换言之，我们是不需要考虑我们维护的区间信息能够由子区间合并得到的（这是本例最关键的一个点，因为 post 这个信息根本没法合并，如果从正常线段树的角度考虑，我们一般维护的是某个区间上海报的种类数，而不是某个区间上到底是哪种海报），而 DOWN 操作则是必须的（比如我们先改父区间，再改子区间，如果我们不下传父区间标记，改子区间的时候，父区间 post 会保留，最后查询答案的时候下传父区间标记，反而会覆盖子区间，从而出错）

最后，本例还有一个比较易错的地方，就是海报区间的维护，对于范围为 [1,2] [6,7] [1,7] 这样的三张报纸，本来我们贴完，客观上讲这三张报纸是都可以被看到的，但是我们在进行离散化以后（本例数据范围一定要离散化）就变成了 [1,2] [3,4] [1,4] 相当于[1,4] 那张报纸会被剩下两张报纸挡住，对于这种情况，我们需要为每个节点维护一个后缀节点，也就是 1 位置对应多维护一个 2 位置，2 位置对应多维护一个 3 位置，这样我们原有的位置是 1 2 6 7，我们每个位置都维护一个后缀位置就变成了 1 2 3 6 7 8，离散化以后三张报纸就是 [1,2] [4,5] [1,5]，这样 [1,5] 中间会露出一个 3 位置，也就是我们在 2 和 6 之间插入的 2 的后缀，防止离散化以后二者相邻，就可以避免这个问题

最后，我们查询的时候，把所有 post 标记都下放到叶子节点，对于每个叶子节点，找一个集合 set 记录所有颜色总数即可，在修改过程中，post == 0 的区间有两个含义：这个区间从来没贴过报纸，或这个区间的报纸信息已经下传到它的子区间中表示

HDU4614

【权值线段树+二分】

思路

本例的模型可以进行如下的简化：给定一个序列，由 0 和 1组成，我们以序列的某个位置作为左端点，向右开始查找，要求找到至少 k 个为 0 位置，问能找到的最靠左的右端点在哪？

显然，我们找到的右端点越靠右，可能在区间内包含的 0 就越多，反之，右端点越靠左，可能包含的 0 就越少，因此右端点位置和答案之间具有单调性，我们可以通过二分答案的方式，来确定右端点的真正位置

如果只是单纯用二分答案，对于我们二分到的一个位置 mid，在 l 到 mid 之间 0 的统计（也就是对应原题中空位置的统计）的时间复杂度是 O(n) 级别的，单组查询还好，如果是多组查询一定会超时，因此我们需要把这个统计进行优化，也就是查询一段区间内 01 序列的区间中 0 的个数，显然可以用线段树进行处理，把单次查询优化到 O(logn) 级别

此外我们还需要额外处理两个边界：我们开始放置花朵的起点可能是不合法的，我们要找到从给定起点开始第一个为 0 的区间位置，我们最后一个放置花朵的终点可能也是不合法的（有可能右端点到 0 所有的花朵都放不完，则需要找到从终点开始，第一个为 0 的位置），这两个位置的查找都可以用类似的二分答案实现，具体见代码注释即可

势能分析

我们在初学线段树的时候就知道，线段树能维护区间信息的前提条件为：

  1. 上级区间的信息可以由左右子区间信息在 O(1) 复杂度内得到
  2. 对某个区间的修改可以在 O(1) 的复杂度内正确映射到当前区间所维护的信息上

比如线段树可以维护的信息（区间和+区间整体加c，对区间的修改可以通过 c × 区间长度映射到区间和上），无法维护的信息（区间翻转，此时如果只用线段树只能在 O(区间长度) 复杂度内完成区间逆序）

对于一些特殊的区间操作，我们确实无法通过 O(1) 的复杂度正确维护，只能把他下降到每个叶子节点（也就是长度为 1 的元区间）进行暴力修改，但是此时我们可以通过一些剪枝操作，来控制我们进行这种暴力修改的次数，进而保证一个比较优秀的时间复杂度，常见的操作有开方，取模等操作：

  1. 区间开平方：给区间内每一个数都开平方，每次向下取整
  2. 区间取模：给区间内的每一个数都关于给定的数 x 取模

区间开平方

开方会使得一个数值快速下降，以下面的题目数据为例，10^12^ 量级这么大的数字，最多开 6 次平方，也就会变成 1，而我们知道，1 无论怎么开平方都不会改变，也就是说，如果一个区间上所有的数字都变成 1，我们再对这个区间或者它的子区间进行区间开平方的时候，就可以直接把这个操作在停在当前区间，因为区间里所有的数都是 1，没必要再开平方了

换言之，每个区间上的数字都变成 1，也就是区间的最大值是 1，我们需要维护一个区间最大值，用于作为重要的剪枝：当到达某个节点的时候，发现这个节点对应的区间最值为 1，此时直接停止操作

我们分析这么操作的时间复杂度，首先每个数字最多进行 6 次开方操作就会变成 1，而我们进行暴力的区间修改，会把区间修改下沉到每个叶节点进行操作，每次对一个叶节点进行一次开平方操作需要 O(树高) 的复杂度，这个叶节点进行最多 6 次开方操作，如果总节点数为 n，则总体暴力修改的复杂度不会超过 O(k×n×logn)，其中 k 是一个小常数，之后我们所有区间的最大值都会变回 1，也就是可以在近似 O(1) 的复杂度内完成剩下的操作

区间取模

取模会使得一个数值快速下降，对于数字 x 而言，如果对比他小的数字取模，每次至少会使得 x 缩减一半，这是取模的一个重要性质，换言之，对于数值 x 而言，如果他对 v 取模，最多进行 logv 次取模操作，x 就会变成 0，此时就不需要继续进行取模操作了

假设我们需要支持这样的区间操作：每次对区间 l 到 r 范围内的所有数字对 v 进行取模，我们首先注意到，如果这个区间上所有的数都小于 v，那么取模相当于没取，这个操作就会自动停止在当前区间，因此我们仍然需要维护一个区间最大值，当某个区间上的最大值已经小于我们这个区间的模数的时候，我们就可以停止后续操作，进行剪枝

而对于需要取模的区间，我们还是需要把操作暴力下放到叶节点的元区间上，对于模数 v，其中一个叶子结点对其取模一次的时间复杂度为 O(树高)，最多要取模 O(logv) 次，而节点总为 n 的时候，总共需要暴力的操作次数不会超过 O(n×logn×logv)，这个复杂度也是完全可以接受的（对应区间取模操作的题在 CF438D，因为和下例思路几乎相同，不再展示代码）

势能

在线段树中，一个节点最多被进行多少次操作，后续就不需要再进行这种操作，这种性质就称为这个节点的势能（比如开多少次平方变成 1，取多少次模数变成 0），如果我们发现某些区间操作只能暴力完成，可以考虑，如果这些区间操作只会减少区间内元素的数值，也就是降低节点的势能，就可以考虑从势能分析的角度，考虑是否能够通过剪枝（记录区间内的最大值等）策略，在有限次数内完成所有区间的暴力操作，使得线段树的总体性能仍然保持较好的状态

在利用势能分析求解的线段树中，所有区间操作要么通过剪枝直接剪掉，要么会暴力下放到叶子节点，此时是不会涉及到 LazyTag 以及 DOWN 方法的
