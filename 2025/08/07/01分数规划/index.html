01分数规划模型

01 分数规划基本模型：给定 n 个数据，每个数据都有（a，b）两个维度，并且没有负数，现在需要你从中选择 k 个数据，要求 k 个数据 a 值的和与 k 个数据 b 值的和的比值尽可能大，返回这个最大的比值

先给出一种错误的贪心思路：选择 a / b 最大的前 k 个数据，反例：（5,5）（0,1）（2,6）要求选 2 个数据，其实也很好理解，显然 x1 / y1 + x2 / y2 与 x1 + x2 / y1 + y1 是不等价的

暴力求解思路：每个数据都枚举选和不选两种情况，总方案数为 2^n^

正解可解的前提：计算机中有关浮点数的精度是有限的，或者说题目对于最大比值的要求有精度范围（比如只保留前两位小数，或前三位小数），此时我们可以用二分答案来求解，保证在规定精度内正确即可

求解思路

对于 01 分数规划的基本模型，假设在最优情况下，我们选择的 k 个元素的比值达到 x，那么则有 (a1 + a2 + … + ak) / (b1 + b2 + … bk) == x，我们对该式进行同分移项则有：a1 - xb1 + a2 - xb2 + … + ak - xbk == 0

此时我们定义结余 c = a - x × b，其中 x 是我们待求解的比值，那么我们 n 组数据就会得到 n 组结余

规定最小精度单位为 eps（这里根据题目要求保留的精度位数来决定 eps 的具体值），对最终的比值 x 进行实数域的二分答案，实数二分的逻辑如下：

  1. 初始边界 l = 0，r = 所有 a 值的和
  2. 二分边界：r > l && r - l >= eps，每次都令 x = (l + r) / 2，然后根据这个 x 取计算一次结余，选出前 k 个最大的结余，求一次结余累加和
  3. 如果结余累加和 >= 0，则更新答案为 x，接着去 [ l + eps, r ] 范围上二分
  4. 如果结余累加和 < 0，则不更新答案，接着去 [ l, r - eps ] 范围上二分

这么做的原理是，在最优决策下，假设我们此时的比值是 x，那么对于我们选出的 k 个数据，一定有：a1 - xb1 + a2 - xb2 + … + ak - xbk == 0 成立，而 x 选大的话，上式会变小，x 选小的话，上式会变大，显然具有单调性，而我们希望留下的是使得结余和非负的那些 x 中最大的，也就是实数域上 10 二分的模型

P4377

思路

首先我们要搞清楚 01 分数规划的本质，更多的时候，它只是用来做数据转化，使得原本无法处理的数据变得可以处理

本例要求我们求出一个选择的方案，使得所选元素总和不小于 w 的前提下，所有元素的 talent 之和与 weight 之和的比值最大，假设题目要求最大化的不是加和的比值，而是比值的加和，也就是要求每个元素的 talent / weight 的比值之和进行最大化，同时要求所选元素总和不小于 w，那么这个题我们肯定能想到用背包去做

现在我们之所以没法转化成背包，或者说是贪心，主要是因为我们要最大化的比值的表达式中，各个元素之间是有相互依赖的（归根结底还是 x1 / y1 + x2 / y2 与 x1 + x2 / y1 + y2 的不等价性），此时我们就考虑用 01 分数规划进行处理，因为它能够通过其二分得到的 x 值，把比值表达式转化为若干个 a - xb 形式的求和，这样我们就实现了数据拆分的过程，因为每个 a - xb 都是独立的，不会受到其他元素的影响

因此在本例中，01 分数规划最大的作用其实是实现了一种对数据的等价变形，通过二分比值 x 和结余的表示，我们的元素就变成了：重量为 w，结余值为 c = a - bx，要求我们选择若干个元素，总重量不小于 m 的前提下，所得到的结余和最大（因为我们判断结余和 >= 0 取的一定是最优情况下的结余和）

那么剩下的就是背包问题了，这里要求的是总重量不小于 w，此问题是总重量恰好为 w 的变形，我们定义 dp [ i ] [ j ] 表示前 i 件物品，总重量恰好为 j 的时候，所得到的最大结余和，最后只需要遍历所有的 dp [ n ] [ m ] 到 dp [ n ] [ Wmax ] 取最优解即可

然而，我们观察数据范围，发现每个物品的重量 w 达到 1e6，那么如果我们按照正常的背包去遍历，时间复杂度会达到 O(nw)，250 × 1e6 不管时间还是空间都开不下，我们注意到要求的总体积 m 只有 1000，因此这里我们需要用到一个很经典的 dp 定义变形，也就是压缩最后一维 dp 表达式的定义，我们规定：

  1. dp [ i ] [ j ] (i < m): 前 i 个元素，重量恰好为 j 的最大结余和
  2. dp [ i ] [ m ] : 前 i 个元素，重量**大于等于 m **的最大结余和

也就是说，我们把所有 j >= m 的状态全都压缩到 m 对应的状态，这样我们就解决了空间上的问题，对于时间复杂度的求解，我们考虑从 i-1 层到第 i 层的转移：

本来我们对于 j 的枚举是要从 0 到 w（w 是最大物品重量），但是实际上，我们的有效转移最多只有 m 种，换言之，也就是说 j 对应的范围上，真正有效的转移实际上是很稀疏的，从 i-1 层到 i 层的转移，假设第 i 个物品重量为 wi，那么实际能转移到的重量也只有 a+wi，b+wi，c+wi，…，g+wi 这 m 种情况

因此这里我们的状态转移不是枚举第 i 层的体积能由第 i-1 的哪些体积转移来，而是直接枚举第 i-1 层已有的体积，看它能转移到第 i 层的哪些体积上，这样我们的时间也优化成 O(nm) 的级别

最后按照 01 背包模型优化掉第一维，倒序遍历体积即可，整个题目从头到尾写下来有一种很自然的感觉

poj 2728

【最优比率生成树】

思路

01 分数规划不仅可以求最大比值，也可以求最小比值，只是求最小比值的时候，对应的二分答案框架和结余和求解要从求最大比值的时候反过来，本例就属于求最小比值的情况

首先要求修路条数足够少，也就是无向图取联通，相当于求原图的最小生成树，其次是要求总花费 / 总距离最小，我们还是考虑用 01 规划对数据进行变形，把总比值拆分成每一条边上的 cost - x × dist，用这个结余值作为边的边权

因为这里要求的是最小比值，我们的最优决策就要向着结余值之和尽可能小的方向考虑，很容易想到最小生成树算法（能够得到最小的边权和），因此我们就有了以下的处理流程：

规定最小精度单位为 eps（这里根据题目要求保留的精度位数来决定 eps 的具体值），对最终的比值 x 进行实数域的二分答案，实数二分的逻辑如下：

  1. 初始边界 l = 0，r = 经验值（z 的范围 / xy 的范围）
  2. 二分边界：r > l && r - l >= eps，每次都令 x = (l + r) / 2，然后根据这个 x 取计算一次结余，以此时的结余作为边权，在原图上求解一次最小生成树，返回得到的边权和
  3. 如果边权和 <= 0，则更新答案为 x，接着去 [ l, r - eps ] 范围上二分
  4. 如果结余累加和 > 0，则不更新答案，接着去 [ l + eps, r ] 范围上二分

P3199

思路

本例用到和平均值有关的一个常见转化，取一个数列 a1 ~ an 的平均值 x，等价于取 a1 - x ~ an - x 求和以后和 0 进行比较，如果转化后的数列和值小于 0，则说明取大了，如果和值大于 0，则说明取小了

这个转化来自下式：

细观察我们发现，如果令 01 分数规划中的 b 数组全为 1，就是平均值的求解公式，因此平均值天生就隐含 01 分数规划的思想

而对于本例来说，我们要求的不是线性序列上的平均值，而是一个图中，全部环上的平均值，并且是各个环的平均值中的最大值，我们考虑对于每次二分得到的可能平均值 x，我们都让所有边权减去 x，那么环上的边权对应也会减去 x，我们考虑 x 对于边权的影响：

  1. 如果 x 恰好是最大的平均值，所有边权都减去 x 以后，原图中环上边的边权和都 >= 0，也就是说不会有负环
  2. 如果 x 比最大平均值小，所有边权都减去 x 以后，原图中环上边的边权和都 > 0，也不会存在负环
  3. 如果 x 比最大平均值大，所有边权都减去 x 以后，原图中环上边的边权和会存在 < 0 的情况，也就是说会出现负环

因此本例的 check 函数需要设计为对于当前新的边权 w - x，在有向图中进行负环的判定，这里可以用 spfa 算法，我们再介绍一种新的判断负环的方法

我们需要维护两个信息：

  1. path [ u ] : 在当前深搜的路径上，u 节点是否正在被搜索，如果我们当前到达了 u，并且正在从 u 向下搜索别的节点，此时 u 的 path 值设为 1，如果我们已经走完了 u 的所有路径，并且从 u 回退到它的父节点，就设为 0
  2. value [ u ] : 从初始源点搜到 u 位置，路径上的累加和

首先我们设置一个超级源点，从超级源点依次向 1 ~ n 中的每一个点进行深搜，而对于当前节点 u，它能够继续向 v 节点搜索的条件是u 的 value 值 + uv 之间的边权必须严格小于 v 本身的 value 值，此时才能从 u 向 v 扩展，并且更新 v 对应的 value 值，这是一个很重要的剪枝，因为如果从 u 走到 v 会使得路径的累加和变大的话，那么即便 u 和 v 之间有负环，等到从超级源点走到 v 的时候，也一定能找到这个负环；而如果 u 和 v 之间确实没有负环，这一层搜索就直接被剪掉了

上面考虑的其实是这种情况，当前我遍历到 2 和 3 之间这条边，发现 value(2) + w(2, 3) > value(3)，此时我们就不再从 2 向 3 搜索，但是 2 和 3 确实能构成一个负环，此时这个负环也不会被漏掉，因为我们还会从超级源点遍历到 3，3 号节点会依次搜索到 4 3 6，等 6 号节点再次到 2 号节点的时候，value(2) > value(6) + w(6, 2) 就可以把 2 的 value 值更新成 -6，而 3 此时的 value 值是 0（从超级源点第一个搜到 3 位置），此时 2 就可以去更新 3，而 3 的 path 值设成 True，也就是说，我们在保证 value 更新值单调递减的前提下，在路径上遍历到 path 值为 1 的节点（已经搜到过），此时就表示我们找到了负环

换言之，我们在 u 处找到负环的充要条件为：u 首先可以搜索到 v（w 和 value 满足条件），并且 u 遍历到一个 path == 1 存在的 v，或者 v 处能找到负环
