网络模型

通过阅读本文，您可以使用 pytorch 搭建简单的网络模型

nn.Module

nn 是神经网络（Neural network）的缩写，而nn.Module则是 nn 中 Containers（容器）中为所有神经网络所设计的通用架构，我们所搭建的神经网络都可以从 nn.Module 模版进行继承，只需要在其上进行修改补充即可

一般我们需要重写的两个方法为：

  1. __init__()：初始化方法
  2. forward：神经网络的前向传播方法

下面我们给出一个最简单的神经网络的示例，后续我们搭建的复杂网络都是基于此架构来实现的

这里我们定义了一个最简单的网络，输出 = 输入 + 1，需要注意的是，这里我们的 input 和 output 都是张量类型，执行的 +1 操作即为张量中每个元素都 +1

常用神经网络结构

下面我们简单介绍nn.Module模块中，和神经网络有关的相关结构与用法，这里我们会涉及到toech.nn和torch.nn.functional两个库，其中前者是对后者的顶层封装，后者是对前者的底层实现

Convolution Layers

Convolution Layers 是神经网络中的卷积层，在 nn 模块中，我们最常用的卷积操作是conv2d，也就是对于二维图像的卷积操作

nn.functional.conv2d()

该函数是二维卷积操作的最底层实现，接收以下几个参数：

  1. input：卷积层的输入，要求必须是一个四维的 tensor，tensor 的各个维度分别表示 minibatch（一次性最少训练的数据个数），in_channels（输入数据的通道数），iH（输入图像的高），iW（输入图像的宽）
  2. weight：卷积核，要求必须是一个四维的 tensor，tensor 的各个维度分别表示 out_channels（输出的通道数，也就是使用卷积核的数量），输入数据的通道数与分组数的比值，一般 group 默认设置为 1，所以这里可以看成是输入数据的通道数，kH（卷积核的高），kW（卷积核的宽）
  3. bias：偏置系数，一般默认为 True
  4. stride：卷积的步长，默认为 1，可以认为修改增大卷积的补偿
  5. padding：是否需要边缘填充，默认不填充，可以人为设置在边缘进行全 0 的填充

nn.conv2d()

该类是对 functional 中的 conv2d 函数的顶层封装，二者实现的都是对于二维图像的卷积化操作，但是接收的参数形式有所不同，nn.conv2d()接收的参数列表如下：

  1. in_channels：输入图像的通道数量，如果是一般彩色图片则通道数为 3
  2. out_channels：输出图像的通道数量，也就是卷积核的数量
  3. kernel_size：卷积核的大小，传入一个整型 x，表示卷积核大小为 x×x，也可以传入一个元组（x，y）表示一个不规则卷积核，长度为 x 宽度为 y
  4. stride：卷积的步长大小，传入一个整型 x，表示横向和纵向的步长都是 x，也可以传入一个元组（x，y）分别表示横向步长为 x，纵向步长为 y
  5. padding：边缘填充的层数，传入一个整型 x，表示横向和纵向的边缘填充都是 x，也可以传入一个元组（x，y）分别表示横向填充层数为 x 纵向填充层数为 y
  6. padding_mode：边缘填充的内容，一般都选择zero使用 0 填充

对于这张图片中的模型，因为只有一个卷积核，则 in_channels 和 out_channels 都是 1，但是如果我们使用多个卷积核，那么对应的卷积后的输出也有多个（每个卷积核内容 / 大小都可以不相同），此时 out_channels 也会对应增加

Pooling Layers

Pooling Layers 称为池化层，我们最常用的模块为MaxPool2d，该模块称为最大池化，也就是我们说的下采样

nn.MaxPool2d()

该模块是卷积神经网络中最常用的池化层模块，其接收下面几个参数：

  1. kernel_size：池化核的大小，可以传入一个整型 x，表示 x×x 大小的池化核，也可以传入元组（x，y）表示池化核的高为 x 宽为 y
  2. stride：池化的步长，含义和卷积步长相同，只是卷积步长默认为 1，池化步长默认为 kernel_size
  3. padding：边缘填充，默认为不填充，设置为 True 时填充值默认为 0
  4. Ceil_model：因为 stride 默认为池化核的大小，因此我们在移动过程中，很可能出现池化核部分覆盖输入图像的情况，当 Ceil_model 设为 True 的时候，部分覆盖的情况也要进行池化得到结果，为 False 的时候则不需要保留这部分结果

原理讲解

池化过程和卷积过程相类似，只是卷积操作会把卷积核与输入图像的每一个位置做哈达玛积，而池化操作的池化核中没有具体的值，只是用来限定数据处理的位置，比如 3×3 大小的池化核，每次就会在大小为 3×3 范围的输入内选择一个数值作为池化结果

而最大池化指的就是我们每次在这个 3×3 范围内是按照最大值来选取的，也就是选取范围内的最大值作为这个范围的代表元素，与之类似的还有最小池化（选取最小值作为代表），平均池化（选取平均值作为代表）等

输入和输出还是各自有 4 个维度，其中推导过程和卷积层完全相同

使用池化的目的是，在保证能够保留图像主要信息的前提下，还能尽可能降低图像的尺寸大小，从而加快模型的训练速度（可以类比 1080P 的视频降低到 720P，不影响视频主体信息的获取，但是能使得播放更加流畅）

Non-linear Activations

Non-linear Activations 称为非线性激活函数，非线性激活函数对网络性能的改善至关重要（因为多个线性函数的叠加仍然等价于线性函数，不会从根本上改善网络的性能，必须引入非线性激活函数，才能化腐朽为神奇），在神经网络中，我们常用relu和sigmoid函数作为激活函数

所有的线性函数在实例化的时候接收的参数都相同，只接收一个 inplace 参数，该参数默认为 False，表示不修改原有 input，而是返回一个新的 output 作为结果；如果设为 True，则表示函数本身无返回值，而是把修改重新作用在 input 上

其中 Linear Weight 即为 k 值，来自 -√k 到 √k 的均匀分布；而 Linear bias 即为 b 值，来自 -√k 到 √k 的均匀分布，其中 k 是 in_featrures 的倒数

nn.Linear()

Sequential

在 PyTorch 中，nn.Sequential 是一个顺序容器（sequential container），用于将多个神经网络层（或其他模块）按顺序组合在一起，形成一个完整的模型，它可以简化模型的定义和正向传播（forward pass）的代码，特别适用于简单的线性堆叠结构

损失函数与反向传播

Loss function

Loss function 即为神经网络中的损失函数，为了提高模型性能，不管我们的损失函数具体如何定义，我们都希望它应该越小越好，损失函数的作用如下：

  1. 计算实际输出和目标之间的差距
  2. 为更新输出提供依据（也就是神经网络的反向传播过程）

在 nn 模块中已经为我们准备好了许多损失函数，我们直接调用即可

L1Loss

该损失函数用于计算输出每一个维度的分量和目标差值的绝对值，接收以下两个参数：

  1. input = (N，*)：输入的张量必须给出 batch_size=N，其余维度不做要求
  2. target = (N，*)：目标张量必须给出 batch_size=N，其余维度需要和 input 保持相同

在使用 L1Loss 实例化 l1loss 对象的时候，我们可以传入其对结果聚合的形式：

  1. reduction=“sum”：对各个维度差的绝对值求和
  2. reduction=“mean”：对各个维度差的绝对值求平均值

MSELoss

该损失函数用于计算输出每一个维度的分量和目标差值的平方，接收以下两个参数：

  1. input = (N，*)：输入的张量必须给出 batch_size=N，其余维度不做要求
  2. target = (N，*)：目标张量必须给出 batch_size=N，其余维度需要和 input 保持相同

使用 MSELoss 实例化 mseloss 的时候，默认会对所有差值的平方先求和再除以特征维度的总数，也就是所谓的均方误差

Optim

所有的优化器模型都集成在torch.optim库中，在上文中我们提到，损失函数的第二个作用是通过反向传播为网络模型进行调参，而具体采用什么算法进行参数调整，则需要我们选择合适的优化器来完成，比较常用的优化器有SGD（随机梯度下降优化器）

SGD优化器

我们以随机梯度下降优化器为例，展示使用优化器结合反向传播进行网络调参的大致流程，该优化器通过torch.optim.SGD()方法进行初始化，并且接受以下两个参数：

  1. parameters：网络参数，通过调用网络实例化对象的parameters()方法即可获取
  2. lr：学习率，一般在初始学习阶段，我们都设置为 0.01（lr 设置太大或太小都不好），在 SGD 中我们可以近似认为学习率就是梯度下降的步长

不同的优化器的参数可能不尽相同，但是上述两个参数parameters和lr则是优化器中一般都会设置的两个通用参数

流程演示

在调参的过程中，我们一般都会设置为内外两层循环，其中外层的 epoch 循环表示参数调整的轮数（一共进行多少组完整的参数调整），内层的 data 循环表示在单次调整中，我们要用到整个 data_loader 中的所有数据

对于每一个 epoch，我们都设置一个running_loss表示这一个轮次的训练中，所有数据的loss_function损失值的累加和，对于有效的训练，随着训练轮次的增加，running_loss的值应当递减，在当前训练轮次中：

  1. 每进入一组数据 data 的训练，我们通过optim.zero_grad()方法把上一组数据的梯度清零，防止之前残留的梯度影响本组数据的训练
  2. 接着我们通过交叉熵损失的backward()反向传播方法，完成参数对应梯度的求解
  3. 最后我们通过optim.step()方法，完成网络参数中梯度的更新，并且把当前这一组数据的损失值累加到总体的损失值上
