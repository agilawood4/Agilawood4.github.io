Destiny2025-12-052026-01-17

存储管理与索引

DBMS 两大功能：查询控制（查询处理与优化，查询执行）和存储管理（索引 / 文件管理，缓冲区管理，物理存储管理，事务处理）

DBMS 主要的实现技术：

物理存储系统

存储分级

我们把存储自底向上分为磁盘（Disk），内存（Memory），CPU 三部分

  • 数据只有放入内存才能被处理
  • DBMS 设定数据库的基本存储是在磁盘上，DBMS 管理内存与外存数据的交换

DBMS 存储管理的目标：最小化磁盘和主存之间传输存储块的数量，即最小化磁盘存取次数，实现手段是在主存中保持尽量多的块，使得上层要访问一个块时，它在主存中的概率最大

磁盘

磁盘结构如下图所示：

磁盘块 / 块：一个磁盘块由若干连续的扇区构成，焊死存储分配和检索的逻辑单元，大小一般为 4k~16k，数据以块为单位在磁盘和主存之间传输，页面通常指块

磁盘访问时间：从发出读写请求到数据传输完成

  • 寻道时间：磁盘臂定位
  • 旋转时间：等待被访问的扇区出现在读写头下方
  • 传输时间：从磁盘读取数据或向磁盘存储数据

物理存储管理器

物理存储管理器也称为磁盘管理器，是 DBMS 结构的最底层，隔离上层模块和底层的软、硬件平台

物理存储管理器以页 / 块组织数据，负责数据在磁盘和主存之间的移动，主要操作包括页 / 块的读、写、申请、释放

数据存储结构 / 物理结构

数据库存储结构主要是文件的组织结构，数据库文件的存储介质结构组成如下：

  • 数据库：由若干文件构成（数据库中的一个表对应一个文件）
  • 文件：由若干定长的存储块 / 页构成
  • 块 / 页：存储分配和数据传输的单位，一个块包含若干个记录，每个记录有唯一的标识符 ID（由块号和块中相对位置组成）
  • 记录：由若干数据项组成

文件组织：

  • 一个文件在逻辑上被组织为记录的序列，记录被映射到磁盘块上
  • 文件中的记录有多种存储方式，支持记录在磁盘上的定位

文件的磁盘块分配

数据库中的一个表对应一个文件，一个文件包含若干磁盘块，这些磁盘块的常见分配方式如下：

  • 连续分配：数据块被分配到连续的磁盘块上
  • 链接分配：数据块中包含指向下个数据块的指针
  • 聚簇分配：簇是连续的几个磁盘块，簇之间指针连接
  • 索引分配：索引块中存放指向数据块的指针

块 / 页：数据库中的页 / 块结构常用分槽页结构（slot）

记录：记录是字节序列，DBMS 负责把该序列解释为属性类型和值

  • 记录头部：包含元组的元数据，例如加锁信息
  • 记录数据：属性的实际数据，属性一般按表中的顺序存储，多数 DBMS 不允许一个记录大小超过一个页

文件中记录的组织方式

一个文件由若干磁盘块组成，每个磁盘块又包含若干条记录，因此文件也可看成由记录组成

  • 堆（Heap）：记录可以放在文件空间中的任何位置
    • 链表方式：在文件开始维护一个 Header 页，该页存储了空白页链表头指针和数据页链表头指针，每个页记录了当前包含的空槽数
    • 页目录方式：DBMS 维护特殊页保存在文件中的数据页的位置，并记录每个页中的空槽数
  • 顺序（Sequential）：基于每个记录的搜索码值顺序排列
    • 搜索码：用于在文件中查找记录的任意属性或属性集合
    • 通过指针把记录链接起来，每个记录的指针指向按搜索码排列的下一条记录
    • 可以高效按某个搜索码处理记录
  • 索引（Indexing）：按某种顺序有序存储
    • 索引指的是记录的关键字与相应记录的存储地址的对照表
    • 索引文件由主文件和索引表构成，索引表必须按关键字有序，主文件本身可以按主关键字有序组织（顺序索引文件）或无序组织（索引非顺序文件）
  • 散列（Hashing）：按搜索码上的 hash 函数，计算出记录在文件中存放的块
    • 散列文件的存储单位称为桶（Bucket）
    • 应对哈希冲突：在散列值的个数多于一个桶时，形成一个主桶和多个溢出桶的列表
    • 二次检索：先用哈希函数确定项所在的主桶，再根据链表逐一找到每个溢出桶
  • 聚集（clustering）：将有联系的记录存储在同一个块上，以最小化 I/O 次数
    • 聚集码：聚集码是一种属性，用于定义哪些记录被存储在一起
    • 多表聚集：把多个关系存储在一个文件中，在每个块中存储两个或更多关系的相关记录，可以加快特定的连表查询，但会使单表访问变慢

缓冲区管理

（1）基本概念

缓冲区：是主存中可以存储磁盘块副本的区域，也是以块 / 页作为基本存储单位（块 / 页是存储分配和数据交换的单位）

缓冲区管理器：负责缓冲空间分配，内外存交换

缓冲区管理目标：最小化磁盘和主存之间传输数据块的数量，即最小化磁盘存取次数，实现手段是在主存中保持尽量多的块

（2）工作原理

执行引擎需要操作某磁盘块时，通过资源管理器调用缓冲区管理：

  • 如果该块在缓冲区中，则缓冲区管理器返回该块在内存中的地址
  • 如果不在缓冲区中，则缓冲区管理器为该块在缓冲区中分配空间，将块从磁盘读进缓冲区，并将内存地址返回给调用者

（3）缓冲区的组织与管理

缓冲区被组织为一个固定大小的页面数组，每个元素被称为帧，存放磁盘上的一个块 / 页

缓冲区元数据为页表

缓冲区设置了共享锁和排它锁，用于实现多线程访问

眼存取的替换策略为最近最少使用（LRU）算法

索引

索引的基本概念

索引文件的构成：

  • 索引记录 / 索引项：索引文件中的记录，包括两个域
    • 索引域 / 搜索码：存储数据文件中一个或一组域（属性）
    • 指针：指向索引域值为 K 的记录所在磁盘块的地址
  • 索引表中的部分属性进行组织或排序，使得 DBMS 的执行引擎利用这些属性能够快速有效进行表的访问
  • DBMS 会在执行查询时使用最恰当的索引

索引的分类

（1）两种基本类型

  • 排序索引：索引项是排序的
  • 哈希索引（无序索引）：索引项是乱序的，使用索引域上的哈希函数确定位置

（2）聚集索引和非聚集索引

  • 聚集索引：索引域 / 搜索码值的排列顺序与记录在文件中的排列顺序一致，称为主索引
  • 非聚集索引：索引项排列的次序与文件中记录的排列顺序不同，称为辅助索引

一个表可以有多个辅助索引，但只能有一个主索引

（3）稠密索引和稀疏索引

  • 稠密索引：对于文件中的每个搜索码都有一个索引项
  • 稀疏索引：只有部分索引域 / 搜索码值有索引记录

注意：稠密索引是对于每个搜索码而言的，而不是对关系中的每一条记录而言，下面这个例子也是稠密索引

稀疏索引的例子：

只有在文件记录以索引域排序的时候，才可以用稀疏索引，此时使用稀疏索引查找索引域为 K 的记录，可以现在索引中定位小于 K 的最大索引域值，从该索引域指针指向的记录开始顺序查找

相比稠密索引，稀疏索引占空间小并且维护代价低，但是定位记录慢，非聚集索引一定都是稠密索引（因为文件记录没有按索引域排序，此时稀疏索引会失效）

（4）二叉树索引和多叉树索引（都属于多级索引）

  • 二叉树索引：每个节点只有一个关键字（即搜索码），由两个指针，分别指向关键字值小于或大于当前节点值的节点
  • 多叉树索引：每个节点有 D 个关键字，进而分割出 D+1 个关键字去吃去就按，因此也有 D+1 个指针，每个指针指向相邻两个关键字（左右边界设成 -inf 和 inf）之间的取值空间

为了防止极端情况下，二叉树退化成链表，我们引入基于平衡树的 B+ 树

B树和B+树

（1）B树（平衡树）索引

限制每个节点防止关键字与指针的最小和最大个数：

  • 根节点拥有 [2,n][2,n][2,n] 个子节点
  • 中间节点有 [n/2,n][n/2,n][n/2,n] 个子节点（上取整）
  • 叶节点有 [(n−1)/2,n−1][(n-1)/2,n-1][(n−1)/2,n−1] 个记录指针（上取整）

其中 n 的值相对于特定的树是给定的

（2）B+树

B+树是 B 树的改进，把树中所有关键字都按照递增次序从左到右安排在叶节点上，并且链接起来，B+树能同时进行随机查找和顺序查找

B+树节点结构：每个叶子节点最多包含 n-1 个搜索码 / 索引码 / 关键字值 K1,K2,…,Kn−1K_1,K_2,\dots,K_{n-1}K1​,K2​,…,Kn−1​ 以及 n 个指针 P1,P2…,PnP_1,P_2\dots,P_{n}P1​,P2​…,Pn​

  • KiK_iKi​ 是索引码的值
  • PiP_iPi​ 对于非叶子节点是指向子节点的指针，对于叶子节点是指向记录或记录桶的指针，PnP_nPn​ 指向下一个叶子节点
  • 节点中的各索引码的值满足 K1<K2<⋯<Kn−1K_1<K_2<\dots<K_{n-1}K1​<K2​<⋯<Kn−1​

B+树的查询

从树的根节点开始，通过比较查询码值 vvv 与节点的kik_iki​值，向下遍历树，直到到达包含指定值的叶节点为止，如果在叶子节点中找到 ki=vk_i=vki​=v，则返回目标记录指针 pip_ipi​，否则关系中不存在该值的记录

B+树插入节点

先使用查询算法定位要插入的叶节点 lll，如果还有空间就插入，否则需要拆分

具体的拆分过程为将节点 lll 的 nnn 个码值的前 [n/2][n/2][n/2] 放在 lll 中，剩下值创建一个新节点存储，把新节点插入 lll 的父节点中，此时父节点中也要多加一个码值（因为子节点变多，分割域必然变多），递归向上处理，直到插入不再产生拆分或建立了一个新根节点为止

B+树删除节点

先使用查询算法定位要要删除的叶节点 lll 进行删除，如果 lll 的码值个数低于下限，可能导致节点合并与重新分配

若下溢则先尝试与兄弟结点重新分配（借关键字）并更新父结点分隔键；若兄弟也在下限无法借，则与兄弟合并，并在父结点中删除相应分隔键与指针；父结点若因此下溢，按相同规则递归向上处理；根结点若只剩一个孩子则该孩子上升为新根，树高减少

B+树也可以用于组织文件（上面讲的是用 B+树组织索引），此时叶子节点存的是记录本身而不是记录的指针

Hash 索引

（1）哈希索引的实现

  • 哈希函数：把索引码 KeyKeyKey 映射到较小的 Hash(Key)Hash(Key)Hash(Key)，这个映射函数就叫哈希函数，哈希表中的键是 Hash(Key)Hash(Key)Hash(Key)，值是索引码 KeyKeyKey 对应的具体记录
  • 哈希方案：解决一个哈希值对应多条记录，常用溢出链接法实现

（2）静态哈希和动态哈希

  • 静态哈希：哈希表大小固定
  • 动态哈希：允许哈希表大小动态修改
