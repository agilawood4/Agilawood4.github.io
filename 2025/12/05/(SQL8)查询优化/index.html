查询优化

查询优化分类：

  • 代数优化：也称为逻辑优化，是指对关系代数表达式的优化，即按照一定的规则，对关系代数表达式进行等价变化，改变关系代数操作的次序和组合，使得查询执行更高效
  • 物理优化：也称为非代数优化，是指存取路径和底层操作算法的优化

关系护具库管理系统执行查询的步骤：

  • 查询分析：对语句的检查，词法检查，语法检查
  • 查询检查：对数据库的检查，语义分析，符号名转换，合法性检查，视图转换，安全性和完整性初步检查
  • 查询优化：选择一个高效执行的查询处理策略
  • 查询执行

查询算法

选择操作的实现

（1）全表扫描方法

  • 适用于小表，不适合大表
  • 适用于表中绝大多数数据都满足要求的情况，对最大值 / 最小值类查询不适合

（2）索引扫描方法

  • 前提是选择条件中的属性上有索引（Hash 索引或 B+ 树索引），可以按照索引进行查找，根据索引的选择分为两步，先根据索引得到目标元组的地址，再根据地址得到目标元组本身

连接操作的实现

连接操作是查询中最耗时的操作之一，我们只讨论等值连接（自然连接）实现所用的算法

我们以查询 S(Sno,Snmae)S(Sno,Snmae)S(Sno,Snmae) 和 SC(Sno,Cno)SC(Sno,Cno)SC(Sno,Cno) 中的学生姓名和选课课号的连表查询为例，此时连接的依据是 S.Sno=SC.SnoS.Sno=SC.SnoS.Sno=SC.Sno

（1）嵌套循环算法 O(n2)O(n^2)O(n2)

  • 两层循环，依次遍历 SSS 表和 SCSCSC 表中的每一条记录，比较他们的 SnoSnoSno 项是否相同，若相同，则作为连表后的一条记录输出

（2）排序合并算法 O(nlogn)O(nlogn)O(nlogn)

  • 如果两个表都没有排序，则分别把它们按照连接属性 SnoSnoSno 进行排序，此时二者的 SnoSnoSno 都是递增的，则可以用双指针扫描法，在线性的复杂度内，完成匹配

（3）索引连接算法（O(nlogn)O(nlogn)O(nlogn)）

  • 假设在 SCSCSC 表上已经建立了 SnoSnoSno 的索引，此时我们可以线性遍历 SSS 表中的每一个 SnoSnoSno，根据索引查找 SCSCSC 表中对应 SnoSnoSno 的地址，从而找到对应的元组进行连接
  • 索引建立本身也需要一次排序

（4）Hash  JoinHash\; JoinHashJoin 法（O(n)O(n)O(n)）

  • 把连接属性作为 Hash 码，用同一个 Hash 函数把 SSS 表和 SCSCSC 表中的元组散列到 Hash 表中
  • 划分阶段：对包含元素较少的表做 HashMap 映射，遍历所有元组，填写 HashMap 键为 Hash(Sno)Hash(Sno)Hash(Sno)，值为该 SnoSnoSno 对应的元组
  • 连接阶段：对包含元素较多的表做 HashMap 查找，遍历所有元组，对于每个 SnoSnoSno，使用相同的 Hash 函数，在 HashMap 中查找 Hash(Sno)Hash(Sno)Hash(Sno) 对应的元组，把两个元组连接后输出即可
  • 算法前提：较小的表在划分阶段生成的 HashMap 可以在内存中完全存下（这里把 SnoSnoSno 映射为 Hash(Sno)Hash(Sno)Hash(Sno) 主要是为了减小桶的下标范围）

查询优化

关系数据库管理系统通过某种代价模型计算出各种查询执行策略的执行代价，然后选取代价最小的执行方案

  • 在集中式数据库中，执行代价主要包括：磁盘存取块数（I/O 代价），处理机时间（CPU 代价），查询的内存开销，其中主要的代价是 I/O 代价

  • 在分布式数据库中，执行代价主要包括：

因为 I/O 操作涉及机械动作比较耗时，在计算查询代价时一般用查询处理读写的块数作为衡量单位，查询优化的总目标是算则有效的策略，求得给定的关系表达式的值，使得查询代价最小

查询过程中，开销代价大的是连接操作，为了实现查询优化，我们需要让参与连接运算两侧表中的行数越少越好

具体而言，有选择操作和连接操作时，我们先做选择操作，这样参与运算的表的行数会大大减少，这就是代数优化

代数优化

（1）投影的串接定律

πA1,A2,…,An(πB1,B2,…,Bm(E))≡πA1,A2,…,An(E)\pi_{A_1,A_2,\dots,A_n}(\pi_{B_1,B_2,\dots,B_m}(E))\equiv\pi_{A_1,A_2,\dots,A_n}(E) πA1​,A2​,…,An​​(πB1​,B2​,…,Bm​​(E))≡πA1​,A2​,…,An​​(E)

当且仅当 {A1,A2,…,An}\{A_1,A_2,\dots,A_n\}{A1​,A2​,…,An​} 是 {B1,B2,…,Bm}\{B_1,B_2,\dots,B_m\}{B1​,B2​,…,Bm​} 的子集，使用后者代替前者可以减少一次对表的扫描

（2）选择的串接定律

\delta_{F_1}(\delta_{F_1}(E))\equiv\delta_{F_1\and F_2}(E)

使用后者代替前者可以减少一次对表的扫描

（3）选择与投影的交换律

δF1(πA1,A2,…,An(E))≡πA1,A2,…,An(δF1(E))\delta_{F_1}(\pi_{A_1,A_2,\dots,A_n}(E))\equiv\pi_{A_1,A_2,\dots,A_n}(\delta_{F_1}(E)) δF1​​(πA1​,A2​,…,An​​(E))≡πA1​,A2​,…,An​​(δF1​​(E))

当且仅当条件 F1F_1F1​ 涉及的属性是 {A1,A2,…,An}\{A_1,A_2,\dots,A_n\}{A1​,A2​,…,An​} 的子集，先做选择操作能够优化连接查询

（4）选择与笛卡尔积的交换律

如果 FFF 中涉及的属性都是 E1E_1E1​ 中的属性，则有：

δF(E1×E2)≡δF(E1)×E2\delta_{F}(E_1×E_2)\equiv\delta_{F}(E_1)×E_2 δF​(E1​×E2​)≡δF​(E1​)×E2​

如果 F=F_1\and F_2，且 F1F_1F1​ 只涉及 E1E_1E1​ 中的属性，F2F_2F2​ 只涉及 E2E_2E2​ 中的属性，则有：

δF(E1×E2)≡δF1(E1)×δF2(E2)\delta_{F}(E_1×E_2)\equiv\delta_{F_1}(E_1)×\delta_{F_2}(E_2) δF​(E1​×E2​)≡δF1​​(E1​)×δF2​​(E2​)

如果 F=F_1\and F_2，且 F1F_1F1​ 只涉及 E1E_1E1​ 中的属性，F2F_2F2​ 同时涉及 E1,E2E_1,E_2E1​,E2​ 中的属性，则有：

δF(E1×E2)≡δF2(δF1(E1)×E2)\delta_{F}(E_1×E_2)\equiv\delta_{F_2}(\delta_{F_1}(E_1)×E_2) δF​(E1​×E2​)≡δF2​​(δF1​​(E1​)×E2​)

尽可能使得选择操作在笛卡尔积之前先做，能够优化连接查询

（5）选择运算对并运算，差运算，自然连接运算具有分配律

δF(E1∪E2)≡δF(E1)∪δF(E2)δF(E1−E2)≡δF(E1)−δF(E2)δF(E1⋈E2)≡δF(E1)⋈δF(E2)\delta_F(E_1∪E_2)\equiv\delta_F(E_1)∪\delta_F(E_2)\\ \delta_F(E_1-E_2)\equiv\delta_F(E_1)-\delta_F(E_2)\\ \delta_F(E_1\bowtie E_2)\equiv\delta_F(E_1)\bowtie\delta_F(E_2) δF​(E1​∪E2​)≡δF​(E1​)∪δF​(E2​)δF​(E1​−E2​)≡δF​(E1​)−δF​(E2​)δF​(E1​⋈E2​)≡δF​(E1​)⋈δF​(E2​)

尽可能使得选择运算在并，交，自然连接之前，能够优化相关查询

（6）投影对笛卡尔积的分配律

设 E1,E2E_1,E_2E1​,E2​ 是两个关系表达式，A1,A2,…,AnA_1,A_2,\dots,A_nA1​,A2​,…,An​ 是 E1E_1E1​ 的属性，B1,B2,…,BmB_1,B_2,\dots,B_mB1​,B2​,…,Bm​ 是 E2E_2E2​ 的属性，则：

π1,A2,…,An,B1,B2,…,Bm(E1×E2)≡πA1,A2,…,An(E1)×πB1,B2,…,Bm(E2)\pi_{_1,A_2,\dots,A_n,B_1,B_2,\dots,B_m}(E_1×E_2)\equiv\pi_{A_1,A_2,\dots,A_n}(E_1)×\pi_{B_1,B_2,\dots,B_m}(E_2) π1​,A2​,…,An​,B1​,B2​,…,Bm​​(E1​×E2​)≡πA1​,A2​,…,An​​(E1​)×πB1​,B2​,…,Bm​​(E2​)

同理，投影对并运算也有分配律，先做投影运算，能够优化连接查询
