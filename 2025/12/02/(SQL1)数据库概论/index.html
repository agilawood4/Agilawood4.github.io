数据库概论

数据库系统概述

基本概念

数据：数据是数据库中存储的基本对象

数据库：数据库是长期存储在计算机内，有组织的，可共享的大量数据的集合（数据库的三个特点）

数据库管理系统：数据库管理系统又称为 DataBase Management System，即 DBMS

数据库管理系统所实现的功能：数据定义功能；数据组织、存储和管理；数据操纵功能；数据库的事务管理和运行管理；数据库的建立和维护功能

数据管理技术的产生与发展

数据管理技术经历了人工管理，文件系统，数据库系统三个发展阶段，数据库的发展紧跟计算机的发展历程

（1）人工管理阶段

人工管理阶段数据管理特点：

  • 数据不在计算机上保存
  • 没有软件系统对数据进行管理，由应用程序管理数据的逻辑结构和物理结构，数据和程序不具有独立性
  • 基本没有文件概念，数据组织方式由程序员自行设计
  • 一组数据对应一组数据，数据不共享，面向应用存在

人工管理阶段数据管理缺点：数据的逻辑结构或物理结构发生变化以后，应用程序必须做相应的修改，这会加重程序员的负担

（2）文件系统阶段

文件系统数据管理特点：

  • 数据以文件形式保存在外存上，可以长期保存
  • 文件多样化
  • 数据的存取基本上以记录为单位
  • 数据和程序由一定的独立性

文件系统数据管理缺点：

  • 文件分散，相互独立，数据冗余度大，数据和程序缺乏独立性
  • 文件与应用程序基本上是一一对应，数据共享性差，冗余度大
  • 容易造成数据不一致性
  • 数据维护，程序维护难度大

（3）数据库系统阶段

数据库系统中的核心软件：数据库管理系统 DBMS，在 OS 的支持下工作

数据库技术解决的基本问题：

  • 集成数据的表示
  • 统一管理下的数据共享

解决基本问题的核心技术：

  • 数据模型
    • 数据模型 = 概念模型 + 逻辑模型（层次 / 网状 / 关系模型）+ 物理模型
  • 数据独立性
    • 数据独立性指的是应用程序与数据结构之间相互独立的关系

数据库系统数据管理特点：

  • 面向全组织的复杂的数据结构：在描述数据时，不仅描述数据本身，还描述数据之间的联系，使整个组织的数据结构化
    • 数据结构化是数据库主要特征之一，是数据库与文件系统的根本区别
  • 数据冗余度小，易扩充：数据库从整体观点描述数据。数据不再面向某个应用，而是面向整个系统，从而大大减小数据的冗余度
  • 具有较高的数据和程序的独立性
    • 数据的物理独立性：数据的存储结构（物理结构）改变时，数据的逻辑结构可以不变，从而应用程序也不必改变
    • 数据的逻辑独立性：数据的逻辑结构改变时，应用程序可以不变
  • 统一的数据控制功能
    • 数据的安全性控制
    • 数据的完整性控制
    • 并发控制
    • 数据库恢复
  • 数据的最小存取单位是数据项

数据模型

数据模型（Data Model）也是一种模型，用于抽象和表示现实世界中的数据和信息

数据模型的分类

根据模型应用的不同目的，数据模型可以分为概念模型和逻辑模型、物理模型两大类

【注意】航子的课把逻辑模型叫数据模型，它课件里的数据模型指的是逻辑模型，而不是这里我们顶层的数据模型

（1）概念模型

  • 概念模型也称为信息模型，它是按用户的观点来对数据和信息建模，主要用于数据库设计

  • 概念模型的常用表示方法：实体—联系方法（E-R）法

  • 概念模型中包含的有关概念：实体，属性，码，域，联系

（2）逻辑模型和物理模型

  • 逻辑模型：主要包括层次模型（树型结构），网状模型（图结构），关系模型（二维表），面向对象模型，对象关系模型，它是按计算机系统的观点对数据建模，主要用于 DBMS 的实现
    • 其中网状模型可以最直观表示实体之间的多对多关系
  • 物理模型：对数据最底层的抽象，它描述数据在系统内部的表示方法和存取方法，在磁盘或磁带上的存储方式和存取方法，是面向计算机系统的

数据模型的组成要素

数据模型是严格定义的概念集合，这些概念准确地描述系统中数据的静态特性，动态特性，完整性约束

数据模型的三要素：数据结构，数据操作，完整性约束

（1）数据结构

数据结构描述数据库组成对象以及对象之间的联系，数据结构所描述的对象类型的集合，是对系统静态特性的描述

数据结构是刻画数据模型最重要的方面，通常按照数据结构的类型命名数据类型

（2）数据操作

数据库主要有查询和更新（包括插入，删除，修改）两大类操作，数据操作是对系统动态特性的描述

（3）数据的完整性约束条件

数据的完整性约束条件是完整性规则的集合，完整性是给定的数据模型中数据及其联系所有的制约和依存规则，以确保数据的正确性，相容性（二者统称为数据的完整性）

完整性约束条件包括：

  • 符合这种数据模型所必须遵守的基本的通用的完整性约束条件（实体完整性，参照完整性）
  • 针对具体数据的特定语义约束条件（用户定义完整性）

数据库的系统结构

数据库模式的概念

模式（Schema）是数据库中全体数据的逻辑结构和特征的描述，同一个模式可以有多个实例

关系模式描述的是表的结构（规定了一张表的表头都有哪些列），而关系实例描述的是表的内容（具体往表里填写了什么内容）

数据库的结构

数据库采用的是三级模式两级映像的结构，三级模式指的是：外模式，模式，内模式（物理模式）；两级映像指的是：外模式/模式映像，模式/内模式之间的映像

三级模式

（1）模式（Schema）

模式也称为逻辑模式 / 概念模式，是数据库中全体数据的逻辑结构和特征的描述，是所有用户的公共数据视图

模式是三级模式的核心

它是数据库系统模式结构的中间层，既不涉及数据的物理存储细节和硬件环境，也与具体的应用程序、所使用的应用开发工具，以及高级程序设计语言无关

（2）外模式（External Schema）

外模式也称为子模式（Subschema）/ 用户模式，它是数据库用户（包括写入数据的用户和查询数据的用户）能够看见和使用的局部数据的逻辑结构和特征的描述，是数据库用户的数据视图，是与某一应用程序相关的数据的逻辑表示

  • 外模式通常是模式的子集，一个数据库可以有多个外模式
  • 统一外模式可以为某一用户的多个应用程序所使用，但一个应用程序只能使用一个外模式
  • 外模式是保证数据库安全性的一个有力措施， 每个用户只能看见和访问对应的外模式中的数据，数据库中的其余数据是不可见的

（3）内模式（Internal Schema）

内模式也称为存储模式（Storage Schema），一个数据库只有一个内模式，它是数据物理结构和存储方式的描述，是数据在数据库内部的表示方式

两级映像与数据独立性

数据库的三级模式是对数据的三个抽象级别，它把数据的具体组织留给 DBMS 管理，使得用户能够逻辑地抽象地处理数据，而不必关心数据在计算机中的具体表示方式和存储方式

数据库管理系统在这三级模式之间提供了两层映像：

  • 外模式 / 模式映像
  • 模式 / 内模式映像

两层映像确保数据库系统中的数据能够具有较高的逻辑独立性和物理独立性

（1）外模式 / 模式映像

模式描述的是数据的全局逻辑结构，外模式描述的是数据的局部逻辑结构

当模式改变时，由数据库管理员对各个外模式 / 模式的映象作相应改变，可以使外模式保持不变，应用程序是依赖数据的外模式编写的，从而应用程序不必修改，保证了数据与程序的逻辑独立性，简称数据的逻辑独立性

（2）模式 / 内模式映像

数据库中只有一个模式，也只有一个内模式，所以模式 / 内模式的映像是唯一的，它定义了数据全局逻辑结构和存储结构之间的对应关系

当数据库的存储结构改变时（比如选择了另一种存储结构，比如改变存储在磁盘的位置），由数据库管理员对模式 / 内模式映像作相应改变，可以使模式保持不变，从而应用程序也不必改变，保证了数据与程序的物理独立性，简称数据的物理独立性

三级模式两级映像的结构优点

（1）保证数据的独立性

  • 模式与内模式分开：数据的物理独立性
  • 模式与外模式分开：数据的逻辑独立性

（2）简化用户接口，方便用户使用

  • 用户只按照玩模式操作，无需了解数据库的总体逻辑结构与物理存储结构

（3）有利于数据共享

  • 从模式产生不同的外模式，外模式可以相互覆盖

（4）有利于数据的安全保密

  • 应用程序只能操作其对应的外模式

数据独立性与数据共享

数据的逻辑独立性使得数据库逻辑结构变化后，程序中使用的数据结构不变

  • 应用中使用的数据结构可以与数据库统一的数据逻辑结构不同
  • 数据库统一的逻辑结构可以对应不同应用使用的多个数据结构
  • 数据库可以支持多种特定的数据表示和操作，从而实现数据共享

数据库系统的组成

数据库系统一般由数据库，数据库管理系统（及其开发工具），应用系统和数据库管理员构成
