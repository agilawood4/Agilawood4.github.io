Transformer变种

BERT模型

BERT（Bidirectional Encoder Representations from Transformers）是由谷歌于 2018 年提出的双向 Transformer 编码器模型，在自然语言处理领域取得了显著的成就

该模型的核心结构为两个 Transformer 编码器的堆叠，主要工作流程为：预训练 + 微调两步过程

模型结构

BERT 的主要特点在于其双向性，利用双向语言表示来训练模型，对于传统的 Transformer 架构，对于 iii 位置输出的预测，模型认为其只和 1∼i−11 \sim i-11∼i−1 范围内的输出有关，而 BERT 模型则同时使用 $1 \sim i-1 ;and;i+1 \sim n $ 两部分的输出进行预测

从直观上来讲，BERT 不仅考虑了上文对当前输出的影响，同时考虑了下文对当前输出的影响，兼顾上下文进行输出预测，很大程度上能提高预测的准确性

如上图所示，将 BERT 与另外两种模型对比可知，iii 时刻的预测同时参考了它之前和之后的输出的共同影响，也即：

Embedding词嵌入

Embedding（词嵌入过程）用于把文本中的词语（或字符）转化为向量表示，BERT 中的词嵌入通过预训练得到的现成的词向量矩阵实现，用于捕捉词语间的语义关系

BERT 词嵌入后得到的向量是实数向量（可运算），可以通过计算向量间的距离，反映两个词语义之间的近似程度

BERT 最终得到的词向量由 Token Embeddings，Segment Embeddings，Position Embeddings 这三部分向量相加所组成：

  • Token Embeddings：词向量，起始单词为 CLS，可用于分类任务
  • Segment Embeddings：用于区别不同的句子
  • Position Embeddings：位置向量，通过模型学习得到（区分 Transformer 通过正余弦函数分配位置向量，BERT 拥有较大的学习语料库，模型可以自行学习生成位置向量）

预训练 - Masked语言模型

Masked 的核心目的是使得模型具有双向性，也就是同时捕捉上下文信息的能力，通过随机掩盖一些词语的学习语义关系，可以训练模型更好地理解上下文信息

对于上图而言，在 input 中，黄色位置为被掩盖的信息，当该位置被掩盖以后，其对应的输出就只能通过它的上下文信息进行推断

在具体的训练过程中，我们一般随机选择 15% 的输入 token 进行掩盖（Masked）操作，而不是把每个 token 都掩盖一遍，最终的损失函数的计算只参考被掩盖掉的那些 token 的预测结果

同时，在被 Masked 的那些 token 中，有 10% 的单词会被替换为其他单词，10% 的单词不会被替换（保持原词不变），剩下 80% 的单词会被替换为 Masked 标记

预训练 - 下一句预测

为了满足面向问答和自然语言处理任务的需要，模型需要被训练用于理解两个句子之间的关系，也就是成对输入句子 A 和 B，其中 B 有 50% 的概率作为 A 的上下文出现，模型需要预测 B 是否为 A 的上下文

BERT 在训练时选择的预料是文档级别的，最终预训练以后可以达到 97 ~ 98% 的准确率

Fine-Tuning 精调

有时候为了使得 BERT 模型适用于特定的任务，我们需要在普遍适用的模型基础上，对其中的参数进行细微地调整，使得其可以提高我们所指定的某一类任务下的准确率，这一过程就称为微调（或精调）

微调过程一般只会更新 BERT 模型中的最后一层，其中对于句子级别的分类任务，增加 softmax 层即可：

P=softmax(CWT)P=softmax(CW^T) P=softmax(CWT)

大部分的微调过程都需要在保留已有预训练模型的基础上，为模型增加新的一层即可，因此其训练速度较预训练过程要快得多

模型评价

BERT 模型的优点：效果显著、能力强、通用性强

缺点：计算复杂度高，训练数据量大（需要大量高质量语料作为训练支持），解释性差

T5模型

T5（Transfer Text-to-Text Transformer）是谷歌于 2019 年提出的一种重要的 Transformer 模型的变体，因为名称中含有 5 个 T而得名

在我们所介绍的 Transformer 模型中，BERT 模型主要依赖编码器，而 GPT 系列主要依赖解码器，T5 模型则同时使用了编码器和解码器

基本思想

T5 模型的基本思想在于，把所有的 NLP 任务都转化成 Text-to-Text（文本到文本）的任务

模型结构的选择

T5 考虑了三种比较主流的模型结构，分别是编解码器结构（Encoder-Decoder），语言模型（Language Model），基于前缀的语言模型（Prefix LM）

编解码器结构（Seq-to-Seq 模型）：其中 Encoder 模块可以同时参考前后文信息（主要部分），而 Decoder 模块只能参考前文信息

语言模型（只有 Decoder）：每个 Decoder 模块都只能参考前文信息

前缀语言模型：每一层都可以看成是 Decoder 和 Encoder 的结合体，在同一层内，一部分可以同时参考到前后文信息，一部分只能参考到前文信息

这三部分模型结构的核心区别在于对注意力机制的掩码操作，三个模型对应的注意力机制掩码操作分别如下：

编解码器 - 双向可见

语言模型 - 单向可见

前缀语言模型 - 前缀双向可见（单向可见和双向可见的混合体）

预训练策略

T5 模型在预训练阶段关键步骤的探索流程，主要分为以下四个方面的比较：

（1）自监督的预训练方法，这一部分又有三种处理思路（语言模型，BERT模式，顺序还原），假设我们要处理的一句话是Thank you for inviting me to your party last week，三种思路的处理流程如下：

  • Prefix language modeling：只给出句子的前面几个词，让模型去预测后面待填写的部分
  • BERT-style：先给出完整的句子，然后随机把其中的某些单词进行掩盖（Masked），让模型尝试去还原完整的文本
  • Deshuffling：先给出完整的句子，然后把文本顺序打乱，让模型尝试找到文本正确的顺序

实验结果证明，使用 BERT-style 进行随机掩码预训练得到的模型效果最好，这也是 T5 采用的预训练方法

（2）在选用了 BERT 进行随机掩码预训练流程以后，具体的掩码方式也有三种处理思路（Mask，Repkace Spans，Drop）：

  • Mask：直接把文本中的部分 token 替换为 Mask 标记（我喜欢吃苹果，还喜欢跑步 -> 我喜欢吃【M】果，还喜欢【M】步）
  • Replace Spans：直接把文本中完整的词语替换为 Mask 标记（比如对上例中，苹果或跑步会作为完整的词被整体替换）
  • Drop：不进行替换，而是随机选择部分词，直接把这些词丢弃掉

实验结果证明，使用 Replace Spans（随机小段替换）进行处理的效果最好

（3）在选择 Replace Spans 作为具体掩码方式后，接着要选择进行掩码操作（文本破坏）的比例，T5 模型在这里考虑了四个比例值：10%，15%，25%，50%，最终实验结果证明，选择 15% 比例的模型效果最好，这也是 BERT 所选择的掩码处理的比例

（4）最后还需要确定文本小段替换的长度，T5 模型考虑了 2 3 5 10 这样四个替换长度，实验结果证明，替换长度为 3 的时候，模型效果最好

T5 模型预训练所选用的数据集为 C4（Colossal Clean Crawled Corpus）数据集，共有 750G 的预训练数据，数据进行简单的启发式处理（比如多条重复数据中只保留一条）以后，即可用于模型训练

模型版本

T5 模型在预训练以后发布了 5 个模型，参数量越大的模型对应的效果也会越好（heads - 注意力头，layers - 网络层数，parameters - 参数量）
