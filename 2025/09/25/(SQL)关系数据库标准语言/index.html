关系数据库标准语言 SQL

数据库发展：层次 / 网状 -> 关系数据库

SQL（structured query language）特点：

  • 综合统一（数据操纵，数据定义，数据控制）
  • 高度非过程化
  • 面向集合的操作方式
  • 以同一种语法结构提供两种使用方式
  • 语言简捷，易学易用

SQL基本概念

基本表与导出表：

  • 基本表（Base Table）：是实际存在的，在物理存储中有对应的存储结构和存储文件

  • 导出表（Extracted Table）：是从基本表中导出的表，有视图（View）和快照（Snapshot）

    • 快照是静态的，表示某个时刻数据库的状态

    • 视图是一个虚表，即视图所对应的数据不会实际存储在数据库中，只在数据库的数据字典中存储视图的定义

    • 视图一经定义就可以和基本表一样进行查询等操纵，也可以用来定义新的视图

基本表相当于三级模式中的模式，视图相当于三级模式中的外模式，viewviewview 是 SQL 中定义外模式的一种方式，我们可以操作视图，也可以直接操作基本表：

SQL数据查询功能

查询的基本结构：

  • 查询块的结果仍是一个表
  • 查询块执行的过程是在表的水平方向上按 WHERE 给出的检索条件选取元组（完整的一行就是一个完整的关系），又在垂直方向上按 SELECT 指定的列进行投影
  • 查询块可以进行关系代数中的投影，选取，连接等操作的组合

投影检索

SELECT 不会像投影运算一样自动去重，需要设置DISTINCT手动实现去重，之所以会用到去重，是因为有可能我们从完整的关系中选出某几列之后，这些列相互之间可能就不再具有唯一性，需要做去重处理

选取检索

  • 可以用（）进行优先级控制
  • 如果要控制输出的列是表中的所有列，可以用SELECT *实现，否则我们只需要在SELECT后面跟上要选的列，列之间用逗号分隔即可

排序检索

通过ORDER BY子句接在SELECT FROM WHERE查询块之后，可以把结果按指定列排序

格式：ORDER BY 列名1 [ASC|DESC], 列名2 [ASC|DESC], ...

  • 其中ASC表示升序排序，DESC表示降序排序，默认为升序排序
  • 可以是单列排序或多列排序，待排序的列名可以用逗号分隔
  • 该子句在 SELECT 子句中作为最后一个子句出现

排序的关键字顺序从前往后，优先级顺序依次降低，在上例中，第一排序关键字为SD ASC，第二排序关键字为SA DESC

子查询嵌套检索

WHERE子句中可以包含另一个查询块，该查询块称为子查询或嵌套查询，包含子查询的语句称为外部查询

外部查询利用子查询来获取检索条件的条件值，检索条件根据子查询的结果来确定外部查询的结果数据，也就是说子查询嵌套检索的查询条件不再是单一的比较表达式，而是另一个查询得到的结果

子查询按照与外部查询的联系不同，分为普通子查询和相关子查询：

  • 普通子查询：和外部查询无关，可单独执行得到一组值
  • 相关子查询：把外部查询的列值作为检索条件的条件值（可以类比函数调用时的传参，外部的列值会传入子查询参与比较）

用 EXISTS 检索

EXISTS 表示存在量词 ∃\exists∃，表达式 EXISTS (子查询)，当且仅当子查询结果非空时为真

库函数检索

  • COUNT()COUNT()COUNT()：统计列中不同值的个数，其中 COUNT(∗)COUNT(*)COUNT(∗) 为对行计数，也即统计关系表格中行的数量（每一个元组本身就是互斥的）
  • SUM()SUM()SUM()：对列值求和
  • AVG()AVG()AVG()：对列值求平均
  • MAX()MAX()MAX()：在列值中找最大值
  • MIN()MIN()MIN()：在列值中找最小值

分组检索

按属性列（列组）将关系的元组分组，每组在这些属性列（列组）上具有相同值，对每一组执行 SELECT 操作

WHERE 子句和 HAVING 子句：

  • WHERE：针对行进行操作，用于去掉不符合条件的若干行

视图的作用：
