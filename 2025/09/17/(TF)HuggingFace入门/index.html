HuggingFace入门

通过阅读本文，您可以大致了解 HuggingFaceHuggingFaceHuggingFace 入门有关的基础知识，所有模型 / 数据查看与下载的网址：https://huggingface.co

基础组件

Pipeline

  • 流水线，用于模型推理，封装了完整的推理逻辑，包括数据预处理，模型预测以及后处理

Tokenizer

  • 分词器，用于数据预处理，将原始文本输入转化为模型的输入，包括 input_ids，attention_mask，token_type_ids

Model

  • 模型，用于加载，创建，保存模型，对 Pytorch 中的模型进行了封装，同时更好地支持预训练模型

Datasets

  • 数据集，用于数据集加载与预处理，支持加载在线与本地的数据集，提供了数据集层面的处理方法

Evaluate

  • 评估函数，用于对模型的结果进行评估，支持多种任务的评估函数

Trainer

  • 训练器，用于模型训练，评估，支持丰富的配置选项，快速启动模型的训练流程

Pipeline

Pipeline 称为流水线 / 管道，具体由数据预处理，模型调用，结果后处理三部分所组成，使得我们能够直接输入文本即可获得最终的答案，相当于对一些预训练模型的高度集成封装：

  • 数据预处理：把文本输入 Raw text 经过 Tokenizer 序列化得到由若干个 token id 所组成的 input_ids
  • 模型调用：把 input_ids 作为模型输入，模型经由内部处理逻辑，得到 Logits 输出
  • 结果后处理：对 Logits 进行归一化，得到最终用于预测 / 分类的概率值

Logits：模型最后一层（输出层）的原始输出分数，还没有经过 softmax 归一化处理，即最后一层 Linar 的输出，softmax 的输入

Tokenizer

Tokenizer 用于对数据进行预处理，以文本数据为例，我们的模型并不能直接处理人类世界的文本序列（中文 / 英文），而是需要把他们转化成特定的数字 id 序列，才可以被模型识别，而 Tokenizer 即可实现这样的功能

一般而言，我们的数据预处理可以分为以下几步：

  • Step1 分词：使用分词器对文本数据进行分词（把句子分解成单词）
  • Step2 构建词典：根据数据集分词的结果，构建词典映射（每个词和它对应的 token id 的映射），如果使用预训练的词向量，词典映射则要根据词向量文件进行加载
  • Step3 数据转换：根据构建好的词典，把分词后的数据做映射，把文本序列转化为 token id 序列，具体而言即把每个单词映射为一个对应的数值 id
  • Step4 数据填充与截断：如果我们想以一个 batch 进行批次处理，则需要保证每个批次内加载的 token 序列长度相同，这就需要对过短的数据进行填充，对过长的数据进行截断，保证数据长度处于模型能够接受的范围，同时 batch 内数据维度大小一致

现在这些功能，Tokenizer 都可以帮我们实现！

数据映射通常和上一节提到的 Tokenizer 配合使用，用于对数据集中的每一条数据做序列化，通过调用map()方法，其中传入指定的函数 f，f 的作用即可广播到数据集中的每一条数据上:

更常用的情况是对 dataset 中的每一条数据做序列化，假设我们当前的 DatasetDict 中的 feature 字段为 title 和 content，我们现在希望对 content 和 title 分别做序列化，并且有以下要求：

  • 把 title 序列化得到的 inputs 中的 input_ids 项作为 labels 存入 content 序列化得到的 inputs 中，即建立新的映射
  • 删除掉原数据集中的 title 和 content 列，只保留序列化后得到的新列

这里 map 方法中还支持传入几个参数：

  • batched=true：按批进行处理，可以加快 map 的映射速度
  • num_proc：设置多进程加载数，可以加快 map 的映射速度
  • remove_columns：在完成映射后的数据集中，删除指定列，可以通过传入一个列表，列表中存指定列的名称（features 字段中的名称），也可以传入dataset['train'].dolunm_names表示删除 dataset 中原有的字段
