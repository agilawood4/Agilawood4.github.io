强化学习中的基础概念

强化学习中的基础概念：

  • Environment：环境，智能体所处的外部世界或任务场景
  • Agent：智能体，即学习和做出决策的实体，其职责为观察环境，选择动作，获得奖励，学习改进
  • State：状态，环境在某一时刻的完整描述，包含做出最优决策所需的全部信息
  • Observation：观察，智能体实际能感受到的环境信息（某些情况下，我们可以近似认为 State = Observation）
  • Action：动作，智能体对环境施加的影响
  • Reward：奖励，环境对智能体动作的即时反馈，引导智能体学习好的行为

ActionSpace：可选择的动作空间，如 {left, up, right}

Policy：策略函数，输入当前智能体所处状态 State，输出每个候选的 Action 的概率分布，一般使用 π 表示，如下所示 $$ \pi(left|s_t) = 0.1\\\pi(up|s_t)=0.2\\\pi(right|s_t)=0.7 $$ 表示在当前状态下，智能体选择向左，向右，向上的概率分别是 0.1 0.2 和 0.7，在强化学习中，我们一般不会直接选择概率最高的动作作为下一步决策，而是根据各个动作的概率分布进行采样，即按照它们各自的概率，随机抽取一个动作作为决策，这有助于模型学习到更多可能的决策

Trajectory：轨迹，用 τ 表示，表示一连串状态和动作的序列，也称为 Episode 或 Rollout，通常表示为 {s0, a0, s1, a1, …}，对于给定的当前状态和当前动作，能够得到的下一个状态可能是确定的也可能是未知的，如下所示 $$ s_{t+1}=f(s_t,a_t) \;\;\;确定\\s_{t+1}=P(*|s_t,a_t)\;\;\;随机 $$ 比如对于大模型而言，给定已有输出序列和当前生成的 token，下一步输出的序列就是确定的，而对于某些预测类任务而言，当前状态推出的下一步状态则是随机的

Return：回报，指的是从当前时间点到整个过程结束的 Reward 的累加和，对于每个状态 state，我们选择某个 action 并不是希望这一步动作所获得的 reward 最大，而是从全局考虑，希望从当前状态开始到结束的整个过程中的 Return 值最大

强化学习训练目标的两种等价表述：

  • 训练一个 policy 神经网络 π，在所有状态 s 下，给出相应的 action，各自得到的 return 取期望最大
  • 训练一个 policy 神经网络 π，在所有的 Trajectory 中，各自得到的 return 取期望最大

在这里，我们会用到一个近似推导： $$ E(x)_{x\sim p(x)}=\sum_{x}x*p(x)\sim \frac{1}{n}\sum_{i=1}^n x\;_{x\sim p(x)} $$ 即当 N 足够大时，某个分布的期望值就等于从这个分布中做 N 次随机抽样所得到的抽样值取平均的结果

PPO推导过程

假设我们所训练的 Trajectory 是 τ，其对应的 Return 值为 R(τ)，τ 所服从的分布为 P(θ)，其中 θ 为我们所训练的策略网络中的参数，我们的训练目标为调整参数 θ，最大化 R(τ) 的期望，也即最大化： E(R(τ))t ∼ Pθ(τ) = ∑τR(τ)Pθ(τ) 这里我们能改变的只有策略网络中的参数 θ，无法改变环境给每个状态的 Reward，因此我们考虑使用梯度上升策略，对 θ 求梯度： $$ \nabla E(R(\tau))_{t\sim P_{\theta}(\tau)}=\nabla\sum_{\tau}R(\tau)P_{\theta}(\tau)\\=\sum_{\tau}R(\tau)\nabla P_{\theta}(\tau) $$ 接下来进行等价数学变形： $$ =\sum_{\tau}R(\tau)\nabla P_{\theta}(\tau)\frac{P_{\theta}(\tau)}{P_{\theta}(\tau)}\\ =\sum_{\tau}P_{\theta}(\tau)R(\tau)\frac{\nabla P_{\theta}(\tau)}{P_{\theta}(\tau)} $$ 这样即可再次得到一个期望形式的表达式，我们使用上面提到的近似推导，将期望转化为采样均值： $$ =\frac{1}{N}\sum_{n=1}^NR(\tau^n)\frac{\nabla P_{\theta}(\tau^n)}{P_{\theta}(\tau^n)} $$ 接着逆用对数函数的求导公式，即 $\nabla \log f(x)= \frac{\nabla f(x)}{f(x)}$ 继续变形： $$ =\frac{1}{N}\sum_{n=1}^NR(\tau^n)\nabla \log P_{\theta}(\tau^n) $$ 下面我们考虑对 Pθ(τ) 的表示，这里我们假设，一条轨迹中的每一个状态，都仅和上一个状态与当前采取的动作有关，当前 Trajectory 概率就等于这条轨迹中每一个 state 和在其状态下给出的 action 的连乘，即 $P(\tau^{n})=P(s_1^{n})\prod_{t=1}^{T_n}P(a_t^n|s_t^n)$，代入梯度表达式则为： $$ =\frac{1}{N}\sum_{n=1}^NR(\tau^n)\nabla \log\prod_{t=1}^{T_n}P_{\theta}(a_n^t|s_n^t)\\ =\frac{1}{N}\sum_{n=1}^NR(\tau^n)\sum_{t=1}^{T_n} \nabla\log P_{\theta}(a_n^t|s_n^t)\\ =\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}R(\tau^n) \nabla\log P_{\theta}(a_n^t|s_n^t)\\ $$ 最终得到的表达式即为，对所有可能的 Trajectory 期望的最大梯度，使用这个梯度去乘以学习率，即可更新神经网络中的参数，这就是policy gradient 梯度策略算法

去掉表达式中梯度，我们要求的即为使得该表达式最大的参数 θ 值： $$ =\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}R(\tau^n) \log P_{\theta}(a_n^t|s_n^t)\\ $$ 该表达式的直观意义为：如果某条轨迹所得到的回报为正，那么就增加当前轨迹中，每个状态 state 下采取当前 action 的概率；反之如果某条轨迹所得到的回报为负，那么就减小当前轨迹中，每个状态 state 下采取当前 action 的概率

值得注意的是，在训练阶段，我们给出的每条轨迹中的状态和动作都应该是确定的，其最终产生的回报也是确定的，我们相当于是用这些确定的轨迹，去动态调整网络参数 θ，使得正回报对应的轨迹中，每一步的概率都增加，负汇报对应的轨迹中，每一步的概率的都减少

PPO训练过程

首先我们定义损失函数，即在目标函数前加上负号，来使用优化器最小化这个损失函数： $$ Loss=-\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}R(\tau^n) \nabla\log P_{\theta}(a_n^t|s_n^t)\\ $$ 朴素的训练方式是构建一个神经网络，输入层为当前的 state，输出层为当前状态可能选择的各个动作空间 action 的概率（经过 SoftMax 处理），后续我们让网络进行 n 轮训练，即可得到每个 τn 对应的 Return 值：

这样的训练存在两个问题：

  • 每个动作 action 只会对当前状态 state 之后到结束这段范围内的 reward 所产生影响，而不会影响 state 之前的 reward
  • 每个动作 action 对当前 state 之后状态的影响应该是逐步衰减的，也就是距离当前状态越远的状态，受到的影响也会相应更小

对此，我们把 R(τn) 的表达式进行修改： $$ R(\tau^n) \rightarrow\sum_{t'=t}^{T_n}\gamma^{t'-t}r_{t'}^n = R_t^n $$ 其中 γ 为衰减因子，表示影响呈现指数级别的衰减，为此我们得到的新的目标函数为： $$ \frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}R_t^n\nabla\log P_{\theta}(a_n^t|s_n^t) $$ 此外，我们的模型还存在一个问题，就是在好的环境下，不论我们选择什么动作，有可能得到的回馈都是正向的，而在差的环境下，不论我们选择什么动作，有可能得到的回馈都是负向的，这可能导致我们的模型收敛缓慢，延长训练时间

为此，我们可以给各个动作的返回值统一加上一个 baseline 或者统一减去一个 baseline 常数，增加不同动作的区分度：

因此我们的表达式变形为： $$ \frac{1}{N}\sum_{n=1}^{N}\sum_{t=1}^{T_n}(R_t^n-B(s_n^t))\nabla\log P_{\theta}(a_n^t|s_n^t) $$ 这里的 baseline 也需要通过训练得到

PPO概念补充

Action − Value Function

动作价值函数：在表达式中，Rtn 每次都是一次随机采样，如果只做一次采样会导致方差很大，训练不稳定，要想使得采样逼近期望，则需要无限次采样

我们用 Qθ(s,a) 表示在状态 s 下，做出动作 a 所期望得到的回报，这个函数就被成为动作价值函数

State − Value Function

状态价值函数：我们用 Vθ(s) 表示在状态 s 下，无论做任何动作，所能得到的回报的期望值

Advantage Function

优势函数：我们用 Aθ(s,a) = Qθ(s,a) − Vθ(s,a) − Vθ(s) 表示在当前状态 s 下，做出动作 a，比其他动作能多带来的优势，在数值上表示为动作价值函数和状态价值函数的差值

在原本表达式中，我们的 Rtn − B(snt) 所表示的其实就是优势函数的含义，因此我们的表达式进一步变形为： $$ \frac{1}{N}\sum_{n=1}^{N}\sum_{t=1}^{T_n}A_{\theta}(s_n^t,a_n^t)\nabla\log P_{\theta}(a_n^t|s_n^t) $$ 我们把优势函数的表达式进行代入，同时对 Qθ(st,a) 做近似估计： Qθ(st,a) = rt + γ * Vθ(st + 1) 对于当前状态 s 做出 a 动作，所能期望得到的回报值，就应该等于当前时刻 t 所得到的回报值，加上 t+1 时刻对应的状态价值函数，乘以一个衰减系数 γ

这样我们的优势函数即可写成： Aθ(st,a) = rt + γ * Vθ(st + 1) − Vθ(st) 其中如果我们对于 Vθ(st + 1) 也做和 Qθ 同样的估计，即可得到如下的递推表达式： $$ A_{\theta}^1(s_t,a)=r_t+\gamma*V_{\theta}(s_{t+1})-V_{\theta}(s_t)\\ A_{\theta}^2(s_t,a)=r_t+\gamma*r_{t+1}+\gamma^2*V_{\theta}(s_{t+2})-V_{\theta}(s_t)\\ \cdots $$ 采样的步数越多，方差越大，偏差越小；采样的步数越少，方差越小，偏差越大，在上述函数中，Aθ1, Aθ2, …可以看成是一步采样，两步采样…所得到的结果

由此我们得到 GAE 优势函数，即 GeneralizedAdvantageEstimation，其表达式为： AθGAE(st,a) = (1−λ)(Aθ1+λ*Aθ2+λ2*Aθ3+⋯) 这里我们设置 λ 值为 0.9，这是一个常用的经验值
