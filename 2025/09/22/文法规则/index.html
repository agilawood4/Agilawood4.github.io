文法规则

Destiny2025-09-222025-10-19

基本约定

  • 只有int这一种变量类型
  • 函数的返回值类型只有void和int两种
  • if条件语句内部不能有嵌套的括号，if((case1) || (case2))或者if((case1) && (case2))都是非法的
  • for循环语句中不能定义变量，for(int i=0;...;...)是非法的，循环变量必须定义在循环体外部，但可以在循环体内重新赋值
  • 当前文法不支持自加和自减，i++和i--都是非法的
  • if和for语句中不能有空代码块，但可以只加一个;
  • 除定义语句可以由连续多个,分隔之外，其他语句不能由,连接，a=1, b=2;是非法的

常量的定义

注意：

  • 常量定义后必须赋值，const int a;这种形式不合法，常量数组也同理
  • 定义在全局中的常量数组可以只初始化部分值，其他部分默认为 0

函数定义

函数的定义和调用需要覆盖以下情况：

  • 有返回值int对应return x;和无返回值void对应return;
  • 形参列表中无参数，有一个参数，有两个参数，有多个参数；形参列表中有数组
  • 实参列表中为空，为常数数字，变量，变量表达式，数组名，数组元素值，函数返回值
  • 普通变量形参按值传递，数组变量接收数组的地址，普通常量变量可以作为函数参数，但常量数组不可以作为函数参数

语句块和语句

语句块：

需要考虑空语句块，在main函数中加一对空的{}即可

语句：

语句需要考虑以下几种情况：

  • 单个的赋值语句，左值可以是变量 / 数组元素，右值可以是已经赋值过的变量，常数，变量表达式，函数返回值，常量，注意必须是单个语句，不能用,连接多条语句
  • 单条printf()语句
  • 单个;组成的空语句，比如if (condition){;}；或者由一个表达式（只计算不赋值）加;组成的语句，比如a+b;
  • 条件语句
  • 循环语句
  • break;语句
  • continue;语句
  • return x;或return;语句

表达式

  • 普通表达式 Exp：可替换为加法表达式 AddExp

  • 条件表达式 Cond：可替换为或表达式 LorExp

  • 左值表达式 LVal：可替换为普通变量，常量，一维数组元素

    注意这里的左值分为可修改左值和不可修改左值，其中不可修改左值可以作为常量标识符，如 const int a=5;b=a这里的 a 就是不可修改左值

  • 基本表达式 PrimaryExp：可替换为 (Exp)，左值表达式，数字，注意只有这里引入了括号

  • 一元表达式 UnaryExp：可替换为基本表达式，函数调用，单目运算符+一元表达式

  • 单目运算符 UnaryOp：可替换为加号 +，减号 -，非 !

    单目运算符可以重复使用，如a=+-+-1是合法的

  • 乘除表达式 MulExp：可替换为一元表达式，乘除表达式 * 一元表达式，乘除表达式 / 一元表达式，乘除表达式 % 一元表达式

  • 加减表达式 AddExp：可替换为乘除表达式，加减表达式+乘除表达式，加减表达式-乘除表达式

  • 关系表达式 RelExp：可替换为加减表达式，关系表达式 > 加减表达式，关系表达式 >= 加减表达式，关系表达式 < 加减表达式，关系表达式 <= 加减表达式

    如果我们连续替换，可能会出现这种a>b>c的情况，这种情况是合法的，语义层面会被解释成a>b的布尔值再和c比较

  • 相等表达式 EqExp：可替换为关系表达式，相等表达式 == 关系表达式，相等表达式 != 关系表达式

    这里可能会出现形如a==b>c<d!=e>5这种复杂的格式，其中 > < >= <= 的优先级高于 == !=，因此相当于计算a==((b>c)<d)!=(e>5)，相同优先级从左到右计算，最后会全部转化为布尔值0和1的比较

  • 与表达式 LAndExp：可替换为相等表达式，与表达式 && 相等表达式

  • 或表达式 LorExp：可替换为与表达式，或表达式 || 与表达式

  • 常量表达式 ConstExp：可替换为加法表达式，但其中的符号必须都是常量标识符

    常量表达式用于：定义数组大小，为数组赋初值，为变量 / 常量赋初值

    常量表达式可以由 数字常量、常量标识符，以及它们通过加减乘除等运算组合而成的表达式

具体来讲推导链为：条件表达式 -> 或 -> 与 -> 相等 -> 比较 -> 加减 -> 乘除 -> 一元 -> 基本 -> (加减) -> (乘除) -> (一元) -> (基本) -> ((加减))…

注意：if条件语句中如果出现嵌套括号，括号中的内容必须是AddExp且必须只能有AddExp，以下是一些常见的合法性判断

左值和右值

左值（lvalue）的定义：能够表示对象在内存中的位置（有地址），可以出现在赋值号左边（注意不是通常理解上的等号左边的就是左值，后者只是前者的子集）

右值：一般表示临时值或字面量，没有独立地址，不能单独被赋值

左值又分为可修改左值和不可修改左值：

  • 可修改左值：普通变量，数组变量，如int x; x=10;
  • 不可修改左值：const修饰的对象，const int a=5; int b=a;这里a就是不可修改左值，它有存储空间和地址，但是不能被赋值

条件语句

条件语句需要考虑以下几种情况：

  • 只有if没有else；同时有if和else，注意else if是非法的
  • if()括号中的内容为Cond条件表达式，可以按照表达式规则进行替换
  • 需要考虑短路求值的原则，比如if(1||...)或者if(0&&...)的情况
  • if()中的条件可以为常数，变量，数组元素，表达式，函数返回值

字符串常量

我们目前支持的字符串为以下几种：

  • %d
  • ASCII为 32,33,40-126 的字符
  • \n

static关键字

static 只修饰静态局部变量，不会在全局变量定义中出现
