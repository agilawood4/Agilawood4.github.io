注意力机制

生物学中的注意力

深度学习中的注意力机制借鉴了人类生物学上的注意力机制（视觉注意力机制），人类视觉通过快速扫描全局图像，寻找重点关注目标区域，也就是注意力焦点，进而对这一区域投入更多的注意力资源，以期获取有关注意力目标的更多信息，同时抑制其他无关信息

这一过程帮助人类利用有限的注意力资源，从大量目标中快速抽取出高价值信息

深度学习中注意力机制的核心目标：从众多信息中选择出对当前任务目标更关键的信息

编解码器结构

一般来讲，大多数注意力机制都附着在 Encoder - Decoder 框架下，但是需要注意，注意力机制是一种思想，其本身并不依赖于任何框架，只是用 Encoder - Decoder 来实现会比较方便，效果也相对较好

这种框架本质上即为两个串联的 RNN 结构，能够实现从序列到序列的变换，但是这种结构存在一个致命问题：即信息瓶颈问题，编码器和解码器之间的联系只有一个固定长度的隐层向量 C，随着历史序列长度的增加，向量 C 无法很好地变表示全局的语义信息

由此我们引入注意力机制，考虑上下文向量 C 应该能访问输入序列的所有部分，而不仅仅是最后一个，因此我们需要为每个时刻设置不同的语言编码向量 Ci，表示不同的关注区域

编解码器中的注意力机制

下面我们将给出从 Decoder 到 Encoder，由输入序列得到输出序列的过程：

（1）在编码器阶段（下半部分），输入序列 x1, x2, …, xn 依次进入 RNN 网络，每个时间步得到一个隐状态： h1, h2, …, hn 这组隐状态 {hj}j − 1T 即为输入序列的表示，保存了输入的上下文信息

（2）在解码器阶段（上半部分），解码器要生成输出序列 y1, y2, …, yn，在生成第 i 个输出时，会先计算打分函数 eij： eij = attentionnet(yi − 1,hj) 解码器会取上一个时刻的输出 yi − 1，把它和编码器的第 j 个隐状态 hj 放在一起，通过一个注意力网络（可能是前馈网络，点积等）计算得到一个分数，这个分数 eij 表示：当前解码器要生成第 i 个词时，输入位置 j 对这个词的重要程度

（3）接着对所有的 eij 进行 Softmax 归一化得到注意力权重 α： $$ a_{ij} = \frac{\exp(e_{ij})}{\sum_{k=1}^{T_x}\exp(e_{ik})} $$ 也就是把 i 位置的输出和所有 j 位置的输入所对应的隐层之间的打分 ei1, ei2, …, eiTx 做 Softmax 归一化，得到一组权重系数，其中 aij 表示生成第 i 个输出时，应该给 j 位置的输入多大的注意力权重

（4）接着计算上下文向量 zi，用刚才得到的权重 aij 对编码器隐层状态 hj 加权求和，得到的 zi 是一个动态摘要，它主要包含与第 i 个输出相关的输入信息，相对于传统 RNN 的中间向量 c，前者能够更好表示历史序列中的相关信息

（5）最后解码器生成 yi，把上下文向量 zi 和编码器得到的隐状态 hi 一起送入解码器的 RNN，得到新的隐层状态并且最终输出 yi，这个 yi 也会作用在下一步中 yi + 1 的求解中

注意力的可视化

下图是对注意力权重矩阵（也就是上面我们求解的 aij）的直观表示，直观来讲，每个黑白色块表示的就是其横轴和纵轴所对应单词之间的关联度，也就是所谓的注意力权重大小，颜色越亮表示权重越高，对应单词之间的关联度也就越大

注意力本质上就是在学习源语言和目标语言之间的对齐关系（aligment），其中注意力矩阵中的 aij 表示：解码器在生成第 i 个词时，对编码器第 j 个词的注意力权重，下图中每一行对应目标语言中的一个词，每一列对应源语言中的一个词

图右下角红框显示，英语 “the man said” 对应了法语的 “a dit l’homme”，注意力分布显示出这种跨词对齐关系

注意力的计算

注意力就是衡量编码器各个隐状态与前一时刻解码器输出对齐的分数，可以用小型神经网络进行计算，也可以有其他各种各样的计算方式：

自注意力机制

Self-Attention：序列自身的注意力 Self − Attentionnet(x,x)，该注意力机制用于寻找一个序列本身内部的关系，也就是序列中词之间的相似关联程度

注意力机制的优点：

  • 解决了编码器到解码器之间信息传递的瓶颈问题
  • 建立了编码器状态和解码器间的直接联系，消除了梯度消失的问题
  • 提供了更好的可解释性

transformer：编码器到解码器两个序列之间的一个转换器，注意力机制在某种程度上就是 transformer（前者强调功能，后者强调结构，但二者本质上是一个东西）

键值对注意力和多头注意力

传统注意力机制工作流程

传统的注意力机制如下图所示：

  • t − 1 时刻隐层的输出 yt − 1 会和每个输入的 xj 所得到的隐层信息 hj 结合，计算关联度得分 eij
  • 每个关联度得分经过 Softmax 归一化后得到一组概率值 aij，进而 每个 aij 和对应的隐层 hj 做加权求和得到上下文信息 zi，作为 t 时刻的上下文输入
  • 由 t − 1 时刻1隐层输出和当前上下文信息，共同得到 t 时刻对应的隐层输出，完成一轮计算

经典注意力机制存在如下的缺陷：

  • 注意力分数 eij 会仅基于每个隐层 hj 在序列中的相对位置而不是他们的具体内容，限制了模型关注相关信息的能力

键值对注意力

为解决传统注意力机制的缺陷，我们借助数据库中有关键值对（Key-Value Pair）的概念，引入了键值对注意力模型，使得模型能够学习到输入和输出之间更加复杂、更有意义的对齐（Alignment）

其中红色的 query 模块地位相当于传统注意力机制中的 yt − 1，只不过这里其定义更加灵活，可以来自解码器上一时刻的输出，也可以来自编码器层（自注意力机制）

这里我们不再使用隐藏层 hj，而是通过对 input 做其它线性变换，从每个输入中抽取出一对信息，也就是 key 和 value 信息，接着我们保留 value 信息，只把 key 信息和当前的 query 拿去计算注意力分数，得到一组 eij，后面的处理和传统注意力机制相同，这样我们实现了相似性和具体内容的分离

键值对注意力分数的计算

我们需要把键（Key）、值（Value），查询（Query）分别映射到向量空间中，得到对应的矩阵 Q，K，V，具体的映射关系为： Q = XWQ, K = XWK, V = XWV. 其中 X 是编码器输入，W 是参数矩阵，其具体参数值通过模型训练和学习得到，有了三个矩阵以后，我们即可计算注意力得分： $$ Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})V $$ 其中分子可以看成是 Q 和 K 矩阵相似度的计算，分母上 dk 是模型转化出的向量维度，二者相除相当于归一化，整体进行 softmax 以后和 V 矩阵相乘，即可得到注意力得分

显然当 key 和 value 生成的向量相同时，该模型就退化成了传统的注意力机制，换言之，整个模型相较于传统模型在结构上其实没有大的变化，只是通过把键和值进行分离，为处理带来更多的灵活性

多头注意力机制

多头注意力机制（Multi-Head Attention）是对键值对注意力机制的改进，相当于在其基础上，引入多个查询向量 Query，分别从不同角度提取 input 向量中的信息

普通的 Scaled Dot-Product Attention 在计算时，只有一组 Q, K, V，这使得模型只能在一个角度下去关注输入序列中的相关性，而用户需求 / 语言语义往往是多层次的，因此我们需要多头注意力机制，让模型用多种不同的投影方式去平行地坎同一个序列，从而捕捉更加丰富的依赖信息

首先我们还是需要对输入向量（如编码器输出序列）做三组线性变换，得到 Q, K, V，每一个头都会有自己的投影矩阵 WiQ, WiK, WiV，这其实也好理解，因为要提取不同特征，对应的参数矩阵必然不同

多头并行计算注意力，每一个头都要单独计算一次 Scled Dot-Product Attention，也就是 Attention(QWiQ,KWiK,VWiV)，得到多个 head 的输出 {head1, head2, …, headh}

接着把所有的 head 结果在最后一维拼接起来，拼接的结果最后通过一个投影矩阵 WO，得到最终的输出，也就是： $$ MultiHead(Q,K,V)=Concat(head_1,head_2,\dots,head_h)W^O\\ $$ 其核心思想是并行多视角关注，不同的投影矩阵让每个头从不同角度看待序列，多头输出拼接再投影，提升模型捕捉复杂依赖关系的能力

自注意力分数

我们以序列中连续的两个单词 think 和 machine 为例，现在 think 要计算其和序列中全部单词（包括它自己）的注意力分数，此时的注意力分数用向量点积结果进行量化：

我们用 think（待考察词向量）的 query 向量和输入序列中全部的词向量的 key 向量分别计算点积，也就是上图中，假设序列中一共只有两个词，则我们需要分别计算： $$ query_{think} * key_{thinking}\\ query_{think} * key_{machine} $$ 二者做点积得到的结果分别是 112 和 96，接着我们要进行归一化操作，也就是把向量点积的结果除以 key 键向量维度的平方根，也就是 $\sqrt{d_k}$，接着使用 Softmax 函数进行归一化处理，使得梯度更加稳定

这里得到的 Softmax 后的结果就是词 think 和序列中其他词的注意力分数，也就是反应了当前词和序列中其他词的相似关联程度

接着我们进行注意力加权求和，也就是把上一步 Softmax 后得到的注意力分数和序列中每个单词的 value 向量相乘后进行加权求和： z1 = α11 * value1 + α12 * value2 其中 z1 是 think 这个单词得到的上下文向量输出，α 值是单词之间的注意力分数，最终 z 值会发送回前馈神经网络

自注意力矩阵的运算

在实际求解中，为了提高计算效率，我们不会对序列中的每个词分别进行计算，而是会以矩阵的形式，一次性完成整个序列中每个词对应的 z 值输出的计算：

具体来讲，我们会取出序列中每一个词，拿它的 query 去和序列中包括它在内的所有词的 key 进行相乘，先得到注意力分数矩阵 Aij，其中 aij 表示 i 词和 j 词之间的相似关联程度，接着再由 A 矩阵和各个词的 value 向量相乘，最终得到 Z 矩阵，其中的每一个行向量 zi 表示第 i 个词由自注意力机制，在原序列中得到的上下文相关信息

而对于我们开篇提出的那个问题，显然求完注意力矩阵以后，ait ∼ animal 的分数值应该是最大的

Transformer模型

模型结构

Transformer 的模型核心由编码器和解码器所构成，编码器将输入数据编码成为一系列向量，解码器则使用这些向量生成输出序列，二者之间通过注意力机制进行交互，使得模型能够通过输入序列生成输出序列

该模型最早应用于机器翻译领域，实现从一种语言的输入到另一种语言的输出的转换

从宏观上来看，模型由两个编码器和六个解码器分别串联而成

所有编码器的结构都相同，但是权重参数并不共享，每个编码器由两个子层所组成：

  • Self-Attention 层（自注意力机制）：让序列中每个词都能和序列中其他词建立相似关联度
  • Feed Forward 层（前馈神经网络）：对每个位置的向量独立进行非线性变换，相当于在注意力之后再增加一层映射，增强表示能力，通常是两层全连接网络，中间带有一层激活函数（RELU / GELU）

多个编码器结构进行堆叠，每一层的输出会传递给下一层，而每个解码器则由三个子层所组成：

  • Self-Attention 层（掩码自注意力机制）：跟编码器的 Self-Attention 类似，但会对未来词进行遮盖（mask），保证在预测第 t 个词时只能看到前面 t-1 个词
  • Encoder-Decoder Attention 层（交互注意力）：让解码器在生成目标序列时能够对编码器输出的上下文信息进行关注
  • Feed Forward 层（前馈神经网络）：和编码器一样，对每个位置的表示做非线性变换，提升表达能力

解码器同样是堆叠多层这样的结构块，逐步生成目标序列

编码器结构

在自然语言处理中，每个词在处理前都要嵌入成为一个词向量，一般设置转化后的向量维度为 512 维，这里我们假定句子中每一个词转化为向量的最大表示维度不会超过 512 维

每个词的信息在编码器中的流动路径都是相互独立的，词语之间的关系在自注意力层中完成计算，而前馈网络层中没有交互，因此可以并行计算，最后我们得到的编码器结构如下：

也就是说我们会有多套参数矩阵 WQ, WK, WV 进而生成多组 Q, K, V，从不同的角度进行相似度关联计算，得到若干组不同的 zk 矩阵，再最后经过映射矩阵 WO 统一映射为上下文信息输出矩阵 z

这里的多头机制类似于 CNN 中不同卷积核的设置，能够捕获输入中不同的注意力权重，更加全面地提取信息并且进行相似度计算

位置嵌入

为了让模型理解单词在句子中的顺序，我们需要增加位置编码向量，其中单词最终的向量化结果由 Positional Encoding 和 Embedding 两部分生成的向量相加得到：

  • Positional Encoding：使用word2vector或者glove等现成模型即可实现
  • Embedding：需要根据每个单词在输入序列中的位置 position，按照下面的公式分别进行正余弦编码

$$ PE(pos,2i)=\sin(\frac{pos}{10000^{2i/d}})\\ PE(pos,2i+1)=\cos(\frac{pos}{10000^{2i/d}}) $$

其中 pos 表示单词在序列中的位置，d 表示词嵌入向量的维度（通常设置为 512 维），其中奇数位置用余弦转换，偶数位置用正弦转换

因为 Transformer 并不适用 RNN 结构，而是直接参考输入全局信息，因此其无法获取单词本身的顺序信息，因此需要额外加入位置编码，来保存单词在序列中原始的相对结构

残差与层归一化结构

在每个编码器和解码器的子层中都使用了残差连接，具体来讲，对于 Self-Attention 自注意力机制层，以及 Feed Forward 前向传播层中，我们都加入了残差连接的机制，这样有助于网络学习更加复杂的特征，避免梯度消失和爆炸，同时使得训练更加稳定

具体来讲，残差连接通过将模型的输入（input）和输出（output）直接相加，有效加快模型收敛速度

此外，我们在每一个编码器的输出前，都设置了层归一化操作，也就是（Normalization 层），这样的设计同样有利于模型收敛，加快模型的训练速度

解码器结构

解码器在编码器结构的基础上，添加了和编码器输出的交互层（也就是解码器的中间层），具体来讲，每个解码器都需要考虑编码器的输入作为自己的输入，进而用以输出序列中下一个单词的表示，通过二者的连接，模型可以通过编码器对输入序列的理解，从而生成更加准确的输出序列

同时连接编码器和解码器还可以避免信息丢失的问题，提高模型整体的稳定性

全局架构

输入阶段（input）：

  • 原始输入序列经过 Embedding 和 PositionEmbedding 最终得到每个单词的表示（词向量 + 位置向量）

编码器（Encoder）：

  • 输入序列经过多个 Encoder 层的堆叠，每一层包括 Self − Attention 和 FeedForward
  • 自注意力层：学习输入序列中单词之间的关系
  • 前向传播：非线性映射，增强表达能力

最终编码器输出的是整个序列的上下文表示，即对每个单词都融合了全局语义的向量

解码器（Decoder）：

  • 解码器在生成目标序列时，按时间步逐词生成
  • MaskedSelf − Attention：解码器的自注意力层只允许看到已生成的词，屏蔽掉未来的词（避免偷看答案）
  • Encoder − DecoderAttention：解码器利用注意力机制，把编码器输出的上下文 (K,V) 和当前解码器自注意力机制生成的隐状态 h 结合，进一步得到最终的隐状态 h′
  • FeedForward：对结果进行非线性映射
  • Linear + Softmax：最后通过线性变换+Softmax 得到一个概率分布，预测最可能的下一个词

交叉注意力

我们说解码器（Decoder）有两个注意力子层：

  • MaskedSelf − Attention：关注目标序列已经生成的部分
  • Encoder − DecoderAttention：关注整个输入原序列

交叉注意力就是第二个子层，它负责把编码器全局信息 (K,V) 和解码器当前步查询 Q 相结合：

  • (K,V)：来自编码器输出 K = HencWK, V = HencWV，其中 Henc 是编码器最后一层输出，投影矩阵为可学习参数，整段解码过程中所看到的 (K,V) 都是输入序列全句映射得到的结果，形状为 (S,dmodel)
  • Q：来自当前解码步的隐状态 Qt = htdecWQ，其中 htdec 表示目标序列第 t 步在 Decoder 中的表示（经过解码器的自注意力机制得到），每个解码步都会得到不同的 Qt

换言之：编码器只有一套自注意力机制，而解码器有一套（Masked）自注意力机制+一套交叉注意力机制，值得注意的是，解码器预测单词时，不是把自注意力和交叉注意力结果直接拼接或加和，而是一个串联的过程：

  • 首先编码层做自注意力机制，上一层解码器层（多个解码器串联而成，如果为第一个解码器，则使用目标序列 Embedding+Position Embedding 得到的结果）的最终输出 X(t−1) 作为自注意力机制的输入，通过投影得到 Q, K, V，解码器内部处理得到隐层输出 ht(1)
  • 这个 ht(1) 再作为交叉注意力机制中的 Query，和由整个编码器处理原序列得到的结合了全局序列信息的上下文输出 z1, z2, …, zn 映射得到的 (K,V) 做交叉注意力机制，得到处理后的隐层输出 ht(2)
  • 这个隐层输出 ht′ 接着经过 Linear + Softmax 层处理，作为当前解码器层的最终输出

在所有自注意力机制中，Q, K, V 都来自于同一段序列 X 的映射，只不过在编码器中， X 某个位置的词得到的 Q 向量能够看到全局的 K, V，但是在解码器中，X 某个位置的词得到的 Q 向量只能看到它时刻之前的 K, V，对于层间串联的编码器 / 解码器而言，这里的序列 X 指的都是上一级编码器/ 解码器的输出，而对于第一级编码器 / 解码器而言，则为原始序列 S 经过嵌入得到的词向量

总而言之，注意力机制的核心就是 Q, K, V，只有有了这三者，我们就能够进行相似关联度计算，这部分打分机制是相同的，我们需要控制的就是 Q, K, V 的不同来源，最后需要区分的就是第一层编码器和第一层解码器的原始输入，分别是原序列和目标序列进行 word_embedding 和 pos_embedding 得到的结果

线性层和softmax层

编码器在输出序列的每个位置的原始输出是一个浮点数值，通过线性层和 softmax 层，我们可以得到最终该位置预测的单词

首先经过线性层（通常为 LinearLayer）：其输入为来自解码器最后一层的隐藏向量（通常维度为 dmodel，512维），接着通过一个权重矩阵把隐藏向量投影到词表大小 V 的维度上，得到 logits 向量，注意：这里解码器最后一层输出的向量只代表序列中的一个词，而不是整个序列

接着通过 Softmax 层，把 logits 向量映射为概率分布，其中每个概率值就代表词表中对应位置的词应当填写在当前位置的概率得分，我们选择得分最高的词填写在当前位置即可

模型评价

Transformer 模型特点：

  • 与 RNN 相比更容易并行化训练，考虑序列的全局信息，本身无单词顺序，需要加入位置嵌入来表示单词位置信息
  • 重点是自注意力结构（多头注意力）和 QKV 矩阵的构建，多头注意力包含多个自注意力模块

随着 2017 年 Transformer 模型被首次提出，深度学习网络模型中的参数量呈现指数级别的上升，谷歌于 2018 年提出的 Switch Transformer 模型的参数量已经达到了惊人的 1.6T：
