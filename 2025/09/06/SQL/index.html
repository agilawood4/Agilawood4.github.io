数据库概述

数据库（DataBase - DB）：存储数据的仓库，在数据库中，数据有组织的进行存储

数据库管理系统（DataBase Management System - DBMS）：操纵和管理数据库的大型软件

SQL（Structured Query Language - SQL）：操作关系型数据库的编程语言，定义了一套操作关系型数据库的统一标准

启动数据库：以管理员身份运行cmd，键入net start mysql80，数据库默认开机自行启动

关闭数据库：以管理员身份运行cmd，键入net stop mysql80

MySQL 客户端链接：

  • 把 MySQL 的 bin 文件添加到环境路径中以后，在 Windows 命令行直接键入mysql -u root -p，然后输入密码
  • 直接在 MySQL 自带的命令行（MySQL 8.0 Command Line Client）中输入个人设置的数据库密码

能显示 MySQL 版本则说明客户端链接成功

关系型数据库（RDBMS）：

概念：建立在关系模型基础上，由多张相互连接的二维表所组成的数据库

特点：

  • 使用表结构存储数据，格式统一，便于维护
  • 使用 SQL 语言操作，标准统一，使用方便

SQL通用语法

SQL 通用语法：

  • SQL 语句可以单行或多行书写，以分号进行结尾
  • SQL 语句可以使用空格 / 缩进来增强语句的可读性
  • MySQL 数据库的 SQL 语句不区分大小写，关键字建议使用大写来确保可读性
  • 注释：单行注释使用--或者#，多行注释使用/**/

数据类型及案例

SQL 中支持的常见数据类型分为三种：数值类型，字符串类型，日期时间类型

数值类型：

其中最后一类 DECIMAL 分为精度和标度，比如 123.45 这个数字，精度就是总长度 5，标度就是小数点后的长度 2

注意 DOUBLE 类型的用法，在定义时需要传入两个参数，分别表示浮点数的最大长度，浮点数支持的小数点位数（比如我们定义精确到小数点后一位的百分制分数，最大数值为 100.0，因此可以使用 DOUBLE(4, 1) 来进行定义）

字符串类型：

其中常用的是 CHAR 类型和 VARCHAR 类型，二者在使用的时候需要传入一个字节数 len，这个类型所能存储的字节数就不能超过 len：

  • 对于 CHAR 类型而言，不论其实际存储的数据长度为多少，申请的空间都为 len
  • 对于 VARCHAR 而言，会动态申请其实际分配的空间

一般来讲，VARCHAR 的性能不如 CHAR，因为前者需要动态计算实际分配的空间

时间日期类型：

其中 YYYY-MM-DD 表示年月日，HH-MM-SS 表示时分秒
