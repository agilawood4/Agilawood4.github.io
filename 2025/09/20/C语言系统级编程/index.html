对象

C语言中对象的概念

C语言中的对象（object）：一个在内存中拥有特定大小的、连续的、可标识的数据存储区域，其内容可以用来表示一个或多个值，该定义有两个关键点

  • 存储区域（Region of Storage）：这段区域是一个具体的、连续的控件，控件大小由对象类型所决定
  • 值的表示（Representation of Values）：存储在该区域内的二进制数据（字节序列）被解释为特定的值，这种解释方式取决于对象的类型

对象（object）是 C 语言中最核心的概念，没有之一

对象类型

一个对象一般有一个对象类型（Object Type），形式化定义为 T（某些对象可能没有对象类型），根据程序员是否可知对象大小，我们将对象分为完全对象类型和不完全对象类型：

  • 完全对象类型：程序员可以获知其大小
  • 不完全对象类型：程序员无法获知其大小

（注意：对象分配后大小就是确定的，不完全对象类型也有一个确定的大小，只是这个值程序员不可知）

对象值和对象表示

一个对象的各个 bit 所组成的二进制串就是这个对象的对象表示（Object Representation），而这个对象中所存储的内容则为对象值（Object Value）

对象表示和对象值通过对象类型进行映射

如果有两个类型相同的对象：

  • 如果对象表示一样，则对象值一定一样
  • 如果对象值一样，则对象表示不一定一样

这是因为组成对象表示的 bit 中，包括用于表示实际内容的 value bits 和没有意义的 padding bits（比如用于实现结构体对齐的占位符），一个对象可能包含若干个无意义的 padding bits

（注意：value bits 和 padding bits 的构成也是由对象类型决定的）

对象的size

对于任何一个对象而言，如果它包含的连续字节的个数为 n (n >= 1)，那么这个对象的 size 值即为 n，这个对象所包含的比特数量就是 n×CHAR_BIT 个（注意这里答案不是 8n）

一个对象类型可以形式化定义成为一个 T，对于对象类型我们有两个函数：sizeof(T)sizeof(T)sizeof(T) 和 alignof(T)alignof(T)alignof(T)：

  • sizeof(T)sizeof(T)sizeof(T)：规定了一个 T 类型的对象其大小是多少，如果 T 是不完全对象类型，则无法用 sizeofsizeofsizeof 求大小，也就没有对应的 sizesizesize 值
  • alignof(T)alignof(T)alignof(T)：规定一个 T 类型的对象的缺省对齐要求是多少，这意味着对象的对齐要求可能可变

类型本身没有大小和对齐要求，只有对象才有大小和对齐要求，因此 sizeof(T)sizeof(T)sizeof(T) 和 alignof(T)alignof(T)alignof(T) 本质上是在求 T 类型的对象的大小和缺省对齐要求

对象地址和对象的对齐要求

对象地址（Address）：对象所占用的连续字节中 lowest 字节的编号

对齐要求（Alignment）：要求对象的地址能够被对齐要求整除，比如 int 类型对象的地址必须要能被 4 整除

类型

类型（Type）是用来理解一个对象或函数的重要依据，这里我们将 Type 做以下三种分类：

  • 算术类型（Arithmetic Type）
  • 派生类型（Derived Type）
  • 限定类型（Qualified Type）

算术类型

算术类型都是完全对象类型，除枚举类型外，其他算术类型都是 C 语言的原生 / 基础数据类型（Build-in）

C 语言的基础类型中不包含枚举类型

Characyer Related

char 类型，signed char 和 unsigned char 是三种不同的类型，编译器会指定 char 类型和 signed char 或者 unsigned char 中的一种行为一致

有符号整数类型

对于任何一个标准的有符号整数类型 T 的对象而言，所有的 bits 分成三部分：1 个 sign bit，若干个 value bits，若干个 padding bits

假设 sign bit 加上 value bits 的个数为 N，那么 N 就是这个对象类型的宽度（Width），而 N-1 就是这个类型 T 的精度（Precision）

编译器对于有符号整数类型的宽度定义中，其中 signed char 的宽度一定要等于 一个 CHAR_BIT，而其他四种类型我们只规定宽度的最小值，编译器可以把他们设置成更大的值

但是需要保证，signed char 到 short int 到 int 到 long int 到 long long int 的宽度必须是单调不减的，比如我们把他们宽度都设成 64

数组类型

数组类型的形式化定义：T[N]T[N]T[N]

  1. T：元素类型（Element Type），必须是一个完全对象类型
  2. N：元素类型的个数（Number of Elements）

N 是一个表达式：

  1. 如果没有 N，单纯的 T[  ]T[\;]T[] 也是一个数组类型，该数组类型是一个不完全对象类型
  2. 有 N 的情况下，如果 N 是一个整数常量或整数常量表达式，则 T[N]T[N]T[N] 是普通数组类型
  3. 否则 T[N]T[N]T[N] 是边长数组类型（Variable Length Array Type），边长数组的合法性在 C 语言中目前存在很大争议

数组类型也是一个合法对象类型，对应也有大小和对齐要求：

  • sizeof(T[N])：等价于N * sizeof(T)，T[]没有 size
  • align(T[N])：等价于alignof(T)，T[]没有 alignment
