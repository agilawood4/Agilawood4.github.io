LC214

【KMP 算法求两个字符串的重叠部分（交集）】

思路

我们考虑把原串翻转后得到 rs，那么问题就转化成：我们希望把 rs 和 s 进行前后拼接，但是 rs 和 s 的公共部分只能出现一次，问拼接后得到的新串，也就是求两个字符串的公共部分，比如 rs = aaacecaa，s = aacecaaa，公共部分为 aacecaa，那么拼接后得到的回文串就是 aaacecaaa，其中两个串的公共部分的求解也是 KMP 算法的一个典型用法

因为我们只能在原串之前添加字符，也就是说 rs 必须在前，s 必须在后，也就是把 s 拼在 rs 上，我们以 rs 作为文本串，s 作为模式串，跑一遍 KMP 算法，rs 最后一个字符所能匹配到的 s 串的最大长度，即为二者的公共部分

其实我们可以在 KMP 的流程中，记录文本串的每一个位置所能匹配到的模式串的最大下标（等价于最大匹配长度），这里因为更新顺序的缘故，最后 j+1 的值即为文本串的最后一个字符所能匹配到模式串的最大下标位置

P2375

【nxt 变形】

引理

本例考察对 KMP 算法中 nxt 数组的理解和变形，首先我们重新回看我们 getNxt 的过程：

我们关注 j 指针的变化，对于每一个 i 位置的 nxt 值完成求解以后，j 位置会停留在它上次被更新完的位置，对于下一个 i 位置到来以后，它要么只向后 +1，要么接着向后跳，因此在整个过程中，最坏的情况就是 j 每次都 +1，总复杂度也不会超过 O(n)，而 j 的回退也不会超过 -1，因此该过程的总复杂度就是线性的，其中 j 指针的更新机制对于我们本例的优化思路至关重要

以下是本例的思路：

本例希望我们对 nxt 数组进行变形，求出一个字符串 s 所有不重复的前后缀个数，比如对于 aaaaa 这个字符串，我们使用 KMP 算法会找到两组前后缀：第一个 a 和最后一个 a，前两个 a 和后两个 a，对于 3 个 a 的前后缀的情况，因为题目要求不能重复，因此不计入答案

我们考虑满足条件的前后缀组数和 nxt 数组的关系，先不考虑不能重复的要求，对于 aaaaa 这个字符串，它的 nxt 数组分别是 -1 0 1 2 3，以 4 位置作为结尾的后缀，可以分别对应以 3 2 1 0 位置结尾的后缀，因此有 4 组前后缀关系，对应的也就是 4 位置根据 nxt 数组向前跳，需要跳 4 步才能到达 -1，我们找到一个大致的关系：以某位置作为结尾的后缀能够匹配到的前缀的数量，等于从这个位置开始，由 nxt 数组向前跳，跳到 -1 位置所跳跃的步数

下面我们证明这个结论：我们可以由递推法证明，假设 i 位置对应的 nxt 位置为 j，也就是说 i 向前跳一步可以到达 j 位置，假设 j 位置作为结尾的后缀可以匹配的前缀数量为 x，而 i 位置作为结尾的后缀又可以匹配 j 位置作为结尾的前缀，而 j 位置能匹配到的前缀都是 i 位置后缀的子串，因此 i 位置作为结尾的后缀可以匹配的前缀数量就是 x+1

因此我们可以在更新 nxt 数组的同时，更新每个 i 位置跳跃到 -1 所需要的步数 step 数组，满足：step[i] = j == -1 ? 1 : step[j] + 1的关系

下面我们考虑前后缀不能重复的限制，假设 i 位置作为结尾的后缀首先匹配到了 j 位置作为结尾的前缀，也就是 i 通过一步 nxt 跳到 j 位置，那么 j 位置作为结尾的前缀长度为 j-0+1，要想前后缀不重复，必须保证 i 位置作为结尾的后缀的起始位置在 j 位置之前，也就是 i-j > j-0+1，也就是必须满足 i >= 2×j+1 才可以，所以我们在 i 通过 nxt 往前跳的时候，需要一直跳到 i >= 2×j+1 才可以，因此我们可能会写出这样的代码：

同时本例给我们一个很有意思的结论，也就是以 i 位置作为结尾的后缀，所能匹配到的前缀的个数，其实是和 nxt 数组的值有关，i 位置每经过 nxt 数组向前跳一次，就能匹配一个前缀，直到跳到 -1 位置为止

KMP & DP

KMP 和 DP 结合类的题目，大概会给一个长串（不确定）和一个短串（确定），然后要求短串在长串中的出现次数（不出现，只出现一次，出现两次），问满足这样条件的长串的种类有多少

从数据范围来看，一般满足长串的长度很长（10^9^），短串的长度很短（近似于常数），同时会给定组成字符种类的限定（全是字母或全是数字）

P3193

思路

我们定义二维 DP 状态：dp [ i ] [ j ] 表示**长串（文本串）长度为 i 的子串，和短串（模式串）**长度为 j 的子串恰好匹配，所能得到的方案数，对于该状态我们有以下几个约定

  1. 文本串和模式串都从起始位置开始，分别以第 i 个和第 j 个字符结尾
  2. 除当前匹配之外，文本串前 i 个字符不会和模式串有其他的匹配

接下来我们考虑状态转移，如果是想从当前状态的子状态进行转移，也就是从 dp [ i -1 ] [ k ] 的某个状态转移，换言之也就是文本串的前 i - 1 个字符和模式串的前 k 个字符完成匹配的前提下，双方各自读进来一个字符，模式串的匹配位置变成了前 j 个字符，根据 KMP 算法的性质，因为每读进来一个字符，j 位置不是简单的 +1，而是有可能通过 nxt 数组向前跳，也就是说 k 有可能比 j 大也有可能比 j 小，因此我们是没法根据 j 的位置反推得到 k 的位置的，所以从子状态向当前状态转移的思路不现实

因而我们考虑，当前状态可能更新哪些状态，利用当前状态去更新其他的状态（DAG 的更新形式），此时我们的转移思路变成了：当前的 dp [ i ] [ j ] 表示文本串前 i 个字符和模式串前 j 个字符完成了匹配，此时文本串读进来第 i + 1 个字符 ch，我们根据这个 ch 的种类，结合 nxt 数组就可以让 j 对应模式串中的下标 p 往前跳，它具体跳到哪我们并不关心，这是由 ch 和 nxt 数组共同决定的，我们只知道，当前的状态 dp [ i ] [ j ] 会对 dp [ i + 1] [ p + 1 ] 有贡献，其中 p 是 j 位置对应的模式串下标，根据 ch 由 nxt 数组跳转后得到的待更新的模式串状态的下标，此时我们就完成了当前状态对它可能有贡献的状态的更新

P10581

思路

本例和上例相比，对模式串 S 的出现次数有额外的要求，要求 S 在文本串中出现两次，因此我们还需要对 dp 数组多开一维，维护模式串出现的次数，即定义 dp [ i ] [ j ] [ k ] 表示文本串长度为 i 部分，恰好匹配了当前正在匹配的模式串长度为 j 的部分，已经匹配了 k 个完整的模式串，所对应的方案数（注意当前正在匹配的那个模式串不计入已经完整匹配的模式串的数量）

对于匹配规则的约定和上例完全相同

对于状态转移，还是 DAG 更新形式，考虑当前状态可以用来去更新哪些状态，这里因为多了一维 k 的状态，我们需要考虑这一维状态怎么转移，需要修改的就是在上例中，当 p+1 == m 的时候，也就是当前我们匹配到了一个完整的模式串，因为上例我们要求不能出现完整的模式串，这个状态会被直接跳过，而在本例中，这个状态是合法且必须要被更新的

首先，我们顺利匹配到一个完整的模式串，那么第三维度 k 会被更新成 k + 1，即完整匹配的模式串的个数会 +1

此外，我们下一次在模式串中匹配的下标位置应该是当前 p 对应的 nxt [ p ] 位置，因此对应的长度应该是 nxt [ p ] + 1，之所以这么处理，可以参考 KMP 查找所有子串匹配位置时，找到一个完整子串以后，对 p 位置的更新，主要是为了处理文本串有重复的时候，比如模式串是 abab，而文本串是 ababab，当完成第一组匹配的时候，i 在文本串的 3 位置，p 在模式串的 3 位置，此时 i++ 到文本串的 4 位置，p 应该通过 nxt 数组跳到模式串的 1 位置，这样才能匹配到第二个 abab，如果 p 直接回到 -1 的话，下一次就会从 0 位置开始匹配，这样我们是找不到第二个 abab 的，因为文本串有重叠，如果每次匹配完成都从 -1 开始重新更新 p 的话，这些重叠的部分就无法被考虑了

注意：对于上两例而言，目前的做法都会超时，用到的矩阵快速幂的优化技巧会后续补充

Z函数

基础知识

Z函数：字符串与其每一个后缀的最长公共前缀（LCP）长度（抽象定义）

具体来讲，我们会维护一个 z [ i ] = j 数组，表示原字符串从 i 位置开始，和它从 0 位置开始，分别依次向后匹配，所能匹配到的最大长度，比如这里 S 的 4 位置开始，能够得到子串 ab，和 S 从 0 位置开始的 ab 恰好匹配，因此 z [ 4 ] = 2

显然 z [ 0 ] = len(S)，因为从 0 开始前缀能够和自身匹配

求解过程

对于我们当前遍历到的 i 位置，假设 i-1 及之前位置的 z 函数都完成了求解，且我们维护了 0 ~ i-1 中的一个 l 位置，使得 l + z [ l ] 的值最大，也就是令 r = l + z [ l ]，其中 r 的的范围最远，那么 l ~ r 会对应一段前缀 0 ~ r - l，使得这两段内容是完全相等的，而 i 如果处在 l ~ r 之间，那么 i 也会相应的对应一个 i - l 的位置，如下图所示，这两部分是对应完全相等的

而 0 ~ r - l 范围内的 z 函数都是已知的，我们接下来考虑，当 i 在 l 到 r 范围之间的时候，先利用已知的 i - l 位置的 z 函数更新 i 位置的 z 函数，分为以下两种情况讨论：

  1. z [ i - l ] < r - i + 1 的时候，此时因为 i 位置和 i - l 位置作为起点的两段绿色串完全相同，而前者能匹配的前缀没有超过红色位置，那么后者所能匹配的前缀也必然不会超过红色位置，因此 z [ i ] = z [ i - l ]
  2. z [ i - l ] > r - i + 1 的时候，此时我们考虑反证法，如果 z [ i ] > r - i + 1 也成立，那么也就是说，紫色部分和黄色部分中有相等的段，那么我们之前维护的 r 位置必然不是能够匹配到的最远位置，因此产生矛盾，所以 z [ i ] 必然不可能超过 r - i + 1

因此我们有结论：当 r >= i 的时候，我们可以初步限定 z [ i ] 的值必然不会小于 z [ i - l ] 和 r - i + 1 中的较小值

接下来，我们考虑 z [ i ] 有没有可能继续扩展，直接暴力 z [ i ] 和 z [ i ] + i 位置的字符是否相等，不断扩展 z [ i ] 的值即可

最后如果 i 位置所能扩展到的右端点 i + z [ i ] - 1 超过了原本我们维护的最远的 r 位置，则对 l 和 r 进行更新，得到更远的 r 位置（l 和 r 位置可以初始化为 -1）

复杂度分析

整体求解的复杂度为 O(n)，其中 n 为序列长度，这是因为我们所有的求解过程，不论是 i 位置在 s 串上的滑动，还是最远边界 r 位置的确定，都是不回退的，没有进行重复计算，因此可以认为是线性的复杂度

简单来讲，Z 函数可以求解某个字符串 S 中，以任意 i 位置作为起始位置的子串，所能够匹配到 S 的最大前缀的长度

LC28

思路

可以看成是 Z 函数的一个简单用法，我们把 needle 拼在 haystack 前面，二者中间加一个特殊分隔符 #（这是为了防止 needle 为 aaa，haystack 为 aaaaaa 这样的情况），那么 needle 就成了我们想要匹配的前缀，haystack 中可能存在的那个和 needle 匹配的不分就成了我们和 Z 函数要找的和前缀匹配的那个子串

我们只需要跑一遍 Z 函数，然后遍历每个位置，检查是否存在某个位置的 z 值是 needle 的长度，如果存在则表示匹配到了 needle 前缀，也就是 haystack 中存在和 needle 匹配的子串
