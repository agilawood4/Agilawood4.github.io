二分专题

基础知识

二分的条件

什么样的题目可以用二分来解？

  1. 存在单调的映射关系（单调函数，数组……），只不过函数是通过计算从 x 得到 y，而数组是通过地址索引从 x 得到 y，二者本质上是没有区别的
  2. 已知映射关系为 f(x) = y，现在给出 y 值，要求求出 x 值，当满足由 x 求 y 好求，但是通过 y 反求 x 困难，此时考虑使用二分（如果我们有一种数据结构可以支持快速从值索引下标，则我们就不再需要二分）

函数由计算实现映射，占用时间资源，数组由地址索引实现映射，占用空间资源，二者的等价性满足了算法中的时空可交换性

使用二分的关键：寻找条件和答案之间的单调性

二分查找的库函数

lower_bound 和 upper_bound 函数：二者都是在有序序列中才能生效

lower_bound：查找序列中第一个不小于 x 的位置，返回的是这个位置的地址

upper_bound：查找序列中第一个严格大于 x 的位置，返回的是这个位置的地址

返回的地址有时候会差个 1，用的时候最好自己调一下

381E - 11/22 Subsequence

思路

怎么快速得到一个 11/22 字符串的长度，我们观察该字符串的性质，中间是一个 /，左右是数量相等的 1 和 2，我们考虑使用贡献法，统计一定包含每个 / 的 11/22 串的长度，也就是要统计它左边和右边的 1 和 2 的数量

我们考虑一个子问题：给定一个 [ l , r ] 的区间，其中内部有一个 /，那么包含它的 11/22 字符串的最大长度即为 l 到 / 中 1 的数量，以及和 / 到 r 中 2 的数量中取一个最小值，那么问题又变成了快速统计一段区间内 1 或者 2 的数量，因此我们想到可以用前缀和进行问题优化

接下来我们考虑二分的过程，我们这里二分的是所有 / 的位置，我们首先遍历序列，O(N) 复杂度求出所有 / 位置的下标，对于每一段 L 到 R 的区间查询，我们都二分 / 的位置 ind，现在考虑二分的具体实现：

  1. 如果 ind 小于 L，那么应该去 ind 的右边接着二分
  2. 如果 ind 大于 R，那么应该去 ind 的左边接着二分
  3. 如果 ind 是 L 到 R 之间的一个合法位置，统计答案以后考虑，如果左侧 1 的个数更多，则希望把 / 左移，在 L 到 ind 之间接着二分；如果右侧 2 的个数更多，则希望把 / 右移，在 ind 和 R 之间二分，因为我们希望两侧 1 和 2 的数量是差不多的，这样我们取二者最小值能得到一个较大的答案

这里我们做的实际上是二分查找，查找的是使得答案最大的 / 的位置，也就是最合适的 ind 的位置

388G - Simultaneous Kagamimochi 2

思路

本例也需要在 O(1) 复杂度内实现对一组区间的快速求解

我们首先证明一个问题：如果对于一个区间 L 到 R，假设存在一个 k 使得我们能够做出来 k 组镜饼，我们一定可以选择序列中的前 k 小和后 k 大进行两两组合，这一定是一组合法答案，也就是说每个位置的元素，都向后取 R - k + 1 - L 个位置就能确定唯一一个和他对应的位置，与它形成组合，也就是说我们把一组合法方案进行具象化，在一组合法解中，每个元素都一定和他向后 R - k + 1 - L 个位置的元素进行组合，我们可以反证法证明，假设后 k 个最大的元素都没法和前 k 个最大的元素组合，那往前选别的元素只能小不能大，更凑不出合法方案，就和我们的假设矛盾了

现在我们把上面的分析反过来，对于一个 k，我们如何检验一段 L 到 R 的区间内，是否可以找到 k 组元素拼成镜饼，首先我们一定是选前 k 个和后 k 个进行匹配，前 k 个元素中的每个元素，它应该放的位置我们是知道的，因为在上面的分析中它的位置是唯一确定的，第 i 个元素与之对应的位置就是 i + R - k + 1 - L，那么我们需要知道前 k 个元素中，每个元素离他最近的那个是他二倍的元素的位置，即判断这个位置是否会超过它应该放的位置，假设 pi 存的是第 i 个位置的元素离他最近的是他二倍的元素的位置，那么必须要求 pi <= i + R - k + 1 - L，也就是 pi 不能超过它的指定对应位置（可以往前不能往后），因为如果可以靠前的话我们直接强行把他移动到它自己的指定位置就行，但是如果靠后的话说明它的指定位置是不满足条件的，只能往后取更大的位置，但是我们后面只有 k 个位置和前面 k 个位置一一对应那个，相当于它向后会占了别人的位置，那么剩下的位置势必会不够用，那么后面就会有溢出 R 的情况，此时就是不满足的，也就是我们要检验前 k 个元素中的每一个位置 i 是否满足：

对该公式进行变形则得到： pi - i <= R - k + 1 - L，右侧是和 i 位置无关的恒量

那么我们就需要维护 L 到 (R - L + 1) / 2 区间内每个位置 pi - i 的最大值（我们选的上面的位置不可能会超过区间长度的半数，因为 N 个元素最多也就是做出来 N / 2 个镜饼 ），只要这个最大值都小于右侧含 k 的表达式，而维护区间内的最大值，可以借助 st 表快速查询实现

st [ i ] [ j ] : 每个以 i 开头，长度为 1 << j 的区间内 px - x 的最大值：

接下来分析二分的具体流程：对于一段 L 到 R 的区间，我们二分选择的镜饼的组数 k，用来二分的范围即 0 到 区间长度的半数，对于每遍历到的一个 k，我们都希望整个区间的前 k 个和后 k 个进行匹配，那么我们就需要判断需要判断 L 到 L + k - 1 这段区间内所有的点对应的 pi - i 值是否不会超过 R - L - k + 1，如果最大值都没有超过，那么当前 k 就是合法的，对于单调性的证明，显然如果 k 选择越大，那么每个位置需要往后的就越多，就越有可能挤掉后面的位置，使得 pi - i 的值溢出，k 变小的时候同理，因此二分的单调性得证

二分内容的设计

有时候如果我们直接对题目要求的答案进行二分，是不满足单调性的，此时我们需要改变二分的内容，找一个既能反映出答案（或求出答案），又具有单调性的量进行二分

CF2051E - Best Price

思路

首先我们不能二分商店的定价，因为定价是高低和收益高低不满足单调性，定价太低的话收益肯定低，定价太高的话可能超出顾客上限，使得购买的人减少，也可能会使得收益变小

另一个思路尝试是直接二分商店的总收益，但是这样 check 函数中没法知道每个顾客到底花多少钱买，以及买还是不买，因此也行不通

正解思路：本例二分的对象比较特殊，它并不是直接对答案进行二分，而是在求解答案的过程中用到了二分，首先我们证明，商店定价的最优解一定是所有顾客的 a 值和 b 值中第一个数，不可能在此之外（我们可以这么想，既然我们没办法通过二分得到答案，那得到商店定价的办法就只剩下枚举了，既然是枚举就要确定其大致范围，因此可以猜测就在已有的数据中取）

下面我们证明：最优解的商店定价一定是所有的 a 或者 b 中的某个值

假设不是，那么一定存在一个离他最近的且比它大的 a 值或 b 值，我们把定价就调整到这个离他最近的值，商店获得的收益一定会变大（因为是调整到离他最近的 a 或者 b 值中的较小者，所以既不会失去顾客，也不会获得更多的差评）

因此我们只需要枚举所有的 a 和 b，现在确定了定价 x 以后，我们考虑怎么算出来收益，我们可以快速求出以下两个值：

  1. 在当前定价下，会进行购买的人数 - 在 b 序列中查找第一个不小于 x 的位置
  2. 在当前定价下，会进行购买，且不会给差评的人数 - 在 a 序列中查找第一个不小于 x 的位置

那么差评数目就是上面两者作差，所以我们的二分是用在了上面两个查找中

易错：不要认为把 a 序列排好序以后，a 序列中不小于 a [ i ] 的数字个数就是 n - i + 1，因为 a 序列有重复，最好就是直接二分查找！（ b 序列也同理）我在这里卡了好久没调出来

序列第 m 数问题

二分答案的一种用法，其核心问题在于：给定一个序列，序列中的每个元素按照某种构造规则又可以衍生出新的元素（比如按等差数列构造），现在要求在所有元素的序列中找到第 m 大 / 小的数，其中所有元素的总数很大，元素取值范围也很大

ACWing - 4645

本例是十三届蓝桥杯省赛 C 组倒数第二题

思路

每个技能都可以被使用多次，每次使用都会在初始值上减掉一个损失量，也就是说每个技能都可以衍生出来一个等差的技能序列，现在题目要求在所有（这里的所有包括初始技能和等差构造出来的衍生技能）中取 M 个最大的，问得到的收益是多少

最朴素的想法：把所有技能以及它能构造出来的等差技能序列中的所有元素全都存起来，然后从大到小排序，取前 M 个最大的就行（可以过 40% 数据）

单调队列优化：因为我们只要前 M 个最大的，而且我们又知道，对于同一个等差数列中的技能，前面的技能没被选的时候，由它减去损失量得到的技能一定也不会选，因此我们考虑使用一个优先队列，先维护 N 个技能的初始值，每次选择一个最大的，选择完之后再把它减掉损失量之后重新入队（直到为 0），重复 M 次即可，复杂度为 O(MlogN)，可以过 60% 数据

最优解：本例中 M 的范围很大，要想卡过时间，我们设计的算法必须和 M 无关，或者把 M 压缩到 log 级别

我们假定所有元素加上他的等差数列形成的总序列我们已经得到了，因为我们是从大到小选择序列中前 M 个最大的数，我们考虑这 M 个数中最后一个被选择的数 x 是多少，x 的范围是 0 - 1e6，又因为 x 的选择和我们得到答案中的个数具有单调性关系，x 越大，比 x 还大的数就越少，就越不可能选够 M 个；x 越小，比 x 还小的数就越大，就越可能选够 M 个，而我们要求的恰好是序列中第 M 大的数，也就是最大的满足大于等于 x 的数的个数大于等于 M 个的 x，因此可以对 x 进行二分

check 函数的设计：给出我们所选择的最小的数 x 之后，相当于每个元素形成的等差数列中，知道了首项，又知道了下界，可以 O(1) 求出每个等差数列中可选的元素个数，则 check 函数可以在 O(N) 复杂度内完成

结尾边界的判断：我们考虑边界情况，假设很多元素所形成的等差数列中都有 x 这一项，则一共有 a 个 x，假设所有严格大于 x 的项为 b，则最后可能会存在 b < M , b + a > M 的情况，也就是说并不是所有的 x 都会被选，可能我们选到最后的时候，只选了其中几个 x 就已经到达 M 个数，但是我们按照等差数列公式算出的个数是每一个等差数列中的 x 都会被选上，这势必会导致我们的答案变大，多出来的就是 x 序列中本来不该选但选上的数，也就是说多的数全都是 x，因此我们只需要记录我们在等差数列公式计算时实际取的元素个数，它和真实 M 的差值即为多选的 x 的个数

本例和上例的形式很相近，每个物品都可以买 1 到无穷多个，给出一个花费上限，要求我们求解可购买的最大数量

我们把这个差价对应的价格看成商品，也就是说对于一个商品 p 而言，它衍生出来的序列为 p，3p，5p，7p……，这样我们就把同一件商品只能选一种变成了可以依次选择的形式，选一件就是 p，选两件就是 p + 3p = 2^2^p ……依次类推

有了商品序列和同一件商品的递推规则，我们考虑怎么二分的问题，因为我们买的商品一定是按价格递增的，优先买入的肯定是便宜的商品，因此我们可以二分买入商品序列中最贵的那个商品的价格 x，检验在当前 x 值下，所有不超过 x 价值的商品都买入，所需要的总花费是否会超过上限 M，考虑是否答案具有单调性，x 选的越大，能买的商品就越多，花费就越大，越容易超过 M，x 选的越小，能买的商品就越少，花费就越小，越不会超过 M，因此答案具有单调性，可以对 x 进行二分

结尾边界的判断：我们买完所有价格不超过 x 的物品以后，我们手头的总钱数可能还有剩余，此时我们要继续买价格为 x+1 的物品，现在我们证明 x+1 的物品一定可以买

  1. 一定存在价格为 x+1 的物品可以被买，否则我们二分 x 和 x+1 是没区别的（ x+1 的物品买不了，上界就相当于是 x ），又因为我们二分的是最大的 x，则会不停向后继续更新，答案不会是 x
  2. 价格为 x+1 的物品一定购买，我们考虑会不会出现最后剩的钱数够买 x+1 价格的物品，但实际上我们每个物品能构造出来的价格为 x+1 的物品没有那么多，这种情况也是不会存在的，因为既然我们无法把答案更新成 x+1，而可以更新成 x，就说明一定是因为 x+1 价格的物品使得我们在买入的时候，因为买入 x+1 的物品使得总钱数超过 M 上界而变得不合法了，也就是说我们的总钱数一定是只能买 x+1 价格的物品中的一部分，而无法买入全部

因此我们最后可以放心用 sum剩余 / x+1 得到我们可以买入 x+1 的部分物品的数量

二分其他应用

二分答案还有一个很重要的作用就在于，它可以把对未知量的求解转化为对已知量的检验，也就是说我们证明中相当于多了一个条件，或者说也是一种反证法的思路

368C - Triple Attack

思路

这是一大类很典型的题，这类题做的一般就是对一个序列依次做一些有规律的操作，最后使得这个序列具有某种性质，然后题目会问我们在操作的过程中对序列操作的个数，或者是操作过程中得到的最大收益之类的问题，操作量一般都很大，这时候我们可以考虑用二分答案来解

具体对本例来说，我们要求的就是使得序列中所有元素都非正的时候的 T 对应的最小值，T 的范围一定是很大的，因为每个人的血量为 1e9，一共 1e6 个人，大概会做 1e15 次操作，因此这个量是一定是要二分出来的

对于我们当前二分到的一个 T 值，我们考虑 check 函数的设计：我们根据当前第一个人的血量以及 T 在攻击这个人之前的初值来进行分类，因为一个血量何时减成 0，既和它的初值有关，也和它碰到三倍暴击的时刻有关，因此需要同时以二者来分类
