背包模型练习

通过阅读本文，您可以大致了解一些简单的背包问题模型的应用，包括但不限于 01 背包，分组背包，多重背包，完全背包

P4026

思路

本例需要先进行建模和思路转化，我们不从相互还钱考虑，而是让三个人都把自己拥有的钱放到桌子上，然后每个人各自拿走自己应得的那些钱（也就是还完钱之后自己剩下的钱），三个人总共拿取钱的总次数 / 2 其实就是相互还钱的次数

因而我们有了对应的 dp 状态，定义 dp [ i ] [ x ] [ y ] 表示前 i 种面额的钱币，a 拿走 x 这么多钱，b 拿走 y 这么多钱，所对应的最小拿取次数，这里因为总钱数是固定的，全局最优拿取次数下 a 和 b 拿走的钱数也是确定的，那么剩下的钱一定被 c 拿走，因此 c 拿走的钱数被隐式确定，不需要额外表示

而对于状态转移，我们考虑第 i 种面额的钱币，a 和 b 分别拿了几张，假设 a 拿了 p 张，b 拿了 q 张，而桌子上总共有的第 i 种面额的钱币的个数应该为 a [ i ] + b [ i ] + c [ i ] = s [ i ]，a 原本自己拥有的钱数是 a [ i ]，b 原本拥有的钱数为 b [ i ]，而现在 a 有的钱数为 p，b 有的钱数为 q，相当于 a 从桌子上拿了 p - a [ i ] 张，b 从桌子上拿了 b [ i ] 张（这里最优情况下三个人显然不会重复拿放，比如 a 先把第 i 种钱放一张到桌子上再拿回去，因此我们只需要看当前每个人的钱数和它原本拥有钱数的差值即可）

那么在当前状态下，a 和 b 多拿或少拿的钱，一定来自于 c，即如果 a 和 b 拥有的钱数变少，那么这些钱一定放到桌子上最终被 c 拿走，如果 a 和 b 拥有的钱变多，多的钱也一定来自 c，因此 c 从桌子上拿取的钱数就是 p + q - a [ i ] - b [ i ]，因此我们可以在当前状态下枚举 p 和 q，那么我们每次转移的代价C = abs(p - a[i]) + abs(q - b[i]) + abs(p + q - a[i] - b[i])，转移到的子状态就是dp[i - 1][x - p * mz[i]][y - q * mz[i]]

我们初始化所有状态为 -INF，最后 a b c 三个人应有的前应该是自己原本有的钱，减去自己还别人的，加上别人还自己的钱，因此我们只需要判断 dp [ 6 ] [ suma ] [ sumb ] 是否被正确更新过，即可判断是否有可行解，最后需要注意交换次数是拿取总次数的半数

P2967

思路

本例属于有依赖的背包模型，具体来讲就是若干个游戏为一组，如果想购买组内的游戏，必须要先买这一组所对应的机器，且机器不会产生价值，只会带来代价

首先我们考虑，每一组内的最优答案构成 01 背包模型，组与组之间的最优答案相互独立，也就是说对于某个机器下的所有游戏而言，在一定要选这台机器的前提下，选择其中的前 k 个游戏，最大代价为 j 所带来的价值是确定的，且和其他组无关

而对于整体答案而言，整体的最优答案就是各个组内的最优答案共同作用的结果，因此我们考虑按机器进行分组，定义状态表示 dp [ i ] [ j ] 表示前 i 台机器，最大花费代价为 j 的时候，所能得到的最佳收益，接下来我们考虑状态转移，对于当前遍历到的第 i 台机器，我们有两个决策：

  1. 买不买这台机器，如果不买这台机器，dp [ i ] [ j ] 就直接转移到 dp [ i-1 ] [ j ]，也就是根本不考虑第 i 台机器
  2. 如果买这台机器，在其内部选哪些游戏，这是一个独立的子问题，对于第 i 台机器下的 k 个游戏而言，我们可以单独用 01 背包求出一个子dp‘ [ k ] [ j ]表示对于第 i 台机器下的前 k 个游戏，最大花费代价为 j 时，所能得到的最大收益，其中dp’的第一维可以被优化掉

因此本例是一个 dp 内部嵌套 dp 的问题，为了便于进行状态转移，我们再额外定义一个辅助的 dpt 数组，用来完成每一台机器内部游戏的决策：

  1. dp [ i ] [ j ] : 表示前 i 台机器，最大花费代价为 j 的时候，所能得到的最佳收益
  2. dpt [ i ] [ j ] : 表示前 i 台机器，不考虑购买机器本身的花费，最大花费代价为 j 的时候，所能得到的最佳收益

因此在本例中，我们先用 dp [ i-1 ] [ j ] 更新 dpt [ i ] [ j ]，表示不买第 i 台机器的基础情况，接着在 dpt 中做一次 01 背包，求解出第 i 台机器内游戏的购买情况，最后用 dpt [ i ] [ j ] 更新 dp [ i ] [ j ] 的值，这里注意有一个 cost 的偏移量，也就是从 dpt 更新回 dp 的时候，要考虑购买这台机器本身花的前，因此每一个 dp [ i ] [ j ] 其实是由 dp [ i ] [ j+cost ] 更新得来的

在本例中，我们在 01 背包和 dpt 更新 dp 的两轮循环中都用到 j 这个变量，这两个变量的含义其实是不同的，前者表示的是只针对第 i 台机器中的那些游戏，仅购买这些游戏的最大花费代价，而后者则表示的是前 i 台机器中，包含购买机器和所有游戏的全局最大花费代价

同时，本例相当于做了两次空间优化，首先是对于机器编号 i 的优化，其次是对于每一轮做 01 背包时，遍历每一个机器中所有游戏编号 k 的优化，因此后两轮循环的更新过程都需要倒序遍历

P1064

思路

在写完 P2967 以后再看 P1064，不能说是一模一样也只能说是大差不差，这两例都属于有依赖的背包模型，其中物品分成两种，主件可以单独购买，附件要想购买就必须先买对应的主件，只是本例区别于上例的地方在于，本例的主件也是有价值的，对收益有贡献，我们要做的只是在用 dpt（不考虑第 i 个主件，只考虑第 i 个主件的附件）通过第 i 个主件的花费平移更新 dp（考虑第 i 个主件，前 i 个主件的收益）的时候，加上第 i 个主件自身的价值即可

此外本例需要单独收集所有的主件物品，对于每个主件维护他们各自的附件，在遍历的时候，最外层遍历只需要遍历所有主件即可，这里有一次下标映射，第 i 个主件（用于状态转移）-> 主件在所有物品中原本的下标 -> 索引到主件的 v 和 p 值

P2854

思路

补充：本例的数据范围大概是 n L B 三个参数都在 10000 左右

本例很容易想到想到二维费用背包，也就是定义 dp [ i ] [ l ] [ b ] 表示前 i 根线段，覆盖区间长度为 l，花费最大费用为 b 所能得到的最大收益，然后发现这样是三层循环，提交会超时

我们考虑怎么优化掉一维枚举，思考后发现本例并不是严格的二维费用背包，因为题目要求线段覆盖不能重叠与缺失，那么我们真正所能用到的有效长度 l 其实是很有限的，因为很多 l 值其实是非法的，我们所有能枚举的 l 不会在集合 X 之外，X 是所有线段的终点位置，因此我们先枚举线段编号再枚举覆盖长度 l 其实是很浪费的做法，因为很多 l 都是无效的长度

因此我们考虑对所有线段按终点排序，按顺序遍历每一条线段，重新规划 dp 状态为 dp [ i ] [ j ] 表示覆盖范围为 0 ~ i，花费恰好为 j 时，所能得到的最大收益，因此我们的状态转移就变成了若干单点之间的转移，不会涉及到状态的继承，我们每遍历到一条线段 L，我们直接从它的终点处减去线段长度，作为前一个转移状态，因为我们对终点进行了排序，能够转移到当前线段的子状态如果存在，则先前一定会在当前线段终点之前更新对应的 dp 数组，因此上述转移方式是无后效性的

这里因为我们是按照线段离散式转移的，因此最后覆盖所有区间时，花费不一定恰好为 B，而是不超过 B，因此最后还需要遍历所有 dp [ L ] [ i ] 来进行答案统计
