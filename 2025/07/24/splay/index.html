伸展树 splay

通过阅读本文，您可以大致了解伸展树的实现原理，结构定义，操作以及简单的应用

实现原理

splay 树基于 BST 树实现，通过核心的 splay 操作，能够确保每次操作（插入、删除、查询）以后，树的高度都维持在 log(n) 左右，其中 n 是树上的总节点数

结构定义

基础的 splay 树（不支持重复词频，相同权值开辟多个节点存储）通常需要维护以下信息：

  1. 根节点 head，当前节点总数 cnt
  2. fa [ i ]：i 号节点的父节点（根节点的父节点默认为 0）
  3. sze [ i ]：i 号节点做根的子树的大小
  4. ls [ i ] and rs [ i ]：i 号节点的左右儿子节点编号
  5. key [ i ]：i 号节点的权值

此外我们还需要两个简单的辅助函数，帮助我们快速实现后续功能：

  1. UP()：当 i 号节点做根的子树结构发生变化时，使用 i 号节点的左右儿子的 sze 值更新 i 号节点的 sze 值
  2. lr()：快速返回 i 号节点是它的父节点的左儿子还是右儿子（1 - rs，0 - ls）

左旋 & 右旋

所有的平衡树结构保持平衡的关键都是通过左旋（zig）和右旋（zag）限制树高来实现的，splay 的核心伸展操作也是通过基本的左旋和右旋实现的，我们通过调用rotate()方法，把当前节点 i 进行左旋或右旋

具体操作分为以下两种情况：

  1. 如果 i 是其父节点 f 的左儿子，则把 i 右旋，首先更新 f 的左儿子为 i 的右儿子，如果 i 的右儿子不为 0，则更新其父节点为 f，然后更新 i 右儿子为 f
  2. 如果 i 是其父节点 f 的右儿子，则把 i 左旋，首先更新 f 的右儿子为 i 的左儿子，如果 i 的左儿子不为 0，则更新其父节点为 f，然后更新 i 左儿子为 f
  3. 最后更新 f 的父节点为 i

此时我们改变了 i 和 f 的父子关系，那么对于 f 的父节点（i 祖先节点）g，也要更新它的儿子结构，如果祖先节点 g 存在，又分成两种情况：

  1. 如果 f 是 g 的左儿子，那么旋转后，更新 g 的左儿子为 i
  2. 如果 f 是 g 的右儿子，那么旋转后，更新 g 的右儿子为 i
  3. 最后更新 i 的父节点为 g

此时 f 和 i 的子树结构发生变化，所以最后我们要依次对 f 和 i 进行 UP 操作，更新其最新的 sze 值

伸展操作

伸展操作是 splay 的核心操作，我们一般通过splay(i, goal)函数实现，该函数可以通过循环调用rotate()方法，逐渐把 i 节点旋转上提，直到成为 goal 的子节点，在旋转的过程中我们会遇到以下三种情况：

  1. 设 i 的父节点为 f，f 的父节点为 g，当 g == goal 时，只需要旋转一次 i 节点本身即可
  2. 一字型结构：当 i 是 f 的左节点，f 是 g 的左节点，或 i 是 f 的右节点，f 是 g 的右节点时，成为一字型结构，此时需要先旋转 f，再旋转 i
  3. 之字型结构：当 i 是 f 的左节点，f 是 g 的右节点，或 i 是 f 的右节点，f 是 g 的左节点时，成为之字型结构，此时需要对 i 旋转两次

每一轮旋转之后，都需要相应的更新 i 的父节点 f 和 祖先节点 g，直到 f == goal 操作结束，此时记得特判，如果 goal == 0，则 i 成为新的根节点 head

以上两个操作就是 splay 树的核心操作，下面展示和 splay 的插入，删除，查询等有关的操作，这里我们需要注意的是，除了 remove，Key2Rank 等少数操作外，splay 树绝大多数状态在操作完成后，都需要对最后一次遍历到的节点 last 进行一次 splay 操作，让这个节点成为根节点

这其实是用到了局部性原理，我们假定最后一次遍历到的这个节点，在不远的将来还会被频繁访问

插入节点

我们通过调用add()方法，向树中插入节点，此时遵循 BST 插入节点的规则，每次根据 num 和 key [ i ] 的大小关系，决定 i 去往左子树还是右子树插入，插入完成以后，记得把新节点 splay 成新的根节

查询操作

基础的 splay 树中支持维护以下三个信息：

  1. index：节点编号
  2. rank：节点按照权值大小的排名（根据题目要求不同，相同的权值可以返回最小排名，也可以返回最大排名）
  3. key：节点权值

以上三个信息都可以两两转化，我们只需要设计以下三个函数，就可以实现任意两个信息之间的求解，这三个函数的基础都是通过 BST 树左右子树的跳转来实现的

Rank2Index

通过调用Rank2Index()可以实现由节点排名获得节点编号，对于当前遍历到的节点 i，我们通过比较 i 的左子树的节点数 sze [ ls[i] ] 和待查找的 rank 的大小，决定去左子树还是右子树继续查找

该函数不需要进行 splay 操作，找到目标节点后直接返回！

Key2Rank

通过调用Key2Rank()可以实现由节点权值获得节点排名，这里根据题目要求，相同的权值，我们可以查询最小排名，也可以查询最大排名，查询最小排名时，我们统计树上所有严格小于 num 的节点数量；查询最大排名时，我们统计树上所有不超过 num 的节点数量

最后记得把查询到的节点 last 进行 splay 到根节点

Rank2Key

通过调用Rank2Key()方法可以实现由节点排名查询其权值，该方法其实最好实现，因为我们可以由排名获取编号，再通过 key 数组直接索引到权值即可

最后记得把查询到的节点 ind 进行 splay 到根节点

总结

基础的 splay 所维护的 index，key，rank 信息，只需要通过Rank2Inedx(), Rank2Key(), Key2Rank()两两之间都可以进行转化：

  1. index -> key：直接通过 key 数组索引
  2. index -> rank：先通过 key 数组索引到权值，再调用 Key2Rank
  3. key -> index：先调用 Key2Rank，再调用 Rank2Index
  4. key -> rank：直接调用 Key2Rank
  5. rank -> index：直接调用 Rank2Index
  6. rank -> key：直接调用 Rank2Key

删除操作

首先我们先介绍一个引理：给定数值 num，如何找到 num 在树上的后继位置插入新的元素

我们可以查询 num 的编号 i，以及 num+1 的编号 j，这里因为 num 和 num+1 是严格的前驱和后继关系，我们把 i 通过 splay 提到根节点，把 j 通过 splay 提到 i 的子节点时，i 和 j 之间一定不会再有其他节点，也就是说，j 节点此时一定没有左儿子，所以 j 节点此时空置的左儿子位置，就是我们新元素即将插入的位置

这里我们的删除操作即可根据此引理完成，假设我们要删除 num 数值，首先我们通过查询 num 和 num+1 的排名，来检查 num 在树上是否存在，如果存在我们就尝试删除这个节点

首先我们查找 num 的编号为 i，如果 i 没有左儿子，我们删除 i 以后直接让其右儿子作为根节点；如果 i 没有右儿子，我们删除 i 以后直接让其左儿子作为根节点

如果 i 的左右儿子都存在，我们就需要先找到 i 节点的排名 kth，和他的下一个节点，也就是找到排名 kth+1 的节点对应的编号 j，把他 splay 到 i 的右儿子，此时删掉 i 以后，i 的左儿子过继给 j，让 j 称为新的根节点即可（设置 head = j，同时 j 的父节点为 0）

前驱 & 后继

这里我们定义 num 的前驱为严格小于 num 的权值中，最大的权值，num 的后继严格大于 num 的权值中，最小的权值（num 本身在树上不一定存在）

需要注意的是，对于我们查询到的前驱和后继编号，我们都需要把对应节点 splay 到根节点

splay 维护区间信息

基本思想：类比线段树的懒标记，在 splay 中我们仍然考虑利用 LayTag 标记，维护我们对区间的操作

至于怎么把我们要操作的那个序列提出来，可以找到对应排名 l 到 r 区间外延边界 l-1 和 r+1，通过 Rank2Index 找到他们对应的节点编号 i 和 j，通过提根把 i 提为根节点，j 提为 i 的右儿子，那么我们要操作的那个区间就存在 j 节点的左子树中

  1. 区分排名和编号：

在本例中，需要我们能区分排名和编号，也就是 rank 和 index，在一次区间翻转中，我们给出的 l 和 r 相当于是排名，而我们提根操作中操作的 i 和 j 是编号，因此我们需要实现 Rank2Index 把排名转成节点编号后，再进行操作

这里可能会有疑问：我们不是按照序列的下标建立的 splay 吗，那么我们要反转序列的 l 到 r 范围，这个 l 和 r 不就是序列的下标吗，那应该和 splay 中的节点编号一一对应，为什么还要 Rank2Index 转化呢？

在我们刚刚建好树的时候，序列下标和节点编号确实是一一对应的，但是我们进行区间翻转时，如果做了 LazyTag 的下放，则会交换某个节点的左右节点，相当于对应编号的节点被打乱了，对应到原序列而言，假设我们的原序列为：

现在我们交换 2 ~ 4 范围的序列：

此时我们假设还输入 l = 2 r = 4 时，其实它对应我们翻转过一次的序列下标应该是 l -> index = 4，r -> index = 2，因此在做完区间翻转时，输入的序列下标和我们在 splay 中维护的节点编号就不再是一一对应的

所以我们的 splay 中序遍历到底维护的是什么：严格来讲，应该是当前序列中的第几个数，比如我们在翻转过一次的 splay 中查询排名 rank = 2 的数，查到的数字就应该是 d，也就是当前序列（已经反转过）的第二个数字

这个排序和 rank 是对应的，但是和原始 index 以及树上的节点编号并不是对应的，因为我们的翻转打乱过编号和排名的对应关系

  1. 懒标记的下发

首先是懒标记的下发，只要涉及到查询操作，都需要先下发懒标记，这一点和线段树完全相同，具体的下发过程也和线段树相同：分别把懒标记作用于左右儿子，然后清除自身的懒标记

其次是如何找到待操作的目标序列，假设我们要操作的序列为 l r，因为我们建立了两个哨兵，因此实际操作的应该是 l+1 和 r+1 范围，而我们想把这段区间放在两个节点之间，也就是说需要扩出边界 l-1 和 r+1，找到 l-1 排名对应的节点编号 i 和 r+1 对应的节点编号 j，把 i 提成根节点，j 提成 i 右儿子，那么待操作的区间恰好卡在 i 和 j 之间，也就是 j 的左儿子，对应的也就是right[left[head]]对应的节点

我们把 LazyTag 打在这个节点上，只要后边又遍历到这个节点，我们就把对应的 LazyTag 进行下放即可

  1. 简易建树

对于这种中序遍历维护的是下你选哪个序初始下标有序的 splay，我们初始建树的时候因为树上节点编号 cnt 和序列中节点编号 ind 是一一对应的，因此也就是说每次建立的节点的遍历顺序都是递增的，初始的树一定是一条长链，因此我们可以直接让新加入的节点作为当前根节点的右节点，然后直接提根，把新增节点提成根节点即可

P2596

【区间移动 / 单点移动】

思路

使用 splay 维护线性序列时，需要区分以下几个概念：

  1. 树上节点的编号：初始化时，节点编号和线性序列的原始下标序列一一对应
  2. 中序遍历顺序：和每一次查询给出的当前的线性序列的下标一一对应，如果进行了区间移动，则和树上节点编号不等价
  3. 树上节点的权值：查询的信息，通过 key 数组借助节点编号可以快速查询

我们用 splay 维护的核心在于：不管我们对区间做什么样的移动（区间翻转，区间平移），对 splay 中序遍历得到的序列，和线性序列做对应区间操作后得到的序列相同

  1. splay 维护区间移动 / 单点移动

本质上是一样的，单点可以看成是左右端点重合的区间，假设我们现在想把当前线性序列中下标 l1 到 r1 的区间，平移到 l2 到 r2 位置：

比如我们要把 3 到 5 的区间平移到 6 到 7 位置，则得到如下的序列：

首先我们要操作的区间为 l1 到 r1，也就是在 splay 中的排序为 l1 和 r1，这里的排序也就是 splay 中中序遍历的 rank 值是 l1 和 r1，我们为了操作完整区间，还是把端点外延，得到 l1-1 和 r1+1

考虑对开区间端点做提根操作，分别通过 Rank2Index 查询到 l1-1 和 r1+1 排名对应的树上节点的编号 i 和 j，仍然把 i 提成根节点，j 提成 i 的右儿子，那么此时 j 的左儿子所在的子树，就是我们要移动的区间，我们记录该子树的根节点为 target，断开 target 和 j 的关系，即可从原 splay 树中去掉这段区间

接着我们通过 Rank2Index 查询到 l2 和 l2 + 1 排名对应树上节点的编号 i 和 j（这里复用 i 和 j），仍然把 i 提成根节点，j 提成 i 的右儿子，这里因为 j 是 i 严格的后继（j 的中序排序只比 i 大一位），因此此时 j 一定没有左儿子，j 空置的左儿子位置，就是我们要插入 target 的位置，此时即可把 target 对应的区间插入到 i 和 j 之间

注意移动前后都要更新对应 i 和 j 节点的 sze 信息，也就是需要调用 UP 更新子树关系

只要涉及到区间操作，我们就可能会操作到 l-1 和 r+1 位置，因此我们通常都会在 1 位置和 n+2 位置垫两个哨兵，真实的有效序列被平移到 2 ~ n+1 位置

  1. 本例操作的分析

本例题干描述的书籍编号，其实是我们维护的节点权值，这里不要混淆

对于中序遍历维护线性序列顺序，而不是权值有序的 splay 树，我们在维护相关函数的时候会有所改变，下面会详细解释

首先我们需要维护一个反向索引 pos，pos [ num ] = ind 表示权值 num 在 splay 上对应的节点编号为 ind，比较好实现，只需要在 add 操作中，key [ ct ] = num 时反向建立 num 和 cnt 的索引即可

下面是该 splay 用到的相关操作：

  • Add()：

只会在初始化建立 splay 的时候执行节点插入操作，且这里我们中序遍历维护的是原始线性序列下标，下标一定是随插入顺序递增的，因此我们可以每次直接插在原树根节点的右儿子上，然后直接旋转到根即可（因为按照中序遍历递增的形式，建立出来的树本身就是一条长链）

  • TOP() -> move(rank，2)：

把排序为 rank 的节点移动到排序为 1 的位置，因为我们垫了一个哨兵，实际移动到的是 2 位置

  • Bottom() -> move(rank，n-1)：

把排序为 rank 的节点移动到排序为 n 的位置，因为我们垫了一个哨兵，实际移动到的是 n-1 位置

  • Insert() -> move(rank，rank+t)：

把排序为 rank 的节点，向后移动 t 位，移动到 rank+t 位置

以上三个函数调用我们前面所讲的move()函数，当成区间移动即可，下面是本例的重点：

  • Ask() -> Key2Rank()：

查询权值 num 所对应的中序遍历顺序，注意这里我们中序遍历不再按照权值递增维护（区分模版中的平衡树），而是维护当前线性序列的下标顺序，因此原本的 Rank2Key 函数失效

我们需要首先通过 pos 数组，把权值 num 反向索引到节点编号 ind，现在我们想知道 ind 的中序遍历顺序，只需要通过 splay 操作，把 ind 旋转到根位置，统计它左子树的节点数量即可

那么 ind 的排名 / 中序遍历顺序即为sze[ls[ind]] + 1，注意这个排名是算上队首哨兵的排序

  • Query() -> Rank2Key()：

查询排名 rank 对应的权值 num，该函数可以直接复用，先通过 Rank2Inex 得到对应的节点编号，再根据 key 数组直接所引到权值即可
