矩阵快速幂

基础知识

矩阵乘法

矩阵 A 和 B 能够相乘的前提条件是 A 的列数等于 B 的行数，即 A 是 n×k 的矩阵，B 是 k×m 的矩阵，则相乘后的结果为 n×m 的矩阵，做一次矩阵乘法的时间复杂度是 O(nmk)，需要经过三重循环

动态数组开辟

其中vector<vector<int>>ans(n, vector<int>(m, 0))用来初始化一个有 n 行，每一行有 m 列，每个位置数字默认为 0 的二维数组，等价于写成int ans[n][m] = {0}的形式，相当于动态数组的开辟，当我们想根据 n 和 m 的具体值开辟相应大小的空间时，而不是像常规数组一样把空间写死，就可以采用 vector 这种写法

矩阵快速幂

只有方阵才可以进行矩阵快速幂，在乘法快速幂中，我们用 res = 1 记录初始答案，在矩阵快速幂中，我们相应地使用单位矩阵记录初始答案

时间复杂度分析：进行一次矩阵乘法的复杂度为 O(n^3^)，总共进行了 logb 次，因此总体时间复杂度就是 O(logb × n^3^)

矩阵快速幂优化 DP 问题

一维 k 阶递推

对于**一维 k 阶固定地推关系的 DP **问题，可以使用矩阵快速幂进行优化，具体操作过程如下，以 Fibonacci 数列的递推过程为例进行演示：

易知 DP 的转移方程即为 Fibonacci 数列的递推通项：f(n) = f(n - 1) + f(n - 2)

那么我们矩阵快速幂的形式一定是：一个 1×2 的答案矩阵，每次和一个 2×2 的转移矩阵相乘，得到一个新的 1×2 的答案矩阵

且对应位置的答案是有顺序的，初始的答案矩阵从左到右分别是 f(1) 和 f(0)，经过与转移矩阵的一次相乘，新的答案矩阵从左到右就变成 f(2) 和 f(1)，也就是得到 f(2) 对应的答案

接下来是转移矩阵参数的确定，则和递推式的系数有关，a 的值就是 f(n-1) 项的系数，c 的值就是 f(n-2) 项的系数，因为这里二者都是 1，那么第一列的两个系数 a 和 c 就分别都是 1，而第二列两个系数 b 和 d 可以通过前几项递推代数的方式代出来

最后是求解次数的问题，由第一项推第二项，乘了一次转移矩阵，再推第三项，乘了两次转移矩阵，所以求第 n 项的时候，需要乘 n - 1 次转移矩阵，注意这里是因为基础项中最大是 f(1) 所以才 -1，如果是 3 项递推，最大项就应该是 2（根据 0 1 2 推 3），那么这里就应该变成 -2

一般式推导

注意矩阵快速幂只能优化这种具有规定格式的，一维 k 阶的 DP 线性递推问题，下面我们再以 5 阶递推关系为例，进行一次通用演示：

递推式：f[n] = 3×f[n - 1] + 2×f[n - 2] - 4×f[n - 3] + f[n - 4] - 2×f[n - 5]

因此我们设计的转移矩阵关系应该为：一个 1×5 的答案矩阵，每次和一个 5×5 的转移矩阵相乘，得到一个新的 1×5 的答案矩阵，因为是 5 阶递推，初始值 0 1 2 3 4 必须已知

其中 k(n - 1) 到 k(n - 5) 分别对应递推式中 f [n - 1] 到 f [n - 5] 的系数，也就是 3，2，-4，1，-2

因为求 f(5) 需要乘一次转移矩阵，求 f(6) 需要乘两次转移矩阵，因此求 f(n) 需要乘 n-4 次转移矩阵，也就是说需要乘转移矩阵的 n-4 次幂

常见题目

一维 k 阶问题的递推式比较简单，DP 入门中的 Fibonacci 数列，跳台阶等问题，都可以看成一维 k 阶递推的简单应用

k 维一阶递推

首先我们解释下什么是 k 维一阶问题，对于我们每个状态的 dp [ i ]，第 i 个状态的 dp 值中有 k 组答案，而第 i 个状态的 dp 值仅和第 i-1 个状态的 dp 值有关，这类 dp 的递推式成为 k 维一阶问题，也可以使用矩阵快速幂进行优化，具体什么是 k 维度一阶 DP 问题，最常见的就是用字符 / 短串拼长字符串的二维 DP 问题，我们通过以下两道例题来具体展示

LC1220

思路

这种形如用小串拼大串的题（这里的小串可以看成是 5 类单个字符），我们通常都可以定义 dp 表达式为 dp [ i ] [ j ] 表示构造长度为 i 的字符串，结尾字符为 j 所能形成的方案数，本例比较好的一点是，每个字符后面能拼接那些字符已经明确给出，我们对偶一下就可以得知每个字符前面能拼接哪些字符，换言之，也就是知道每个字符可以拼在哪些字符后面，即当前状态可以由哪些子状态转移而来

因为题目给出的是每个字符后面可以拼接哪些字符，本质上这还是一个 DAG 形式的更新关系，即每个字符结尾的状态可以更新出哪些状态，而不是看每个状态能被哪些状态更新出来，因此我们有以下关系：

  1. a 字符结尾：可以更新 e 字符结尾
  2. e 字符结尾：可以更新 a，i 字符结尾
  3. i 字符结尾：可以更新 a，e，o，u 字符结尾
  4. o 字符结尾：可以更新 i，u 字符结尾
  5. u 字符结尾：可以更新 a 字符结尾

也就是说我们每一维 i 对应的状态有 a e i o u 这五种结尾的情况，也就是 k 维一阶中的 k 值是 5，同时第 i 个状态的各个结尾情况，可以由第 i-1 个状态的各个结尾情况完全决定，因此我们说这种推导是一阶的，我们考虑 i-1 状态向 i 状态方案数的贡献矩阵：

这里还是推荐用向后转移的思想更新转移矩阵，也就是看当前以某个字符作为结尾，对下一位以哪些字符结尾的方案数有贡献，比如当前以 a 字符作为结尾，因为 a 后面只能拼 e，因此对下一位的结尾情况而言，只有 e 结尾位置为 1，其余位置都为 0，其余字符的贡献推导情况同理

初始化情况是长度为 1，也就是 i = 1 的情况，以单个字符结尾的方案数都是 1，要求长度为 n 的情况，则需要中间经过 n-1 次转移矩阵，因此需要乘转移矩阵的 n-1 次幂即可

LC552

思路

本例所抽象出的模型和上例完全相同，给出 A L P 这样三个字符，要求用这三个字符拼一个长度为 n 的字符串出来，同时对字符的使用有约束条件：

  1. 整个序列中最多有 1 个 A
  2. 整个序列中最多有两个连续的 L

因为对两个字符分别有要求，所以我们需要在上例的基础上增加一维，分别表示长度，结尾 L 的情况和 A 使用的个数，dp [ i ] [ j ] [ k ] 表示组成长度为 i 的长串，已经使用了 j 个 A 字符，结尾位置有 k 个连续的 L 字符所对应的方案数

因为矩阵快速幂优化的一般都是二维 dp 居多，我们考虑能不能把后两维进行压缩，因为这里其实 j 和 k 的取值范围其实很小，因此我们构造以下的映射：

  1. dp [ i ] [ 0 ] [ 0 ] -> dp [ i ] [ 0 ]：0 次缺勤，最后连续 0 次迟到
  2. dp [ i ] [ 0 ] [ 1 ] -> dp [ i ] [ 1 ]：0 次缺勤，最后连续 1 次迟到
  3. dp [ i ] [ 0 ] [ 2 ] -> dp [ i ] [ 2 ]：0 次缺勤，最后连续 2 次迟到
  4. dp [ i ] [ 1 ] [ 0 ] -> dp [ i ] [ 3 ]：1 次缺勤，最后连续 0 次迟到
  5. dp [ i ] [ 1 ] [ 1 ] -> dp [ i ] [ 4 ]：1 次缺勤，最后连续 1 次迟到
  6. dp [ i ] [ 1 ] [ 2 ] -> dp [ i ] [ 5 ]：1 次缺勤，最后连续 2 次迟到

接下来我们考虑各个 DP 状态可以转移到哪些 DP 状态，并且据此写出转移矩阵：

  1. dp [ i ] [ 0 ]：可以转移到 dp [ i+1 ] [ 0，1，3 ]
  2. dp [ i ] [ 1 ]：可以转移到 dp [ i+1 ] [ 0，2 ]
  3. dp [ i ] [ 2 ]：可以转移到 dp [ i+1 ] [ 0，3 ]
  4. dp [ i ] [ 3 ]：可以转移到 dp [ i+1 ] [ 3，4 ]
  5. dp [ i ] [ 4 ]：可以转移到 dp [ i+1 ] [ 3，5 ]
  6. dp [ i ] [ 5 ]：可以转移到 dp [ i+1 ] [ 3 ]

以第一个转移为例，0 次迟到 0 次缺勤的基础上，下一天可以是 A P L 中的任意一个，因此可以转移到 0 次迟到 1 次缺勤，1 次迟到 0 次缺勤，0 次迟到 0 次缺勤这三种情况下的任意一个，由此构建出的转移矩阵如下：
