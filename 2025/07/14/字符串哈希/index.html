字符串哈希

基础知识

基本思想：哈希的本质是从高维空间向低维空间的映射（字符串 -> 整数值），对于其他维度空间的信息，只要我们能够设计合理的哈希函数，都可以已完成类似的映射（(x, y) -> a×x + b×y 也可以看成是一个简单的哈希）

倒序维护哈希函数的优he点：

  1. 方便结合定长窗口的滑动，实现相邻窗口之间的线性推导
  2. 方便由前缀哈希值求解区间哈希值

例题

P3370

【模版】字符串哈希

思路

字符串哈希的模版，为了避免哈希冲突，我们可以使用不同的 base 值建立两套哈希（分别选取 base 值为 13131 和 9973），因此考虑可以把哈希函数进行简单的封装，得到 StringHash 类即可

本例中封装的 StringHash 类可以留待后用

LC1044

思路

看到题目询问可能存在的子串最长长度，考虑是否可以对答案进行二分，因为答案具有单调性，如果存在较长的两个重复子串，那么对这两个子串分别再取任意子串，一定可以得到长度不超过当前长度的重复子串，因此二分是可行的

而 check 函数的设计只需要根据我们二分得到的长度 len，线性扫描原字符串，存每一个长度为 len 的子串对应的哈希值，检查是否有哈希值出现两次以上即可

LC3292

思路

本例如果我们从 target 入手，实际上就是在问，如果把 target 这个字符串分成若干段，使得每一段都是来自于 words 中某个字符串的前缀，问最小的拆分数为多少，而拆分数越小，对应的也就是每一段的分段长度越大，也就是说对于以 i 位置作为起点的一段，我们希望其向后延伸的 len 值尽可能大

我们维护这样一个结构，dst [ i ] = j 表示 target 字符串中，以 i 位置作为起点，能够分割出的最大子段的结束的下一个位置在 j，假设 target 是 abcdefg，从 1 位置开始，假设能找到的 words 中的最大前缀是 bcde，那么 dst [ 1 ] 的值就是 5，也就是 f 对应的位置

首先是 dst 数组的求解，这里很容易想到应用上例的思想，以 i 位置向后延伸，假设存在一段长度为 len 的子串可以被某个前缀满足，那么这个子串的子串也一定可以被当前前缀对应的子串所满足，也就是说对应的 dst 值是具有单调性的，所以对于每个位置的 dst 值可以用二分来求解

而 check 函数的设计则涉及到检查 dst [ i ] = j 对应的 i ~ j-1 段的子串是否是某个 words 的前缀，我们不可能每次 check 的时候都遍历一遍所有的前缀，考虑先预处理出所有可能的前缀，找一个 set 存起来，每次去 set 中查找即可

而有了 dst 数组以后，我们就考虑怎么利用 dst 数组所维护的以 i 位置作为起点的最长分割子串求出 target 的最小分割数，这里可以考虑贪心思想，对于分割出的每一段，只要我们能往后找到更远的地方，我们就让这一段尽可能长

分析到这里，我们就可以类比之前贪心一章中分析过的下面这道题目来做了：

LC45

思路

本例是很典型的贪心思想，和原题一样，我们记录 dist [ i ] = j 表示 i 位置向后跳转最远能到达的位置是 j 位置

假设我们当前处在 i 位置，i 位置向后最远能跳到的位置是 j 位置，那么我们下一步跳转就可以在 i ~ j 之间任选一个位置，至于怎么选，我们显然要选一个 k 位置，使得 k 位置再向后跳一步能到达的新位置在 i ~ j 之间最远（这里我们看的是 dist [ k ] 而不是 k 本身的大小，因为即便我们选 j 位置，如果 j 向后能跳的位置很小，我们再跳一步就一定不是最优的）

因此我们每轮都以这样的方式更新当前位置 cur，并且同步更新它再跳一步所能到达最远的位置 nxt，跳跃结束的条件有两个：

  1. 无法跳到 n-1 位置，也就是说对于当前的 cur 和 nxt，他们之间所有的 k 位置所能跳到的下一个位置都在 nxt 之内，也就是说跳不过 nxt 的限制，此时一定到不了终点，最好的情况就是在 cur 位置起跳到 nxt，然后被卡在 nxt 位置
  2. 能够跳到终点，假设对于某个 cur 位置而言，它对应的 nxt 值已经超过了 n-1，那么显然它再跳一步就能够达到终点，此时我们直接退出，并且另 cnt 值加一即可（加上最后超过 n-1 要跳的一步）

在有了这道题的分析之后，我们思考会发现，字符串的分割其实和跳跃的原理是相同的，假设我们的字符串是 abcdefg，从 b 位置开始，我们能找到的最长前缀是 bcde，也就是 dist [ 1 ] = 5，我们完全可以等价地认为，从 1 位置开始跳，向后所能跳到的最远位置是 5 位置，因为我们最大取前缀为 bcde 时，下一次分割的起点就是 5 位置，如果我们取 bcde 的前缀，下次切割的起点就是 1 ~ 5 之间的某个位置

因此这两例题其实完全是等价的两个模型

唯一需要注意的是对于边界条件的特判，上一例要求我们恰好跳到 n-1 位置就行，对于 n-1 位置那一段本身我们是不管的，而本例要求我们把整个字符串都分割完，也就是说，假设我们跳到 n-1 位置，我们也要再跳一步把 n-1 这个位置的子串分割出来，所以我们的边界条件会变成 nxt < n，此外跳 0 步是分成一个子串，跳 1 步是分成 2 个子串，最后返回值和跳跃步数相比还是要 +1

LC1297

思路

本例涉及到重复子串的一个重要性质：求出现次数可能最多的子串，给定长度下界和长度上界，我们只需要定目标长度为下界即可，因为越短的字符串越容易重复，假设较长的字符串出现了，那么对应的它的子串一定也会出现，而子串必然可以从上界长度减到下界长度，因此本例根本不需要考虑长度上界

此外对于子串中不同字符个数的限定，我们考虑用一个滑窗，在遍历的时候，每次加入头部元素，删除尾部元素，O(1) 动态维护窗口内不同元素的数量总和即可
