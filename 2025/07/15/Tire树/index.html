字典树及可持久化

Tire 树

基础知识

树的节点代表集合，树的边代表关系

核心操作主要有某个序列（字符串、整数的二进制表示，数列）的插入操作 insert，以及对某个序列的查询操作 query

一些需要注意的地方：

  1. 一般规定 0 号节点为根节点，所有单词路径初始化都是 p = 0 继续向下搜索
  2. 开点大小：一般开一个二维数组 ch，第一维大小为序列总数 × 单个序列长度，或者题目会直接给出所有序列总长度，第二维大小为最大分支数量，常见的有三种（全是小写字母的序列 - 26；01Tire - 2；数列 - 数列中最大元素大小）

Tire 树例题

LC820

【Tire 树处理前缀问题】

思路

本例描述的很抽象，大概翻译一下就是，对于原集合中的字符串，如果某个字符串是另一个字符串的后缀，我们就扔掉这个字符串，问最后集合中会剩下多少字符串，返回剩下字符串的总长度+总个数

首先我们显然把后缀问题转化成前缀问题，也就是把所有字符串先翻转，那么问题就变成了，在插入过程中，我们要舍弃那些作为别人前缀出现的字符串，那么就会有两种情况（假设 a 是 b 的前缀）：

  1. a 先插入，b 后插入
  2. b 先插入，a 后插入

显然第一种情况好处理，因为我们只需要在插入 b 的时候判断，是否存在某个节点，之前已经插入过字符串（我们记录一个 ds 数组在每个字符串结尾位置对应的节点处存这个字符串），如果存在，它一定是 b 字符串的前缀，直接把该处的 ds 设成空即可

因此我们要保证先插入短串，后插入长串，对所有字符串升序排序即可

LC1233

【Tire 树处理前缀问题】

思路

本例和上例完全是对偶问题，上例是具有前缀关系的时候，舍弃短串保留长串；而本例是具有前缀关系的时候，保留短串舍弃长串，做法完全相同，我们还是先插入短串，后插入长串，当插入长串过程中遇到某个作为其前缀的短串已经存在的时候（ds 数组不为空），直接从 insert 函数中返回，也就是停止插入即可

本例还有一个易错点，就是 a / b / c 和 a / b / cc 在 Tire 树中具有前缀关系，但是实际上并不是文件包含关系，对于这种特判的处理，我们可以给每个字符串末尾都加上一个 /，变成 a / b / c / 和 a / b / cc / 再处理即可

LC677

【数据结构设计】

思路

本例其实和下面那道题思路很类似，都是要查询以某个前缀开头的满足要求的字符串（半开放性约束，前面指定前缀，后面无要求），比较朴素的思路是 Tire + dfs，先按照前缀向下遍历到前缀结束的节点 p，再以 p 作为起点向下搜索，利用 dfs 找到后面所有的合法字符串，每个字符串的权值就存在它结束位置对应的节点上

我们也可以利用节点信息进行优化，我们定义 value [ i ] = j 表示以 i 号节点作为结尾对应的前缀，其对应的权值和为 j，这样我们相对于上面的思路则做了两点改变：

  1. insert 的时候，val 权值不再是只在结束位置赋值，而是在路径上的每个节点都需要加上，因为如果 apple 这个字符串对应权值为 10，那么 a，ap，app，appl，apple 都可以看成这个字符串的前缀，那么这些前缀都可以引出 apple，因此他们对应的权值也都要+10
  2. query 的时候，因为我们的权值已经同步更新到每个前缀，就不需要从前缀再去搜索它能得到的每个字符串了，通过 Tire 树遍历到目标前缀结尾的节点，直接返回这个节点的 value 值即可

LC1268

【字典树维护更多节点信息】

思路

在字典树中，我们所维护信息的种类一般有两种：

  1. value 类：只在完整字符结尾的节点处赋值，表示该节点处有几个完整的字符串作为结尾，或以该节点作为结尾的字符串的具体内容
  2. cnt 类：对每个节点都赋值，表示在 insert 过程中，这个节点被用到的次数

对于本例而言，我们的思路是先按照 products 把字典树建立起来，对应的每个节点我们都维护一个 vector，存三个字典序最小的字符串（因为字典树本身的性质，能出现在同一个深度为 h 的节点位置的字符串必然是前 h 位前缀相同的），也就是对于某个 s，假设 s = abcd，我们在 a，ab，abc，abcd 四个前缀对应的节点上，都把 abcd 这个字符串存一遍，接着我们用 searchWord 去卡这些字符串，以 searchWord 作为 target 去查询，对应路径上的节点对应的 vector 中有内容就是有答案，没有就说明不存在以 searchWord 当前子串为前缀 products 字符串

这里就体现出我们按照 cnt 类型存节点信息的好处了，因为我们的 searchWord 是以前缀子串的形式逐个进行查询的，我们希望在查询的每一步都能直接得到答案，因此给每个节点都存上信息显然是更好的选择

LC2261

【使用 Tire 树去重】

思路

字典树其实还可以用来去重

对于本例而言，我们把所有满足条件的子数组都当成一个字符串插入字典树中，如果我们插入了一个之前插入过的子数组，那么它所经过的每一个节点都不会是新的，也就是全程中没有创建过新的节点，此时就可以知道这个子数组出现了重复，而最后 Tire 树中的节点数，就是去重后满足条件的子数组的个数，因为我们的子数组是逐个元素扩展的，假设我们目前有了 1 2 3，下一次我们压入一个 4，得到 1 2 3 4 这个子数组，那么向 Tire 树中插入时，前三个节点都存在，只有 4 对应的节点会开辟，节点数+1，对应找到一个新的子数组

HZOJ-282

【01字典树】

思路

所谓 01 字典树，就是把一个整数的二进制序列从高位到低位当成一个字符串来存，因为二进制位只有 0 和 1 两种情况，因此 Tire 树此时只有两个分支，故而称为 01 字典树

对于本例，我们依次把遍历到的数字 x 插入到 Tire 树中，对于当前的数字 x，我们去 Tire 树中已经维护的数字中，找尽可能和 x 对应位置 01 数位恰好相反的数字即可

可持久化字典树

字典树支持的查询：在第 1 ~ n 次插入的单词中，是否存在单词 x

可持久化字典树支持的查询：在第 i ~ j 次插入的单词中，是否存在单词 x，换言之，可持久化字典树可以支持当我们已经进行了 n 次插入以后，我们可以得到任意第 k（k <= n）次插入完成时，树上的所维护的单词信息

重点：

  1. 插入版本（历次插入后信息）的维护
  2. 区间信息的统计技巧（P4375 例题）

节点插入

可持久化字典树再插入新单词的时候，所有新单词路径有关的节点都会重新生成（不管他的前一个版本中有没有这条路径，如果有，两个版本会都经过这条路径下沉一层，如果没有，那么上一个版本再下沉一次直接指向 0 号节点，后续不再提供有效的节点信息继承），其他节点则会继承上一个版本的旧的节点信息

比如我们在 3 号版本的基础上生成 4 号版本的时候，4 号版本要插入的单词的第一个字符是 a，即便第 3 号版本已经有字符 a 的路径，为了不破坏第 3 号版本所维护的结构，4 号版本要重新生成字符 a 的路径，同时两个版本都沿着字符 a 的路径向下，接着 4 号版本添加第二个字符 b 的路径，同时继承 3 号版本所有其他的路径，也就是 3 号版本的 d 路径，最后两个版本都沿着字符 b 的路径向下，4 号版本添加最后一个字符 g 的路径，同时继承 3 号版本其他的路径，也就是字符 c 的路径

区间查询

对于区间查询，我们需要重点考虑以下三个思想：

  1. 前缀和思想：l 到 r 之间的版本中某个 s 出现次数的查询，等价于 r 版本查询到的出现次数，减去 l-1 版本查询到的出现次数
  2. 层的思想：r 和 l-1 版本始终是层数对应的，二者会沿着待查询字符串 s 的路径，共同从他们的根节点向子节点，同步转移
  3. cnt 数组的含义：我们引入 cnt 数组，cnt [ i ] 表示编号为 i 的节点出现的次数，假设 r 版本中，i 号节点出现了 a 次，l-1 版本中，i 号节点出现了 b 次，那么 l ~ r 版本中 i 号节点就出现了 a-b 次，而 i 号节点是由其父节点经过唯一确定的路径走到的（因为 r 版本和 l-1 版本都在走相同路径下降，可能他们走到 i 号节点的父节点不同，但是走的路径一定相同，比如都是 a 字符），因此节点的出现次数就等价于特定路径出现的次数，如果 a-b > 0，则说明 l ~ r 版本之间有 a 这条路径加入，那么目前为止以 a 作为结尾的前缀就存在于两个版本之间

对于上图而言，除了每个版本根节点标注的是节点编号之外，其他节点标注的信息就是每个节点对应的 cnt 值，我们以 4 号和 1 号版本为例，看第二层的节点，3 - 1 = 2，说明在 2 ~ 4 版本内，以 a 作为前缀的字符串被插入了两次，而事实也确实如此

P4375

【可持久化字典树模版】

思路

本例是可持久化 Tire 树的典型应用

首先看到区间异或和，很容易想到前缀和求解，我们记录 S 数组是原数组 a 的前缀异或和，那么原题待求解表达式就转化成为S[N] ^ S[p - 1] ^ x，而对于一次查询而言，S[p - 1] ^ x是定值可以记为常值 C，因此我们需要在区间 l-1 ~ r-1 范围内找一个 S [ p ]，使得 S [ p ] 与 C 异或得到的结果最大（这里之所以范围是 l-1 ~ r-1，是因为 p 的范围是 l ~ r，而 p 是区间异或和的起点，转化为前缀和端点需要前移一位）

如果没有对 p 位置的区间限制，那么本例完全等价于上例，加上区间限制以后，我们考虑用可持久化 Tire 树来求解，我们记 Tire 的每一个版本对应一个区间范围，版本 1 对应 S 数组的 11 区间，版本 2 对应 S 数组的 12 区间，版本 k 对应 S 数组的 1~k 区间，依次类推，这也是可持久化字典树维护区间信息的常用表示方式

接下来我们仿照上例的思路，对于常数 C，我们在可持久化的 l-1 到 r-1 版本中，从数位的高位到低位，依次找尽可能和 C 对应数位 01 相反的数字，这里因为可持久化的缘故，我们不能再简单的根据 ch 的信息判断是否存在满足要求的数位，而是需要再次使用前缀和的思想，我们用 cnt 数组记录每个节点出现的次数，假设我们当前要查找的区间范围是 a 到 b 版本，a-1 版本和 b 版本目前所处的节点分别是 ra 和 rb，我们当前想向下查找 1 的路径是否存在，也就是 a 到 b 之间，从 ra 和 rb 这一层向下，是否有新的 1 插入，我们只需要用ch[rb][1] 对应的 cnt 值和ch[ra][1]对应的 cnt 值作差即可，如果差值大于 0，则说明 ra 和 rb 之间存在这样的路径

换言之，我们现在想在 a 和 b 版本之间找到满足条件且异或值尽可能大的一条路径，只需要在对应层（也就是对应的 ch 节点）依次把 b 版本的 cnt 值和 a-1 版本的 cnt 值作差，即可作为路径选择的判定依据，因为我们在每一层都需要统计 0 和 1 的数量，因此我们的 cnt 数组是按层记录的，它表示的某个节点出现的次数，也就等价于上层指向这个节点的 0 或者 1 的路径出现的次数，这里要和我们常规意义下的 value 数组记录的完整单词进行区分
