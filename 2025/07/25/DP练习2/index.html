DP 练习2

通过阅读本文，您可以大致了解状压 DP 和树形 DP 相关的简单应用

UVA12186

（如果懒得注册 UVA 账号可以做 P3408，几乎是一模一样的题）

思路

本例属于树形 dp 的一个小变形：dp + 贪心，对于每一个节点 u，他在自己的子节点 v 中进行决策的时候，需要进行一些简单的贪心，完成每一层自下向上的推导

我们定义 dp [ u ] 表示 u 节点能够向上反馈请愿书，所需要的最少员工数，那么假设 u 有子节点 v1，v2……vk，这些子节点不管是谁向上反馈请愿书，他们的贡献都是相同的，都是 +1 的贡献，但是他们的代价不同，显然他们每个人的代价都是 dp [ vi ]

因此我们有简单的贪心：对于每一层都把当前节点 u 的所有子节点 v 按照 dp 值从小到大排序，依次选 dp 值小的节点来提交请愿书，此外我们还需要统计与每个节点直接相连的子节点个数 sze，只要提交请愿书的节点数 / 总子节点数超过阈值以后，就停止 dp 的累加

这里我们需要在每一层遍历子节点的同时，对当前层下一层的子节点做一次排序，整体复杂度为 O(nlogn)

P5911

思路

相比于单点的状态压缩，本例可以看成是多点状态压缩的模版，一般的状压 DP 我们都是每次向集合中加入一个点（比如 TSP 问题中我们每次把新到达的那个单点加入到我们已经走到过的集合中）

而在本例中，我们用 st 的二进制表示已经过河的人的序列，1 表示都已经过河，0 表示还没有过河，最后要求的状态就是 st [ (1 << n) - 1 ]

我们考虑这个状态怎么转移得到，很显然它可以从它的任意子集转移而来，比如如果我们当前要求的状态为 111，那么它可以由状态 000,001，010,100,011,110,101 等状态转移而来，对应这些子状态需要叠加的状态就是 111,110,101,011,100,001,010，也就是说我们需要枚举当前状态的子集，这里每经历一次转移，新加进来的元素，我们就让这些人分成单独的一组过桥

比如我们让 111 从 010 转移过来，那么新加进来的状态就是 101，也就是我们要让第 1 个和第 3 个人一起过桥，因此我们需要先预处理出来任意分组情况下，这些人过桥的总重量和总时间，只有重量不超过 m 的分组，才可以作为转移媒介，向子状态进行转移
