DP练习1

通过阅读本文，您可以大致了解和区间 DP 与数位 DP 有关的一些简单题目

HDU3506

思路

题意翻译：每个位置都对应一个数值，合并两组元素的代价就是这两组元素的数值之和（比如合并 ab 和 cd 这两个区间，代价就是 ab 和 cd 上所有元素的数值之和），序列是成环的（也就是首尾相连，第一个元素和最后一个元素也可以合并）

其实就是合并石子的加强版，我们只需要破环为链，把序列平移复制一倍以后，做区间 DP 即可，最后的最优解在所有的 dp [ i ] [ i+n-1 ] ( i from 1 to n ) 中产生，这么做我们发现，序列长度为 1000，复制一份后为 2000，朴素的合并石子的区间 DP 的复杂度为 O(n^^3) 显然会超时，这里我们引入四边形不等式进行优化

※ 四边形不等式

四边形不等式优化（Quadrangle Inequality Optimization，也叫四边形不等式单调性优化）通常用于 区间 DP 或 序列划分型 DP 中，可以将原本的 O(n^3^) 降为 O(n^2^)

对于区间 DP 中的 dp 式，dp [ i ] [ j ] 一般表示处理从 i 到 j 范围的区间到目标状态的代价，因此我们可以将其等价为一个二维的代价函数 w( i, j )，对于该代价函数w(i,j)，若满足对所有i ≤ a ≤ b ≤ c ≤ d ≤ j，都有：

w(a,c) + w(b,d) ≤ w(a,d) + w(b,c)成立

或者有一种更常见的判别方式，对于任意的 i 和 j 都有：

w(i,j) + w(i + 1,j + 1) ≤ w(i,j + 1) + w(i + 1,j)成立

我们就称该代价函数 w 满足四边形不等式，以下是常见的满足四边形不等式的代价函数的示例（可以记住前两个比较常见的代价函数）：

代价函数 w(i,j)w(i, j) 应用场景
(Sj - Si)^2或[sum(ai)]^2，其中 S 是前缀和 区间平方差代价划分，比如数列平方和最小化
sum(ck), k from i+1 to j ，且 ck 单调不减 区间和成本，单调性有助于满足四边形不等式
variance(i,j)或其他凸代价 需要最小化方差或偏差
Huffman 编码的代价w(i,j)=freq[i,j] * depth Optimal Merge Pattern、编码等问题

而满足四边形不等式的模型有一个重要的性质：决策点具有单调性，也就是我们记 s [ i ] [ j ] = k 表示：通过子区间合并 i 到 j 区间时，我们选择 k 作为区间分点，用 ik 和 k+1j 这两个子区间合并得到 ij 区间，所得到的代价最小，那么我们在合并 ij 区间，枚举分点 k 的时候，不再需要从 i 到 j 范围枚举，而是可以直接从s[i][j-1]到s[i+1][j]范围枚举即可

P1005

思路

首先给出本例贪心错误的反例（贪心的思路是对于一个序列每次从两侧选一个较小的取出，用小的数对应小的权重，利用排序不等式求得最值，该思路求解不了形如 3 1 2 3 即两侧数字相同的序列）

本例是典型的区间 DP结合逆向思维构造序列的题目，即题目要求我们从两侧开始取数，我们可以转化成优先构造子区间，扩展到完整区间的模型，这样我们每次拿首尾的两个数之一，就等价转化成子区间 i~j 每次选择 i-1 位置或 j+1 位置的两个数之一进行扩展的问题，最后需要注意，我们构造区间和从区间取数的顺序是相反的，因此对应的权值也要反过来对应，先构造的数对应大权值，后构造的数对应小权值

同时，显然可以证明，每一行之间的的收益值是独立的，因此我们可以按照每一行先单独求解，再统一贡献答案即可

P4805

思路

本例也可以看成石子合并的变式，添加了以下两种额外的考虑：

  1. 我们合并两段区间的前提是这两段区间的区间和值相等，如果区间和值不相等，我们不能进行合并
  2. 本例加入了合并三段区间的情况，也就是对于某个区间 i~j，需要在其中找两个分点 l r，把 i 到 l，l+1 到 r，r+1 到 j 这三段进行合并

对于 1 的处理，我们需要给区间合并加上一个判断条件，当我们合并 i 到 j 区间，以 k 作为分点时，需要判断 i ~ k 和 k+1 ~ j 范围的区间和值是否相等，只有相等时才能合并，而那些无法完成有效合并的区间，我们就让区间和值默认为 0，表示这段区间无法合并，其上代表的最大值是无效的

对于 2 的处理，通过本例区间合并必须要求子区间和值相等的特殊要求，我们发现：

  1. 对于区间 [ l, r ]，我们如果想把它和其他区间合并，一定要取它之上的所有数字之和，来和其他区间的和值进行比较，此时区间和值具有单调性，区间越长，对应的和值也一定越大

  2. 对于一个确定的左端点 l，那么对于右端点 r1 < r2 < r3 <…，有 [ l, r1 ]< [ l, r2 ] < [ l, r3 ] <…，即区间值具有单调性，确定右端点同理

所以，我们可以得到下面的性质：

  1. 在一个区间内，只要找到一种满足题目要求的合并方案，就可确定这段区间的值，直接跳出本区间的循环，从而减少循环次数

  2. 在合并三个饭团时，若枚举到的中间区间为[ k, t ]，我们让 k 与 t 分别向中间靠近，[ l, k−1 ]与 [ t+1, r ] 两个区间逐渐增大，在此过程中，如果区间可以合并，那么一定能够通过分别扩大两个子区间，找到这个可行的状态，因此可以使用双指针分别控制 k 和 t 的移动，减少一层循环，达到优化效果

HDU4734

题意翻译：给一个上限 F(Up)，问从 1 到 n 中有多少个数，满足这个数的数位按照 F(A) = An × 2^n-1^ + An-1 × 2^n-2^ +……+ A2 × 2 + A1 × 1 不超过给定的上限 F(Up)，问满足这样条件的数的个数，多组测试数据

思路

这种类型的数位 DP 就很难求出对应的 dp 预处理数组，因为一般来讲，如果我们想用 dp 预处理做的话，限定条件一般都是不含哪些数字，或者不含哪些连续的数字，此时我们一般定义 dp 数组含义为：dp [ i ] [ j ] 表示以数字 i 开头，长度为 j，除第一位以外剩下的数位随便填，所得到的数字数量，而在本例中，最大的问题就在于我们每次给出的上限 F(Up) 是不确定的，因此我们没办法得到统一的 dp 预处理数组，因此考虑记忆化搜索的形式

那么我们设计搜索状态，首先有 ind 表示当前遍历到了上界的哪一位，free 表示是否可以任意填数，因为我们要判断 F(A) <= F(Up)，我们还要边构造边记录我们当前构造的数字的权重数位和 sum，如果 sum 超过限制，就直接剪枝，因此我们很容易就设计出如下的搜索状态，使用三个参数即可完全表示：

接下来我们考虑记忆化加速搜索过程，这里数位 dp 记忆化的经验就是，只记录 **free == 1 **的状态，因为这些状态可以随意填数，对应的就是预处理出来通用的 dp 数组，而 free == 0 的时候，此时的方案严格收到我们给出当前的上界的限制，如果换一个上界的话显然就是另一回事了，我们记这样的状态意义不大还容易出错

因此我们记录 dp [ ind ] [ sum ] 表示当前可以任意填写数字，填到数位的第 ind 位，数位权重累加和为 sum 时，后面构造数字的方案数，注意因为 Up 的不同，我们上限得到的 limit 也不同，因此每一次搜索的 sum 对应的 dp 状态都不同，所以每次搜索前都需要清空 dp 数组进行初始化

我们把这版代码提交，发现会超时，原因就在于我们每次都需要清空 dp 数组，直接拉高了我们的时间复杂度，因此我们考虑能否设计出一个通用的状态，类似于我们的预处理 dp 数组，只初始化一次即可，也就是要排除掉 sum 和 上界 limit 限制之间的关系，其实这个思路很好想，我们现在每次都是从 0 开始，逐渐累加 sum 到 limit 上界，我们可以把这个过程反过来，从 limit 开始，逐渐刨除 sum 直到 0，因为 0 是一个确定的下界，那么我们的 dp 状态定义可以修改为：当前数位为 ind，还剩 sum 大小的余量可以使用，能够任意填写，能够构造出的数字的个数，这样我们就统一了边界为 0，不同的上界 limit 进来之后只会更新上层的 dp，完全可以复用子状态中的 dp

P4999

思路

本例要求的就是从 L 到 R 这个区间内每一个数字的所有数位之和，本例也是一个很难用预处理 dp 数组进行递推的题目，因此我们还是考虑记忆化搜索的解法，我们定义搜索状态为当前搜索到的数位为 ind，当前是否可以任意填写数字的标志为 free，以及当前已经填好的数位的数位累加和为 sum，除此之外本例再没有其他限制，只要搜索到数位的最后一位，对应的 sum 值就是当前数字的数位累加和，我们直接返回即可

接下来我们考虑记忆化优化，本例需要记忆的状态只有一个数位 ind，我们记 dp [ ind ] 表示从数位 ind 位置开始，如果可以随意填写数字，后面填写的那些数位的累加和，那么如果我们搜到一个已经存在的数位状态，假设此时已有的数位和是 sum（我们通过本次搜索搜到的），后面累加的数位和是 dp，值得注意的是，这两部分答案并不是对等的，前者只是一个数字部分数位和，而后者则是这些数位上所有数位所能填写出来的数字的数位和，因此前者要和后者结合，则需要把前者已有的数位和和后者的每一种情况加以组合，假设后者的数位位数为 ind+1，那么对应的数字情况就是 10^ind+1^，因此此时返回的记忆化状态应该是：
