AI 提示工程

提示工程 - prompt engineering

在与 AI 大语言模型交流的过程中， 提示指的就是用户输入给大模型的问题或语句，大模型会根据提示给出相应的回答，而提示工程就是用来研究如何提高和 AI 的沟通质量与效率的

提示规范

对于提示词的设计，可以参考 OoenAi 官网文章《用API进行提示工程的最佳实践》，对应网址为：

https://help.openai.com/en/articles/6654000-best-practices-for-prompt-engineering-with-openai-api

以下是一些可能的提示规范：

  1. 使用最新的模型（数字越大，开头字母字典序越大一般代表模型版本更高）
  2. 把指令放在提示的开头，并且用 ### 或者 “““ 来分隔指令和上下文
  3. 尽可能对上下文和输出的长度，格式，风格等给出具体，描述性，详细的要求
  4. 通过一些具体的实例来阐明你想要的输出结果或格式
  5. 先从零样本提示开始，如果效果不好，再选用小样本提示
  6. 减少空洞，泛泛而谈，不严谨的描述
  7. 尽量告知 AI 应该要做什么，而不是告诉他不要做什么

JSON

JSON 中常见的数据结构有两种：JSON 对象 {} 和 JSON 数组 []，其中 JSON 对象可以转化成 Python 中的字典，而 JSON 数组可以转化成 Python 中的列表

JSON 对象

对象以 {} 作为开头和结尾，内部的每个元素都是键值对，元素之间以逗号分隔，格式和 Python 中字典格式相同

JSON 对象中的键必须是字符串，且字符串必须以双引号包含，不能以单引号包含

JSON 对象中的值必须是以下几种：

  1. 字符串："star"
  2. 数字：31
  3. 布尔值：true或false
  4. 数组：["hi", 7]
  5. 对象：{"age", 25}
  6. 空值：null

JSON 数组

数组以 [] 作为开头和结尾，内部的元素也可以是字典类型，元素之间以逗号分隔，格式和 Python 中列表格式相同

LangChain

在使用 LangChain 前，请确保已经安装了 langchain 和 langchain_openai 库

LangChain 消息列表

LangChain 中的模型主要有 LLM（语言模型）和 Chat Models（聊天模型）两种，前者以字符串作为大模型的输入和输出，后者以消息列表作为大模型的输入，以消息作为大模型的输出

而消息列表中所包含的消息又分为以下三种：

  1. 系统消息：SystemMessage，表示对 AI 系统的指令
  2. 人类消息：HumanMessage，表示来自用户的聊天消息
  3. AI消息：AIMessage，表示 AI 大模型的回答

其中这三个消息模块都可以从库langchain.schema.messages中导入，并且进行相应的对象实例化，目前只需要传入 content 一个参数，表示这条消息的具体内容：
