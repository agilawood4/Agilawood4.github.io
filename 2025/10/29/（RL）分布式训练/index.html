分布式训练

在大模型训练的过程中，我们需要同时考虑计算效率和显存效率两方面的问题：

  • 计算效率：大模型的训练依赖于海量的训练数据，海量的训练数据带来了海量的计算需求
  • 显存效率：大模型主要体现在模型的参数规模上，参数规模逐渐变大的模型对显存的依赖逐渐加剧

所谓的分布式训练，就是解决怎么让我们的模型能够在多张 GPU 显卡上同时运行的问题，通过阅读本文，您可以大致了解和分布式训练以及accelerate模块有关的相关内容

分布式训练基础

分布式训练简介

分布式（Distributed）是指系统或计算机任务被分布到多个独立的节点或计算资源上进行处理，而不是集中在单个节点或计算机上

分布式模型训练是一种机器学习和深度学习领域中的训练方法，它通过将计算任务分发到多个计算资源或者设备上来加速模型的训练过程，分布式训练通过并行计算的方式，将数据和计算任务分配到多个节点上，从而提高训练速度和处理大规模数据的能力

常见的分布式训练策略

假设我们的数据集是Data，模型是一个[L0,L1]的两层网络，常见的分布式训练策略如下：

（1）数据并行（Data Parallel，DP）：每个 GPU上都复制一份完整模型，但是每个 GPU 上训练的数据不同，即把整个数据集划分为多份，在不同 GPU 上运行不同的子数据集

此时要求每张卡都能够完整加载整个模型，如果模型本身参数量很大，一张卡显存不够，该方法同样失效

（2）流水并行（Pipeline Parallel，PP）：把模型按层拆分，每个 GPU 上都包含模型的部分层，保证能够正常训练，此时不要求每张卡内都可以执行完整的训练流程

（3）张量并行（Tensor Parallel，TP）：把模型每层的权重拆开，对于每一份权重，每个 GPU 上各包含一部分，保证能够正常训练，此时不要求每张卡内都可以执行完整的训练流程

（4）3D并行：我们可以考虑把上述三种并行方式混合使用，比如在下图中，我们分别采用了 2 路数据并行，4 路流水并行，4 路张量并行的实现思路，总共需要用到2*4*4=32张 GPU 显卡

Data Parallel

这里我们先介绍基于nn.DataParrallel所实现的数据并行方法

Data Parallel 基本流程

Data Parrallel 大致可以分为以下几个流程：

  • GPU0 加载 model 和 batch 数据
  • 把 batch 数据从 GPU0 均分到各卡
  • 把 model 从 GPU0 复制到各卡
  • 各卡同时进行前向传播
  • GPU0 收集各卡上的输出，并且计算 Loss（实际上 Loss 值是在各卡上直接算好的，GPU0 负责统一回收）
  • 把 Loss 分发到各卡，进行反向传播，计算梯度
  • GPU0 收集各卡的梯度，进行汇总
  • GPU0 更新模型

Data Parallel 推理

实际运行中，如果我们使用 nn 模块中的 Data Parallel 调用多 GPU 进行训练，训练速度其实没有多大的提升，甚至有可能下降，因而其实我们很不推荐使用 nn 模块中的 Data Parallel 进行分布式训练

nn.DataParallel存在的问题：

  • 其本身实现是一个单进程，多线程的过程，由于 GIL 锁的问题，不能充分发挥多卡的优势
  • 由于 Data Parallel 本身训练策略的问题，各个副卡只负责前向计算和反向传播，而优化器等全部部署在主卡上，会出现一个主节点占用远高于其他节点的情况
  • 效率较低，每次训练开始都要重新同步模型，大模型的同步时间会比较难接受，尤其是设置 batch_size 较小的时候
  • 只适用于单机训练，无法支持真正的分布式多节点训练

实际上，Data Parallel 更多被用在并行推理验证的过程中，因为在推理过程中，我们不会对模型进行更新，因此也就省去了每一轮推理都要把模型从主卡复制到多卡，进行重新部署所花费的时间，这是我们节省时间的主要原因

Distributed Data Parallel

Distributed Data Parallel 基本流程

Distributed Data Parallel 才是真正的数据并行化训练，其大致流程如下：

  • 使用多个进程，每个进程都加载数据和模型
  • 各个进程同时前向传播，得到输出
  • 各个进程分别计算 Loss，反向传播，计算梯度
  • 各进程间通信，将梯度在各卡同步
  • 各进程分别更新模型

相当于每张卡上都会有单独的前向和反向传播，只是会在过程中把各张卡的梯度进行同步

分布式训练中的基本概念

分布式训练中的基本概念如下：

  • group：进程组，一个分布式任务对应一个进程组，一般就是所有卡都放在一个组里
  • world_size：全局的并行数，一般等于总的 GPU 卡数
  • node：节点，可以是一台机器，或者一个容器，一个节点内包含多张 GPU 卡
  • rank / global_rank：整个分布式训练任务内的进程序号
  • local_rank：每个 node 内部的相对进程序号

下面以 2 机 4 卡分布式训练为例进行说明：

  • node=2；world_size=4（这里每两张 GPU 卡共同完成一个训练流程）；每个进程占用两张 GPU

分布式训练中的通信

通信的概念：在分布式训练中，通信是不同计算节点之间进行信息交换以协调训练任务的关键组成部分

通信类型：

  • 点对点通信：将数据从一个进程传输到另一个进程
  • 集合通信：一个分组中所有进程的通信模式称为集合通信

常见的集合通信：

  • Scatter，Gather，Reduce，All Reduce，Broadcast，All Gather

其中 Scatter 和 Gather 是一组对偶的操作，把信息从一台机器分发到多台机器的过程即为 Scatter，把信息从多台机器汇总到一台机器的过程即为 Gather

而 Reduce 则是在 Gather 的基础上，不仅汇总信息，同时对汇总到的信息做聚合计算（示例中为加法），All Reduce 则是在此基础上，把聚合后的信息从一台机器分发到剩下的机器上

而 Broadcast 则是把一台机器上的同一个信息复制多份，然后分发到其他各个机器上，All Gather 则是在 Gather 的基础上，把各个机器的信息汇总后，再更新给所有的机器，而不仅仅是一台机器

Accelerate

Accelerate 基础

Accelerate 简介：

Accelerate 是 HuggingFace 生态中针对分布式训练推理提供的库，目标是简化分布式训练的流程，这个库本身不提供分布式训练的内容，但是其内部集成了多种分布式训练的框架，如 DDP，FSDP，Deepspeed 等

Accelerate 库提供了统一的接口，一套代码搞定多种分布式训练框架，只用在源代码上加几行代码，即可让单机训练程序变为分布式训练程序

Transformers 库中也是通过 Accelerate 集成的分布式训练，因此 Accelerate 库的学习非常有必要！
