机器阅读理解任务

机器阅读理解任务简介

机器阅读理解任务（Machine Reading Comprehension，MRC）是一项通过让机器回答基于给定上下文的问题来测试机器理解自然语言程度的任务，简单来说即给定一个或多个文档 P，以及一个问题 Q，输出问题 Q 的答案 A

机器阅读理解任务的形式是较为多样化的，常见的类型包括完形填空式（让模型去填空），答案选择式（让模型从候选答案中选择），片段抽取式（答案在描述文档中供模型提取），自由生成式（模型自由发挥给出答案），我们下面将以片段抽取式为例进行演示实现

我们的核心任务建模为：定位答案在文档中的起始位置和结束位置

  • context文本描述：一段包含答案的提示文本

  • question 问题：我们希望模型回答的问题

    • text参考答案内容：相当于训练模式下的标签，和答案起始位置一一对应

机器阅读理解评估指标

机器阅读理解任务常用的评估指标有两个:

  • 精准匹配度（Exact Match，EM）：计算预测结果与标准答案是否完全匹配
  • 模糊匹配度（F1）：计算预测结果与标准答案之间字级别的匹配程度，F1 的计算需要以下两个值的求解
    • 召回率（Recall）：正确答案中，被模型预测出来的字数，简记为 R
    • 准确率（Accuracy）：模型预测的答案中，在正确答案之内的字数，简记为 A
    • 计算公式：F1=2∗R+A1R+1AF1=2*\frac{R+A}{\frac{1}{R}+\frac{1}{A}}F1=2∗R1​+A1​R+A​

我们通常需要把数据处理成如下的形式：

其中两个特殊标识符含义如下：

  • [CLS]：是 BERT / Transformer 系列放在序列最前面的特殊标记
  • [SEP]：分割标记，用于把不同的句子 / 段落隔开，在这里第一个[SEP]用于分隔问题 / 提示文本，第二个[SEP]表示序列结尾

答案位置的定位：

  • start_positions / end_positions：在分词后的 Token id 序列中用于标识答案范围的索引，我们训练的目标就是让模型预测出这两个位置，从而圈定答案（特别地，如果某个切片中不含答案，常见做法是把start=end设为[CLS]的位置，告诉模型本切片无答案）
  • offset_mapping：一张字符级别 <=> Token id 级别的对其表，用于把原始文本的字符起止位置和分词后 Token id 的起止位置进行映射，实现训练时根据数据集中给出的字符级答案 span，找到对应的 Token 级start/end；预测时把 Token 级的答案反查回原文的字符片段，保证抽取结果与原文对齐

Context过长问题的解决（模型通常有max_seq_length的上限，而问题+上下文+特殊符号往往超过这个上限，此时必须裁剪或切片）：

  • 直接截断：把上下文裁掉（通常保留前面的一段），简单易实现、吞吐高，会丢失被裁掉部分里本来包含的答案，导致这些样本无法定位答案（只能标为无答案），训练/评估也会因此受到影响
  • 滑动窗口（Sliding Window）：把长上下文按窗口长度带重叠地切成多个片段（用 doc_stride 控制重叠长度），对每个片段都构造一条样本：[CLS] 问题 [SEP] 片段i [SEP]，可以大幅降低答案被截断的概率，如果答案跨越某个边界，重叠区域能保证它完整落在至少一个片段里，但是实现更复杂，计算量增加，每个样本只有片段级上下文，全局上下文略有缺失

多项选择任务

多项选择任务

多项选择任务：多项选择任务是机器阅读理解中的一种任务形式，相较于之前的机器阅读理解定义而言，多项选择任务需要提供答案的候选项，即给定一个或多个文档 P，以及一个问题 Q 和对应的多个答案候选，输出问题 Q 的答案 A，A 是答案候选中的某一个选项

预训练模型（重要）

预训练

预训练（Pretrain）：是指通过自监督学习从大规模数据中获得与具体任务无关的预训练模型的过程，最终产出为预训练模型（Pretrained Model）

具体来讲，即希望通过预训练过程，把知识压入模型，让模型具备一定的通用（不针对某一特定任务）的能力，常见的预训练模型大致分为以下三类：

  • 掩码语言模型，自编码模型（Only Decoder）：将一些位置的 Token 替换为特殊的 Mask 字符，预测这些被替换的模型
  • 因果语言模型，自回归模型（Only Encoder）：将完整序列输入，基于上文的 Token 预测当前 Token
  • 序列到序列模型（Decoder + Encoder）：任务较为多样化，只是采用编码器解码器的方式，预测部分放在解码器中

掩码语言模型

掩码语言模型将一些位置的 Token 替换为特殊的[MASK]字符，预测这部分的字符，我们只需要计算掩码部分的 Loss，其余部分不参与损失的计算（会被填充为 -100，计算 Loss 的时候会默认忽视这部分的 Token）

因果语言模型

把完整的序列输入，模型会基于上文的 Token 预测当前的 Token，也就是t位置的 Token 用于预测t+1位置的 Token，换言之，即t+1位置的真实 Token 作为t位置预测结果的 label，在计算 Loss 的时候会有一种错位感

因为模型是逐个 Token 进行生成的，因此为了确保模型不会无休止输出，需要在结尾位置加入额外的<EOS>作为结束符号

序列到序列模型

并不常用，也不是本章我们学习的重点

生成式对话机器人

对话机器人简介

对话机器人：本质上是一个用来模拟人类对话或聊天的计算机程序，接收人类的自然语言作为输入，并给出合适的回复

按照任务类型的划分，对话机器人可以大致划分为：闲聊机器人，问答机器人，任务型对话机器人

按照答案产生的逻辑划分，对话机器人可以大致划分为：检索式对话机器人，生成式对话机器人

这里我们重点关注基于生成式的问答机器人的实现
