从 PPO 到 GRPO

PPO 是一种 actor-critic 强化学习算法，被广泛应用在 ChatGPT 等 LLM 的强化学习微调阶段，它通过最大化以下代替目标来优化大语言模型：

由于 PPO 中使用的价值函数通常是于策略模型大小相当的另一个模型，因此会带来大量内存和计算负担，此外，在强化学习训练期间，价值函数被视为计算优势以减少方差的基线

然而，在 LLM 的背景下，通常只有最后一个 Token 会被奖励模型赋予奖励分数，这可能会使得在每个 Token 上都准确的价值函数的训练变得复杂，为了解决这个问题，我们提出了组相对策略优化，即 GRPO，如下图所示，GRPO 不需要像 PPO 一样进行额外的价值函数近似，而是使用针对同一问题生成的多个采样输出的平均奖励作为基线，即群体计算

群体计算是 GRPO 中的一个关键步骤，即策略模型会根据输入 qqq 从旧策略中采样处一组动作 o1,o2,…,oGo_1,o_2,\dots,o_Go1​,o2​,…,oG​，对于每个动作，奖励模型都会得到一个对应的奖励 r1,r2,…,rGr_1,r_2,\dots,r_Gr1​,r2​,…,rG​，GRPO 会把所有动作的优势函数和奖励一起计算，进行群体层次的优化，而不仅仅是单个动作

LiGRPO+KL=1Ti∑t:Mi,t=1(−min⁡(ri,tAi,clip(ri,t,1−ε,1+ε)Ai)+βk3i,t)L_{i}^{GRPO+KL}=\frac{1}{T_i}\sum_{t:M_{i,t}=1}(-\min(r_{i,t}A_i,\text{clip}(r_{i,t},1-\varepsilon,1+\varepsilon)A_{i})+\beta k3_{i,t}) LiGRPO+KL​=Ti​1​t:Mi,t​=1∑​(−min(ri,t​Ai​,clip(ri,t​,1−ε,1+ε)Ai​)+βk3i,t​)

L=1N∑i=1NLiGRPO+KLL=\frac{1}{N}\sum_{i=1}^NL_i^{GRPO+KL} L=N1​i=1∑N​LiGRPO+KL​

GRPO与PPO的对比

二者在 “最终损失形式（PPO clipped surrogate）” 上几乎一致，损失函数表达形式均为：

L=−E[min⁡(rtA, clip(rt,1−ϵ,1+ϵ) A)]L=-\mathbb{E}\Big[\min\big(r_tA, \ \text{clip}(r_t,1-\epsilon,1+\epsilon)\,A\big)\Big] L=−E[min(rt​A, clip(rt​,1−ϵ,1+ϵ)A)]

区别主要在优势函数 AAA 的构造方式与问题建模

PPO（经典做法）

  • 粒度：token / 时刻级 AtA_tAt​

  • 构造：有价值网络（Critic），用 GAE(λ) 做时序差分（即按照时间步 t 向后逐步采样）：

    AtGAE=∑l=0∞(γλ)l(rt+l+γV(st+l+1)−V(st+l))A_t^{\text{GAE}}=\sum_{l=0}^{\infty}(\gamma\lambda)^l\big(r_{t+l}+\gamma V(s_{t+l+1})-V(s_{t+l})\big) AtGAE​=l=0∑∞​(γλ)l(rt+l​+γV(st+l+1​)−V(st+l​))

  • 特点：需要训练 Critic、向后回溯、时间信用分配

GRPO（Group Relative Preference Optimization）

  • 粒度：句子 / 序列级 AiA_iAi​（同一条回复所有 token 共用一个优势）

  • 构造：对同一 prompt 的组内多条候选做线性加权奖励后组内标准化：

    Ai=ri−μgσg+ε,ri=∑kwkri(k)A_i=\frac{r_i-\mu_g}{\sigma_g+\varepsilon},\quad r_i=\sum_k w_k r_i^{(k)} Ai​=σg​+εri​−μg​​,ri​=k∑​wk​ri(k)​

  • 特点：不需要 Critic / GAE，把序列当成上下文 bandit，用“组内相对好坏”替代时序优势

统一范式

后边我们还会接触到多个 PPO 模型的变体，DeepSeek 提供了一个统一的范式来分析不同的训练方法，如SFT、RFT、DPO、PPO、GRPO，并进一步进行实验以探索统一范式的要素，一般来说，关于训练方法参数 θ\thetaθ 的梯度可以写成：

∇θJA(θ)=E[(q,o)∼D](1∣o∣∑t=1oGCA(q,o,t,πrf)∇θlog⁡πθ(ot∣q,o<t))\nabla_{\theta}J_{A}(\theta)=E[(q,o)\sim D]( \frac{1}{|o|}\sum_{t=1}^{o}GC_{A}(q,o,t,\pi_{rf})\nabla_{\theta}\log\pi_{\theta}(o_t|q,o_{<t}) ) ∇θ​JA​(θ)=E[(q,o)∼D](∣o∣1​t=1∑o​GCA​(q,o,t,πrf​)∇θ​logπθ​(ot​∣q,o<t​))

这里存在三个关键组成部分：

  • E[(q,o)∼D]E[(q,o)\sim D]E[(q,o)∼D]：其中 D 是数据集，它决定了训练数据，通常，这些数据是从环境中采样的状态 q 和动作 o 对
  • GCA(q,o,t,πref)GC_{A}(q,o,t,\pi_{ref})GCA​(q,o,t,πref​)：其中 πrf\pi_{rf}πrf​ 是奖励函数，它是训练奖励信号的来源它提供了如何根据状态-动作对 (q,o) 计算奖励值 r 的方法
  • 算法 A：它处理训练数据和奖励信号，得到梯度系数 GC，该系数决定了数据的惩罚或强化的幅度

训练流程

GRPO 的训练流程和 PPO 的训练流程几乎完全相同，大致分为以下几个部分：「采样 → 打分 → 优势计算 → 拼接 → 损失 → 训练循环」

有所不同之处在于：

  • 模型选择：PPO 需要四个模型配合工作，而 GRPO 只需要两个模型（actor_model - 策略模型，ref_model - 参考模型）配合若干奖励函数即可完成整个训练流程
  • 打分阶段：GRPO 会在打分阶段对同一条 Prompt 做一次num_generation的扩展，接着以组为单位计算组内句子级相对优势，最后把各组的优势进行跨组拼接，即返回[bsz*num_generation, ...]形状的张量，进而参与损失函数的计算
