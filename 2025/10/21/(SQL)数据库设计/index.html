关系数据理论

判断数据库设计好坏的指标：关系数据理论

数据依赖：一个关系内部属性值之间相互依赖又相互制约，可以分为函数依赖（Functional Dependency），多值依赖（Multivalued Dependency），我们只学习函数依赖有关内容

函数依赖

设 R(U) 是属性 U 上的关系模式，X Y 是 U 的子集，r 是 R 的任意一个具体关系，t s 是 r 中任意两个元组，如果对于任意的 t[X] = s[X] 成立时，都可推出 t[Y] = s[Y] 成立，则称 X 函数确定 Y 或者 Y 函数依赖于 X，记作 X→\rightarrow→Y

特殊情况：如果关系 r 中任意两个元组的 X 都不相等，那么此时 X 可以确定任意的 Y，不满足前置条件时，推导式一定成立

平凡 & 非平凡依赖

对于函数依赖 X$ \rightarrow$Y，如果 Y 是 X 的子集，则称为平凡函数依赖，如果 Y 不是 X 的子集，则称为非平凡的函数依赖

对于函数依赖 X->Y，X 称为决定因素，Y 称为被决定因素

函数依赖的补充说明

函数依赖是语义范畴的概念：只能根据语义来确定一个函数依赖，而不能形式化证明一个函数依赖成立

函数依赖不随时间变化：如果关系 R 具有函数依赖 X->Y，即便关系 R 随着时间变化，但 X->Y 不变（比如对于学号->姓名这个函数依赖，不论是哪一届的学生，这个推导都成立）

函数依赖和属性间的联系类型

  • 1:1 联系：如果 X 和 Y 是一对一，则 X->Y 等价于 Y->X，比如学生的学号和身份证号
  • 1:m 联系：如果 X 和 Y 是多对一，则只存在 X->Y（注意：这里是多端决定一端），比如学生的学号和系号（学号是多，系号是一，应该是学号决定系号，多个学号可以属于同一个系号）
  • n:m 联系：如果 X 和 Y 是多对多，则 X 和 Y 之间不存在任何函数依赖，比如学生的学号和课程名

三种函数依赖

完全函数依赖 & 部分函数依赖：在 R(U) 中，如果 X->Y，且对于任意 X 的真子集 X‘，都有 X’ 无法推出 Y，则称 Y 对 X 完全依赖，记作 X -> Y(上标加 f)，否则称为 Y 对 X 部分函数依赖，记作 X->Y(上标加 p)

传递函数依赖：在 R(U) 中，如果 X->Y，Y->Z，且 Y 无法推出 X，则称 Z 对 X 传递函数依赖，记作 X->Z(上标加 t)，注意：这里之所以限定 Y 无法推出 X，如果没这个限定，那么 X 和 Y 可以看成是等价的属性，那么 X->Z 就可以看成直接函数依赖，传递函数依赖强调的是 X 无法直接推出 Z，而是必须先经过 Y，才能推出 Z

关系键的形式定义

区分主码（主键）和主属性：主码（主键）是候选码中之一，主属性是包含在任意候选码中的属性，非主属性是不包含在任意候选码中的属性

候选码（候选键）和主码（主键）：设 K 是 R<U,F> 中的属性或属性集合，若 K->U(上标加 f)，则称 K 为 R 的候选码，若候选码多于一个，则选定其中一个作为主码，即候选码就是能推出整个关系 U 的属性集合 K

码的性质：唯一标识性 & 最小性

主属性和非主属性：包含在任意一个候选码中的属性，称为主属性；不包含在任何码中的属性，称为非主属性（某个属性不能同时满足主属性或非主属性）

外部码：关系模式 R 中属性或属性组 X 并非 R 的码，但 X 是另一个关系模式的码，则称 X 是 R 的外码，对外部码的语义约束条件称为参照完整性

函数依赖公理系统

函数依赖公理系统：以数理逻辑为基础的函数依赖公理系统

函数依赖的逻辑蕴含：关系模式 R<U,F> 中（U 是属性集合，F 是 U 上存在的函数依赖关系的集合），X Y 是 R 的属性集合，如果从 F 中的函数依赖能够推出 X->Y，则称 F 逻辑蕴含 X->Y（F 中可能本身不包含 X->Y，但基于其中已有的函数依赖，我们可以推出 X->Y 在 R 上也成立）

函数依赖集 F 的闭包：在关系模式 R<U,F> 中，为 F 所逻辑蕴含的函数依赖的全体称为 F 的闭包，记作 F^+^（F 本身和它推出的函数依赖的集合）

Armstrong公理系统

Armstrong 公理系统，对于 R<U,F>，有如下规则：

  • A1 自反律：如果 Y⊆X⊆UY\subseteq X\subseteq UY⊆X⊆U，则 X->Y 为 F 所蕴含
  • A2 增广律：若 X->Y 为 F 所蕴含，且 Z⊆UZ\subseteq UZ⊆U，则 XZ->YZ 为 F 所蕴含
  • A3 传递律：若 X->Y，Y->Z 为 F 所蕴含，则 X->Z 为 F 所蕴含

Armstrong 公理的推论：

  • 合并规则：由 X->Y，X->Z，可以推出 X->YZ
  • 伪传递规则：由 X->Y，WY->Z，可以推出 WX->Z
  • 分解规则：由 X->Y，Z⊆YZ\subseteq YZ⊆Y，可以推出 X->Z

属性集的闭包

属性集的闭包：属性集 X 关于函数依赖集 F 的闭包（X~F~^+^）

Armstrong 公理系统的有效性与完备性

Armstrong 公理系统是有效且完备的：

  • 有效性：由 F 出发根据 Armstrong 公理推导出来的每个函数依赖一定在 F 所蕴含的函数依赖的全体之中

  • 完备性：F 所蕴含的函数依赖的全体中的每一个函数依赖，必定可以由 F 根据 Armstrong 公理导出

换言之，就是说正确的函数依赖一定能用 Armstrong 推出，Armstrong 推出的依赖也一定能保证正确性

函数依赖集等价与覆盖

函数依赖集等价：

  • 函数依赖集 F，G，若 F^+^=G^+^，则称 F 和 G 等价
  • 如果 F 和 G 等价，则称 F 覆盖 G 同时 G 也覆盖 F，即 F^+^=G^+^ 等价于 F ⊆\subseteq⊆ G^+^ 且 G ⊆\subseteq⊆ F^+^

函数依赖集的最小依赖集：

  • 若函数依赖集 F 满足下列条件，则称 F 为一个极小函数依赖集，也称为最小依赖集和最小覆盖：
    • 对于 F 中任意函数依赖 X->A，A 必定是单属性（不能是属性集合），即右部单属性化
    • F 中不存在这样的函数依赖 X->A，使得 F 与 F - {X->A} 等价，即没有冗余的 FD
    • F 中不存在这样的函数依赖 X->A，在 X 中有真子集 Z，使得 F 与 F - {X->A} + {Z->A} 等价，即每个 FD 左部都没有多余属性，也就是说 U 中 X-Z 这部分属性是必不可少的

函数依赖集 F 的极小化处理：

定理：每个函数依赖集 F 都等价于一个极小函数依赖集 Fm，此 Fm 为 F 的最小依赖集

范式

如果一个关系满足某个指定的约束集，则称它属于某种特定的范式（Normal Form）

  • 满足最低要求约束的称为第一范式，即 1NF

1NF

当一个关系只包含原子值这一约束时，称为 1NF，原子值即为二维表的每一行和每一列的交叉位置上总是精确的存在一个值，而不是值的集合，即不能存在表中有表的情况

满足原子值这一约束条件的关系称为规范化关系，简称范式，在关系数据库中，都是规范化的关系

2NF

如果 R∈1NF，且每个非主属性（非主属性指的是不在候选码中存在的那些属性）完全依赖于码，则称 R∈2NF

  • 如果 R 的全体属性都是 R 的主属性（没有非主属性），那么 R∈2NF
  • 从 1NF 中消除非主属性对码的部分函数依赖，则可以获得 2NF 关系
  • 在 2NF 中，允许主属性部分函数依赖于码（2NF 只要求了非主属性必须完全依赖于码）

关系模式 R<U,F> 中，若不存在这样的码 X 和属性组 Y，以及非主属性组 Z(Z ∉ Y)，使得下式成立：X->Y，Y->Z，Y无法推出 X，则称 R ∈ 3NF，也就是说，3NF 中不允许传递依赖的存在

或定义为：若关系模式 R ∈ 2NF，且每个非主属性都不传递依赖于 R 的任何码，就称 R ∈ 3NF

BCNF：

若关系模式 R<U,F>∈1NF，如果对于 R 的每个函数依赖 X->Y，且 Y 不是 X 的某个子集（非平凡函数依赖），可以推出 X 中必定含有码（X 本身就是码，或者 X 的某个子集是码），则 R<U,F> ∈ BCNF

BCNF 关系模式具有下面三个性质：

  • 所有非主属性都完全函数依赖于每个候选码（证明：假设存在某个部分依赖 X->Y，那么 X 中包含码，且 X 中有某个真子集，X1->Y，且 X1 中也包含码，那么 X1 和 X 之间违反码的最小性，产生矛盾）
  • 所有主属性都完全函数依赖于每个不包含它的候选码
  • 没有任何属性完全函数依赖于非码的任何一组属性（反证法：证明 “若R∈BCNF，X->Y(上标加 f)，且 X 不是码” 这个命题不成立，也就是证明，若 R∈BCNF，X->Y(上标加 f)，X 本身必然就是码，此时 X 中必定含有码 X1，使得 X1->Y(上标加f) 成立，此时和 X->Y(上标加 f) 冒充，因此命题不成立）

X->Y 则 X 中必须包含码；X->Y(上标加f) 则 X 必须是码，注意区分前提条件

全码（以所有的属性作为码）中不存在函数依赖关系，一定是 BCNF

规范化目的与基本思想

范式之间的关系：4NF⊂BCNF⊂3NF⊂2NF⊂1NF4NF \sub BCNF\sub 3NF \sub 2NF \sub 1NF4NF⊂BCNF⊂3NF⊂2NF⊂1NF

  • 在关系数据库中，对关系的最基本要求是满足第一范式，这些关系常有一些异常或冗余等弊病，规范化的目的就是要消除这些弊病
  • 规范化的基本思想是逐步消除数据依赖中不合适的部分，使数据库中各关系模式达到某种程度的分离，使得一个关系只描述一个实体或实体间的一种联系，即一事一地的设计原则，规范化的本质是概念的单一化

模式分解的定义理论

关系模式分解

模式分解的定义：函数依赖集合 F_i=\{X\rightarrow Y | X\rightarrow Y\in F^+ \and XY \subseteq U_i\}，则称 F~i~ 为 F 在 U~i~ 上的投影，有两个约束条件，其一是 X 和 Y 必须是 U~i~ 的属性，其二是 X->Y 必须属于 F 的函数依赖闭包

关系模式 R<U,F> 的一个分解 ρ\rhoρ 是指 ρ={R1<U1,F1>,R2<U2,F2>,…,Rn<Un,Fn>}\rho=\{R_1<U_1,F_1>, R_2<U_2,F_2>,\dots,R_n<U_n,F_n>\}ρ={R1​<U1​,F1​>,R2​<U2​,F2​>,…,Rn​<Un​,Fn​>}，其中各个 U~i~ 的并集能够得到属性全集 U，且不存在某个 U~i~ 是另一个 U~j~ 的真子集

分解的无损连接性

设 ρ={R1,R2,…,Rn}\rho=\{R_1, R_2,\dots,R_n\}ρ={R1​,R2​,…,Rn​} 是 R<U,F> 的一个分解，r 是 R<U,F> 的一个关系，定义 mρ(r)=⋈∏Ri(r)m_{\rho}(r)=\bowtie \prod_{R_i}(r)mρ​(r)=⋈∏Ri​​(r)，其中 ∏Ri(r)={t[Ui]∣t∈r}\prod_{R_i}(r)=\{t[U_i]|t \in r\}∏Ri​​(r)={t[Ui​]∣t∈r}，即 mρ(r)m_{\rho}(r)mρ​(r) 是 ρ\rhoρ 中各个关系模式投影上的连接，若对于 R<U,F> 的任何一个关系 r，都有 r=mρ(r)r=m_{\rho}(r)r=mρ​(r)，则称分解 ρ\rhoρ 具有无损连接性，简称为无损分解

说通俗点，如果能把一张大表分解成若干小表之后，拼起来还能恰好是这张大表，属性恰好不多不少，就说明这是一个无损分解

分解的无损连接性判定准则：

R<U,F> 的一个分解 ρ={R1<U1,F1>, R2<U2,F2>} 具有无损连接性的充分必要条件是 U1 ∩ U2 -> U1 - U2 ∈ F^+^ 或者 U1 ∩ U2 -> U2 - U1 ∈ F^+^，即 R1，R2 的共同属性至少构成 R1 R2 二者之一的候选码
