RLHF

通过阅读本文，您可以大致了解和人类偏好学习（RLHF）以及近端策略优化（PPO）的相关原理与推导，本文是 LLM 篇的第一节

RLHF（Reinforcement Learning from Human Feedback）是大语言模型在生成领域的新训练范式，即以强化学习方式依据人类反馈优化语言模型

在过去的模型训练中，我们对生成结果的评估一般是主观和依赖上下文的，比如，我们希望模型生成一个有创意的故事，一段真实的信息性文本，或者可执行的代码片段，这些结果难以用现有的基于规则的文本生成指标进行衡量（如 BLEU，ROUGE），除了评估指标，现有的模型通常以预测下一个词的方式和简单的损失函数进行建模，没有显式地引入人类偏好和主观意见

RLHF 的核心思想在于：使用强化学习的方式i直接优化带有人类反馈的语言模型，它能够使得在一般文本数据语料库上训练的语言模型性能和复杂的人类价值观所对齐，即使用人类对生成文本的反馈作为性能衡量标准，或者进一步用该反馈作为损失来优化模型

RLHF+PPO

RLHF 是一项涉及多个模型和不同训练阶段的复杂概念，大致可以分为三个步骤：

  • 预训练一个语言模型 LM
  • 聚合问答数据并且训练一个奖励模型 RM（Reward Model）
  • 用强化学习 RL 方式微调 LM

预训练语言模型

首先，使用经典的预训练目标训练一个语言模型，由于 RLHF 还是一个尚待探索的领域，对于哪种模型适合作为 RLHF 的起点目前还没有明确的答案

接着，我们基于预训练的 LM 来生成用于训练奖励模型 RM（也称为偏好模型）的数据，并在这一步引入人类的偏好信息

训练奖励模型

RM 是 RLHF 区别于旧范式的开端，奖励模型 RM 把 “人类更喜欢哪种输出” 变成一个标量奖励；有了这个 “可微 / 可用的奖励”，我们才能把任意 RL 算法（如 PPO）无缝接到语言模型（LM）上进行偏好对齐训练

RM 的实现有两种体系结构：

  • 端到端集成进 LM（二者使用同一骨干，共享参数或共享表示）
  • 模块化独立存在（单独设置一个打分器）

关于模型选择方面，RM 可以是另一个经过微调的 LM，也可以是根据偏好数据从头开始训练的 LM，这里对 RM 的训练可看做有监督的训练，我们用prompt-respone+score这样的人工标注数据，训练 RM 在输入prompt-response以后，能够基于对这条 prompt 的 response 打出符合人类偏好的分数

在训练文本方面，RM 的prompt-response文本是从预定义数据集中采样生成的，并且使用初始的 LM 给这些 prompt 生成多条候选回答；在训练奖励数值方面，需要人工对 LM 生成的回答进行排序 / 偏好比较（这里之所以进行排名而不是打分，是因为绝对分数因为不同标注者的价值观不同而难以校准，而排名看相对优劣可以消除线性变换歧义）

对于具体的排名方式，目前比较成熟的方式是对不同 LM 在相同提示下的输出进行比较，然后使用 Elo 系统为每一组 prompt 下的不同 response 建立一个完整的排名，这些不同的排名结果将被归一化用于训练的标量奖励值

在具体实现层面，目前成功的 RLHF 系统使用了和生成模型具有不同大小的 LM，可能原因是偏好模型和生成模型需要具有类似的能力来理解提供给它们的文本，直观来讲就是，RM 需要有足够的能力来理解 LM 所输出的结果

强化学习微调

使用强化学习训练 LM 需要克服的一大难点在于，微调 LM 中全部的 10B+ 个参数的成本过高，目前的可行方案在于，使用策略梯度强化学习算法（Policy Gradient RL）中的近端策略优化（PPO，Proximal Policy Optimization）微调初始 LM 的部分或全部参数

PPO 算法已经存在了相对较长的实践，并且有大量的使用文档，所以大家更倾向于在 RLHF 中选择 PPO，目前 RLHF 的许多进步大多是和如何把熟悉的 RL 算法应用到更新参数量较大的模型方面有关

在讲解具体算法之前，我们把微调任务表述为强化学习问题：

  • 策略 policy：一个接受 prompt 并且返回一些列文本或文本概率分布的 LM
  • 行动空间 action apace：当前策略的行动空间是 LM 词表中所对应的所有 Token，一般在 50k 数量级
  • 观察空间 observation space：可能的输入 Token 序列，一个很大的空间，大小为词汇量^输入Token的数量
  • 奖励函数：偏好模型和策略转变约束 Policy shift constraint 的结合

PPO 算法确定的奖励函数具体计算如下：把 prompt xxx 输入初始的 LM 和当前微调的 LM，分别得到了输出文本 y1,y2y_1, y_2y1​,y2​，将来自当前策略的文本传递给 RM 得到一个标量奖励值 rθr_\thetarθ​，将两个模型的生成文本进行比较，计算差异的惩罚项，这个惩罚项通常设计为输出词分布序列之间的 KL 散度的缩放，即

r=rθ−λrKLr=r_\theta-\lambda r_{KL} r=rθ​−λrKL​

这一项被用于惩罚 RL 策略在每个训练批次中生成大幅度偏离初始模型的情况，以确保模型输出连贯且合理的文本，如果去掉这一惩罚项，可能会导致模型在优化中生成乱码文本来欺骗奖励模型以提高奖励值

最后根据 PPO 算法，按当前批次数据的奖励指标进行优化，这来自于 PPO 算法的on-policy特性，PPO 算法是一种信赖域优化 TRO（Trust Region Optimization）算法，它使用梯度约束确保更新步骤不会破坏学习过程的稳定性

用这个梯度更新神经网络的参数就是 Policy Gradient，即梯度策略算法，基于这个思想，我们可以定义损失函数：

L=−1N∑n=1N∑t=1TnR(τn)log⁡Pθ(atn∣stn)L=-\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{Tn}R(\tau^n)\log P_{\theta}(a_t^n|s_t^n) L=−N1​n=1∑N​t=1∑Tn​R(τn)logPθ​(atn​∣stn​)

有了 Loss 以后，即可对神经网络进行训练，一般是采集一批数据，训练网络，用训练好的网络再执行 N 次序列，得到数据以后再训练网络，这称为 On-Policy，也就是说采集数据所用的策略网络和这些数据训练的网络是同一个，这会造成大部分时间都花费在采集数据上，训练非常慢

对于上面的公式 1N∑n=1N∑t=1TnR(τn)log⁡pθ(ant∣snt)\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}R(\tau^n)\log p_{\theta}(a_n^t|s_n^t)N1​∑n=1N​∑t=1Tn​​R(τn)logpθ​(ant​∣snt​) 可以做如下理解：该表达式可以分为两部分，第一部分 R(τn)R(\tau^n)R(τn) 表示第 n 条策略的整体回报 Return，第二部分 Pθ(ant∣snt)P_{\theta}(a_n^t|s_n^t)Pθ​(ant​∣snt​) 是第 n 条策略中，在第 t 步时，根据当前 state 做出 action 的概率，我们的目的是让这个表达式的值尽可能大，也就是：

  • 如果一个轨迹的 Return 值大于 0，就增大这个轨迹中所有状态下采取当前动作的概率，也就是增大每一个 Pθ(ant∣snt)P_{\theta}(a_n^t|s_n^t)Pθ​(ant​∣snt​)
  • 如果一个轨迹的 Return 值小于 0，就减小这个轨迹中所有状态下采取当前动作的概率，也就是减小每一个 Pθ(ant∣snt)P_{\theta}(a_n^t|s_n^t)Pθ​(ant​∣snt​)

我们目前推出的模型存在以下两个问题：

  • 每一个动作按理说只能影响这个动作之后的 Reward，而不是影响整个轨迹中这个动作之前的 Reward
  • 当前动作对于它之后几步的影响可能会比较大，但对于相距比较远的 Reward 影响应该比较小，因此需要设置影响逐渐衰减

下面我们对此进行修正：

首先对 Reward 进行求和，对于当前步骤 t，并不是对整个轨迹的 Reward 求和，而是对之后步骤对应的 Reward 求和，并且使用 γ∈(0,1)\gamma\in(0,1)γ∈(0,1) 表示当前动作对之后的影响逐步衰减：

R(τn)→Rtn=∑t′=tTnγt′−trt′nR(\tau^n)\rightarrow R_t^n=\sum_{t'=t}^{T_n}\gamma^{t'-t}r_{t'}^nR(τn)→Rtn​=∑t′=tTn​​γt′−trt′n​

其中 RtnR_t^nRtn​ 表示第 t 步动作对于整个轨迹的影响

另一个需要考虑的问题是，在好的局势下，无论做什么动作都会带来正向收益（正的 Reward），此时算法会增加所有动作的概率，Reward 大的动作的概率会增加的大一些，Reward 小的动作增加的概率会小一些；在坏的局势下，无论做什么动作都会带来负向收益（负的 Reward），这会导致算法收敛变慢

而我们期望的现象是相对好的动作概率得以增加，相对差的动作概率得以减小，这样可以加速收敛，一个很自然的想法就是减去一个基线的 baseline，即 B(snt)B(s_n^t)B(snt​)，这样可以反映出当前动作相对于其他动作的价值

此时整体的公式可以变换为：

接下来我们引入几个额外的概念：

  • 动作价值函数 Action Value Function：之前 RtnR_t^nRtn​ 是累加之后的 Reward，但都是采样得到的，会造成方差比较大，训练不稳定，理想情况需要采样无限多次，这里引入动作价值函数 Qθ(s,a)Q_{\theta}(s,a)Qθ​(s,a)，表示在状态 s 下，做出动作 a 的期望，进而可以估计当前的动作能得到的回报的期望
  • 状态价值函数 State Value Function：表示在状态 s 下，做任何动作，得到的回报的期望值，表示当前状态的价值，用 Vθ(s)V_{\theta}(s)Vθ​(s) 表示
  • 优势函数 Advantage Function：在状态 s 下，做出动作 a 比其他动作能多带来多少用时，公式表示为 Aθ(s,a)=Qθ(s,a)−Vθ(s)A_{\theta}(s,a) = Q_{\theta}(s,a)-V_{\theta}(s)Aθ​(s,a)=Qθ​(s,a)−Vθ​(s)，即做出某个动作得到的回报相对于这个状态期望的回报的差值，表示了当前动作相对其他动作的优势

上述公式中当前动作所带来的回报减去基线其实就是优势函数的含义，那么可以直接用优势函数进行替换：

下一步，我们需要探究这个优势函数该如何计算，首先有如下公式：

Qθ(st,a)=rt+γ∗Vθ(st+1)Q_{\theta}(s_t,a)=r_t+\gamma*V_{\theta}(s_{t+1}) Qθ​(st​,a)=rt​+γ∗Vθ​(st+1​)

即当前动作期望得到的 return 值等于当前这一步动作得到的 Reward 加上下一状态的状态价值函数，并且要带上衰减系数，将其代入优势函数，可以得到：

Aθ(st,a)=rt+γ∗Vθ(st+1)−Vθ(st)A_{\theta}(s_t,a)=r_t+\gamma*V_{\theta}(s_{t+1})-V_{\theta}(s_t) Aθ​(st​,a)=rt​+γ∗Vθ​(st+1​)−Vθ​(st​)

这里只剩下状态价值函数 VθV_{\theta}Vθ​，因此只需要训练一个代表状态价值函数的网络即可，我们对轨迹的下一步动作和 Reward 进行采样，即可得到近似公式：

Vθ(st+1)≈rt+1+γ∗Vθ(st+2)V_{\theta}(s_{t+1})≈r_{t+1}+\gamma*V_{\theta}(s_{t+2}) Vθ​(st+1​)≈rt+1​+γ∗Vθ​(st+2​)

类似的，我们可以对优势函数进行多步采样，即反复利用上式（展开次数分别为 1,2,…T）进行展开：

Aθ1(st,a)=rt+γ∗Vθ(sy+1)−Vθ(st)Aθ2(st,a)=rt+y∗rt+1+γ2∗Vθ(st+2)−Vθ(st)Aθ3(st,a)=rt+γ∗rt+1+γ2∗rt+2+γ3Vθ(st+3)−Vθ(st)…AθT(st,a)=rt+γ∗rt+1+γ2∗rt+2+γ3∗rt+3+⋯+γT∗rT−Vθ(st)A_{\theta}^1(s_t,a)=r_t+\gamma*V_{\theta}(s_{y+1})-V_{\theta}(s_t)\\ A_{\theta}^2(s_t,a)=r_t+y*r_{t+1}+\gamma^2*V_{\theta}(s_{t+2})-V_{\theta}(s_t)\\ A_{\theta}^3(s_t,a)=r_t+\gamma*r_{t+1}+\gamma^2*r_{t+2}+\gamma^3V_{\theta}(s_{t+3})-V_{\theta}(s_t)\\ \dots\\ A_{\theta}^T(s_t,a)=r_t+\gamma*r_{t+1}+\gamma^2*r_{t+2}+\gamma^3*r_{t+3}+\cdots+\gamma^T*r_{T}-V_{\theta}(s_t) Aθ1​(st​,a)=rt​+γ∗Vθ​(sy+1​)−Vθ​(st​)Aθ2​(st​,a)=rt​+y∗rt+1​+γ2∗Vθ​(st+2​)−Vθ​(st​)Aθ3​(st​,a)=rt​+γ∗rt+1​+γ2∗rt+2​+γ3Vθ​(st+3​)−Vθ​(st​)…AθT​(st​,a)=rt​+γ∗rt+1​+γ2∗rt+2​+γ3∗rt+3​+⋯+γT∗rT​−Vθ​(st​)

易知采样次数越多，方差越大，偏差越小，这是因为随着 T 的增加，目标中包含的随机回报项变多（rt+2,rt+3,…r_{t+2}, r_{t+3},\dotsrt+2​,rt+3​,…），这些噪声在求和时会积累，使得方差增加，同时，引导项的权重 γn\gamma^nγn 变小，对近似 VθV_\thetaVθ​ 的依赖变弱，因此偏差变小

这里我们定义一个中间变量 δtV\delta_t^VδtV​，表示在第 t 步带来的优势：

δtV=rt+γ∗Vθ(st+1)−Vθ(st)δt+1V=rt+1+γ∗Vθ(st+2)−Vθ(st+1)…\delta_t^V=r_t+\gamma*V_{\theta}(s_{t+1})-V_{\theta}(s_{t})\\ \delta_{t+1}^V=r_{t+1}+\gamma*V_{\theta}(s_{t+2})-V_{\theta}(s_{t+1})\\\dots δtV​=rt​+γ∗Vθ​(st+1​)−Vθ​(st​)δt+1V​=rt+1​+γ∗Vθ​(st+2​)−Vθ​(st+1​)…

则原式可以化简为更加简洁的形式：

Aθ1=δtVAθ2=δtV+γ∗δt+1VAθ3=δtV+γ∗δt+1V+γ2∗δt+2V…A_{\theta}^1=\delta_t^V\\ A_{\theta}^2=\delta_{t}^V+\gamma*\delta_{t+1}^V\\ A_{\theta}^3=\delta_{t}^V+\gamma*\delta_{t+1}^V+\gamma^2*\delta_{t+2}^V \\\dots Aθ1​=δtV​Aθ2​=δtV​+γ∗δt+1V​Aθ3​=δtV​+γ∗δt+1V​+γ2∗δt+2V​…

在优势函数计算时，GAE（Generalize Advance Estimation）对所有步骤都进行采样，通过对所有步骤（采样一次，两次，…乃至采样 T 次）进行加权平均，得到最终的 GAE 优势函数：

AθGAE(st,a)=(1−λ)(Aθ1+λ∗Aθ2+λ3Aθ3+… )=(1−λ)(δtV+λ∗(δtV+γ∗δt+1V)+λ2(δtV+γ∗δt+1V+γ2∗δt+2V)+… )=(1−λ)(δtV11−λ+γ∗δt+1Vλ1−λ+… )=∑b=0inf⁡(λγ)bδt+bVA_{\theta}^{GAE}(s_t,a)=(1-\lambda)(A_{\theta}^1+\lambda*A_{\theta}^2+\lambda^3A_{\theta}^3+\dots)\\ =(1-\lambda)(\delta_t^V+\lambda*(\delta_{t}^V+\gamma*\delta_{t+1}^V)+\lambda^2(\delta_{t}^V+\gamma*\delta_{t+1}^V+\gamma^2*\delta_{t+2}^V)+\dots)\\ =(1-\lambda)(\delta_t^V\frac{1}{1-\lambda}+\gamma*\delta_{t+1}^V\frac{\lambda}{1-\lambda}+\dots)\\ =\sum_{b=0}^{\inf}(\lambda\gamma)^b\delta_{t+b}^V AθGAE​(st​,a)=(1−λ)(Aθ1​+λ∗Aθ2​+λ3Aθ3​+…)=(1−λ)(δtV​+λ∗(δtV​+γ∗δt+1V​)+λ2(δtV​+γ∗δt+1V​+γ2∗δt+2V​)+…)=(1−λ)(δtV​1−λ1​+γ∗δt+1V​1−λλ​+…)=b=0∑inf​(λγ)bδt+bV​

这表示在状态 s 的第 t 步所做动作 a 的相对优势，并且平衡了采样不同步骤（采样一次、两次、…乃至采样 T 次）带来的方差和偏差的问题，这时再次更新策略梯度的优化目标函数：

1N∑n=1N∑t=1TnAθ(snt,ant) ∇log⁡Pθ(ant∣snt)→1N∑n=1N∑t=1TnAθGAE(snt,ant) ∇log⁡Pθ(ant∣snt)\frac{1}{N}\sum_{n=1}^{N}\sum_{t=1}^{T_n} A_{\theta}(s_n^t, a_n^t)\,\nabla \log P_{\theta}(a_n^t \mid s_n^t) \\\rightarrow \frac{1}{N}\sum_{n=1}^{N}\sum_{t=1}^{T_n} A_{\theta}^{\mathrm{GAE}}(s_n^t, a_n^t)\,\nabla \log P_{\theta}(a_n^t \mid s_n^t) N1​n=1∑N​t=1∑Tn​​Aθ​(snt​,ant​)∇logPθ​(ant​∣snt​)→N1​n=1∑N​t=1∑Tn​​AθGAE​(snt​,ant​)∇logPθ​(ant​∣snt​)

这里的状态价值函数 Vθ(st)V_{\theta}(s_t)Vθ​(st​) 一般用一个神经网络来拟合，可以和策略网络共享同一部分参数，最后一层不同即可。这里的状态价值函数只需要最后输出一个标量来代表当前状态的价值即可，监督信号则是当前步到轨迹结束所有的 Reward 的加权和。

之前 On-Policy 模式下，用于采集训练数据的网络和被训练的网络是同一个，这样做的另一个缺点是：采集的数据只能使用一次，下一次训练就需要重新采集，这会导致训练效率低下

因此我们希望采集数据的模型和我们最终训练的模型不是同一个，并且采集的数据可以用于多次训练，这启发了 Off-Policy 模式的产生

首先我们介绍重要性采样的概念，求 f(x)f(x)f(x) 的期望，其中 x 服从分布 p(x)p(x)p(x)，则有：

E(f(x))x∼p(x)=∑xf(x)∗p(x)=∑xf(x)∗p(x)q(x)q(x)=∑xf(x)p(x)q(x)∗q(x)=E(f(x)q(x)p(x))x∼q(x)=1N∑n=1Nf(x)p(x)q(x),  x∼q(x)E(f(x))_{x\sim p(x)}=\sum_{x}f(x)*p(x)\\ =\sum_xf(x)*p(x)\frac{q(x)}{q(x)}\\ =\sum_{x}f(x)\frac{p(x)}{q(x)}*q(x)\\ =E(f(x)\frac{q(x)}{p(x)})_{x\sim q(x)}\\ =\frac{1}{N}\sum_{n=1}^Nf(x)\frac{p(x)}{q(x)},\;x\sim q(x) E(f(x))x∼p(x)​=x∑​f(x)∗p(x)=x∑​f(x)∗p(x)q(x)q(x)​=x∑​f(x)q(x)p(x)​∗q(x)=E(f(x)p(x)q(x)​)x∼q(x)​=N1​n=1∑N​f(x)q(x)p(x)​,x∼q(x)

这里我们的核心变换步骤是引入 q(x)q(x)\frac{q(x)}{q(x)}q(x)q(x)​ 以后，对表达式形式进行变形，将原期望转化为求 f(x)p(x)q(x)f(x)\frac{p(x)}{q(x)}f(x)q(x)p(x)​ 的期望，其中 x 服从 q 分布，可以近似成在 q 分布下做 N 次采样，对采样结果计算均值，因此我们就把 p 分布下的均值，转化为了 q 分布下的均值，进而转化为了 q 分布下的采样，这是变形的关键

有了重要性采样的概念，就可以再次更新策略梯度表达式：

1N∑n=1N∑t=1TnAθGAE(snt,ant)∇log⁡Pθ(ant∣snt)=1N∑n=1N∑t=1TnAθ′GAE(snt∣ant)Pθ(ant∣snt)Pθ′(ant∣snt)∇log⁡Pθ(ant∣snt)\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}A_{\theta}^{GAE}(s_n^t,a_n^t)\nabla\log P_{\theta}(a_n^t|s_n^t)=\\ \frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}A_{\theta'}^{GAE}(s_n^t|a_n^t)\frac{P_{\theta}(a_n^t|s_n^t)}{P_{\theta'}(a_n^t|s_n^t)}\nabla\log P_{\theta}(a_n^t|s_n^t) N1​n=1∑N​t=1∑Tn​​AθGAE​(snt​,ant​)∇logPθ​(ant​∣snt​)=N1​n=1∑N​t=1∑Tn​​Aθ′GAE​(snt​∣ant​)Pθ′​(ant​∣snt​)Pθ​(ant​∣snt​)​∇logPθ​(ant​∣snt​)

这样我们就把 On-Policy 的训练替换为 Off-Policy，这里引入参考策略 θ′\theta'θ′ 来进行采样，计算参考策略 θ′\theta'θ′ 的优势函数，然后用其更新训练策略 θ\thetaθ，只需要加上重要性采样系数 Pθ(ant∣snt)Pθ′(ant∣snt)\frac{P_{\theta}(a_n^t|s_n^t)}{P_{\theta'}(a_n^t|s_n^t)}Pθ′​(ant​∣snt​)Pθ​(ant​∣snt​)​（训练策略做某个动作概率除以参考策略做某个动作的概率）来调整优势函数即可

再次使用 ∇log⁡f(x)=∇f(x)f(x)\nabla \log f(x) = \frac{\nabla f(x)}{f(x)}∇logf(x)=f(x)∇f(x)​ 对原式进行化简，即可得到：

原式=1N∑n=1N∑t=1TnAθ′GAE(snt∣ant)Pθ(ant∣snt)Pθ′(ant∣snt)∇Pθ(ant∣snt)Pθ(ant∣snt)=1N∑n=1N∑t=1TnAθ′GAE(snt∣ant)∇Pθ(ant∣snt)Pθ′(ant∣snt)\text{原式}=\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}A_{\theta'}^{GAE}(s_n^t|a_n^t)\frac{P_{\theta}(a_n^t|s_n^t)}{P_{\theta'}(a_n^t|s_n^t)}\frac{\nabla P_{\theta}(a_n^t|s_n^t)}{P_{\theta}(a_n^t|s_n^t)}\\ =\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}A_{\theta'}^{GAE}(s_n^t|a_n^t)\frac{\nabla P_{\theta}(a_n^t|s_n^t)}{P_{\theta'}(a_n^t|s_n^t)} 原式=N1​n=1∑N​t=1∑Tn​​Aθ′GAE​(snt​∣ant​)Pθ′​(ant​∣snt​)Pθ​(ant​∣snt​)​Pθ​(ant​∣snt​)∇Pθ​(ant​∣snt​)​=N1​n=1∑N​t=1∑Tn​​Aθ′GAE​(snt​∣ant​)Pθ′​(ant​∣snt​)∇Pθ​(ant​∣snt​)​

去掉梯度，我们最终得到了 PPO 的损失函数：

Loss=−1N∑n=1N∑t=1TnAθ′GAE(snt,ant)Pθ(ant∣snt)Pθ′(ant∣snt)Loss=-\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}A_{\theta'}^{GAE}(s_n^t,a_n^t)\frac{P_{\theta}(a_n^t|s_n^t)}{P_{\theta'}(a_n^t|s_n^t)} Loss=−N1​n=1∑N​t=1∑Tn​​Aθ′GAE​(snt​,ant​)Pθ′​(ant​∣snt​)Pθ​(ant​∣snt​)​

还有一点需要考虑的则是，不能让训练策略和参考策略在同一情况下给出的动作概率分布差别过大，这里可以使用KL散度或者截断函数来加以限制：

Lppo=−1N∑n=1N∑t=1TnAθ′GAE(snt∣ant)∇Pθ(ant∣snt)Pθ′(ant∣snt)+βKL(Pθ,Pθ′)Lppo=−1N∑n=1N∑t=1Tnmin⁡(Aθ′GAE(snt∣ant)∇Pθ(ant∣snt)Pθ′(ant∣snt),  Aθ′GAE(snt∣ant)clip(∇Pθ(ant∣snt)Pθ′(ant∣snt), 1−ϵ, 1+ϵ))L_{ppo}=-\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}A_{\theta'}^{GAE}(s_n^t|a_n^t)\frac{\nabla P_{\theta}(a_n^t|s_n^t)}{P_{\theta'}(a_n^t|s_n^t)}+\beta KL(P_{\theta}, P_{\theta'})\\ \\ L_{ppo} = -\frac{1}{N}\sum_{n=1}^N\sum_{t=1}^{T_n}\min(A_{\theta'}^{GAE}(s_n^t|a_n^t)\frac{\nabla P_{\theta}(a_n^t|s_n^t)}{P_{\theta'}(a_n^t|s_n^t)},\;A_{\theta'}^{GAE}(s_n^t|a_n^t)clip(\frac{\nabla P_{\theta}(a_n^t|s_n^t)}{P_{\theta'}(a_n^t|s_n^t)},\,1-\epsilon,\,1+\epsilon)) Lppo​=−N1​n=1∑N​t=1∑Tn​​Aθ′GAE​(snt​∣ant​)Pθ′​(ant​∣snt​)∇Pθ​(ant​∣snt​)​+βKL(Pθ​,Pθ′​)Lppo​=−N1​n=1∑N​t=1∑Tn​​min(Aθ′GAE​(snt​∣ant​)Pθ′​(ant​∣snt​)∇Pθ​(ant​∣snt​)​,Aθ′GAE​(snt​∣ant​)clip(Pθ′​(ant​∣snt​)∇Pθ​(ant​∣snt​)​,1−ϵ,1+ϵ))

这里 KL 散度的表达式为 KL(P∣∣Q)=∑x∈XP(x)log⁡P(x)Q(x)KL(P||Q)=\sum_{x\in X}P(x)\log\frac{P(x)}{Q(x)}KL(P∣∣Q)=∑x∈X​P(x)logQ(x)P(x)​，即 P 分布相对于 Q 分布的相似程度，KL 散度的值大于等于 0，如果 KL 散度越接近 0，说明 P 和 Q 越相似，如果 P 和 Q 分布完全一致，则 KL 散度等于 0，且 KL(P∣∣Q)≠KL(Q∣∣P)KL(P||Q)≠KL(Q||P)KL(P∣∣Q)​=KL(Q∣∣P)

训练细节

奖励模型

数据格式：(Prompt, 好的回答 Chosen, 坏的回答 Rejected)，即对于同一条 Prompt，模型刚结婚粗多条候选回答，标注员在其中选出一条更好的回答标为 Chosen，把剩下较差的几条标为 Rejected，相比于给每个回答一个明确的打分，这种三元组的偏好数据更容易获得，也相对客观

虽然训练数据是偏好数据，但建立模型要针对一个问答序列给出一个明确的得分，由于在强化学习中，奖励信号十分重要，用于指明模型的改进方向，因此一般采用能力与当前大模型相当或者更强的模型，这里能力相当的模型也可以用于生成奖励信号，因为评价回答的好坏比生成一个好的回答要容易的多

模型结构：将大模型的预测头 LM Head（权重 shape：[hidden_size, vocab_size]，把hidden_size维度的 Token 投影到vocab_size维度的 logits）改为 Score Head（权重 shape：[hidden_size, 1]，把hidden_size维度的 Token 投影到标量 score），这里输出只有一维，即一个标量打分值，并且只对序列最后一个 token 输出得分，因为只有最后一个 token，能够看到前面所有的 token，可以对整个序列进行评分

整体训练

一共有 4 个模型：

  • 基准模型：用于初始化训练用的策略，将其作为起点的 πθ(0)\pi_{\theta}^{(0)}πθ(0)​，同时在 PPO 中做 KL 参照，限制新策略不能偏离它太远，确保稳定训练，该模型后续不再训练，作为训练模型的参照 / 标尺来使用（[hidden_size, vodac_size]）
  • 训练模型：要用PPO 更新的主角策略 πθ\pi_{\theta}πθ​，结构与 SFT 模型相同，负责逐个 Token 的生成，在给定奖励和 KL 约束下，提高好回答的概率，降低差回答的概率（[hidden_size, vocab_size]）
  • 建立模型：输入整段（prompt, response），输出一个标量奖励值，学习人类偏好，用于给训练模型 / 策略所产生的回答打分，形成某一条完整轨迹 Trajectory 的回报，可以在主干 Transformer 顶层接一个 Score Head 来实现（[hidden_size, 1]）
  • 状态价值模型：我们上面说 Vθ(st)V_{\theta}(s_t)Vθ​(st​) 的求解需要训练一个网络模型，指的就是状态价值模型，可以与训练模型共享主干，在顶部接一个 Score Head 即可，序列生成任务重，每个解码步都有一个状态，对应一个状态价值（[hidden_size, 1]）

在实际训练中，训练模型可以和状态价值模型合并，共用一套参数，最后输出两个头，LM Head 和 Value Head 即可

在当前的 PPO 训练环境下，状态就是截止到当前 Token 的序列，动作就是接下来要输出的序列，大模型就是策略函数，奖励模型需要针对完整的输出，给出一个得分

目前在训练两个模型：文本输出的大模型和输出每个 Token 状态价值的模型，状态价值网络的监督信号有三种实现方式：

  • 蒙特卡洛法：Vlabel(St)=rt+γrt+1+γ2rt+2+…γT−1rtV_{label}(S_t)=r_t+\gamma r_{t+1}+\gamma^2 r_{t+2} + \dots \gamma^{T-1}r_tVlabel​(St​)=rt​+γrt+1​+γ2rt+2​+…γT−1rt​，整个问答序列由随机采样获得，然后统计当前 Token 到结束的 Reward 衰减累计值，由于是随机采样，所以方差很大
  • 时序查分法：Vlabel(St)=rt+γV(St+1)V_{label}(S_t)=r_t+\gamma V(S_{t+1})Vlabel​(St​)=rt​+γV(St+1​)，只采样一步，当前的价值等于采样一步所得到的 Reward 加上衰减过的下一步价值，但依赖当前网络的价值估计会带来偏差
  • 广义优势法：Vlabel(St)=AtGAE+V(St)V_{label}(S_t)=A_{t}^{GAE}+V(S_t)Vlabel​(St​)=AtGAE​+V(St​)，其中 GAE 表示每一步状态做出当前动作的期望回报减去当前状态的期望回报，即整体表示当前步采样当前动作的期望回报，因为 GAE 同时考虑了一步到多不采样，平衡了方差和偏差，一般采用这种方法
