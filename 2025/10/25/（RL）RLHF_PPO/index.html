RLHF

通过阅读本文，您可以大致了解和人类偏好学习（RLHF）以及近端策略优化（PPO）的相关原理与推导，本文是强化学习有关的第一篇内容

RLHF（Reinforcement Learning from Human Feedback）是大语言模型在生成领域的新训练范式，即以强化学习方式依据人类反馈优化语言模型

在过去的模型训练中，我们对生成结果的评估一般是主观和依赖上下文的，比如，我们希望模型生成一个有创意的故事，一段真实的信息性文本，或者可执行的代码片段，这些结果难以用现有的基于规则的文本生成指标进行衡量（如 BLEU，ROUGE），除了评估指标，现有的模型通常以预测下一个词的方式和简单的损失函数进行建模，没有显式地引入人类偏好和主观意见

RLHF 的核心思想在于：使用强化学习的方式直接优化带有人类反馈的语言模型，它能够使得在一般文本数据语料库上训练的语言模型性能和复杂的人类价值观所对齐，即使用人类对生成文本的反馈作为性能衡量标准，或者进一步用该反馈作为损失来优化模型

RLHF+PPO

RLHF 是一项涉及多个模型和不同训练阶段的复杂概念，大致可以分为三个步骤：

  • 预训练一个语言模型 LM
  • 聚合问答数据并且训练一个奖励模型 RM（Reward Model）
  • 用强化学习 RL 方式微调 LM

预训练语言模型

首先，使用经典的预训练目标训练一个语言模型，由于 RLHF 还是一个尚待探索的领域，对于哪种模型适合作为 RLHF 的起点目前还没有明确的答案

接着，我们基于预训练的 LM 来生成用于训练奖励模型 RM（也称为偏好模型）的数据，并在这一步引入人类的偏好信息

训练奖励模型

RM 是 RLHF 区别于旧范式的开端，奖励模型 RM 把 “人类更喜欢哪种输出” 变成一个标量奖励；有了这个 “可微 / 可用的奖励”，我们才能把任意 RL 算法（如 PPO）无缝接到语言模型（LM）上进行偏好对齐训练

RM 的实现有两种体系结构：

  • 端到端集成进 LM（二者使用同一骨干，共享参数或共享表示）
  • 模块化独立存在（单独设置一个打分器）

关于模型选择方面，RM 可以是另一个经过微调的 LM，也可以是根据偏好数据从头开始训练的 LM，这里对 RM 的训练可看做有监督的训练，我们用prompt-respone+score这样的人工标注数据，训练 RM 在输入prompt-response以后，能够基于对这条 prompt 的 response 打出符合人类偏好的分数

在训练文本方面，RM 的prompt-response文本是从预定义数据集中采样生成的，并且使用初始的 LM 给这些 prompt 生成多条候选回答；在训练奖励数值方面，需要人工对 LM 生成的回答进行排序 / 偏好比较（这里之所以进行排名而不是打分，是因为绝对分数因为不同标注者的价值观不同而难以校准，而排名看相对优劣可以消除线性变换歧义）

对于具体的排名方式，目前比较成熟的方式是对不同 LM 在相同提示下的输出进行比较，然后使用 Elo 系统为每一组 prompt 下的不同 response 建立一个完整的排名，这些不同的排名结果将被归一化用于训练的标量奖励值

在具体实现层面，目前成功的 RLHF 系统使用了和生成模型具有不同大小的 LM，可能原因是偏好模型和生成模型需要具有类似的能力来理解提供给它们的文本，直观来讲就是，RM 需要有足够的能力来理解 LM 所输出的结果

强化学习微调

使用强化学习训练 LM 需要克服的一大难点在于，微调 LM 中全部的 10B+ 个参数的成本过高，目前的可行方案在于，使用策略梯度强化学习算法（Policy Gradient RL）中的近端策略优化（PPO，Proximal Policy Optimization）微调初始 LM 的部分或全部参数

PPO 算法已经存在了相对较长的实践，并且有大量的使用文档，所以大家更倾向于在 RLHF 中选择 PPO，目前 RLHF 的许多进步大多是和如何把熟悉的 RL 算法应用到更新参数量较大的模型方面有关

在讲解具体算法之前，我们把微调任务表述为强化学习问题：

  • 策略 policy：一个接受 prompt 并且返回一些列文本或文本概率分布的 LM
  • 行动空间 action apace：当前策略的行动空间是 LM 词表中所对应的所有 Token，一般在 50k 数量级
  • 观察空间 observation space：可能的输入 Token 序列，一个很大的空间，大小为词汇量^输入Token的数量
  • 奖励函数：偏好模型和策略转变约束 Policy shift constraint 的结合

PPO 算法确定的奖励函数具体计算如下：把 prompt xxx 输入初始的 LM 和当前微调的 LM，分别得到了输出文本 y1,y2y_1, y_2y1​,y2​，将来自当前策略的文本传递给 RM 得到一个标量奖励值 rθr_\thetarθ​，将两个模型的生成文本进行比较，计算差异的惩罚项，这个惩罚项通常设计为输出词分布序列之间的 KL 散度的缩放，即

r=rθ−λrKLr=r_\theta-\lambda r_{KL} r=rθ​−λrKL​

这一项被用于惩罚 RL 策略在每个训练批次中生成大幅度偏离初始模型的情况，以确保模型输出连贯且合理的文本，如果去掉这一惩罚项，可能会导致模型在优化中生成乱码文本来欺骗奖励模型以提高奖励值

最后根据 PPO 算法，按当前批次数据的奖励指标进行优化，这来自于 PPO 算法的on-policy特性，PPO 算法是一种信赖域优化 TRO（Trust Region Optimization）算法，它使用梯度约束确保更新步骤不会破坏学习过程的稳定性

我们目前推出的模型存在以下两个问题：

  • 每一个动作按理说只能影响这个动作之后的 Reward，而不是影响整个轨迹中这个动作之前的 Reward

另一个需要考虑的问题是，在好的局势下，无论做什么动作都会带来正向收益（正的 Reward），此时算法会增加所有动作的概率，Reward 大的动作的概率会增加的大一些，Reward 小的动作增加的概率会小一些；在坏的局势下，无论做什么动作都会带来负向收益（负的 Reward），这会导致算法收敛变慢

而我们期望的现象是相对好的动作概率得以增加，相对差的动作概率得以减小，这样可以加速收敛，一个很自然的想法就是减去一个基线的 baseline，即 B(snt)B(s_n^t)B(snt​)，这样可以反映出当前动作相对于其他动作的价值

训练细节

奖励模型

数据格式：(Prompt, 好的回答 Chosen, 坏的回答 Rejected)，即对于同一条 Prompt，模型刚结婚粗多条候选回答，标注员在其中选出一条更好的回答标为 Chosen，把剩下较差的几条标为 Rejected，相比于给每个回答一个明确的打分，这种三元组的偏好数据更容易获得，也相对客观

虽然训练数据是偏好数据，但建立模型要针对一个问答序列给出一个明确的得分，由于在强化学习中，奖励信号十分重要，用于指明模型的改进方向，因此一般采用能力与当前大模型相当或者更强的模型，这里能力相当的模型也可以用于生成奖励信号，因为评价回答的好坏比生成一个好的回答要容易的多

模型结构：将大模型的预测头 LM Head（权重 shape：[hidden_size, vocab_size]，把hidden_size维度的 Token 投影到vocab_size维度的 logits）改为 Score Head（权重 shape：[hidden_size, 1]，把hidden_size维度的 Token 投影到标量 score），这里输出只有一维，即一个标量打分值，并且只对序列最后一个 token 输出得分，因为只有最后一个 token，能够看到前面所有的 token，可以对整个序列进行评分

整体训练

一共有 4 个模型：

  • 基准模型：用于初始化训练用的策略，将其作为起点的 πθ(0)\pi_{\theta}^{(0)}πθ(0)​，同时在 PPO 中做 KL 参照，限制新策略不能偏离它太远，确保稳定训练，该模型后续不再训练，作为训练模型的参照 / 标尺来使用（[hidden_size, vodac_size]）
  • 训练模型：要用PPO 更新的主角策略 πθ\pi_{\theta}πθ​，结构与 SFT 模型相同，负责逐个 Token 的生成，在给定奖励和 KL 约束下，提高好回答的概率，降低差回答的概率（[hidden_size, vocab_size]）
  • 建立模型：输入整段（prompt, response），输出一个标量奖励值，学习人类偏好，用于给训练模型 / 策略所产生的回答打分，形成某一条完整轨迹 Trajectory 的回报，可以在主干 Transformer 顶层接一个 Score Head 来实现（[hidden_size, 1]）
  • 状态价值模型：我们上面说 Vθ(st)V_{\theta}(s_t)Vθ​(st​) 的求解需要训练一个网络模型，指的就是状态价值模型，可以与训练模型共享主干，在顶部接一个 Score Head 即可，序列生成任务重，每个解码步都有一个状态，对应一个状态价值（[hidden_size, 1]）

在实际训练中，训练模型可以和状态价值模型合并，共用一套参数，最后输出两个头，LM Head 和 Value Head 即可

在当前的 PPO 训练环境下，状态就是截止到当前 Token 的序列，动作就是接下来要输出的序列，大模型就是策略函数，奖励模型需要针对完整的输出，给出一个得分

目前在训练两个模型：文本输出的大模型和输出每个 Token 状态价值的模型，状态价值网络的监督信号有三种实现方式：

  • 蒙特卡洛法：Return(St)=rt+γrt+1+γ2rt+2+…γT−1rtReturn(S_t)=r_t+\gamma r_{t+1}+\gamma^2 r_{t+2} + \dots \gamma^{T-1}r_tReturn(St​)=rt​+γrt+1​+γ2rt+2​+…γT−1rt​，整个问答序列由随机采样获得，然后统计当前 Token 到结束的 Reward 衰减累计值，由于是随机采样，所以方差很大
  • 时序查分法：Return(St)=rt+γV(St+1)Return(S_t)=r_t+\gamma V(S_{t+1})Return(St​)=rt​+γV(St+1​)，只采样一步，当前的价值等于采样一步所得到的 Reward 加上衰减过的下一步价值，但依赖当前网络的价值估计会带来偏差
  • 广义优势法：Return(St)=AtGAE+V(St)Return(S_t)=A_{t}^{GAE}+V(S_t)Return(St​)=AtGAE​+V(St​)，其中 GAE 表示每一步状态做出当前动作的期望回报减去当前状态的期望回报，即整体表示当前步采样当前动作的期望回报，因为 GAE 同时考虑了一步到多不采样，平衡了方差和偏差，一般采用这种方法

我们实际使用的其实是广义优势法中给出的回报值的计算公式，和具体推导得到的 Return=δ+VReturn=\delta+VReturn=δ+V 有一定的出入

PPO-Pipeline

模型准备

首先我们需要准备上文中提到的四个模型，即 actor_model 策略模型，ref_model 参考模型，reward_model 奖励模型，value_model 价值模型，这里我们使用Qwen2.5-1.5B作为 actor_model 和 ref_model，使用deberta-v3-base作为 reward_model，而 value_model 则在 actor model的基础上加一个价值头（用于把 actor_model 的 logits 变形为一个标量价值）实现

采样阶段

采样阶段主要指generare_samples()的过程，我们把 Prompts 输入给 actor_model（这里我们先把 actor_model 当做是基础模型），让它对于给定的 Prompts 输出一组 Response（此时 Prompts 和 Response 都是 token id 序列的形式），我们把得到的 Prompt+Response 进行组合，做长度对齐，封装为 Samples（采样样本）传给训练阶段的主流程

时间轴概念：我们得到的action_log_probs, ref_action_log_probs, values, rewards, advantages, returns的形状都是[bsz, num_actions]，这里的 num_actions == response 的 token 数（被 action_mask 过滤后只保留有效位），因此下标 t 与 rewards 的第二维（num_actions/response 时序）严格一一对应；这条时间轴不包含 prompt 段，而是对齐到 response 段的 “有效动作” 序列

打分阶段

我们考虑为每一条 Sample（本质是 Prompt+Response 的组合）进行打分，这里的打分由 KL 散度和奖励模型输出两部分组成，这个打分最终作为 PPO 的回报值 Return 计算过程中的 Reward：

  • KL 散度：我们把每一条 Prompt+Response 对应的 Token 序列作为整体输入分别给 actor_model 和 ref_model，两个模型根据输入进行逐个 Token 的 KL 散度计算
  • 奖励模型输出：调用 reward_model 对于每一条 Prompt+Response 的整体进行打分

KL 散度计算

我们以 actor_model 为例，它会根据输入 Prompt+Response 做逐个 Token 的预测输出，假设输入为[bsz, seqlen]，则模型的输出为[bsz, seqlen, vocab_size]，其中模型在每一条序列的 t 位置的 logits 即为 t+1 位置的 token 出现的概率（未归一化），我们相对这个概率做对数softmax，接着我们以输入的 Prompt+Response 序列中 Response 部分实际出现的每个 Token 作为依据，取出 actor_model 预测这些 Token 的对数概率

比如在 Response 的 t 位置，实际出现的 Token 为 Token_id_3，那么我们就取 actor_model 在 t-1 位置的 lontis 对数归一化以后的结果中，Token_id_3 对应的对数概率，因为我们 actor_model 都是用前序序列预测下一个 Token 的概率，因此最后一个 Token 舍弃不用，做归一化以后，得到 actor_model 的 log_probs，形状为[bsz, seqlen-1]

ref_model 生成的 log_probs 过程同理，形状也为[bsz, seqlen-1]，我们对他们做逐个 token 的 KL 散度计算，即log_ratio = action_log_probs - ref_action_log_probs，这个log_ratio的形状也为[bsz, seqlen-1]

奖励模型输出

这里我们的 reward_model 和 actor_model 所用的模型不同，对应的 Tokenizer 也不同，因此这里我们需要把 Prompt+Response 的 Token id 先解码，再用 reward_model 的编码器重新编码，才能输入到 reward_model 中，该模型会给每一条序列整体打一个标量得分，形状为[bsz, 1]

我们取出log_ratio中最后一个有效 Token 的位置，把这个标量得分只加在最后一个有效 Token 的位置上，即log_ratio的最后一列（有效列）上加上这个标量得分，此时我们即可得到 Loss 计算中的 Reward，即奖励值，形状为[bsz, seqlen]

这是因为后面在计算 GAE 反向递推的时候，最后一个有效动作时刻上的 reward 自然会回传到前面的动作上，而过程惩罚（KL）则每个 token 上都生效，二者共同构成完整的奖励信号

同时需要注意，reward_model 的参数是不会更新的，在 PPO 阶段，会使用with torch.no_grad()对其进行参数冻结，如果想动态更新 RM，需要再 PPO 之外单独增加一条收集偏好+微调 RM 的训练流程

价值计算

除此之外，我们还需要进行价值（Critic）的计算，即 Loss 公式中 V 的计算，我们将 Samples 直接作为输入传给 value_model 即可，该模型输出形状为[bsz, seqlen]的价值，即模型对于每个时刻生成 Token 的得分（这里需要区分 reward_model 和 value，前者是对一个序列整体进行打分，后者是对一个序列中的每个 Token 的生成进行逐个 Token 的打分）

注意：在我们 PPO 的标准公式推导中，Reward 就是奖励模型对整个序列的打分值，这里我们额外加上了一个逐 Token 的 KL 散度约束，这里主要是为了限制 actor_model 在训练过程中，不能偏离 ref_model 太远，防止其出现过于离谱的结果

优势与回报计算

在打分阶段中，我们得到了 valuesvaluesvalues，即 V(t)V(t)V(t)，以及 RewardRewardReward，即 r(t)r(t)r(t)，在有了这两部分以后，我们即可对回报（ReturnReturnReturn）和广义优势函数（GAEGAEGAE）进行求解，这里我们使用上文中提到的反向推导与近似计算思路：

δ(t)=r(t)+γV(t+1)−V(t)GAE(t)=δ(t)+γλGAE(t+1)Returns=GAE+V\delta(t)=r(t)+\gamma V(t+1)-V(t)\\ GAE(t)=\delta(t)+\gamma\lambda GAE(t+1)\\ Returns=GAE + V δ(t)=r(t)+γV(t+1)−V(t)GAE(t)=δ(t)+γλGAE(t+1)Returns=GAE+V

其中假设序列生成的时刻就到 T，则 T+1 时刻的 V 和 GAE 值都假设为 0，即可进行各时刻 GAE 和 V 值的倒推

损失函数与反向传播

在得到 ValuesValuesValues 和 GAEGAEGAE 以后，我们即可进一步构建 PPO 的损失函数，用于对两个核心模型：策略模型（actor_model）和价值模型（value_model）进行参数更新

PPO-Loss

对于 actor_model 的损失函数，即为 PPO 中经典的损失函数，首先我们使用先前构造好的 Prompt+Response 给策略模型，再次做一次前向传播，按照上面打分阶段的思想，依据 Response 中的真实 token id，对模型输出的逐个 Token 的 logits 做对数Softmax 以后，按照真实 Token 的索引，取出策略模型当前的 log_probs

此时我们还需要记录 actor_model 在打分阶段计算 KL 散度时给出的 log_probs，称为 old_log_probs，这就是我们 Loss 计算公式中用于作比的两个对数概率，其中当前的 log_probs 即为 log⁡Pθ(at∣st)\log P_{\theta}(a_t|s_t)logPθ​(at​∣st​)，而 old_log_probs 即为 log⁡Pold(at∣st)\log P_{old}(a_t|s_t)logPold​(at​∣st​)，结合我们之前计算的 GAEGAEGAE，即可计算 actor_model 更新的损失函数：

ratio=exp⁡(logprobs−oldlogprobs)loss=−mean(min⁡(ratio∗AGAE,clpi(ratio∗AGAE)))ratio = \exp(log_probs-old_log_probs)\\ loss=-\text{mean}(\min(ratio*A^{GAE},\text{clpi}(ratio*A^{GAE}))) ratio=exp(logp​robs−oldl​ogp​robs)loss=−mean(min(ratio∗AGAE,clpi(ratio∗AGAE)))

该公式即为对上文中 Loss 的具体实现

这里需要注意的是，在模型前向生成预测 Token 序列时，尽管我们给模型的输入是 Prompt+Response 的完整序列，但是模型的自回归掩码能够保证因果性，即 Causal LM 在位置 t 的隐状态只能看到 <= t 的前缀，这是模型自带的因果掩码作用的结果，模型并不会因为看到真实的 Response 而作弊

Value-Loss

对于 value_model 的损失函数，即用于更新 value_model 的损失函数，首先我们使用先前构造好的 Prompt+Response 给价值模型，再次做一次前向传播，得到模型当前对序列的逐个 Token 的价值

此时我们还需要记录 value_model 在打分阶段求解的 V(t)V(t)V(t) 序列，这里记作 old_values，而模型当前输出的结果称为 values，同时我们还需要记录回报计算过程中得到的 Returns，这三部分将进行 MSE 计算，得到损失函数如下：

LV=(mean)[∑t1t(Vtnew−Returnst)2∑t1t]L_V=\text(mean)[\frac{\sum_t 1_t(V_t^{new}-Returns_t)^2}{\sum_t 1_t}] LV​=(mean)[∑t​1t​∑t​1t​(Vtnew​−Returnst​)2​]

1~t~ 来自 action_mask（只在 response 段的有效 token 上做回归），并做了有效步归一化（这里可以先简单理解为就是对 V 和 Return 做最小均方误差），这里我们把真实奖励与旧的汇报函数进行结合，其中 returnsreturnsreturns 是给 Critic 学习的目标，我们希望其输出的逐个 Token 的 V(t)V(t)V(t) 能够更加接近回报值，这是 PPO 过程对价值函数更新的标准流程
