  • 链路层基本协议

    • 无限制的单工
      • 认为 单向 + 理想信道 + 待发送数据无限 + 接收方总能及时处理
    • 单工停-等
      • 考虑到 接收速度不如发送速度 的问题。
      • 措施：接收方收到一帧给一个回复，发送方拿到回复后再继续发
    • 有噪声信道的单工
      • 考虑到 信道有噪声，数据可能出错 的问题
      • 采用ARQ协议（接发窗口大小均为1 的滑动窗口协议）：校验和 + 确认帧 + 超时重发 + 帧序号（主要是考虑到了出错，希望能出错重发，如果是字符错了，那靠校验和检测出，如果是帧级错误，那靠帧序号+确认帧来超时重发）
    • 滑动窗口协议
      • 考虑到 ARQ信道利用率过低（每个时刻只能有一个数据帧/确认帧在信道上）
      • 思想：允许发送方连续发送多个帧 + 用滑动窗口实现流量控制
      • 应对问题：连续发送的多个帧，中间一个帧出错了，后续帧被正确接收了
        • 后退N帧（GBN）
          • 接收窗口大小等于1，发送窗口大于1
          • 序号位数为n，则发送窗口最大尺寸为2n−12^{n}-12n−1，不然由于序号循环利用，发了2n2^n2n个后，重新发一遍原来的序号，接收方不知道是重传了还是新的数据。
        • 选择重传（无序接收的滑动窗口协议）
          • 要求接收方可以临时缓存接收窗口中的帧
          • 接收方检测到出错后发送一个否定的确认NAK
          • 具有累计确认的功能
          • 序号位数为n，则发送窗口的最大尺寸为2n−12^{n-1}2n−1。这是因为希望接收窗口向前移动的话和旧窗口之间的编号不重合，接收窗口最多一次向前移动发送窗口的大小，加上其本身的大小，需要有接收+发送<2n接收 + 发送 < 2^n接收+发送<2n，同时接收窗口又不应大于发送窗口（不然没啥意义），所以接收一定 小于 2n−12^{n-1}2n−1

  • ppp协议的三个组成部分

    • 将ip数据报封装到单行链路的方法
    • LCP链路控制协议
    • NCP网络控制协议

  • ppp协议的特点

    • 提供的是：有连接的不可靠服务
      • tcp保证可靠传输，链路层的ppp不需要纠错（但是可以检错），不需要设置序号，不需要流量控制
      • ppp不支持多点线路
      • ppp只支持全双工链路
      • ppp能在一条物理链路上支持多种网络层协议
      • 既支持异步链路，也支持同步链路

  • ppp帧的同步方法

    • 字符填充
      • 0x7E => 0x7D, 0x5E
      • 0x7D => 0x7D, 0x5D
      • 0x03 => 0x7D, 0x23 (不止是0x03，信息字段中的任何一个小于0x20的字符都是)
    • 零比特填充
      • 发现5个连续的1就添加一个0

  • 信道利用率公式：U=TDTD+RTT+TAU = \frac{T_D}{T_D+RTT+T_A}U=TD​+RTT+TA​TD​​

  • 链路层介质访问控制 —— CSMA/CD

    • 几个重要策略
      • 载波监听
      • 碰撞检测（发送前和发送中都进行监测）
      • 碰撞强化（发生碰撞后发送干扰信号）
      • 碰撞退避（碰撞后停一段时间再继续发，碰撞多方的等待时间应不相同）
    • 争用期：两倍端到端时延，发送开始后过了一个争用期的时间还没检测到碰撞那就是没有碰撞。
      • 争用期的时间长度内，数据帧不能发完，所以数据帧长度不能太短。如果数据帧变小了，距离要变短来保证帧发送完的时候能反射回来，距离变短没事。

  • CSMA/CD的优缺点：

    • 优点：
      • 负载轻时效率高
      • 实现简单
    • 缺点
      • 负载重时容易碰撞，效率比较低
      • 实时性较差（可能碰撞，不保证数据能在某个ddl前一定送达）

网络层

  • 网络层提供的两种服务

    • 面向连接 的服务 —— 虚电路
      • 开始传数据前，先由网络建立连接，然后通过该连接传，保证了数据传输的可靠性
      • 虚电路只是逻辑连接，不是真的建立了一条物理线路
    • 无连接 的服务 —— 数据报
      • 网络发数据时不用先建立连接，发送的分组可能出错，也不保证分组传送的时限
    • TCP/IP 采用数据报服务

  • 虚电路和数据报的区别

    • 是否有连接建立的不同
    • 可靠性保证对象的不同（前者网络层保证，后者用户主机保证）
    • 转发路径的不同（前者属于同一虚电路的分组均要按照该虚电路进行转发，后者每个分组自由选择路径，到了就行）
    • 分组到达的顺序不同（前者一定是按序抵达，后者不保证按序抵达）
    • 对待节点故障的反馈不同（前者所有经过故障节点的虚电路均瘫痪，后者故障节点可能丢弃分组，网络转发路由可能变化）

  • ip分类（他们的前缀构成了一组台阶）

    • A 0
    • B 10
    • C 110
    • D 1110 组播/多播地址
    • E 1111 保留为今后使用

  • 全0的ip地址表示本主机，全1的ip地址表示广播地址

  • 求ip报中的校验和的算法

    • 包头16位分组
    • 每组取反
    • 循环（最高位进位进到最低位）求和
    • 结果取反，即为校验和

  • 路由器视角下的转发包的过程

    • 收到一个包，把他的目的ip同 自己本身的 各个接口的网络的子网掩码相与，看看目的ip是不是在自己直连的网络中
    • 如果是，就直接转发，不是则继续执行
    • 查自身路由表，看看有没有该目的ip属于的表项，有则按照下一跳进行转发，无则继续执行
    • 看路由表有没有默认路由，有则转，没有继续执行
    • 报错

  • CIDR —— 无分类域间路由

    • CIDR记法（也就是斜线记法）
      • 可以省略点分十进制中低位连续的0（10.0.0.0/10 简写为 10/10）
    • 路由聚合：
      • 也叫“构成超网”
      • 好处：减少路由表中的表项个数，减少路由器之间交换的路由信息量

  • 最长前缀匹配：查路由表时可能有多项匹配，按照最长前缀匹配来做

  • 网际控制报文ICMP：

    • 分装在ip报中，是ip报的数据部分
    • 分为两大类：
      • 差错报告报文
        • 终点不可达（路由器没办法找到路的时候发给主机此报文）
        • 超时（路由器因为TTL=0丢掉报文时向源主机发此报文）
        • 参数问题
        • 源点抑制（路由器因为拥塞丢掉报文时向主机发送这个信息）
        • 重定向（通知主机路由信息改变，之后发别的路由）
      • 询问报文
        • 回声探测（测试网络连通性）
        • 请求时间

  • 路由协议的作用：生成路由信息，而非转发分组

  • 路由算法分类

    • 静态路由选择
      • 人工设置，简单但是不普适
    • 动态路由选择
      • 自适应网络变化，但是开销比较大
      • 典型的两类：
        • 距离向量路由
        • 链路状态路由

  • 自治系统：单一技术管理下的一组路由器，基于此背景路由协议可以分为以下两类：

    • 内部网关协议IGP
    • 外部网关协议EGP

  • 自治系统之间：域间路由选择

  • 自治系统之内：域内路由选择

  • 距离向量路由：

    • 采用算法：贝尔曼福特（Bellman-ford） 算法
    • 缺点：对好消息反应迅速，但是对坏消息反应迟钝
    • rip协议：
      • 距离定义为跳数
      • 距离最大为15
    • rip协议的特点
      • 仅和相邻路由器交换信息
      • 交换的信息是自己知道的全部信息，即自己的路由表
      • 按固定的时间间隔交换信息（而不像ospf仅在链路状态变化时交换）
    • rip协议的优缺点
      • 优点：
        • 实现简单，开销小
      • 缺点：
        • 支持的网络规模有限
        • 交换的是整个路由表信息，开销比较大
        • 网络故障时，要经过很长时间才能将信息传给所有路由器
    • rip交换信息的步骤，收到邻居X发来的rip报文，记录的目的地址为to：
      • to 不在自己的路由表内：
        • 直接加入
      • to 在自己的路由表内：
        • 自己路由表中 to 对应的下一跳是X，则将X新发的信息替换掉
        • 自己路由表中 to 对应的下一跳不是X, 比较距离大小，新的小则替换

  • 链路状态路由：

    • 采用算法：迪杰斯特拉
    • 链路状态的含义：本路由器和哪些相连，以及连的这些链路的代价
    • 链路状态数据库：全网的拓扑结构。其在全网范围内是一致的。
    • ospf：
      • 优点：
        • 收敛速度快：链路状态发生变化时，会用洪泛的方式转发，收敛地快
        • 适用于大规模的网络：只在链路状态变化时才洪泛，不会产生很大的开销
      • 洪泛：结点收到分组时，向除输入链路之外的其它链路都转发出去
      • ospf将AS分区的好处：限制洪泛范围，减少整个网络的通信量
      • 特点：
        • 交换信息的对象：本AS内的所有路由器
        • 交换信息的时机：只有链路状态发生变化时才会交换
        • 交换信息的内容：链路状态（本路由器和谁相邻，相邻的链路的度量如何），只是路由器知道的部分信息。

  • BGP

    • BGP交换的信息：要达到某个网络要经过的路径向量
    • 四种报文：
      • open 用于与相邻的发言人建立关系
      • update 列出要发送的路由信息以及要撤销的多条路由
      • keepalive 周期性确认相邻关系
      • notification 发送检测到的差错
    • 特点
      • 发言人数量很少，和AS数量一个量级，让路由选择不会很复杂
      • BGP支持CIDR
      • BGP刚启动时交换完整的BGP路由表，之后只需要在变化时更新变动部分即可。

  • 路由器的组成部分

    • 分组转发部分
      • 由三个模块组成：
        • 交换结构
        • 输入端口
        • 输出端口
    • 路由选择部分：按照路由协议构建路由表

  • 路由器可以进行差错校验，但是它不能保证传输的ip分组不丢失

  • 路由器转发分组的速率的决定因素：

    • 输入/输出端口的处理性能
    • 交换机构的性能

  • NAT地址转换

    • 专用地址：只能用于机构内部的地址
      • 10/8
      • 172.16/12
      • 192.168/16
    • 互联网中的路由器不转发目的地址为专用地址的包
    • 机构内部用专用地址的好处：
      • 安全性好
      • 减少ip地址空间的占用（不同机构可以在内部各自用各自的专用地址）

    • 隧道

    • 身份认证

传输层

  • 传输层和网络层的对比：

    • 网络层实现 主机 之间的逻辑通信
    • 传输层实现 应用进程 之间的逻辑通信

  • udp和tcp的优劣：

    • udp效率更高，但是tcp更可靠，前者不用建立连接，后者必须先建立连接。

  • 传输层的分用和复用的概念：

    • 分用：传输层数据分到主机的不同应用程序
    • 复用：不同应用程序都可以使用一个运输层协议传输数据

  • 端口号问题

    • 熟知端口号：0-1023
    • 登记端口号：<= 49151
    • 客户端口号：客户进程短暂使用的端口号 >= 49152

  • 为什么传输层用端口而不是进程id来区分上层应用程序：

    • id依赖于特定的操作系统，希望能够在不同操作系统之间流畅通信

  • udp的特点

    • 无连接
    • 尽最大努力交付。即，不管中间会不会出错，我就聚焦于把数据送过去的需求。
    • 面向报文的，没有拥塞控制
    • 支持 n 对 n 的交互通信（n任意取值）
    • 首部开销比较小

  • udp和ip关于校验的区别和联系:

    • udp计算的是整个数据报的校验和，ip仅仅计算包头的校验和。
    • 计算校验和的方法相同，最后求和后记得取一下反码

  • tcp可靠的含义：顺序对，无重复，无差错，无丢失

  • tcp：全双工通信

  • tcp面向的对象：字节流。tcp会认为传输的对象就是无结构的字节流而已。

  • tcp的特点：

    • tcp是虚连接，而不是真正的物理连接
    • 不关心一次传输多长的报文到缓存中
    • tcp可以将一些长的报文划分短一些再传，也可以等待累积有足够多的字节后再构成报文发出去

  • tcp连接的端点是 套接字

  • tcp和udp都有伪首部，且内容是相同的

  • tcp要求接收方必须有累计确认功能，可以减小传输开销

  • tcp的可靠传输的实现方法：

    • 校验和
    • 超时重传

  • tcp流量控制的内涵：控制发送方的速率不要太快，要让接收方来得及接收

  • tcp如何避免双方等待死锁

    • 设置持续计时器，到时间了就发一个零窗口探测报文，接收方给出的回应中包含窗口值，仍然是0的话就重新计时，否则就打破死锁了。

  • 拥塞的产生原因：对资源的需求大于供应

  • 拥塞的特点：

    • 表现为性能下降
    • 常常易趋于恶化（丢弃了部分报文，发送方认为超时了重新传，导致网络更拥堵）

  • 拥塞控制的概念：防止过多的数据注入到网络中而让网络中的路由器或链路不至于过载

  • 拥塞控制和流量控制的区别：

    • 拥塞控制是全局性过程
    • 流量控制是点对点通信量的控制

  • 拥塞控制的一般原理

    • 开环控制：事先预防
    • 闭环控制：事后处理

  • tcp建立连接的必要性

    • 双方能够感知到彼此的存在
    • 双方需要能够一起协商一些参数
    • 需要能够对传输实体资源进行分配

  • tcp为什么握手3次而不是两次：

    • 防止失效的连接请求占用资源
      • 解释：客户端发送的连接请求A滞留了，超时重发了一个B，然后B主导的连接过了一会关闭了。之后A到了服务器端了，如果二报文握手的话，服务器端直接给个答复就建立连接了，但是其实客户这时候并没有连接的意思，就占用资源了。如果三报文的话，滞留的A的回复给到客户端，客户不会理睬，就不会成功三次握手，就不会建立无效的连接。

  • 关于tcp释放连接：连接的双方都可以提出释放。

  • 四报文挥手释放连接：

    • A向B发送连接释放请求
    • B确认A发来的请求报文
    • B等一会确定没数据要发给A了，发给A连接释放报文
    • A确认B发来的释放报文
      • A等待2MSL（报文最大存活时间）后彻底关闭

  • 为什么最后要等2MSL

    • 使得A最后的确认报文能到B
    • 防止失效的连接请求报文段出现在本连接中，再经过2MSL就可以让本连接产生的所有报文从网络中消失。

应用层

  • 网络子系统为应用程序提供调用接口的方式：套接字
  • DNS —— 域名系统
  • SMTP —— 简单邮件传输协议
    • 发送邮件 和 客户读取邮件 协议不同
      • 发送：STMP
      • 读取：POP3
    • 用户代理，指各种接发邮件的app/网站
  • 域名服务器分四类

    • 权限
    • 本地

    • 控制连接
    • 数据连接

背诵总结

各种"报文"格式

总关系：(mac (ip (udp/tcp)))

  • mac帧：| 6 目的mac地址 | 6 源mac地址 | 2 上层类型(ip) | 数据部分(ip报文) | 4 fcs校验序列 |（整个帧之前可能要插入定界符）
  • ip报：| 1个字(0.5B版本号 + 0.5B包头长度单位字 + 1B + 2B 总长度) + 2个字 各种字段 + 1个字 源ip + 1个字 目的ip + 若干字选项 | 数据部分(比如tcp/udp) |
  • 两个传输层（都是 首部 + 数据）：
    • udp：| 半字源端口 + 半字目的端口 + 半字UDP数据报长度 + 半字校验和|
    • tcp：| 半字源端口 + 半字目的端口 + 1字序号 + 1字确认号 + 2字其它 + 变长选项 | 数据部分 |

udp和tcp的应用实例

  • udp：
    • dns
    • rip
  • tcp：
    • bgp
    • http / https
    • ftp
    • SMTP

传输层涉及到的熟知端口号

  • 21 ftp
  • 25 smtp
  • 53 dns
  • 80 http
  • 443 https

udp/tcp的ip protocal字段

ip protocal表示上层协议

  • udp 对应的 protocal 值为17
  • tcp 对应的 protocal 值为6

各个层有的协议

  • 应用层：
    • ftp 文件传输协议
    • dns 域名系统
    • http 超文本传输协议（和www基本划等号）
    • https 安全（safe）的超文本传输协议
    • smtp 简单邮件传输协议
  • 传输层：
    • tcp 传输控制协议
    • udp 用户数据协议
  • 网络层：
    • ip 网际协议
    • arp 地址解析协议
    • rarp 逆地址解析协议
    • icmp 网际控制报文协议
    • igmp 网际组管理协议

各个层的协议数据单元

  • 传输层：报文
  • 网络层：分组（数据报）
  • 数据链路层：帧
  • 物理层：比特

零碎记录

  1. ip包分片的偏移量以8字节为单位。
  2. 描述 ip包头长度/udp数据报头长度 的单位一般就是字（一行）
  3. 集线器起到信号放大和转发的作用
  4. 波特率指的是，码元级别！！单位时间传输的码元个数。
  5. 脉码（脉冲编码）调制 对应 音频
  6. 各种网络在物理层互联时要有相同改的数据传输速率和链路层协议，因为物理层不涉及流量控制和差错处理
  7. 传统的以太网是半双工通信
  8. 卫星通信受气候影响很大。光纤通信不受电磁干扰和噪声影响。
  9. 码分复用 CDM = 码分多址 CDMA
  10. 数据链路层具有纠错功能，比如海明码可以纠正1位的错误
  11. 数据链路层可以定义数据格式（组帧），提供可靠传输（检错纠错），控制介质访问
  12. 流量控制实际上是对发送方的数据流量的控制。
  13. 数据链路层的检错和纠错 并不意味着 可靠传输。还可能会出现帧级别的问题，比如帧重复，帧丢失，帧失序。
  14. internet中所有末端系统和路由器都必须实现《ip》协议
  15. 为什么要分层：
    • 各层间独立，耦合度低，易于实现和维护
    • 分层的话比较灵活
  16. mac地址和ip地址的区别：前者用于物理层和链路层，后者用于网络及以上各层，ip地址是一种逻辑地址。
  17. 为什么要用ip和mac这两种不同的地址：
    • 要是网络都用mac地址的话，会涉及到复杂的硬件地址转换工作。ip地址解决了这一问题，连接到网络的各个主机只要有统一的ip地址，它们之间的通信就会像连接在一个网络上一般简单。
  18. 为什么tcp在进行差错控制/流量控制/拥塞控制时一定要建立连接：
    • 因为进行这些操作的时候，双方要感知到对方的存在，还可能一起协商一些参数等，这种问题需要在两者的连接中解决。
  19. IEEE局域网的数据链路层分为两个子层：逻辑链路控制 + 介质访问控制
  20. ppp提供的是面向连接的不可靠传输服务
  21. arp广播请求分组时用的 目的mac地址 是 FF:FF:FF:FF:FF:FF
  22. RTT 是指 往返时间
  23. ipv6采用 冒号十六进制 的记法
  24. ipv6不用首部校验和
  25. ipv6数据报如果太大而不能被路由器转发，则将会被路由器丢弃
  26. rip是应用层协议，ospf是网络层协议
  27. rip机器在启动时路由表是空的，然后执行以下两步：
    • 首先得到它的直连子网的信息
    • 然后读取邻居给的rip报文信息
  28. tcp和网络层虚连接有什么区别：
    • tcp报文段在传输层的抽象的逻辑信道中传输，对路由器不可见
    • 网络层若采用虚连电路方式，就没办法提供无连接服务了，但是传输层用tcp是不影响网络层提供无连接服务的。
    • 虚电路经过的交换节点都必须保存虚电路状态信息。
  29. 吞吐量：每秒能彻底发送多少比特（有确认帧，考虑到往返时延和发送时延）
  30. 信道利用率反映的是：信道中有多少比例的时间是有数据通过的
