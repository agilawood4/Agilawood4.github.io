上机总结

计网上机考试范围很广，大概率平时也没太多时间去好好学，全靠考前的一两个周去自己整理，如果你的考试时间比较靠后，可以和先考的同学多交流，补全自己的知识储备

路由协议补充

记得补充两条重要的指令，可以免去配很多静态路由：

（1）import-route static

对于 OSPF 协议的一个 Area 而言，对于处于 Area 边界的设备执行 import-route static 指令，可以让整个 Area 中的设备共享其静态路由，对于下面这个组网：

假设 R1，R2，R3 处在 Area 0 中，则按理说需要给 Area 0 中的每个路由器都分别配置到 PCA 和 PCB 的静态路由，但是当做完静态路由引入以后，只需要配 R1 到 PCA 的静态路由，R2 到 PCB 的静态路由，再通过静态路由共享，即可让整个 Area 0 中的设备都学到去 PCA，PCB 的路由

（2）peer x.x.x.x next-hop-loacal & peer x.x.x.x reflect-client

对于 BGP 协议的一个 AS 而言，对于完全处于 AS 内部的设备，一定要配置 peer x.x.x.x next-hop-local，这几乎是一个常识（配置了不会错），而如果一个完全处于 AS 内部的设备需要学到边界设备的静态路由，则可以把它作为反射器，也就是配置 peer x.x.x.x reflect-client，对于下面这个组网：

假设 R1，R2，R3 处于一个 AS 中，整体配置 BGP 协议，则首先 R1，R2 作为边界路由器要配置到 PCA，PCB 的静态路由，而为了让内部的 R2 也学到这个静态路由，并且找到正确的下一跳，给 R2 配置上述的两条指令即可

如何自查配置

因为机房的连线经常接触不良，因此建议每配置好一个设备的基础端口和 IP 都执行 idisplay interface brief 检查端口和 IP 是否配对

全网 ping 通的条件即每个设备的路由表中都有到达各个网段的目的网段，子网掩码，下一跳的表项，可以在每个设备上都通过执行 display ip routing-table 来查看配置，如果最后时间来不及查错，就直接少哪个网段就配哪个网段的静态路由，这样查一圈下来一定能 ping 通

TCP 补充

应用层协议大概率会考 TCP（因为这个最复杂，报文也比较有可分析的地方），又因为这个协议需要上 Linux 虚拟机上去做，有很多易错点：

  • 一开考拿到试卷如果看到后面要考 TCP 协议，一定在电脑开机的时候就把其他虚拟网卡都禁用（仅保留组网配 IP 用到的那一个），这个一定要刚开机的时候做，否则一段时间后就没法禁用了
  • 通常都是在 PCA 和 PCB 全网 ping 通的基础上，继续做 TCP 实验，做之前需要把 PCA，PCB 本地机的网址改成别的，不要和虚拟机网址重复（192.168.1.2 和 192.168.2.2，这两个地址是默认用来做 TCP 发送方和接收方的），否则会产生地址冲突
  • 如果发现 PCA，PCB 上做不了，可以在原来组网的基础上，直接把网口改成 PCC，PCD，在另外两台机器上做
  • 进入 Linux 虚拟机以后记得先右上角选模式为 Tcp_Sent 和 Tcp_Received 再进行发送，对应的脚本运行指令在考试常用指令中会给出，可以直接抄，发送报文的时候一定是接收方先监听，发送方后发送

其它可能会考到的应用层协议还有 FTP 和 DHCP，这两者比较简单，大概流程也是先截获报文然后提交，分析报文

总结

如果 ping 不通不要反复去调路由协议，直接配静态路由，尽快让全网通，因为老师检查的时候大概率不会去看你 display cu 出来的一大堆配置，只会现场让你在本机上 ping 一下，考试时间相当紧张，应用层协议+简答题最起码要留 40 分钟时间去做，最后还要留 10 分钟左右去上传配置

作为本科阶段最后一门上机考试，强度还是很高的，祝大家考试顺利
