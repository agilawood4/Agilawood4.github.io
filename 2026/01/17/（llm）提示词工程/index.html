LLM 基础

LLM 参数的含义

以通义千问的 Qwen-2.5-14B-1M 为例：

  • 2.5：通常表示版本号，版本号越高表示模型发布日期越靠后
  • 14B：模型内部的参数量，1B 表示一亿个参数
  • 1M：模型处理文本时能够处理的最大 token 数（即上下文窗口的长度），1M 表示一百万 token
  • Turbo：可能是模型的一个优化版本
  • Preview：可能是模型的一个预发布或测试版本
  • V：表示模型可能被用于处理视觉数据（图像或视频），如通义千问-QVQ-72B-Preview

LLM 的 API 调用

通过获取 LLM 的 API 秘钥，我们可以实现在本地通过代码对大模型的调用，通过对通义千问大模型的调用，我们演示一个最简单的通过 API 和大模型对话的实例

首先我们需要from openai import OpenAI导入OpenAI库，接着调用一个大模型整体上分为两步：

  • 初始化OpenAI类，创建一个client实例，传入我们要调用的大模型的api_key以及base_url，每个模型对应的base_url可以在对应的官网上查阅获得
  • 调用client.chat.completions.create()方法，传入模型名称model以及我们写好的提示词messages，其中后者为列表，列表中每个元素为字典

调用completion = client.chat.completions.create()的结果为一个json格式的字典，我们通过取completion.choices[0].message.content即可得到大模型的回答

提示词工程

书写简洁清晰，具有良好规范性的提示词有利于大模型更好地了解我们的提问意图，从而给出更高质量的回答

提示词规则

通用模型的提示词：

  • prompt = 角色 + 任务 + 要求 + 细节
  • 其中细节 = 示例 + 约束，即先给出一个正确的示例，然后告知模型在生成时需要满足的约束条件

推理模型的提示词：

  • prompt = 角色 + 背景信息（可选）+ 任务
  • 注意：如果是有推理能力的模型，我们一般不需要告诉他该怎么做，只需要告知它角色和任务目标，让其自行思考即可

提供角色信息的重要性：

  • 指定用户角色和大模型所扮演的角色，比如 “我是一名计算机专业的初学者，你是 LLM 领域的专家，请你为我详细讲解 LLM 的提示词工程有关知识”

这里在调用client.chat.completions.create()时我们引入了两个控制大模型回答发散程度的参数：

  • temperature：取值范围[0,2)，控制模型本身思考的发散程度，取值越大越发散
  • top_k：取值范围(0,1]，控制模型返回答案的个数，模型会把所有候选答案按概率从高到低排序，把top_k作为阈值，若满足P(ans[1])+P(ans[2])+...P(ans[k])<top_k以及P(ans[1])+P(ans[2])+...P(ans[k+1])>top_k，则返回前k个答案作为最终答案

提示词调优技巧

（1）零样本提示（Zero shot）

（2）少量样本提示（Few Shot）

（3）链式思考（Chain-of-thought，COT）

（4）自我一致性（自洽性，Self-Consistency）

（5）思维树（Tree-of-thought，TOT）
