运行时的存储组织及管理

我们在填写符号表的时候，每填入一个符号（变量、函数…）就要为其分配一段存储空间，供未来生成的目标代码在运行时使用

概述

运行时的存储组织及管理

定义：目标程序运行时所需要存储空间的组织与管理，以及源程序中变量存储空间的分配，称为运行时的存储组织及管理

符号表中的符号和数据区中的某一块存储空间是一一对应的

静态存储分配和动态存储分配

（1）静态存储空间：

在编译阶段由编译程序实现对存储空间的管理，为源程序中的变量分配存储的方法

条件：

如果在编译时能够确定源程序中变量在运行时的数据空间大小，且运行时不改变，那么就可以采用静态存储分配方法

但是并不是所有数据空间大小都能在编译过程中确定

（2）动态存储分配：

在目标程序运行阶段由目标程序实现对存储空间的组织和管理，为源程序中的变量分配存储的方法

条件：

  • 在目标程序运行时进行分配
  • 编译时要生成动态分配的目标指令

需要动态存储空间的实例：递归函数（根据递归层数参数 n 的不同，我们需要为递归函数分配的变量和递归栈的深度都不确定，因此在编译时无法确认运行时空间的大小，带参编译，但具体参数值在运行时才会传入）

静态存储分配

分配策略：由于每个变量所需要的空间大小在编译时已知，因此可以用简单方法给变量分配目标地址

  • 首先开辟一块数据区，数据区的首地址在编译过程进入当前模块时确定
  • 按照编译顺序给每个模块分配存储空间
  • 在模块内部按顺序给模块的变量分配存储，此时一般使相对地址，所占数据区的大小由变量类型所确定

静态分配要求语言不存在指针或动态分配，不存在递归调用过程

编译过程除了要给源程序（模块）中的各个变量分配数据区以外，对于子程序而言还要在数据区中保留返回地址，对形式参数液压分配存储以存放相应的实参的信息，另外在编译时还要分配临时变量空间（如存放表达式计算的中间结果等）

因此，对于某个模块，我们会对其相应的数据区进行进一步的划分，如 FORTRAN 会对数据区做如下划分，从低地址到高地址分别是：

  • 隐式参数区：返回地址，函数返回值
  • 形式参数区：函数实参
  • 局部变量和临时变量区：当前模块中定义的变量

动态存储分配

编译程序产生目标码的时候，不仅要产生由源程序本身生成的目标码，同时还要额外生成和存储分配有关的目标码，这部分目标码用于在目标代码执行时为目标程序中的变量等分配存储空间

由于编译时无法具体确定某些数据空间的大小，因此他们分配存储空间必须在程序运行时进行，此时编译程序生成有关存储分配的目标代码，实际上的分配要在目标程序运行时进行，这种分配方式就称为动态存储分配

对于分程序结构（允许 Block 嵌套），且允许使用递归调用的语言，常使用栈式动态存储分配，即使用一个类似于堆栈的运行栈来实现数据区的分配

具体的分配策略为：整个数据区为一个堆栈

  • 进入一个过程时，在栈顶为其分配一个数据区
  • 退出一个过程时，撤销该过程的数据区

活动记录

上图中，AR（active record）称为活动记录，一个典型的活动记录可以分为以下三部分：

  • 局部数据区：存放模块中所定义的各个局部变量，在模块运行时可能会被频繁访问
  • 参数区：存放隐式参数和显式参数，自上向下又分为四部分
    • 形参数据区：每一个形参都要分配数据空间，形参单元中存放实参值或者实参地址，程序中的临时变量或寄存器（现场）存储在参数区中
    • prev abp：abp 通常指向当前正在执行的模块的栈帧的起始位置（基地址），因此 prev abp 就指向调用当前正在执行的模块的那个模块的桢栈的起始位置（注意是调用者而不是被调用者），函数执行完时，释放其数据区，数据区指针指向调用前的位置
    • ret addr：返回地址，本质上是指令地址，即执行完当前函数后下一条待执行指令所处的地址
    • ret value：函数返回值，无返回值则为空
  • display区：存放当前模块各个外层模块活动记录的基地址（因为某个模块可能会访问其外层某个模块中的变量）

首先是最外层的 BBLOCK，它相当于是 main 函数，由操作系统调用，一般没有参数区和 display 区，只有局部变量区，维护其中的局部变量 X Y NAME

接着进入 M1 模块，它的 prev abp 应当执行调用它的模块对应的桢栈的基地址，也就是 AR1 的基地址，它的 ret addr 应当指向它执行结束后将要执行的下一条指令，也就是最外层 BBLOCK 的 END 语句，它的外层模块只有 BBLOCK，因此 display 区只有 AR1 这一个数据区

接着进入 M2 阶段，它被 M1 所调用，因此其 prev abp 指向 AR2 桢栈的起始地址，它的 ret addr 指向 END M1，它的 display 区中只有 AR1 一个数据区

最后进入 M2 内部的 BBLOCK，它的 prev abp 指向 AR3 对应桢栈的基地址，它的返回地址为 END M2，它的 display 区中存 AR1 和 AR3（因为其外层为 BBLOCK 和 M2 两个模块）

补充内容（重要）

首先，根据教材，如果是有返回值的函数，用 Pascal 语言，这个函数在定义的时候会被写成FACTION FACTORIAL():INTEGER，也就是把它写成一个类似变量的形式，函数返回值的时候会写成变量赋值的形式，即FACTORIAL:=N，这表示这个函数有返回值

此时我们需要在调用者函数的模块的活动记录的局部数据区，为这个函数的函数名分配一段存储空间（把它当变量来看），一开始这个函数还没返回值的时候，对应位置写 ？，等它的返回值可以确定以后，就写上返回值

是的没错，实际上的返回值是不写在参数区的，即不会写在 return addr 的下面，也就是我们上面给他分配的那个区域，而是会写在 Pascal 语义下的 “函数名同名的隐式局部变量”，也就是调用者为函数名分配的那段空间中

函数返回值通过函数名同名的隐式局部变量来承载，所以专门给它划了一个局部数据区的槽（标了 FACTORIAL → ?），在执行期间它的值“暂时未知”因此画成 ?；当函数计算完成后，函数尾声会把这个槽中的值作为返回值，书上说只有返回大对象的时候，才会采用调用者提供缓冲区，也就是理论课讲的那种方法

display 区中存的指针应当是指向当前 AR 外层的那些 AR 的 FP（活动记录基地址），这个地址一般不会在 AR 中画出来，所以我们就形象化表述一下，指向基地址，那就是当前 AR 最靠下的地址，就是 return addr 下面的那条线，沙比编书人画图太密集，看上去像指向了 return addr，实则大错特错，和返回地址根本没关系，其实是 return addr 下面那条线

返回地址说的是指令执行层面的事，abp 说的是桢栈跳转之间的事，二者完全没关系好不好！

递的过程：

  • 所有递归函数都看成是被主函数调用的，因为所有递归函数真正的入口都是主函数中的那一个，因此所有递归函数活动记录中的 display 区都应当指向主函数 AR1 中的 return addr 下面的那条线

二元变量地址

根据活动记录的分区规则，我们给出变量在活动分区中的二元变量地址，即：（BL，ON）

  • BL：变量声明所在的层次，可以用它找到该层数据区的开始地址
  • ON：相对于显式参数区开始位置的位移（相对地址）

display 区构建规则

规则（一）：

从外层模块进入内层模块，内层模块的 display 区在继承外层模块所有 dsplay 区的基础上，再加上外层模块本身即可

规则（二）：

从内层模块 i 进入外层模块 j，或同一层前后并列的两个模块 i j（其中 i 靠后 j 靠前），这里我们假设模块编号就是模块的次序，即模块 j 就是第 j 个模块，此时从模块 i 的 display 区构建模块 j 的 display 区，只需要取模块 i 的前 j-1 个模块即可
