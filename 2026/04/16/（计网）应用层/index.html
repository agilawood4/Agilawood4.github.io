应用层

应用层协议是为了解决某一类应用问题

  • 由于网络应用的多样性，应用层协议的一大特定就是种类多

应用层的许多协议都是基于客户 / 服务器方式实现的

  • 客户（client）和服务器（server）都是指通信中所涉及的两个应用进程
  • 客户 / 服务器方式所描述的是进程之间服务和被服务的关系
  • 客户是服务请求方，服务器是服务提供方

套接字编程接口

网络子系统作为操作系统的一部分，以应用编程接口的形式向应用程序提供调用接口

套接字（socket）是最常用的应用层编程接口

经典的 socket 编程接口采用同步调用方式（又称阻塞式），例如调用 recv() 时，调用进程将被阻塞，直到收到数据为止

Windows 这种基于事件驱动的系统中，程序的执行由事件驱动，需要异步事件驱动方式的套接字编程接口

域名系统 DNS

域名系统（DNS，Domain Name System）是 Internet 使用的命名系统

  • Internet 采用层次结构的命名树作为主机的名字，并使用分布式的域名系统 DNS
  • 多个域名服务器上运行专门的域名服务器程序，完成从域名到 IP 地址的解析
  • 域名系统 DNS 基于 UDP 协议实现
    • 应用进程需要进行域名解析时，就调用域名解析程序（resolver），它成为 DNS 的一个客户
    • 向本地域名服务器发送域名解析请求（UDP 报文），其中包含待解析的域名
    • 本地域名服务器在查找域名后，返回应答报文，其中就包含待解析域名对应的 IP 地址

具体过程：应用进程 → 调用域名解析程序 → 称为域名系统的客户 → 向域名服务器发送域名解析请求报文（UDP）→ 报文中包含待解析的域名 → 获得应答报文 → 应答报文中有待解析域名对应的 IP 地址

Internet 的域名结构

Internet 采用层次树结构的命名方法，域名不区分大小写，域名长度不超过 255 字符，级别越高的域名越靠后

…buaa.edu.cn\dots buaa.edu.cn …buaa.edu.cn

其中 cn 是顶级域名，edu 是二级域名，buaa 是三级域名，以此类推

域名服务器

域名服务器用于把域名解析成 IP 地址，因为网络之间通信需要 IP 地址，但用户有时候可能只提供域名，因此需要解析

（1）根域名服务器

  • 根域名服务器知道所有的顶级域名服务器的域名和 IP 地址
  • 当本地域名服务器无法解析域名时，就求助于根域名服务器

（2）顶级域名服务器

  • 负责管理在该顶级域名服务器注册的所有二级域名

（3）权限域名服务器

  • 负责一个区（zone）的域名服务器

（4）本地域名服务器

  • 有时也称为默认域名服务器

域名服务器定期把数据复制到多个域名服务器中保存，其中一个是主域名服务器，其它的是辅助域名服务器

域名服务器的组织方式也是树状结构

域名的查询过程：

  • 主机向本地域名服务器的查询一般采用递归查询
    • 如果本地域名服务器不知道被查询域名的 IP 地址，那么它就以 DNS 客户的身份，向根域名服务器继续发出查询请求报文
  • 本地域名服务器向根域名服务器的查询一般采用迭代查询
    • 当根域名服务器收到本地域名服务器的迭代查询请求报文时，它要么直接给出要查询的 IP 地址，要么告诉本地域名服务器下一步要向哪个域名服务器查询
    • 本地域名服务器根据根域名服务器的提示继续后续的查询

本地域名服务器的迭代查询：

本地域名服务器也可以使用递归查询（较少使用）

本地域名服务器向上层查询域名的过程对于客户端来讲是不可见的，客户端只知道自己和本地域名服务器发生了一次交互，但不知道本地域名服务器拿到 IP 地址的具体策略

  • 一些不常见的域名在跳转到对应的网页速度很慢，可能是因为解析这个域名花费了较长的时间

文件传输协议

文件传输协议（FTP，File Transfer Protocol）使用客户 / 服务器方式：

  • 一个 FTP 服务器进程可以同时服务多个用户进程
  • FTP 服务器进程由两大部分组成：一个主进程，负责接受新的请求；若干个从属进程，负责处理单个请求

FTP 使用两个 TCP 连接：控制连接和数据连接

  • 控制连接：在会话期间一直保持打开，FTP 客户发出的传送请求通过控制连接发送给服务器端的控制进程，但控制连接不用来传送文件
  • 数据连接：收到 FTP 客户发送来的文件传输请求后，服务器端的控制进程创建数据传送进程和数据连接，数据传送进程实际完成文件的传送，传送完毕后关闭数据传送连接并结束运行

万维网

万维网解决的三个问题：

  • URL：标识分布在 Internet 上的万维网文档
  • HTTP：实现万维网上各种超链的链接
  • HTML：存储和表示万维网文档

统一资源定位符 URL

URL（Uniform Resource Locator）是对 Internet 上资源位置和访问方法的一种简洁表示

URL 的一般形式为：

<协议>://<主机>:<端口>/<路径>\text{<协议>://<主机>:<端口>/<路径>} <协议>://<主机>:<端口>/<路径>

其中：

  • 协议：常见的有 ftp 文件传输协议，http 超文本传送协议，News USENET 新闻
  • 主机：存放资源的主机在 Internet 中的域名
  • 端口和路径有时可以省略，如果是 http 协议，如果缺省端口字段，则使用端口号 80，如果缺省路径字段，则指主页（home page）
  • URL 不区分大小写

超文本传送协议 http

HTTP（HyperText Transfer Protocol）基于 TCP 协议实现，是万维网上可靠地址交换文件的重要基础

  • HTTP 是面向事务的客户 / 服务器协议
  • HTTP 是无状态的（stateless），服务器不记录客户端的访问状态（登录 or 未登录）
  • 基本工作原理：Web 服务的熟知端口号是 80，服务器通常在该端口上监听，客户端需要请求某个页面时，与服务器建立 TCP 连接，之后请求传送文件，并进行文件的传送，传送完毕后释放 TCP 连接

HTTP 协议鼠标点击超链接后发生的事件：

  • 浏览器分析超链指向页面的 URL
  • 浏览器向 DNS 请求解析 IP 地址
  • 域名系统 DNS 解析出服务器的 IP 地址
  • 浏览器和服务器建立 TCP 连接
  • 浏览器发出取文件命令：GET/chn/yxsz/index.htm
  • 服务器给出响应，把文件 index.htm 发给浏览器
  • TCP 连接释放
  • 浏览器显示 index.htm 中所有文本

http 版本演进

http 1.0 每传送一个文件都需要建立一次 TCP 连接

http 1.1 使用持续连接（persistent connection）

  • 服务器发送响应后，在一段时间内保持连接，使客户与服务器可以继续传送后续的 HTTP 请求报文和响应报文
  • 这并不局限于传送同一个页面上链接的文档，只要保证这些文档都在同一个服务器上就行
  • 目前主流的浏览器都支持 http 1.1

http 2.1 增加了同时传输多个流的支持

http 3.1 基于 UDP 协议建立

http 报文

http 有两类报文类型：

  • 请求报文：从客户向服务器发送请求报文
  • 响应报文：从服务器到客户的回答

http 是面向正文（text-oriented）的，报文中的字段都是 ASCII 码串（低效但简单），因而每个字段的长度都是不确定的

每个 http 报文都由三部分组成：开始行，首部行，实体主体，在请求报文中，开始行就是请求行

其中方法本质上就是一些命令，URL 即所请求资源的 URL，版本即 http 的版本

而对于应答报文，即把请求报文中 URL 字段替换成状态码字段，即开始行变成状态行，其余保持不变

目前大部分手机端的 APP 和后端服务器的通信仍然采用 http 协议

万维网文档

万维网文档即 HTML（HyperText Markup Language），其中定义了许多用于排版的命令，即标签（tag）

HTML 把各种标签嵌入到万维网的页面中，构成 HTML 文档

  • HTML 文档为文本格式
  • 浏览器从服务器读取 HTML 文档后，按照其中嵌入的各种标签，根据显示器尺寸和分辨率显示页面，仅当 HTML 文档是以 .html 或者 .htm 为后缀时，浏览器才对此文档的各种标签进行解释

https 协议

HTTPS：安全的 HTTP 协议（S 表示 Security），建立在 SSL 基础上

  • 目前 SSL / TLS 已被浏览器广泛支持，很多 Web 应用利用 HTTPS 协议实现安全传输
  • HTTPS 的熟知端口号为 80

SSL：指的是 Secyre Socket Layer，为 TCP 协议提供信息加密和完整性（由此可以推断 HTTPS 协议也是基于 TCP 协议实现的）；而 TLS 是在 SSL 基础上实现的进阶版

数字证书生成：A 支持的加密算法和 B 支持的加密算法取交集，在其中取一个算法作为约定的加密算法，用这个算法生成公钥，同时包含持有者信息，以及网络中的权威机构进行签名

  • 数字证书 = 持有者信息 + 公钥 + 权威机构的数字签名
  • 之所以要有数字签名，是为了防止公钥被攻击，当发生攻击时，公钥中的签名随之发生改变，可以被检测出来，由此检测受到攻击

加密算法分成通用加密算法和公钥加密算法两种：

  • 通用加密算法 / 对称加密算法：使用密码本加密，加密方和解密方使用同一个秘钥，速度快，适合大量数据加密
  • 公钥加密算法：使用公钥和私钥加密，公钥与私钥成对出现，数学相关但不可推，速度慢

电子邮件

发送邮件的协议：SMTP

早期的邮件只支持 7bit 的 ASCII 编码，后面又提出了 MIME 标准，后者可以在邮件首部说明数据类型，支持在邮件中传输文本，声音，图像等多模态数据

读取邮件的协议：POP3 和 IMAP

  • POP3：Post Office Protocol
  • IMAP：Internet Message Access Protocol

邮件发送也是基于 TCP 协议实现的

电子邮件系统中的两类实体：

  • 用户代理 UA（User Agent）：用户与电子邮件系统的接口，即电子邮件客户端软件
  • 邮件服务器：用于发送和接收邮件，按照客户 / 服务器方式工作，发送邮件和接收邮件使用不同的协议，分别为 SMTP 和 POP3 协议

一个邮件服务器既可以作为客户，也可以作为服务器

SMTP 通信的三个阶段：

  • 建立连接：连接在发送主机的 SMTP 客户和接收主机的 SMTP 服务器之间建立，不使用中间邮件服务器
  • 邮件发送
  • 释放连接：邮件完成发送后，SMTP 释放 TCP 连接
