团队项目选题分析

选题

项目名称： 智能学习助手

项目类型： 面向大学生的综合性学习辅助平台

核心定位： 一个集智能问答、知识管理、刷题练习和社区交流于一体的学习平台，旨在通过AI技术提升学生的学习效率和知识沉淀能力。

项目背景

社会调研

在当前的数字化学习环境中，大学生面临着诸多挑战：

  • 信息过载与碎片化：课程教材、课件、习题、笔记往往散落在不同平台，缺乏有效的整合与管理。
  • 答疑成本高：遇到难题时，依赖搜索引擎和通用大模型往往难以获得精准解答，而等待老师或助教回复的周期较长。
  • 复习效率低：面对海量的知识点，学生难以快速梳理逻辑结构，手动整理笔记耗时巨大。
  • 学习孤岛化：缺乏一个能够围绕特定课程进行深度讨论和资源共享的社区空间。

市场与竞品分析

目前市场上的学习类产品主要分为两类：

  • 通用大模型：如 ChatGPT、Gemini，虽然具备强大的生成能力，但缺乏学科专业性和上下文感知，容易幻觉，且无法针对特定课程或教材提供精准答案
  • 垂直型学习平台，如各类题库APP，主要提供课程内容或练习题目，但缺乏智能交互和个性化的知识管理功能，用户需要自己建立学习闭环

竞品实例：【智学北航】提供的“小航AI助手”，只有简单的对话功能，并且没有各个课程的专业课支撑，功能有限，可扩展性差，因此受众约等于0

市场空白： 现有产品未能很好地满足大学生“在特定课程语境下，通过多模态交互获得精准解答，并一键生成结构化笔记”的核心需求。

选题意义

  • 提升学习效率：在特定的课程语境下，通过智能问答和知识管理，帮助学生快速理解难点、整理思路。
  • 构建学习社区：围绕课程建立交流空间，促进同学间的互助与分享。
  • 技术实践价值：探索 RAG（检索增强生成）、多模态处理、结构化输出等前沿技术在教育场景中的应用。

主要功能和创新

主要功能模块

用户模块（P0）

  • 注册与登录：支持邮箱注册，包含验证码校验流程；支持“记住我”功能。
  • 信息管理：允许用户修改昵称、头像和个人简介。
  • 学习数据看板：展示累计学习天数、累计提问数，并配学习任务完成度与学习时长趋势等多维图表。

知识问答模块（P0）

  • 对话管理：支持新建对话、查看历史记录（含课程标签和时间）、搜索及删除。
  • 多模态交互：用户可选择关联课程，支持文本+图片混合提问，降低输入门槛。
  • 笔记生成：用户可通过指令触发，系统结合大模型与知识库检索，生成结构化Markdown笔记，并支持下载为MD或PDF格式。

题库模块（P2）

  • 选课与考纲：用户选择课程后，可查看对应考纲。
  • 智能组卷：基于考纲和题库自动生成练习题
  • 在线答题与批改：支持客观题在线作答，系统自动批改并记录错题。
  • 错题本归档：自动归集错题，便于复习。

学习论坛模块（P1）

  • 课程分区：论坛按课程（如编译原理、操作系统）进行分区浏览。
  • 发帖与互动：支持Markdown编辑器发帖，以及点赞、回复等互动功能。
  • 内容管理：支持编辑/删除自己的帖子，以及举报不当内容。

创新点

  • 课程标签驱动的知识库增强（RAG）：区别于通用AI，系统将回答限定在特定学科语境下，结合本地知识库（如课程PPT、教材）进行检索增强生成，大幅提升回答的精准性和专业性。
  • 多模态学习闭环：从“提问（文本+图片）→ 解答 → 笔记生成 → 题库练习 → 社区讨论”，形成完整的学习闭环，用户无需在不同平台间切换。

产品分发

发布渠道

  • Web端：作为主要入口，面向所有浏览器用户，便于在实验室、图书馆等不同场景下使用。
  • 移动端：未来可扩展为微信小程序或原生App，满足碎片化学习需求。

推广方式

  • 社交媒体：通过微信公众号、群聊、知乎等渠道分享使用案例和功能介绍。
  • 口碑传播：鼓励用户邀请同学使用，形成学习小组。
  • 校园合作：与本校院系学生会部门合作，作为课程辅助工具进行推广。

实现方法

技术架构

  • 前端：基于 Vue 3 ，全面采用 Composition API 与 TypeScript 进行类型安全的开发，样式层面使用 SCSS 实现模块化与主题定制。状态管理统一交由 Pinia 负责，严格遵守“状态与组件分离”原则，禁止在视图组件中直接修改 Store 外部状态，确保数据流清晰可控。所有后端请求均收敛至独立的 api 层进行封装拦截，并在代码提交前强制通过 ESLint + Prettier + TypeScript 检查，保障代码规范与质量

  • 后端：基于 FastAPI 框架构建，利用其原生的异步特性与自动生成 OpenAPI 文档的能力，配合 Docker 容器化部署以实现环境隔离与快速交付。接口调试与文档维护统一通过 Apifox 进行管理，确保前后端协作过程中的接口定义一致性与实时性

  • 数据库：选用 MySQL 作为核心业务数据库，负责存储用户体系、对话记录、笔记元数据等强一致性要求的结构化数据；同时引入 Chroma 作为向量数据库，专门用于存储课程知识库切片后的向量化数据，以支撑 RAG（检索增强生成）的高效语义检索。

  • AI层：核心智能交互基于 GPT 系列模型构建，通过集成 RAG 技术打通“检索-增强-生成”全链路。系统会先将用户的课程资料预处理并存入 Chroma 向量库，当用户发起提问时，后端先在向量库中召回相关知识片段，再将其与大模型 Prompt 动态拼接，从而在保证回答专业性的同时显著降低幻觉率。

开发流程

  • 阶段一：实现用户模块、知识问答模块（文本交互+课程标签+RAG）、对话管理与笔记生成。
  • 阶段二：完善多模态交互（图片上传+OCR）、题库模块（智能组卷+错题本）。
  • 阶段三：开发学习论坛模块，优化UI/UX，增加个性化推荐。

测试与迭代

  • 单元测试：对每个功能模块进行单元测试，确保稳定性。
  • 用户测试：邀请真实用户试用，收集反馈，迭代优化。
  • 性能优化：优化AI响应速度、页面加载速度，确保高并发下的可用性。

预期规模

基于项目的阶段性推广策略，我们对用户规模和核心功能的渗透率进行了如下预估：

阶段 覆盖范围 预估用户量 核心功能渗透率 估算依据
第一阶段（MVP期） 本学院试点 200 - 400 人 60% - 70% 依托课程作业和实验环节，强制或引导本院系学生使用，主要验证“智能问答+笔记生成”核心链路。
第二阶段（成长期） 全校推广 800 - 1200 人 40% - 50% 通过校园论坛、表白墙及跨院系合作，覆盖期末考试季的高频用户，此时题库和论坛功能趋于完善。
第三阶段（稳定期） 多校辐射 5000+ 人 30% - 40% 产品形态成熟，通过开源或SaaS模式向周边高校同类专业推广，形成学习社区生态。

其他

数据安全与隐私

系统将采用业界标准的加密算法对用户敏感数据进行全生命周期保护，包括但不限于密码哈希加盐存储及传输链路的SSL加密，从源头杜绝隐私泄露风险。针对用户上传的图片或文档，服务器端仅在处理瞬间进行OCR识别或内容解析，所有临时文件将在任务完成后立即自动清除，绝不会进行长期持久化存储，确保用户的原始学习资料完全受控于本人，不被系统滥用或作为他用。

版权与合规

平台严格遵循《网络安全法》及相关知识产权法规，在用户上传资料的入口处设置明确的法律风险提示与内容审核机制，严禁传播盗版教材、付费课程录像等侵权内容。

同时，系统通过AI生成的笔记和回答均标注“仅供个人学习交流”，明确限制其商业用途，若未来涉及商业化运营，将预先设计完善的授权协议与收益分配机制，确保平台在法律框架内合规运行。

可扩展性

在技术架构层面，系统采用高度解耦的微服务与模块化设计，后端接口预留了标准化的模型适配层，使得未来可以无缝接入Claude等更多主流大模型，甚至支持用户自定义本地模型。

在业务广度上，系统不仅支持从单一科目向全学科知识库的横向扩展，更规划了“个人云知识库”功能，允许用户在未来的版本中自主上传私有资料，构建专属的AI学习大脑，并具备向考研、考公、职业资格认证等泛教育领域平滑迁移的能力。
