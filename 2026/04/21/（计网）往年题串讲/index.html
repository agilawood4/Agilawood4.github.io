网络结构题

T1 - 交换机 MAC 地址学习

交换机学习 MAC 地址的原则：

  • 当接口 X 收到源 MAC 地址 Y 时，会在转发表中添加字段 (接口, MAC地址)=(X,Y)\text{(接口, MAC地址)}=(X,Y)(接口, MAC地址)=(X,Y)
  • 当查找当前转发表找不到目的 MAC 地址对应的接口时，会向所有接口以洪泛形式发送 ARP 广播帧，查找对应的 MAC 地址对应的接口

（1）A → B

S1 首先学习接口 1 对应 MAC 地址 A，接着查找 MAC 地址 B 对应的接口，发现交换表中没有相关记录，则向接口 2 和 3 发送查询报文，则 S2 的 4 接口也会收到报文信息，即相应学习到 MAC 地址 A 是来自 4 接口

（2）B → A

S1 首先学习接口 2 对应 MAC 地址 B，接着查找 MAC 地址 A 对应的接口，发现交换表在（1）中已经存储了相应记录，接口为 3，直接转发

（3）C → A

S2 首先学习接口 1 对应 MAC 地址 C，然后查找 MAC 地址 A 对应的接口，发现交换表在（1）中已经存储了相应记录，接口为 4，直接转发；到 S1 时，首先学习到接口 3 对应 MAC 地址 C，然后查找 MAC 地址 A 对应的接口，发现交换表在（1）中已经存储了相应记录，接口为 1，直接转发

S1 交换表：

接口 MAC地址
1 A
2 B
3 C

S2 交换表：

接口 MAC地址
1 C
4 A

T2 - MAC 地址转发重写

两个原则：

  • 交换机不会重写 MAC 地址
  • 路由器会重写 MAC 地址

当数据需要跨网段传输时，MAC 地址分为以下三种情况：

  • 从源主机到下一跳路由器：源 MAC 地址为源主机的 MAC 地址，目的 MAC 地址为下一跳路由器的 MAC 地址
  • 从上一跳路由器到下一跳路由器：源 MAC 地址为上一跳路由器的 MAC 地址，目的 MAC 地址为下一跳路由器的 MAC 地址
  • 从上一跳路由器到目的主机：源 MAC 地址为上一跳路由器的 MAC 地址，目的 MAC 地址为目的主机的 MAC 地址

（1）C → S2

源 MAC 地址为 C，目的 MAC 地址为 L1

（2）S2 → S1

源 MAC 地址为 C，目的 MAC 地址为 L1

（3）S1 → R

源 MAC 地址为 C，目的 MAC 地址为 L1

（4）R → F

源 MAC 地址为 L2，目的 MAC 地址为 F

T3 - 路由表的填写

实现 A 和 B 两个网段的通信，路由表中至少需要填写三个字段：

  • A 网段的网段 IP，子网掩码，对应路由器的接口（这个字段也可能是下一跳路由器的网关 IP，视题目要求而定）
  • B 网段的网段 IP，子网掩码，对应路由器的接口（这个字段也可能是下一跳路由器的网关 IP，视题目要求而定）
  • 默认路由（网段 IP 和子网掩码都是 0.0.0.0，对应路由器的接口题目会给出）

首先需要由 A 的 IP 地址和子网掩码，计算得到 A 所在网段的网段 IP 为 211.73.0.0，子网掩码为 255.255.254.0，对应路由表为：

网段 IP 子网掩码 接口
211.73.0.0 255.255.254.0 L1
211.72.1.0 255.255.255.0 L2
0.0.0.0 0.0.0.0 L3

这里题目还给了 A 所在网段的缺省网关，但是没用到，网关即本网段的路由器接入网段所用的 IP，是一个具体的 IP 地址

缺省网关的作用是：主机不知道把数据报发给谁的时候（当前主机和目标不在同一个网段），默认送去的那个出口，即本网段内路由器的 IP 地址，通过路由器实现跨网段通信

T1 - IP 到 MAC 转换

本问考察跨网段传输时，IP 地址到 MAC 地址的转换，需要借助 ARP 协议，具体过程如下：

  • 主机用目的 IP 与自己的子网掩码做与运算，判断目的 IP 与自己不在同一网段

  • 主机查本地 ARP缓存表，看有没有默认网关 IP 对应的 MAC 记录

  • 若有缓存，直接取出 MAC，封装以太网帧发送，流程结束

  • 若没有缓存，主机发送 ARP 广播（目的 MAC = FF:FF:FF:FF:FF:FF）

  • 同网段所有主机都收到广播，只有默认网关回复 ARP Reply（单播），告知自己的 MAC

  • 发送方收到 Reply，将目的 MAC 填入帧，同时缓存该条 ARP 记录

  • 封装以太网帧，目的 MAC 填默认网关的 MAC，直接发送

同理，如果是同网段传输时，只是把查找默认网关变成查找目的主机：

  • 主机用目的 IP 与自己的子网掩码做与运算，判断目的 IP 和自己在同一网段

  • 主机查本地 ARP缓存表，看有没有目的 IP 对应的 MAC 记录

  • 若有缓存，直接取出 MAC，封装以太网帧发送，流程结束

  • 若没有缓存，主机发送 ARP 广播（目的 MAC = FF:FF:FF:FF:FF:FF）

  • 同网段所有主机都收到广播，只有目的主机回复 ARP Reply（单播），告知自己的 MAC

  • 发送方收到 Reply，将目的 MAC 填入帧，同时缓存该条 ARP 记录

  • 封装以太网帧，目的 MAC 填目的主机的 MAC，直接发送

简答题

三大网络分层结构

TCP/IP 体系结构：网络接口层，网际层，传输层，应用层

OSI 体系结构：物理层，数据链路层，网络层，传输层，会话层，表示层，应用层

五层体系结构：物理层，数据链路层，网络层，传输层，应用层

分层的原因：各层之间是独立的，灵活性好，结构上可分割，易于实现和维护，有助于标准化工作

路由聚合算法的优势

  • 缩减路由表规模，减少路由器的存储和查找开销，提高转发效率
  • 减少路由更新流量，网络拓扑变化时只需通告聚合路由，降低了路由协议的开销

路由聚合算法的作用：可以用一个地址块表示更多地址，减少路由器中路由表项的个数，减少路由交换的信息量，提高路由查找效率

RIP 和 OSPF 对比

距离向量算法：每台路由器只知道自己的邻居，不知道全局拓扑，路由器维护一张距离向量表，记录到达每个目的网络的距离（跳数）和方向（下一跳），然后把这张表周期性地告诉自己邻居，邻居根据收到的信息更新自己的表

链路状态算法：每台路由器掌握全局完整拓扑，路由器通过洪泛机制将自己的链路状态（与哪些邻居相连、链路开销多少）广播给全网所有路由器，每台路由器收集齐所有信息后，各自独立运行 Dijkstra 算法计算最短路径

距离向量算法和链路状态算法比较：

比较维度 RIP（距离向量算法） OSPF（链路状态算法）
交换对象 与直连邻居交换信息 通过泛洪与全网所有路由器交换信息
交换时机 周期性交换（每30秒），无论网络是否变化 triggered更新，网络拓扑发生变化时立即触发；此外周期性发送Hello报文维持邻接关系
交换内容 交换完整路由表（到各目的网络的距离和下一跳） 交换链路状态通告LSA（自己与哪些邻居相连、链路开销），不是路由表

延伸对比：

对比项 RIP OSPF
路由算法 Bellman-Ford（距离向量） Dijkstra（最短路径优先）
度量标准 跳数（最大15） 链路开销 Cost（= 10⁸/带宽）
收敛速度 慢，可能计数到无穷 快，拓扑变化立即通告
适用规模 小型网络（≤15跳） 大型网络
传输层协议 UDP，端口520 直接封装在IP中，协议号89
对全局拓扑的了解 不了解，只知道邻居信息 完全了解，每台路由器拥有相同的完整拓扑图

判断子网内能否通信

判断主机 A 和 B 是否能在子网内直接通信（不需要经过路由器）需要两步：

  • A 把自己的子网掩码和 B 的 IP 地址做与运算，判断是否和自己的网段 IP 相同，若相同，则 A 可以直接给 B 发送信息，否则 A 在自己子网中找不到 B，会把信息通过默认网关发送给路由器
  • B 把自己的子网掩码和 A 的 IP 地址做与运算，判断是否和自己的网段 IP 相同，若相同，则 B 可以直接给 A 发送信息，否则 B 在自己子网中找不到 A，会把信息通过默认网关发送给路由器

只有以上两个条件都满足时，A 和 B 才能在同一个子网内直接通信

判断能否连接 Internet

判断某个主机能否通过路由器连接到 Internet 需要两步：

  • 主机到 Internet 配置了路由器（看图即可，一般都会配置）
  • 主机的网关配置正确，即网关和主机在同一个网段（这里的网关即为路由器接入主机所在网段所使用的地址）

具体来讲，需要用主机的网关和主机的地址分别与它所在网段的子网掩码做与运算，如果得到的结果相同，则说明网关配置正确，可以连接到 Internet

RIP 算法考察

T1 - 向量距离表建立

对于路由器 R1 而言，它能直接交付的网段包括：

  • 它自己本身的接口，即图中的 D1，D2，D3
  • 和它自己本身的接口连接在同一条链路上的其他路由器的接口，即图中的 D4，D5

对于这些网段，路由器 R1 的下一跳永远是直接交付，距离永远是 0

对于 R2 而言，它提供了到网段 166.0.4.0 和网段 166.0.6.0 的距离，首先我们需要判断这些网段中有无 D4 所在的网段（因为 D4 相对于 R1 而言为直连，不满足邻居信息跳数 +1 的约定），显然这两个网段都不是 D4，因为它们的子网掩码为 24 位，而 D4 因为和 D2 在同一个网段，二者的子网掩码应该都是 30 位

T1 - 数据链路

从主机 A 启动到 A 与 B 完成通信，首先要通过 ARP 协议解析 MAC 地址，接着通过 TCP/IP 协议建立连接（因为题目中说了使用 TCP 协议），最后才能进行数据传输，因此 F1 ~ F5 分别是：

帧编号 作用
F0 ARP 请求帧，A 广播询问：谁是 211.72.1.2（这里 A 会先检查发现 100.12.1.15 和自己不在同一个子网，因此会改为查找默认网关对应的 MAC 地址），请把你的 MAC 告诉我

简答题

TCP 协议发送窗口的确定

（1）主机根据对方的接收窗口大小和自身拥塞控制算法来计算发送窗口大小

（2）自身发送窗口的上限值，在对方接收窗口宽度和自身拥塞窗口宽度中取较小值

（3）每当收到对方发来的数据段，都根据 TCP 报文段头部的接收窗口大小重新计算以确定是否要调整发送窗口；在传输过程中，每经过一个传输轮次，当拥塞窗口变化时，也需要重新决定是否需要调整发送窗口

ARP 过程

以太网帧中的 MAC 地址通过 ARP 协议获得，首先源主机在本机 ARP 缓存中查找有无目标 IP 对应的 MAC 地址，如果没有则在子网中广播 ARP 请求，目标 IP 设备（可以是网关，或者目的主机）收到请求后返回 ARP 应答，告知自己的 MAC 地址

链路状态路由 vs 距离向量路由

链路状态路由：每个结点通过广播的方式与其他所有结点交谈，但它仅告诉他们与自己直接相连的链路的费用

  • 代表：OSPF 协议

距离向量路由：每个结点仅和它的直接邻居交谈，它为它的邻居提供给自己到网络中其他所有结点的最低费用估计

  • 代表：RIP 协议

IP 地址 vs 物理地址

IP 是逻辑地址，工作在网络层，用于同一地址格式和寻址，全球唯一

物理地址是硬件地址，工作在数据链路层

IP 地址为了解决全球统一编址和路由选择，屏蔽底层不同网络编址的不一致性，物理地址用来表示机器的硬件地址，二者通过 ARP / RARP 协议相互转换

TCP 协议为什么建立连接

TCP 协议要实现差错控制、流量控制、拥塞控制，需要双方维护共同的状态信息，而这些状态信息必须通过建立连接来协商和同步

传输层可靠传输的保障

要实现传输层的可靠传输，需要以下保障：

  • 差错控制：发送方在每个 TCP 段中附上校验和，接收方收到后重新计算，若不一致则丢弃该段。接收方收到正确数据后回复 ACK 确认，发送方若超时未收到 ACK 则重传，以此保证数据不丢失、不损坏
  • 流量控制：接收方在 ACK 中告知发送方自己的接收窗口大小（rwnd），发送方据此控制发送速度，不超过接收方的处理能力，防止接收方缓冲区溢出
  • 拥塞控制：通过慢启动、拥塞避免、快重传、快恢复四种机制配合，发送方从小窗口开始逐步探测网络容量，检测到拥塞（超时或收到 3 个重复 ACK）后主动降低发送速率，避免网络崩溃
  • 连接控制：通信前通过三次握手建立连接，协商双方序列号和窗口大小；通信结束后通过四次挥手释放连接，确保双方数据都已发送完毕再关闭
  • 分组排序与去重：IP 层不保证顺序，TCP 给每个字节编序列号，接收方按序号重新排列乱序数据；同时利用序列号识别并丢弃网络中延迟到达的重复数据包

Tsum=0.08×(1000+3−1)=80.16msT_{sum} = 0.08\times(1000+3-1)=80.16ms Tsum​=0.08×(1000+3−1)=80.16ms
