文本表示

文本表示指的是将自然语言转化为计算机能够理解的数值形式，是绝大多数自然语言处理（NLP）任务的基础步骤

早期的文本表示方法（如词袋模型）通常将整段文本编码为一个向量，这类方法确实简单而且计算高效，但存在明显的局限性——仅统计词频无法表示语序，无法表示上下文信息，因此现代的 NLP 技术主件引入更加精细和表达能力更强的文本表示方法，以更加有效地建模语言的结构和含义

文本表示的第一步通常是分词和词表构建，如下图所示：

其中：

  • 分词（Tokenization）是将原始文本切分为若干具有独立语义的最小单元的过程，这个最小单元就称为 Token，这是所有 NLP 任务的起点
  • 词表（Vocabulary）是由语料库构建出的、包含模型可识别的 token 的集合，即词表中需要包含我们语料库中所有分词得到的 Token，并且为每个 Token 建立一个唯一的索引 ID，支持 Token 和 ID 之间双向映射
    • 词表中必须包含所有 Token，不能出现语料库中的某句话，分词后得到了某个 Token，但是不包含在词表中的情况
    • 一般从 ID 到 Token 的映射称为 Id2Word，用列表实现即可；而从 Token 到 ID 的映射称为 Word2Id，用字典实现即可

在分词和建立词表完成后，我们得到了每个 Token 对应的 ID，第三步我们对每个 Token 做向量化编码，即 Embedding 处理，把每个词真正变成词向量，具体操作为，把每个词对应的 ID（注意，这里传入网络的是 ID 而不是 Token 本身）输入嵌入层 Embedding Layer，转换为低维稠密的向量表示，即词向量

在后续训练或者预测过程中，对于我们输入的某条文本，我们会先对其进行分词，再通过词表得到每个 Token 对应的 ID，将这些 ID 传入嵌入层 Embedding 得到对应的词向量组合，再经过若干中间隐藏层的处理，以文本生成任务为例，模型最终的输出层会针对词表中的每个 Token 生成一个概率分布，表示其作为当前文本的下一个词的可能性，系统通常选取具有最大概率的 ID，通过词表查找索引回 Token，把这个 Token 也拼接到原有文本的末尾，重复上述过程，逐步生成最终的输出文本

因此文本表示的三大步骤分别是：（1）分词；（2）建立词表；（3）生成词向量

分词

不同语言由于语言结构、词边界的差异，其分词策略和算法也不尽相同，这里我们主要讲解英文和中文两种常见语种的分词方式

英文分词

按照分词粒度的大小，可以分成词级分词（Word Level），字符级分词（Character Level），子词级分词（Subword Level），下面逐一介绍

词级分词

词级分词是指将文本按单词进行切分，是最传统、最直观的分词方式，在英文中，空格和标点往往是天然的分隔符

词级分词虽然便于理解和实现，但在实际应用中容易出现 OOV（Out Of Vocabulary，未登录词）问题，即在模型使用阶段，输入文本中出现了在建立词表时没有保存的单词，常见的包括网络热词、专有名词、复合词以及拼写变体等，由于模型无法识别这些词，通常会将其统一替换为特殊标记（如 ），从而导致一信息的确实，影响模型的理解与预测能力

字符级分词

字符级分词（Character-level Tokenization）是以单个字符为最小单位进行分词的方法，文本中的每一个字母、数字、标点甚至空格，都会被视作一个独立的 Token

在这种分词方式下，词表仅有所有可能出现的字符组成，因此词表规模会非常小，覆盖率极高，几乎不存在 OOV 问题，无论输入中出现什么样的新词或拼写变体，只要组成这些词的字符在词表中，则都可以被表示出来

然而，由于单个字符本身语义信息极弱，模型必须依赖更强的上下文来推断词义和结构，这显著增加了建模难度和训练成本，此外，输入序列也会变得更长，影响模型效率

子词级分词

子词级分词是一种结余词级分词和字符级分词之间的分词方法，它将词语切分为更小的单元——子词（subword），例如词根、前缀、后缀或常见词片段，与词级分词相比，子词分词可以显著缓解 OOV 问题；与字符级分词相比，它能更好地保留一定的语义结构

子词分词的基本思想是：即使一个完整的词没有出现在词表中，只要它可以被拆分为词表中存在的子词单元，就可以被模型识别和表示，从而避免整体被替换成

常见的子词分词算法包括 BPE（Byte Pair Encoding），WordPiece 和 Unigram Language Model

其中 BPE 是最早被广泛使用的方法，该方法生成词表的过程大致如下：

  • 先把所有词语都拆成单个字符，然后迭代统计预料中出现频率最高的字符对，将其合并为一个新的子词，并加入词表，该过程持续进行，直到达到设定的词表大小

比如语料库为 [hi, his, hit]，则初始建立的词表为 [h, i, s, t]，接着我们统计语料库中 hh, hi, hs, ht, ih, ii, is, it, sh, si, ss, st, th, ti, ts, tt 出现的频率，发现 hi 出现的频率最高，则我们把 hi 加入到词表中，词表变成 [h, i, s, t, hi]，接着我们再统计语料库中 hh, hi, hs, ht, hhi, ih, ii, is, it, ihi, sh, si, ss, st, shi, th, ti, ts, tt, thi, hih, hii, hi, hit, hihi 出现的频率，发现除了 hi 以外，is 和 it 的出现频率最高，则把 is 和 it 加入词表中，得到 [h, i, s, t, hi, is, it]，如此循环往复，直到子表大小达到我们设定的上限即可

而应用 BPE 进行分词的过程大致如下：

  • 从输入文本的第一个字符开始，优先选择词表中能够匹配的最长子词单元（贪心），然后继续处理剩余部分，直到完成整个序列的切分，如果遇到的首个字符就不在词表中，则直接将其打成 ，接着处理下一个字符即可

子词级分词已经成为现代英文 NLP 模型中的主流方法，如 BERT、GPT 等模型均采用了基于子词的分词机制

中文分词

尽管中文的语言结构与英文存在显著差异，我们仍可以借助 “分词粒度” 的视角，对中文的分词方式进行归类和分析

字符级分词

字符级分词是中文处理中最简单的一种方式，即将文本按照单个汉字进行切分，文本中的每一个汉字都被视为一个独立的 Token

由于汉字本身通常具有独立意义，因此字符级分词在中文中具备天然的可行性，相比英文中的字符分词，中文的字符分词更加语义友好

词级分词

词级分词是将中文文本按照完整词语进行切分的传统方法，切分结果更贴近人类阅读习惯

由于中文没有空格等天然词边界，词级分词通常依赖词典、规则或模型来识别词语边界

子词级分词

虽然中文没有英文中的子词结构（如前缀、后缀、词根等），但子词分词算法（如 BPE）仍可直接应用于中文，它们以汉字为基本单位，通过学习语料中高频的字组合（如 “自然”、“语言”、“处理”），自动构建子词词表，这种方式无需人工词典，具有较强的适应能力，通俗来讲，就是兼顾单个汉字和常见词语的一种分词方式

在当前主流的中文大模型（如通义千问、DeepSeek）中，子词分词已成为广泛采用的文本切分策略

分词工具

目前市面上可用的中文分词工具，按照实现方式可以大致分为如下的两类：

  • 基于词典或模型的传统方法，主要以 “词” 为单位进行切分
  • 基于子词建模算法（如 BPE）的方式，从数据中自动学习高频字组合，构建子词词表

前者的代表工具包括 jieba、HanLP 等，这些工具广泛应用于传统 NLP 任务中

后者的代表工具包括 HuggingFace Tokenizer、SentencePiece、tiktoken 等，常用于大规模预训练语言模型中

词表示

在分词完成之后，文本被转换为一系列的 Token（词，子词或字符），然而，这些符号本身对计算机而言是不可计算的，因此，为了让计算机能够理解和处理文本，必须将这些 Token 转换为计算机可以识别和操作的数值形式，这一步就是所谓的词表示（Word Representation）

词表示的发展经历了从稀疏的 One-Hot 编码，到稠密的语义化词向量，再到近年来的上下文相关的词表示，不同的词表示方法在表达能力、语义建模、上下文适应性等方面存在显著差异

One-Hot 编码

最早期的词向量表示方式是 One-Hot 编码，它将词汇表中的每个词映射为一个稀疏向量，向量的长度等于整个词表的大小，该词在对应的位置为 1，其他位置为 0

One-Hot 虽然实现简单、直观易懂，但它无法体现词与词之间的语义关系，且随着词表规模的扩大，向量维度会迅速膨胀，导致计算效率低下，因此，在实际自然语言处理任务中，One-Hot 表示已经很少被直接使用

语义化词向量

传统的 One-Hot 表示虽然结构简单，但它无法反映词语之间的语义关系，也无法衡量词与词之间的相似度，为了解决这个问题，研究者提出了 Word2Vec 模型，它通过对大规模语料的学习，为每个词生成一个具有语义意义的稠密向量表示，这些向量能够在连续空间中表达词与词之间的关系，使得 “意思相近” 的词在空间中距离更近

如图所示，显然男人和男孩更相似，女人和女孩更相似，因此这两组向量各自的距离相近，我们甚至可以通过向量作差运算得到词表示之间的关系

语义化词向量中最常见的方法即 Word2Vec，下文我们将对其进行重点讲解

Word2Vec

Word2Vec 概述

Word2Vec 的设计理念源自分布假设，该假设认为一个词的含义由它周围的词所决定

基于这一假设，Word2Vec 构建了一个简洁的神经网络模型，通过学习词与上下文之间的关系，自动为每个词生成一个能够反映其语义特征的向量表示

Word2Vec 提供了两种典型的模型结构，用于实现对词向量的学习

CBOW 模型

CBOW，Continuous Bag-of-Words 模型，输入是一个词的上下文（即待预测词前后若干个词），模型的目标是预测中间的目标词，上下文窗口为 k ，分别给出前后各 k 个词

Skip-gram 模型

Skip-gram 模型输入是一个中心词，模型的目标是预测其上下文中的所有词（即中心词前后的若干个词），上下文窗口为 k，模型需要预测前后各 k 个词

只要按照上述目标训练模型，就能得到语义化的词向量

Word2Vec 原理

数据集

Word2Vec 不依赖人工标注，而是直接利用大规模原始文本（如书籍、新闻、网页等）作为数据源，从中自动构造训练样本，因为我们训练的目标本身就是给定上下文预测中心词，或者给定中心词预测上下文，因此直接使用原始语料即可

由于上述两种模型的输入和输出都是词语，因此首先需要对原始文本进行分词，将连续文本转化为 Token 序列

此外，模型无法直接处理文本符号，训练时仍需将词语转换为 One-Hot 编码，以便作为模型的输入和标签参与计算

  • 计算损失，将预测结果与真实中心词 “地铁” 的 One-Hot 向量进行比对，计算交叉熵损失

Word2Vec 词向量获取

词向量的获取通常有两种方式：一种是直接使用他人公开发布的词向量，另一种是在特定语料上自行训练

在实际工作中，无论是加载已有模型还是从零训练，都可借助 Gensim 来完成，它提供了便捷的接口来加载 Word2Vec 格式的词向量，也支持基于自由语料训练属于自己的词向量模型

首先需要执行命令 pip install gensim 来安装 Gensim 库

gensim 提供了方便且统一的训练词向量的 API，即 Word2Vec 接口，具体调用方式如下：

该方法接收如下几个常见参数：

  • sentences：已经完成分词的文本序列组成的二维列表
  • corpus_file：未分词的原始语料，和 sentences 只能二选一传入，不能同时存在
  • vector_size：词向量的维度
  • window：上下文窗口长度
  • min_count：最小词频阈值，原始语料中出现次数小于 min_count 的词会被直接忽略，即不生成对应的词向量
  • sg：控制 Word2Vec 选用的模型，sg=1 表示选用 Skip-Gram，sg=0 表示选用 CBOW
  • workers：并行线程数，用于加速计算
  • alpha：学习率
