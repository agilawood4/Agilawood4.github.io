知识点总结

（1）计算机网络的概念：由一些相互连接的，自治的计算机所组成的集合

（2）计算机网络发展历程：（1）主机 - 终端时代；（2）主机互连时代；（3）网络时代（TCP/IP 的出现）

（3）计算机网络性能评估指标：

  • 性能指标：速率，带宽，吞吐率，时延，时延带宽积，往返时间 RTT，信道利用率
  • 非性能指标

（4）物理层的功能：实现传输介质上透明比特流的传输

（5）物理层的四个特性：机械特性，电气特性，功能特性，过程特性

（6）码元：在使用时间域的波形表示数字信号时，代表不同离散值的波形就称为码元（当 1 码元携带 k 比特数据时，比特速率是码元速率的 k 倍）

（7）通信双方的交互方式：单向通信（单工通信），双向交替通信（半双工通信），双向同时通信（全双工通信）

（8）奎奈斯特公式（理想低通信道）：最大传输速率=2×带宽×log⁡2信号特征数\text{最大传输速率} = 2 × \text{带宽} × \log_2 \text{信号特征数}最大传输速率=2×带宽×log2​信号特征数，单位 bps

（9）香农公式（有噪声信道）：最大传输速率=带宽×log⁡2(1+有效信号功率噪声功率)\text{最大传输速率} = \text{带宽} × \log_2(1+\frac{\text{有效信号功率}}{\text{噪声功率}})最大传输速率=带宽×log2​(1+噪声功率有效信号功率​)，单位 bps

（1）信道的传输速率存在上限原因：无限制提高传输速率，会出现码间串扰现象，使得接收端对码元不可判别

（2）物理层传输介质：

  • 导引型传输介质（有线传输介质）：同轴电缆（50Ω and 75Ω）；双绞线（屏蔽型和非屏蔽型）；光纤（单模光纤和多模光纤）
  • 非导引型传输介质（无线传输介质）：主要指无线电通信，又分为短波通信，微波通信，卫星通信三类

（3）码分复用又称为码分多址，使用扩频通信技术，把 1 bit 的信号转换为 m bit 的码片进行传输，抗干扰能力强，频谱近似白噪声

（4）宽带接入技术：

  • xDSL：基于传统电话线
  • HFC：基于有线电视，光纤混合同轴电缆
  • FTTx：基于光纤的基带接入

（4）数据链路层使用的两类信道：点对点信道（PPP 协议）；广播信道（CSMA/CD 协议）

（5）数据链路层要解决的问题：

  • 封装成帧
  • 差错控制
  • 流量控制与可靠传输
  • 广播信道的介质访问控制

（6）IP 数据报的长度上限：1500B，也称为 MTU，即帧的最大传送单元

（7）常用的封装成帧方法：

  • 字符计数法
  • 比特填充的首尾定界法（同步传输，速率快）
  • 字符填充的首尾定界法（异步传输，速率慢）：0x7D → 0x7D 0x5D；0x7E → 0x7D 0x5E；小于 0x20 的字符 C → 0x7D C+0x20
  • 物理编码违例法

（8）常用的检错方法：

  • 奇偶校验（硬件）
  • 简单累加和（软件）
  • 循环冗余校验（硬件）

（9）常用的纠错方法：海明校验，称为前向纠错

  • 无论是纠错还是检错，都只能做到帧的无比特差错，而不是真的无差错（还要考虑不重复，不丢失，有序性）
  • 纠错比检错需要更多的冗余信息，计算机网络中一般使用检错+重传的机制，深空探测（重传代价大）一般使用纠错机制

（1）基本数据链路协议：

  • 无限制的单工协议
  • 单工停等协议（确认帧）
  • 有噪声信道的单工协议（ARQ 协议：错误检测，超时重传，确认帧，帧序号）
  • 窗口大小为 1 的滑动窗口协议即 ARQ 协议

（2）滑动窗口协议：允许发送方持续发送数据帧，由滑动窗口实现流量控制

  • 回退 N 帧（GBN）：有序传输
  • 选择重传（SR）：无序传输
  • 滑动窗口协议中的 ACK~n~ 含义为：n 号帧以及之前的帧都被接收方正确接收，当前期待收到 n+1 号帧（累计确认机制）

（3）PPP 帧的三个组成部分：

  • 把 IP 数据报封装到串行链路中
  • 链路控制协议 LCP
  • 网络控制协议 NCP

（4）PPP 帧的格式：面向字节传输

  • F：标志字段
  • A：地址字段
  • C：控制字段
  • 协议字段：0x0021 - IP 数据报；0xC021 - LCP；0x8021 - NCP
  • 数据字段
  • FCS 帧检验序列
  • F 标志字段

（5）PPP 协议的工作状态：

  • 两种认证方式：PAP 明文认证，CHAP 挑战握手认证，后者可以防止重放攻击
  • 状态转移图：Dead → Establish → Authenticate → Network → Open → Terminate → Dead
  • PPP 协议支持不同的网络层协议，支持同步和异步链路，在建立 LCP 链路后进行身份鉴别

（6）局域网因为使用广播信道，因此要解决的一个重要问题是：介质控制访问技术

（7）信道划分策略：

  • 静态信道划分：FDM，TDM，CDM，WDM（缺点：不支持动态接入接出，涉及物理硬件开销大）
  • 动态介质访问控制：随机访问（LAN 常用，需要引入碰撞避免策略，如 CSMA/CD）；受控访问（LAN 不常用，令牌环网）

（8）CSMA/CD（以太网的核心）全称为载波监听多点访问/冲突检测，四个重要概念：

  • 载波监听
  • 碰撞检测
  • 碰撞强化
  • 碰撞退避：二进制指数退避算法（2τ×倍数2\tau\times \text{倍数}2τ×倍数）

（9）CSMA/CD 本质上是对带宽的争用，争用期仅和信道传播速度有关，和数据发送速率无关

（1）以太网的争用期为 51.2 us，传输速率为 10Mb/s 的以太网的最短帧长度为 64B，小于这个长度称为无效帧；而对于其他网络，最短帧计算公式为：L=2×信道长度信道上数据传输速度×数据发送速率L = \frac{2\times{\text{信道长度}}}{\text{信道上数据传输速度}}\times \text{数据发送速率}L=信道上数据传输速度2×信道长度​×数据发送速率

（2）IEEE 802 标准把数据链路层划分成两个子层：

  • LLC：逻辑链路控制子层 - IEEE 802.2
  • MAC：介质访问控制子层 - IEEE 802.3（其中 CSMA/CD 也属于 IEEE 802.3）

（3）局域网：在一定区域内由多台计算机互联形成的计算机组，使用广播信道

（4）局域网的分类：

  • 传输介质：有线局域网，无线局域网（IEEE 802.1）
  • 拓扑结构：环形网，星形网，总线网（以太网 - 总线式拓扑），树形网

（5）以太网的物理层：最初使用 50Ω 的同轴电缆，后改为双绞线（10Base-T，其中 10 表示传输速率为 10Mb/s，Base 表示基带传输，T 表示双绞线）

（6）MAC 帧，共 48 bit，高 24 位为厂商标识符，低 24 位由厂商自行分配，全球唯一，以太网帧格式为：

  • 目的 MAC 地址 6B（一定记住以太网帧和其他数据格式是反的，目的地址在前，源地址在后）
  • 源 MAC 地址 6B
  • 协议类型 2B
  • IP 数据报
  • 帧检验序列 4B
  • 在帧的最开始还会有 8B 的帧首序列，包含 7B 的前同步序列和 1B 的帧开始定界符

（7）每个站点都会收到总线上发来的所有帧，但适配器 / 网卡只处理发往本站点的帧，分为单播帧，多播帧，广播帧（目的 MAC 地址为全 1 序列）三类情况，其他帧直接丢弃，也不需要向上报告

（8）以太网常见速度分类：

  • 高速以太网：100 Mb/s，仍使用 CSMA/CD 协议，满足 IEEE 802.3 要求
  • 千兆以太网：1 Gb/s，仍使用 CSMA/CD 协议，满足 IEEE 802.3 要求
  • 万兆以太网：10 Gb/s，传输介质为光纤、双绞线，只在全双工模式使用，不存在争用，不再使用 CSMA/CD 协议

（9）局域网互连：

  • 物理层互连：集线器 / 中继器（不隔离碰撞域和广播域）
  • 数据链路层互连：交换机 / 网桥（隔离碰撞域，不隔离广播域）
  • 网络层互连：路由器（隔离碰撞域和广播域）

（1）网桥在数据链路层对帧进行转发，通过自学习算法建立 MAC 地址转发表，其学习的来源是每一条帧中的源 MAC 地址，对于同一个网段下的单播帧，其转发过程分两种情况：

  • 转发表不存在对应字段时，网桥收到该帧，进行广播
  • 转发表中存在对应字段时，网桥可判断源地址和目的地址在同一网段下，不进行任何转发

（2）不能通过网桥 / 交换机无限扩大局域网原因：网桥 / 交换机不会过滤广播帧，会向各个端口都转发，如果待转发的广播帧较多，会产生广播风暴，引起网络拥塞

（3）虚拟局域网：在局域网的基础上，通过对节点进行分组，即可得到若干逻辑上相互独立的虚拟局域网，其好处为：便于管理，控制网络风暴，实现各个 VLAN 内部的独立性与安全性

（4）帧不会在 VLAN 之间转发，即便是广播帧也不行，帧只能在 VLAN 内部转发

（5）VLAN 之间的通信在数据链路层无法实现，要等到网络层才能实现

（6）冲突域：每个节点都可以收到被发送的帧，即同一时刻只能有一台设备发送信息；广播域：某个站点发出一个广播信号，能收到该广播信号的站点范围构成一个广播域

（7）网络层提供的两种服务：面向连接可靠的服务 - 虚电路，无连接不可靠的服务 - 数据报

（8）网络层的重要协议：网际协议 IP，网际控制报文协议 ICMP，网际组管理协议 IGMP，地址解析协议 ARP，逆地址解析协议 RARP，路由选择协议（OSPF，RIP，BGP）

（9）IP 地址编址的三个阶段：地址分类，子网划分，路由聚合（构成超网，CIDR，无分类域间路由），其中 A 类 - 0，B 类 - 10，C 类 - 110，D 类 - 1110（用于组播），E 类 - 1111

（1）保留地址 / 内网 IP：

  • A 类：10.0.0.0 - 10.255.255.255，共一个网段
  • B 类：172.16.0.0 - 172.31.0.0，共 16 个网段
  • C 类：192.168.0.0 - 192.168.255.255，共 256 个网段

（2）有特殊含义的地址：

  • 网络号全 1 + 主机号全 1：本网段的广播地址，路由器不会进行转发
  • 网络号全 0 + 主机号全 0：本网段表示主机，路由表中表示默认路由
  • 特定网络号 + 主机号全 0：表示特定的网段
  • 特定网络号 + 主机号全 1：表示特定网段下的广播地址
  • 网络号全 1 + 特定主机号：环回地址，用于本地软件测试

（3）路由器转发表中三个核心字段：目的网段 IP，子网掩码，下一跳地址（直接交付 or 对端路由器接口 IP）

（4）对于一个主机而言最重要的三个地址：本机 IP，子网掩码，网关 IP（所在网段的路由器接口 IP）

（5）无分类编址 CIDR 在路由转发时，遵循最长前缀匹配原则，即从所有能匹配的表项中，选择网络前缀（子网掩码中连续的 1）最长的路由进行转发

（6）网际控制报文协议 ICMP 作用：差错报告和网络诊断，9 大类报文如下：

  • 终点不可达：目的端口上没有相应进程接收数据
  • 源点抑制：因为网络拥塞丢包
  • 参数错误：校验和错误
  • 重定向：IP 分组经历的路由发生改变
  • 超时：IP 分组的 TTL 字段变为 0
  • Echo Request / Reply：回声探测，ping 指令原理
  • Timestamp Request / Reply：请求时间，用于时间同步

（7）ICMP 的报文结构：Type，Code，Checksum，Identifier，Sequence Number，其中 Type = 8 表示回显请求，Type = 0 表示回显应答

（8）路由算法生成路由协议，路由协议生成路由转发表，路由器根据路由转发表进行分组转发

（9）路由算法分类：

  • 静态路由选择策略：适用于小网络，人工配置路由，简单开销低
  • 动态路由选择策略：适用于大网络，由专门的算法和协议配置路由，复杂开销高
  • 所有涉及到算法的路由策略都是动态路由选择策略

（1）Internet 采用分层路由原因：Internet 范围大，没有任何一个路由器能维护全网路由信息；接入 Internet 的网络不希望外界了解自己网络内部信息

（2）自治系统（AS）：在单一的技术管理下的一组路由器，这些路由器使用一种 AS 内部的路由选择协议和共同的度量，以确定分组在 AS 内部的路由；同时使用一种 AS 之间的路由选择协议，以确定分组在 AS 之间的路由

（3）路由协议的分类：

  • 内部网关协议 IGP（域内路由选择）：距离向量算法（RIP），状态链路算法（OSPF）
  • 外部网关协议 EGP（域间路由选择）：BGP

（4）路由器：具有多个输入输出端口的专用计算机，其任务为分组转发，两大组成部分：

  • 路由选择：根据路由协议构建路由表
  • 分组转发：交换机构 + 输入输出端口，实现对分组的高效转发

（5）路由器发生丢包的原因：当排队的分组过多，超过路由器缓冲区的负载能力，则路由器会适当丢弃部分数据包（路由器无法保证分组的不丢失）

（6）路由器性能指标：分组的高速转发；路由器性能影响因素：交换机构的性能，输入输出端口的处理能力；评估指标：pps，单位时间内转发的数据包数量，统一采用最小包度量（64B）

（7）网络的控制平面与数据平面：

  • 控制平面：路由选择
  • 数据平面：分组转发

（8）在传统网络中，控制平面和数据平面都集成在路由器中；在 SDN 软件控定义网络中，控制平面和数据平面分离，灵活性强，方便快速控制路由 / 转发策略，适合路由策略频繁变化的网络：

  • 控制平面：控制器
  • 数据平面：路由器，SDN 中把路由器称为交换机

（9）IP 组播：用于实现一对多的数据传输，对于一些网络应用（如视频会议，直播）可以极大减少网络流量

（1）组播的两种情景：局域网内部的硬件组播（需要特定的组播 MAC 地址）；Internet 中的组播（主机通过 ICMP 协议加入组播组，同时需要组播路由器）

（2）NAT 的好处：减少对 IP 地址空间的占用，不同内网可以对同一个专用地址服用；一定程度上提高内网的安全性

（3）NAT 路由器的作用：对于内网发出的数据包，把源地址从内网 IP 改为公网 IP；对于外网返回的数据包，把目的地址从公网 IP 改为对应的内网 IP

（4）虚拟专用网 VPN：

  • 两种使用情形：内部网络之间通过 Internet 互连，远程用户访问内部网络
  • 两种常用技术：IPSee，SSL

（5）网络层：主机之间的逻辑通信；传输层：应用进程之间的逻辑通信，因此传输层是自下向上的第一个端到端通信

（6）传输层的复用与解分 / 分用：

  • 复用：应用层多个应用进程通过传输层发送数据
  • 分用：传输层将收到的数据交付给指定的应用进程

（7）传输层两大核心协议：传输控制协议 TCP，用户数据报协议 UDP

（8）端口：传输层用于区分应用进程的手段；套接字：操作系统提供给应用进程的编程接口，套接字 = { IP 地址 : 端口号 }

（9）传输层使用端口而不是进程 IP 区分应用进程的原因：不同操作系统进程 ID 结构不同；同一个应用进程可能和网络中多个主机进行通信，建立多条连接

（1）传输层使用的端口号（16bit）为软件端口，区分硬件端口，其分类如下：

  • 0 - 1023：熟知端口号
  • 1024 - 49151：登记端口号
  • 49152 - 65535：临时端口号

（2）UDP 协议特点：

  • 无连接，不可靠
  • 进行差错校验，出错即丢弃报文，不重传
  • 允许一对一，一对多，多对多通信
  • 报文段首部仅 8B，开销小
  • 面向报文段进行传输，无拥塞控制，适合多媒体通信
  • 双向通信（因为没有建立连接，所以谈不上全双工）
  • 连接端点：传输层协议的端口

（3）传输层两个协议的校验和都是整个数据报（包含首部，伪首部，数据）的校验和，区分 IP 数据报仅对首部进行校验

（4）UDP 数据报首部：源端口（2B），目的端口（2B），总长度（2B），校验和（2B），最前面还存在伪首部，仅参与校验和计算，不参与实际传输

（5）TCP 协议特点：

  • 面向连接：数据传输前建立连接，传输完成释放连接
  • 面向字节流，以字节为单位传输
  • 只允许一对一通信
  • 确保可靠传输，确保数据不重复，不丢失，不出错，按序到达
  • 全双工通信
  • 连接端点：套接字，即 IP 地址 + 端口

（6）TCP 协议建立的连接本质上是虚连接，并不是物理意义上真正存在的连接

（7）TCP 协议可靠传输的实现：差错控制+超时重传，流量控制，拥塞控制，连接控制，分组排序与去重

（8）TCP 协议超时重传时间计算：RTTO=RTTS+RTTDRTT_O = RTT_S + RTT_DRTTO​=RTTS​+RTTD​，其中 RTTSRTT_SRTTS​ 为加权平均往返时间，RTTDRTT_DRTTD​ 为 RTTRTTRTT 偏差的加权平均值

（9）TCP 协议通过滑动窗口实现流量控制，此时的 ack 表示下一次传输接收方期望收到的数据的首字节序号（和数据链路层的 ACK 区分）

（1）TCP 为每个报文段都设置一个持续计时器，如果发送方收到接收方的零窗口通知，就启动持续计时器，当计时器归零时，则发送一个零窗口探测报文，如果接收方返回窗口大小仍为 0，则重置持续计时器重新计时；否则按照接收方返回的窗口大小继续发送数据

（2）TCP 传输效率控制：（1）缓冲区每存够 MSS 字节的数据就封装成报文段发送；（2）定时发送（最大长度不能超过 MSS）；（3）由应用进程指明发送一次报文段；而 UDP 的最大报文长度是由应用进程所控制的

（3）拥塞：对网络中某种资源的需求量大于该资源提供的可用部分，网络性能随之变差，进而产生拥塞

（4）拥塞往往趋于恶化，原因是：TCP 的超时重传机制（比如路由器因为拥塞丢弃了报文，导致发送方超市重发，网络中因此注入更多报文，加剧了拥塞）

（5）流量控制通常认为是局部概念，是点对点信道 / 端到端的控制过程；而拥塞控制则是全局控制过程

（6）拥塞控制的两种思路：开环控制（提前预防）和闭环控制（发生拥塞再避免）

（7）常见的拥塞控制方法（闭环控制）：慢启动，拥塞避免，快重传，快恢复

  • 慢启动 + 拥塞避免：检测到拥塞，ssthresh 变为当前 cwnd 的一半，cwnd 变为 1
  • 快重传 + 快恢复：检测到三次重复的 ack 序号，ssthresh 变为当前 cwnd 的一半，cwnd 变为更新后的 ssthresh

（8）拥塞避免中用到的两个思想：加法增大与乘法减小

（9）TCP 传输连接的三个阶段：连接建立（三次握手），传输数据，连接释放（四次挥手），连接建立中解决的三个问题：（1）连接双方知晓彼此存在；（2）协商收发窗口大小，同步初始序列号；（3）分配传输过程中需要的实体资源

（1）TCP 传输中客户和服务器的概念：客户主动发起连接请求，服务器被动等待建立连接

（2）采取三次握手而不是两次握手原因：防止失效的连接请求占用服务器网络资源

（3）三次握手的安全问题：TCP SYN Flooding（DDos 分布式拒绝服务攻击的一种），攻击方会进行 IP 地址伪装，发送大量 SYN 报文给服务器，但不响应服务器返回的 SYN+ACK 报文，使得服务器处于半开连接状态，占用大量服务器资源，使得服务器数据结构被堆满，无法相应正常的用户请求

（4）TCP_SYN_Flooding 攻击的解决方法是 Linux 的 SYN_Cookies 机制，把连接实体的分配从第一次握手完成推迟到第三次握手完成

（5）TCP 连接的释放可以由连接双方的任意一方发起，且需要释放两次才能彻底关闭双方的连接

（6）连接关闭前等待 2 MSL 的原因（MSL 为最大报文段存活时间）：（1）确保最后一条 ACK 报文能顺利到达对方；（2）防止旧连接中的失效报文段出现在新连接中

（7）经典的套接字编程接口为阻塞式（同步调用），但 windows 系统是基于事件驱动的系统，套接字编程为异步调用

（8）Internet 采用层次结构命名树作为主机名称，使用分布式域名系统 DNS，由域名服务器运行运行域名服务程序，将域名解析为 IP 地址

（9）域名解析过程：应用进程调用域名解析程序，称为域名系统的客户，向域名服务器发送域名解析请求报文，包含待解析的域名，域名解析服务器返回应答报文，包含对应的 IP 地址

（1）域名长度不超过 255B，级别越高的域名书写越靠后

（2）顶级域名分为国家顶级域名（.cn）和通用顶级域名（.com）

（3）域名服务器分类：

  • 根域名服务器：知晓所有顶级域名
  • 顶级域名服务器：知晓当前顶级域名下的所有二级域名
  • 权限域名服务器：负责一个区的域名解析
  • 本地域名服务器 / 默认域名服务器

（4）主机向本地域名服务器请求域名解析：递归查询，主机只和本地域名服务器交互一次，并不了解本地域名服务器具体解析的过程；本地域名服务器向顶级域名服务器请求域名解析：迭代查询，顶级域名服务器返回对应的 IP 地址或下一次查询该去哪个服务器查询

（5）文件传输协议：一个 FTP 服务器可以同时服务多个用户进程

（6）FTP 协议基于两个 TCP 连接实现：控制连接（在会话期间一直保持打开），数据连接（每传输一次数据打开一次，完成传输后关闭）

（7）万维网（www）解决的三个问题：URL，HTTP，HTML

（8）URL，统一资源标识符：协议://主机:端口/路径，如果是 http 协议，缺省端口为 80，缺省路径为主页（home page）

（9）http，超文本传输协议，面向事务，无状态，基本工作原理为：使用熟知端口号 80，服务器持续监听该端口号，当用户请求页面时，需要与服务器建立 TCP 连接，然后进行文件传送，传送完成后释放连接

（1）http 版本演进：

  • http 1.0：每传输一次数据，就需要建立一次 TCP 连接
  • http 1.1：引入持续连接，建立一次 TCP 连接，可以传输多次数据
  • http 2.1：允许同时传输多个流
  • http 3.1：改用 UDP 协议

（2）http 报文：开始行 + 首部行 + 实体主体，其中请求报文的开始行也叫请求行（方法 + URL + 版本），应答报文的开始行也叫应答行（版本 + 状态码 + 短语），其中状态码：

  • 1 - 通知信息（收到请求或正在处理）
  • 2 - 成功
  • 3 - 重定向
  • 4 - 客户端错误
  • 5 - 服务器错误

（3）https，安全的 http 协议，两种加密算法：通用 / 对称加密算法，公钥加密算法（hyper text transfer protocol secure）

（4）主机 IP 封装成帧后，帧的发送对象：同网段直接发送给目的主机，不同网段需要先交付给路由器

（5）TCP 传输轮次和最大发送序号的关系：如果发生超时，则至少有一个轮次要重传这条报文，因此最大发送序号需要减掉这个报文对应的长度

（6）TCP/IP 发送窗口大小的确定：发送端根据接收端的接收窗口和自身网络的拥塞程度决定发送窗口大小；发送窗口在拥塞窗口和接收窗口之间取较小值；每当收到接收端报文时，就根据 TCP 报文段首部的接收窗口，重新计算是否要更新发送窗口大小；每经过一个传输轮次，也要根据拥塞窗口的变化，重新计算是否要更新发送窗口大小

（7）TCP/IP 向应用层提供的提供的接口是套接字，端口只是 TCP/IP 用于区分不同进程的手段

（8）http 的文档请求时间 = 文档传输时间 + 2 × RTT

（9）以太网交换机如果在转发表中没有找到目的 MAC 地址，则会把帧转发到除本端口以外的所有端口（退化成集线器）

（1）计算机网络中可以没有数据库，但必须有硬件，软件（最基本的：操作系统），协议，一定要与客户机和服务器

（2）数据传输时，不参与封装工作的是物理层，而数据链路层，网络层，传输层都要参与封装工作

（3）电路交换使用的是虚电路连接，并不是真正意义上的物理连接，不等同与物理线路交换

（4）虚电路连接的建立是临时的，但会话结束不一定会释放连接，虚电路能够保证数据的发送顺序与接收顺序相同，而数据报不能保证数据的按序到达

（5）数据链路层不仅可以实现差错检测，还可以提供一定的纠错功能，比如使用纠错编码（海明编码）等

（6）CRC 校验：通信双方在使用 CRC 校验时需要商定使用的多项式编码，一些特殊的多项式因为其良好性质成为国际标准，用于数据通信中的 CRC 校验

（7）以太网地址即通常所说的 MAC 地址，以太网参考模型一般只包括物理层和数据链路层，以太网地址存储在网卡中

（8）HTTP，IMAP，POP3 都可以从邮件服务器中接收邮件

（9）IP 分组的分片可以在主机和路由器上进行，但重组只能在目标主机上完成

（1）路由器检测到拥塞时，会合理丢弃 IP 分组，同时路由器也会对 IP 分组进行差错校验，但是不能保证分组不丢失

（2）路由器的分组转发部分的组成：一组输入输出端口，交换机构

（3）PPP 协议不使用帧的编号和确认机制实现可靠传输：PPP 不用帧编号和确认机制，因为它只在点到点链路上提供简单封装和差错检测，错误帧直接丢弃，可靠传输由上层 TCP 实现（PPP 是有连接，不可靠，无确认）

（4）网络层尽最大努力交付的体现：

  • 不保证按时，不保证按序，不保证无差错，不保证不重复
  • 不故意丢弃数据报（除非校验和错误、发生网络拥塞或路由器缓冲区溢出等异常情况）

（5）UDP 和 IP 都是不可靠无连接的协议，其首部均有校验和，校验和出错时会直接丢弃数据，区别在于：UDP 的校验和是整个数据的校验和，而 IP 的校验和只是首部的校验和

（6）ARP 和 DNS 对比：

  • 相同点：主机发出请求，服务器回应
  • 不同点：ARP 是网络层协议，通过广播的方式把 IP 地址解析为 MAC 地址；DNS 是应用层协议，请求域名服务器把某个域名解析为 IP 地址

（7）TCP 和虚电路对比：

  • TCP 是传输层的逻辑连接，连接信息保存和维护在端系统
  • 虚电路是网络层的逻辑连接，状态信息保存和维护在沿途路由器中

（8）达到最高信道利用率的情景：如果是单工停等协议，发送窗口大小 = 信道的时延带宽积；如果是滑动窗口协议，发送窗口大小 = 2 × 时延带宽积

（9）访问一个网页，从应用层到网络层所使用的协议：

  • 应用层：http 或 https，用于 Web 内容的请求和传输；DNS 用于将域名解析为 IP 地址
  • 传输层：TCP 协议，用于建立可靠的连接和数据传输
  • 网络层：IP 协议，负责数据报的路由和传输；ARP 协议，负责把 IP 地址解析为 MAC 地址

（1）确认报文段丢失并不一定引发数据重传：因为 TCP 采用的是累计确认机制，如果还没来得及重传，就收到了更高序号的确认，则发送方也会认为当前序号对应的数据已经被正确接收

（2）从点击 URL 到显示文档的总时间：假定要解析到所要找的 URL 的 IP 地址共经过 m 个 DNS 服务器，在每个服务器上经历的时间分别为 RTT~1~，RTT~2~，…，RTT~m~，假定要读取的文档请求和应答都只需要一次传输，从本地主机到这个网页的往返时间为 RTT~w~，则总时间为：T=RTT1+RTT2+⋯+RTTm+2×RTTwT = RTT_1+RTT_2+\dots +RTT_m+2\times RTT_wT=RTT1​+RTT2​+⋯+RTTm​+2×RTTw​

（3）在分组交换网络中，n 个分组经过 k 段链路传输的总时间（默认可以进行流水线传输）：T=(k+n−1)×tT = (k + n - 1)\times tT=(k+n−1)×t，其中 k 是从源主机到目的主机经历的链路段数，n 是分组数量，t 分组在一段链路上的发送时间

（4）报文交换：

  • 优点：简单，易于实现，不需要维护分组顺序和重组信息
  • 缺点：整个报文段必须完整接收才能转发，总传输时间长；如果出错要重传整个报文，效率低下

（5）分组交换：

  • 优点：可以并行处理多个分组，减少端到端总传输时间；分组出错时，只需要重传错误的分组，提高传输效率；适应性强，可以更好处理网络中的拥塞和错误
  • 缺点：需要额外的开销维护分组的顺序和重组信息，复杂度高，可能增加网络中的控制信息量

（6）路由器的直连网段下一跳为直接交付

知识点补充

（1）在客户 / 服务器模型（也称为 C/S 模型）中，客户与客户之间不能直接通信，必须经过服务器

（2）在客户 / 服务器模型中，客户必须提前知道服务器的地址，但服务器不需要提前知道客户的地址

（3）客户端主要实现如何显示信息与收集用户的输入（面向用户），而服务器实现数据的处理（面向任务）

浏览器显示的内容来自服务器

（4）客户机通常位于前端，服务器通常位于后端，二者通过网络实现协同计算任务

（5）域名和 IP 地址、MAC 地址、主机之间都不存在一一对应关系（对于某些大型服务器可能存在多个域名，主要是为了负载均衡）

（6）Internet 上提供给看客户访问的主机可以不需要域名（域名只是为了方便访问，不是说一定要有）

（7）同一域名在不同时间可能解析出不同的 IP 地址，多个域名也可能指向同一台主机的 IP 地址

（8）DNS 是基于 C/S 模型的分布式系统，其组成包括：域名空间，分布式数据库，域名服务器

（9）主机在向本地域名服务器查询域名对应的 IP 时，只需要知道本地域名服务器的 IP 即可

（1）权限域名服务器可以把自己管辖的主机名转换为主机的 IP 地址

（2）用户主机向本地域名服务器发出递归查询请求，而本地域名服务器采用迭代查询向其他域名服务器进行查询，二者发出的请求条数分别是 1 条和多条

可以粗略认为，递归查询仅发出一条请求，迭代查询发出多条请求

（3）如果本地域名服务器向上级采用迭代查询，查询 N 级域名，最少需要发出 1 次查询，最多需要发出 N 次查询（最差的情况：每一级域名都需要查询一次）

（4）主机访问某个 http 页面的三个步骤：解析域名；建立 TCP 连接；请求 HTML 页面

（5）FTP 协议的主要特征：允许客户指明文件类型与格式，并允许文件具有存取权限；屏蔽了计算机细节，适合在异构网络中任意计算机 / 操作系统之间传送文件

（6）对于 FTP 协议而言，控制连接先于数据连接建立，并且晚于数据连接释放，控制连接端口号是 21 号，数据连接端口号是 20 号，服务器监听的通常是 21 号端口

（7）因特网用户的电子邮件格式为：用户名@邮箱所在主机的域名

（8）浏览器向邮件服务器发送邮件使用的是 HTTP 协议（区分用户主机向邮件服务器发送邮件使用的是 SMTP 协议）

如果是基于 WWW 万维网的邮件传输过程，发送和接收邮件都可以用 HTTP 协议代替

（9）SMTP 协议仅支持传输七比特的 ASCII 码，因此无需转换即可由 SMTP 直接传输的内容是 ASCII 文本，其他诸如图像，视频，.exe 都需要转换

（2）万维网上的每个页面都有一个唯一的地址，即 URL（统一资源定位符）

（3）http 是无状态协议，如果 web 站点希望识别用户，则需要用到 Cookie（Cookie 在服务器端产生，但是存储在客户端）

PPP 协议两个用处：计算机与 ISP 通信使用的数据链路层协议；两台网络设备之间的直连专用线路

（6）分组交换和报文交换相比的主要改进是传输单位更小且具有固定的最大长度

（1）OSI 七层模型只是理论参考标准，目前实际执行的模型是 TCP/IP 四层模型

（2）OSI 参考模型的三大主要概念：服务，接口，协议

（3）描述网络协议的三大要素：语法（协议报文的字段格式），语义（字段的含义），同步（时序，比如三次握手报文交换的顺序）

（4）负责将比特转换成电信号进行传输的层次是物理层

（5）物理层不存在流量控制的概念，至少要在数据链路层才可以

（6）数据链路层不存在拥塞控制的概念，至少要等到网络层才可以（拥塞控制是全局概念，而数据链路层更偏向点对点传输）

（7）数据链路层功能描述：保证数据的正确顺序和完整性

（8）数据传输过程中，只有物理层不会参与对数据的封装

（9）只要题面中提到端到端通信，默认对应进程和进程之间的通信，即传输层的工作

（1）在 OSI 模型中，网络层同时具有面向连接和无连接的服务，传输层只有面向连接的服务；而在 TCP/IP 模型中，网络层只有无连接的服务，传输层同时具有面向连接和无连接的服务

（2）数据链路层，网络层，传输层这三个层认为具有流量控制功能

（3）互联网采用的核心技术是 TCP/IP 技术，核心协议是 IP 协议

（4）上下邻层实体之间的接口成为服务访问点，各层的服务访问点：

  • 应用层：用户接口 / 应用接口
  • 传输层：端口
  • 网络层：IP 地址
  • 数据链路层：MAC 地址
  • 物理层：机械，电气，功能，过程特性

（5）数据传输的对象：

  • 数据链路层：相邻节点之间
  • 网络层：主机与主机之间
  • 传输层：端到端，进程与进程之间

（6）信息传输速率指的是信道上每一秒传输的比特数，而波特率指的是信道上每一秒传输的码元数

（7）双绞线绞合的目的是为了减少干扰

（8）电缆中采用的屏蔽技术主要是为了减少电磁干扰辐射，同轴电缆相比于双绞线的优势就在于其抗干扰能力更强

（9）不受电磁干扰和噪声影响的传输介质是光纤

（1）中继器扩大网络传输距离的原理是信号再生，而不是简单的信号放大

（2）使用集线器相连的设备所构成的网络结构在逻辑上是总线型，在物理上是星型

（3）中继器连接的不同介质的局域网，要求类型，速率，协议都必须保持一致

（4）中继器会平分带宽，每台设备得到的带宽是总带宽的 1/N，交换机不会平分带宽，每台设备得到的带宽都是总带宽

（5）集线器的端口收到数据后，会从除输入端口之外的其他端口将数据转发出去（因此其不能隔离冲突域）

（6）为终端节点隐蔽物理传输的细节是物理层的功能；控制对物理传输介质的访问是数据链路层的功能（例如 CSMA/CD 协议）

（7）流量控制主要是对发送方所发出数据量的控制

（8）CRC 循环冗余校验机制可以检验出所有的奇数个比特错误，所有的偶数个比特错误，以及所有小于等于校验位长度的连续比特错误

（9）在进行 CRC 的时候，通信双方需要先商定使用的多项式编码

（1）在单工停等协议中，如果应答帧丢失，发送方就会永远等待下去，解决这种死锁现象的方法是超时重传机制

（2）ARQ 协议包含三种：有噪声信道的停等协议，GBN，SR 协议；而连续 ARQ 协议仅两种：GBN 和 SR 协议

（3）媒体共享技术分为静态划分信道（TDM，FDM，CDM，WDM）和动态介质访问（随机接入：ALOHA，CSMA/CD，CSMA/CA；受控接入：轮询）

其中静态的核心思想为：通过分时，分频，分码等方法，将广播信道变为若干点对点信道

（4）MAC 子层（介质访问子层）的功能是：控制和协调所有站点对共享介质的访问

（5）对于时分复用而言，介质的位速率大于单个信号的位速率，介质的带宽大于结合信号的带宽

（6）虽然 FDM 比 TDM 可以更好的利用信道的传输能力，但计算机网络中时分复用技术使用更多，是因为时分复用可用于数字传输，而频分复用则不行

（7）静态信道划分和受控访问都不会发生冲突，而动态介质访问中的随机访问会发生冲突

（8）与 CSMA/CD 网络相比，令牌环网更适合负载高的网络

（9）以太网使用的是分组交换技术

（1）网卡主要工作在数据链路层和物理层

（2）以太网在通信的时候为了同步时钟（自同步功能），采用的是曼彻斯特机制

（3）以太网的 MAC 地址通常存储在网卡中

（4）在以太网中，如果网卡发现某个帧的 MAC 地址不是指向自己，则它会直接丢弃该帧，并且不向网络报告错误信息

（5）在 CSMA/CD 以太网中，站点不可以进行全双工通信，可以进行半双工通信

（6）在 CSMA/CD 协议中，争用期指的是信号从线路一端传输到另一端的时间

（7）在 CSMA/CD 的碰撞退避中，选择二进制指数退避算法的优势：充分考虑了网络复杂对于冲突的影响

（8）高速以太网和传统以太网使用的 MAC 帧格式完全相同，主要是为了版本兼容

（9）无线局域网不适用 CSMA/CD 而使用 CSMA/CD 协议的原因是：难以实现冲突检测（无线局域网的适配器无法实现碰撞检测），且存在隐蔽站和暴露站的问题

（1）在 802.11 协议中，MAC 帧首部的地址字段的含义和作用取决于帧的 To AP 和 From AP 位

（2）对于 CSMA/CA 协议：

  • 接收方收到数据帧后，需要向发送方返回确认帧，发送方未收到确认帧需要重传
  • CA 表示 Collision Avoidance，即尽最大努力避免碰撞，但由于是无线电传输，因此无法检测碰撞（发送时也不会监听信道），也不能完全避免碰撞
  • 发送站点检测到信道空闲后，需要先等待 DIFS 的时间，才能启动发送
  • CSMA/CD 和 CSMA/CA 协议都有碰撞退避算法，且后者的退避算法比前者要复杂得多
  • CSMA/CA 协议可以选择通过 RTS 和 CTS 帧进行信道预约，当然也可以不进行预约
  • CSMA/CA 协议默认使用单工停等协议
  • 三段时间的关系：SIFS < PIFS < DISF（从小到大顺序为 s p d）

（3）处于不同 VLAN 的主机之间如果想通信，则必须在网络层才能实现，数据链路层无法实现

（4）虚拟局域网的实现建立在交换技术的基础上，通过交换机的软件配置和相关协议实现逻辑的分组与管理

（5）CSMA/CD 没有确认帧机制，但是 CSMA/CA 有确认帧机制

（6）广域网采用的也是分组交换技术，局域网和广域网的主要区别除了覆盖范围不同之外，还有使用的协议不同

（7）广域网使用的传播方式的存储转发方式；以太网使用的传播方式是广播式

（8）广域网采用的拓扑结构为网状结构

（9）PPP 协议中的 LCP 协议配置拉数据链路层的相关选项，NCP 协议配置网络层的相关选项

（1）PPP 协议只支持面向点对点的全双工通信，不支持多点通信，也不支持半双工通信，且 PPP 协议两端的网络层可以运行不同的网络层协议

（2）交换机把网络划分为多个网段，一个网段的故障不会影响到另一个网段的运行

（3）交换机可以连接不同的物理层，不同的 MAC 子层，以及不同速率的以太网（不能连接不同网络层协议）

（4）交换机每个接口结点所占用的带宽不会因为接口结点数目的增加而减少，且整个交换机的总带宽会随着接口结点的增加而增加

（5）交换机和集线器相比能提供更好的网络性能，原因在于交换机支持多对用户同时通信（本质是就是隔离了冲突域），即交换机的多个端口可以进行并行传输

（6）以太网参考模型通常只包含物理层和数据链路层

（7）广播式网络即多台主机共享同一条广播信道，其核心特点是不需要路由（即没有网络层）

（8）路由器可以实现不同网络层协议的网络互连，但是以太网交换机不可以（后者工作在数据链路层）

（9）路由器连接的异构网络指的是数据链路层协议和物理层协议至少有一个不同

（1）网络发生拥塞的标志是：随着通信子网负载的增加，网络吞吐量反而降低

（2）路由器连接的多个局域网，可以使用不同的物理层，数据链路层，网络层协议，而网络层以上的高层协议必须相同

（3）路由器在输出分组的第一位之前，必须先收到整个分组，这种机制称为存储转发机制

（4）在路由转发过程中，源主机和中间路由器都不知道 IP 分组到达目的主机需要经过的完整路径

（5）虚电路传输中只有建立连接的分组需要携带源地址和目的地址，其他分组携带虚电路标记即可

（6）SDN 软件控制网络的特点包括：控制和转发功能分离，控制平面集中化，接口开放可编程，不包括 Openflow 取代了路由协议

（7）SDN 不是一种新型的物理网络结构，只是一种设计方法 / 体系结构

（8）虚电路网络只会预先确定传输路径，但不会为线路预先分配带宽，在建立虚电路的时候需要进行路由选择

（9）IPv4 数据报分片后会在目的主机进行重组

（1）不考虑 NAT 的前提下，IP 数据报从源主机到目的主机转发过程中，源地址和目的地址都不会发生改变

（2）把 IP 网络划分为子网，目的是减少广播域的大小

（3）CIDR 技术的作用：有效分配 IP 地址空间，并且减少路由表中表项的数目

（4）可以为主机动态分配 IP 地址的协议为 DHCP 协议

（5）路由器收到数据报后，先对 TTL 减一，再检查是否变为零并判断丢弃

（6）对于已经分片的分组，只对第一个分片产生 ICMP 差错报文，同时对于多播的分组，也不会产生 ICMP 差错报文

（7）与 IPv4 协议相比，IPv6 协议减少了首部字段数量，没有提供校验和字段，使用扩展首部来实现选项功能

（8）IPv6 的首部长度是不可变的（40B），同时不允许在中间路由器进行分片，因而如果一个路由器收到的 IPv6 数据报因为太大而不能转发到链路上，则路由器会把该数据报直接丢弃

（9）IPv4 向 IPv6 的过渡可以采用双栈协议和隧道技术

（1）对于分层路由而言，每个区域中只有处于边界的路由器才知道如何路由到其他区域

（2）分层路由会使得 OSPF 协议更加复杂

（3）路由收敛指的是：网络设备的路由表与网络拓扑结构保持一致

（4）RIP - 距离向量，OSPF - 状态链路，BGP - 路径向量

（5）对于 OSPF 算法而言，划分区域的主要目的是减少洪泛法交换的通信量

（6）BGP 交换的网络可达性信息是：到达某个网络所经过的路径

（7）路由器处理的信息量比交换机多，因此转发速度比交换机慢

（8）路由器在进行路由选择时需要参考不同的路由协议，未必一定会提供延迟最小的最佳路由

（9）路由器转发分组时，根据路由表中的接口标识符确定输出端口

（1）路由器不会转发本网段中的广播帧

（2）使用 IP 地址 + 端口号即可唯一确定互联网上的一个进程（IP 地址在互联网中确定主机，端口号在主机中确定进程）

（3）UDP 传输的可靠性只能由应用层保证

（4）UDP 数据报比 IP 数据报最主要的是增加了端口功能

（5）如果 UDP 校验和计算结果为 0，则对应的校验和字段会填写全 1；如果源主机不使用 UDP 校验和，则对应的校验和字段会填写全 0

（6）不同局域网内的计算机可以使用相同的端口号（可以类比内网 IP 地址）

（7）TCP 协议会强制重传错误分组，并且把消息分成段，在目标主机中进行重组

（8）TCP 报文段中的 RST 字段为 1 时，说明出现严重错误，必须释放连接

（9）TCP 传送大量数据时，以 MSS 的大小将数据进行分割发送

考试总结

一定要看最后一章无线网和 IPv6 协议，重点是 IPv6 的报文段（相比于 IPv4 减少了哪些字段）和 CSMA/CA 协议（相比于 CSMA/CA 的不同），最好让 AI 总结一个比较全面的直接照着背，选择考这两个点很多

填空考点很细，PPT 上有的几乎都会考，甚至考了光纤到家的缩写（FTTH），常见协议的缩写对应的中文名要会背（OSPF，RIP 这些）

简答题很常规，几乎是往年题的原题

计算题很常规，路由表填写要求最大程度的路由聚合，往年题好像没有，不熟悉的同学可以了解一下，流量 / 拥塞控制结合实际数据传输过程和超时重传考察的，不过也比较常规，现场能推出来，也可以做一下王道类似的计算题

整体难度不算大，几乎能被王道的课后题覆盖，考研选手应该很占优势，只看课内 PPT 的话可能需要复习的时候过仔细一点，主播主要是被 IPv6 协议阴了一手，两个有关的多选都当单选做的
