常数初始化

常数初始化，即所有权重参数初始化为同一个常数

在现有 Pytorch 架构中，原有的 ones(), zeros(), constant() 等方法已经废弃，取而代之的是带有下划线的方法，功能完全相同

注意：将权重初始值设为 0 将无法正确进行学习，严格地说，不能将权重初始值设成一样的值，因为这意味着反向传播时权重全部都会进行相同的更新，被更新为相同的值（对称的值），这使得神经网络拥有许多不同的权重的意义丧失了，为了防止 “权重均一化”（瓦解权重的对称结构），必须随机生成初始值

秩初始化

秩初始化，即权重参数初始化为单位矩阵

正态分布初始化

正态分布初始化，即权重参数按指定均值 mean 和标准差 std 进行正态分布初始化

均匀分布初始化

均匀分布初始化，即权重参数在指定区间内按照均匀分布做初始化

Xavier 初始化（Glorot 初始化）

Xavier 初始化根据输入和输出的神经元数量调整权重的初始范围，确保每一层的输出方差与输入方差相近，适用于 Sigmoid 和 Tanh 等激活函数，能有效缓解梯度消失或爆炸问题

Xavier 正态分布初始化：均值为 0，标准差为 2nin+nout\sqrt{\frac{2}{n_{in}+n_{out}}}nin​+nout​2​​ 的正态分布。

Xavier 均匀分布初始化：[−6nin+nout,2nin+nout][-\sqrt{\frac{6}{n_{in}+n_{out}}},\sqrt{\frac{2}{n_{in}+n_{out}}}][−nin​+nout​6​​,nin​+nout​2​​] 区间内均匀分布。

其中 ninn_{in}nin​ 表示输入数，noutn_{out}nout​ 表示输出数

He 初始化（Kaiming 初始化）

He 初始化根据输入的神经元数量调整权重的初始范围，主要适用于 ReLU 及其变体（如 Leaky ReLU）激活函数

He 正态分布初始化：均值为 0，标准差为 2nin\sqrt{\frac{2}{n_{in}}}nin​2​​ 的正态分布

He均匀分布初始化：区间 [2nin,2nin][\sqrt{\frac{2}{n_{in}}},\sqrt{\frac{2}{n_{in}}}][nin​2​​,nin​2​​] 内均匀分布

其中 ninn_{in}nin​ 表示输入数

BatchNormal 参数归一化

Pytorch 中的参数归一化方法以网络层的形式实现，通过调用 torch.nn.BatchNormal1D() 方法可以初始化一个网络层，该方法接收一个参数 num_features 表示待归一化的特征数

常见的变种还有 torch.nn.BatchNormal2D() 和 torch.nn.BatchNormal3D() 方法，目前我们最常用的就是 1D，该方法所接收的输入张量的形状必须是二维或者三维，即形状为 [N, C] 或者 [N, C, L]，其中：

  • C 为特征数，必须和 num_features 值相对应
  • N 为一个批次 batch 的大小，即一组内数据的条数
  • L 为可选项，表示序列长度或时间维度

对于 2D 张量 [N, C]，其归一化规则是对于每一个特征 C，求 N 条数据在 C 特征上的均值和方差，分别对每个 C 进行初始化

对于 3D 张量 [N, C, L]，其归一化规则是对于每一个特征 C，即所有矩阵的第 C 行上的所有列，求解这些数据的均值方差，用来对所有矩阵这一行上所有列进行归一化，假设数据为：

对于通道 0（C=0）：

  • 从样本0，通道0：[1, 2, 3, 4]
  • 从样本1，通道0：[13, 14, 15, 16]
  • 所有数据：[1, 2, 3, 4, 13, 14, 15, 16]，共 8 个值求均值和方差，用来归一化这 8 个数据

对于通道 1（C=1）：

  • 从样本0，通道1：[5, 6, 7, 8]
  • 从样本1，通道1：[17, 18, 19, 20]
  • 所有数据：[5, 6, 7, 8, 17, 18, 19, 20]，共 8 个值求均值和方差，用来归一化这 8 个数据

对于通道 2（C=2）：

  • 从样本0，通道2：[9, 10, 11, 12]
  • 从样本1，通道2：[21, 22, 23, 24]
  • 所有数据：[9, 10, 11, 12, 21, 22, 23, 24]，共 8 个值求均值和方差，用来归一化这 8 个数据

Dropout 随机失活

Dropout 随机失活 / 暂退法是一种在学习的过程中随机关闭神经元的方法，可以通过 torch.nn.Deopout§ 来使用 Dropout，并通过参数 p 来设置失活概率（缺省则默认为 0.5）

所谓失活 / 关闭神经元，即把对应输入到 Dropout c层的张量中的某些位置的值变成 0

传入到 Deopout 中的张量必须是浮点类型

这里设置神经元失活的概率 p 只是期望概率，比如 10 个神经元设置 p=0.5 并不意味着每次一定都关闭 5 个神经元，只是多次测试下的统计规律，但是在关闭一部分神经元的同时，剩下神经元（即张量未清零位置的数值）会翻倍，对应的系数为 11−p\frac{1}{1-p}1−p1​，这是一个确定的绝对数值

损失函数

分类任务损失函数

（1）二分类任务损失函数

二分类任务常用二元交叉熵作为损失函数（Binary Cross-Entropy Loss）

L=−1n∑i=1n(yilog⁡y^i+(1−yi)log⁡(1−y^i))L=-\frac{1}{n}\sum_{i=1}^n(y_i\log \hat{y}_i+(1-y_i)\log(1-\hat{y}_i)) L=−n1​i=1∑n​(yi​logy^​i​+(1−yi​)log(1−y^​i​))

其中：

  • yiy_iyi​ 为真实值，通常为 0 或 1
  • y^i\hat{y}_iy^​i​ 为预测值，表示样本 i 为 1 的概率

在 Pytorch 中可以用 torch.nn.BCELoss() 方法实现二元交叉熵

torch.nn.NCELoss() 允许接收两种形式的标签：

  • 和模型预测输出形状 [N, 2] 相同的独热编码，每一行都是 [0, 1] 或者 [1, 0]，也可以认为是类别标签，标记属于 0 类还是 1 类
  • 和模型预测输出形状 [N, 2] 相同的类别概率标签，每一行都是当前样本属于两个分类类别的参考概率，比如 [0.8, 0.2]

（2）多分类任务损失函数

多分类任务常用多类交叉熵损失函数（Categorical Cross-Entropy Loss），它是对每个类别的预测概率和真实标签之间差异的加权平均

L=−1n∑i=1n∑c=1Cyi,clog⁡y^i,cL=-\frac{1}{n}\sum_{i=1}^n\sum_{c=1}^C y_{i,c}\log \hat{y}_{i,c} L=−n1​i=1∑n​c=1∑C​yi,c​logy^​i,c​

其中：

  • CCC 为多分类总类别数
  • yi,cy_{i,c}yi,c​ 为真实标签值，在本公式中要求为独热编码，即一个长度为 C 的向量，每个位置为 0 或 1 表示是否为此类别
  • y^i,c\hat{y}_{i,c}y^​i,c​ 为预测值，同样是一个长度为 C 的向量，每个位置表示表示样本 i 为此类别的概率

注意：调用 torch.nn.CrossEntropyLoss() 相当于调用了 torch.nn.LogSoftmax() 之后再调用 torch.nn.NLLLoss()，即使用 CrossEntropyLoss() 时上一层的输出不需要 Softmax 激活函数，因为该损失函数内会自动处理

该损失函数允许接收三种形式的类别标签：

  • 和模型预测输出形状 [N, C] 相同的独热编码，每一行都是 [0, …, 1, …, 0] 表示属于某个类别
  • 和模型预测输出形状 [N, C] 相同的类别概率标签，每一行都是样本属于当前分类类别的参考概率，比如 [0.2, 0.1, 0.4, 0.3]（四分类）
  • 每一条样本所属的类别编号，形状为 [N, ]，每个位置是 0 ~ C-1 的一个编号，这种输入形式只有多元交叉熵支持，二元交叉熵是不支持的

回归任务损失函数

（1）MAE

平均绝对误差 Mean Absolute Error，MAE，也称为 L1 Loss，可以通过 torch.nn.L1Loss() 实现

L1n∑i=1n∣yi−y^i∣L\frac{1}{n}\sum_{i=1}^n|y_i-\hat{y}_i| Ln1​i=1∑n​∣yi​−y^​i​∣

L1 Loss 对异常值泛化性好，但在 0 点处不可导

（2）MSE

均方误差 Mean Square Error，MSE，也称为 L2 Loss，可以通过 torch.nn.MSELoss() 实现

L=1n∑i=1n(yi−y^i)2L=\frac{1}{n}\sum_{i=1}^n(y_i-\hat{y}_i)^2 L=n1​i=1∑n​(yi​−y^​i​)2

L2 Loss 对异常值敏感（较大的 X 会被平方项进一步放大），遇到异常值时容易擦生梯度爆炸

平滑 L1，相当于 MAE 和 MSE 结合的产物，通过 torch.nn.SmoothL1Loss() 实现

当误差较小时（∣yi−y^i∣<1|y_i-\hat{y}_i|<1∣yi​−y^​i​∣<1）使用 L2 Loss，使得损失函数平滑可导，当误差较大时（∣yi−y^i∣≥1|y_i-\hat{y}_i|\ge 1∣yi​−y^​i​∣≥1）使用 L1 Loss 降低异常值的影响

参数更新方法

Pytorch 中有关参数更新的方法以优化器（optimizer）的形式实现，常用的优化方法都通过 torch.optim 包进行调用

Pytorch 中反向传播更新参数的流程大致分成以下三步：

在多轮迭代中：

  • 每次加载一个小批量 batch 的数据，作为模型 model 输入得到模型输出
  • 将模型输出和标签传入设定的损失函数，得到 loss_value
  • 损失函数进行反向传播得到模型参数梯度
  • 使用设置好的优化器更新模型参数
  • 清空本轮梯度，防止在下一轮参数更新时本轮梯度累加

遵循这个流程，我们本节需要做的就是选择不同的优化器即可

随机梯度下降 & 动量法

梯度下降法和动量法都可以通过 torch.optim.SGD() 方法实现，后者通过设置 momentum 参数，来设置历史梯度权重，从而实现动量法

该方法接收如下几个常见参数：

  • params：优化器待更新的参数列表，如果有网络模型 Model，通常传入 Model.parameters() 即可，如果只是更新单个张量 X，则需要封装成列表形式
  • lr：学习率，默认为 1e-3
  • momentum：动量法中的历史梯度权重

学习率衰减

学习率衰减优化器 lr_scheduler 并不直接更新 Model.parameters()，而是更新某个已有优化器的参数，因此其接收的参数是 optimizer 而不是 params，在更新优化器参数时，也需要额外调用一次 scheduler_lr.step() 方法才能完成对应优化器学习率的衰减

  • size：发生衰减的间隔
  • gamma：衰减的比例

Adam

通过 torch.optim.Adam() 实现自适应据估计，该方法是 AdaGrad 和 Momentum 两种方法的结合，是目前最常用的参数更新方法

v←α1v+(1−α1)∇h←α2h+(1−α2)∇2v^=v1−α1th^=h1−α2tW←W−ηv^h^v\leftarrow\alpha_1v+(1-\alpha_1)\nabla\\ h\leftarrow\alpha_2h+(1-\alpha_2)\nabla^2\\ \hat{v}=\frac{v}{1-\alpha_1^t}\\ \hat{h}=\frac{h}{1-\alpha_2^t}\\ W\leftarrow W-\eta\frac{\hat{v}}{\sqrt{\hat{h}}} v←α1​v+(1−α1​)∇h←α2​h+(1−α2​)∇2v^=1−α1t​v​h^=1−α2t​h​W←W−ηh^​v^​

其中：

  • η\etaη：学习率
  • (α1,α2)(\alpha_1,\alpha_2)(α1​,α2​)：一次和二次动量系数
  • ttt：迭代次数，从 1 开始

该方法接收三个常见参数：

  • params：优化器待更新的参数列表，如果有网络模型 Model，通常传入 Model.parameters() 即可，如果只是更新单个张量 X，则需要封装成列表形式
  • lr：学习率
  • betas：传入一个元组 (α1, α2)，分别制定一次动量系数和二次动量系数，
