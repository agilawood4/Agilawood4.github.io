网络层

网络层提供的两种服务

（1）面向连接的服务，即虚电路（Virtual Circuit）

  • 通信双方在开始数据传输前，先由网络建立连接，之后的数据均通过该连接进行，由网络保证数据传输的可靠性
  • 虚电路只是一种逻辑连接，分组沿着这条逻辑连接按照存储转发方式传送，而并不是真正建立了一条物理连接
  • 支持方：以电信公司为代表的一派

（2）无连接的服务，即数据报（Datagram）

  • 网络在发送数据时不需要先建立连接，每一个分组在网络中独立传送
  • 网络层不保证服务质量，分组可能出错，丢失，重复，失序，也不能保证分组传送的时限
  • 支持方：以 Internet 为代表的一派

最终第二种服务胜出，TCP/IP 采用数据报服务

网际 IP

网际协议 IP（Internet Protocol）是 TCP/IP 体系中最重要的两个协议之一

与 IP 协议配套使用的还有四个协议：

  • 地址解析协议 ARP：Address Resolution Protocol
  • 逆地址解析协议 RARP：Reverse Address Resolution Protocol
  • 网际控制报文协议 ICMP：Internet Control Message Protocol
  • 网际组管理协议 IGMP：Internet Group Management Protocol

其中 RARP 和 ARP 协议和网络层与数据链路层之间的衔接有关，因此写在靠近网络接口层的位置

网络层传输的数据单元称为数据包（package），也称为数据报，有时候简称为包

IP 地址分类

IP 地址：分配给主机或路由器的标识符，目前使用的是 IPV4 的 32 位 IP 地址（IPV6 是 16 字节地址，IPV4 是 4 字节地址）

  • IP 地址的分配由 ICANN 负责

IP 地址的编址方法经历了三个阶段：

  • 分类的 IP 地址
  • 子网划分
  • 构成超网

分类的 IP 地址：IP 地址被分成 A,B,C,D,E 五类，每一类地址都包含网络号 net-id 和主机号 host-id 两个字段

IP 地址::={<网络号>,<主机号>}\text{IP 地址}::=\{<网络号>,<主机号>\} IP 地址::={<网络号>,<主机号>}

不同类别的 IP 地址的区别主要在于网络号，主机号的长度不同，通俗来讲，每个大机构对应一个网络号，而机构内部的小设备对应不同的主机号

在 A~E 类地址中，真正能够给网络使用的只有 A,B,C 三类地址，它们的网络 IP 开头分别是 0，10，110，因此通过地址开头即可区分它们属于哪一类地址

其中 A 类地址有两个特殊的网络号，即 0 号网络号和 127 号网络号：

  • 0 号：对应网络 IP 为 0.x.x.x，表示本机地址，或保留地址
  • 127 号：对应网络 IP 为 127.x.x.x，表示环回地址

这两个特殊的网络号是不能用的，因此 A 类的网络号会少两个

而对于所有主机号而言，都有两个特殊的主机号，即全 1 和全 0 主机号：

  • 全 0 主机号：表示整个网络本身（或者说是某个网段），一个网段可以包含多个网络地址
  • 全 1 主机号：广播地址

其中 A,B,C 类地址都会各自拿出来一部分地址作为私有地址，私有地址在不同的内网中是可以复用的（接在一个路由器上的内网中地址不能重复），而剩余的地址则是公网地址，公网地址则是全球唯一的

路由器对目的地址是私有 IP 地址的数据报一律不进行转发

网络地址转换 NAT（Network Address Translation）：在专用网连接到因特网的路由器上安装 NAT 软件，安装了 NAT 软件的路由器叫做 NAT 路由器，它至少有一个有效的外部全球 IP 地址

区分 IP 地址与硬件 MAC 地址

  • IP 地址：网络层及以上各层使用的地址，是一种逻辑地址，存放在 IP 包头部
  • 物理地址：数据链路层以及物理层使用的地址，存放在数据链路层的帧中

从上层到下层，待传输的数据是逐渐被封装的，换言之，下层传送的数据中一定包含上层的数据，下层传送的数据由上层的数据加工封装而来，因此，帧中是有 IP 地址的

ARP 与 RARP 协议

广域网的通信使用 IP 地址，局域网内部的通信使用 MAC 地址，局域网之间的通信使用 IP 地址

IP 地址与物理地址的相互转换问题：在下图中，如果主机 H10H_{10}H10​ 想要把 IP 包发送给 H1H_1H1​，路由器 R1R_1R1​ 想要在局域网中把 IP 包发送给主机 H1H_1H1​，需要知道 H1H_1H1​ 的物理地址

这就需要用到 ARP 协议，ARP 协议用于把 IP 地址映射为物理地址（MAC 地址），而 RARP 协议恰好相反，其用于把物理地址映射回 IP 地址

在下图演示的过程中，如果 A 设备想给 B 设备发送 IP 数据，需要借助 ARP 协议得知 B 的 MAC 地址

但是如果每次要发送信息的时候，都要先广播 ARP 帧去查目标站的 MAC 地址，这样会很浪费带宽，且容易造成广播风暴，因此每个主机会设有一个 ARP 高速缓存（ARP cache），存有本地局域网上各主机和路由器的 IP 地址与硬件地址的映射表

当主机 A 想向本局域网上的主机 B 发送 IP 包时：

  • 先在 A 的 ARP 高速缓存中查看有无主机 B 的 IP 地址
  • 如果有，就直接查出它对应的硬件地址，再把硬件地址写入 MAC 帧，通过局域网发送
  • 如果没有，则在网络中广播一个 ARP 请求
  • 当主机 B 收到这个 ARP 请求以后，会向主机 A 返回一个 ARP 应答，告知它自己的 MAC 地址

注意，以上操作发生的前提是 A 和 B 主机处于同一个局域网内（即 ARP 解决的是同一个局域网中的主机或路由器的 IP 地址和硬件地址之间的映射问题），如果 A 和 B 不在同一个局域网内，此时 IP 包需要经过路由器转发，此时局域网内 A 要完成的是路由器 IP 和物理地址之间的映射，而 B 的物理地址则由 B 所在局域网的那个路由器进行查询

  • A：查找 A 所在局域网的路由器的物理地址
  • B 所在局域网的路由器：查找 b 的物理地址

IP 数据报

一个 IP 数据报由头部和数据两部分组成，其中头部是 20 字节的固定字段 + 0 到多个可选字段

（1）第一个字：在 TCP/IP 模型中，每 32bit 被划分为一个字，其中 IP 包的前四个字段分别为：

  • Version 字段：长度为 4bit，IP 协议版本，目前的 IP 协议版本号为 4，即 IPV 4
  • IHL（IP Head Lenth）：长度 4bit，IP 包头的长度，最小为 5 最大 15，单位为 word，也就是 4 字节，因此 IP 数据报头部最长为 60 字节，最短为 20 字节
  • Type of Service：长度 1B，表示服务类型，和数据被传输的优先级有关，但是目前很多路由器都忽略该字段
  • Total Lenth：长度 2B，IP 数据报的总长度，包括包头和数据，单位为字节，根据其位数可知，一个 IP 包的最大长度为 65535 字节（通常不会传这么大的 IP 包，因为底层的数据链路层所能传输的最大数据长度为 1500 字节）

（2）第二个字：全部和 IP 包的分片有关（较大的 IP 包需如果超过了数据链路层的 MTU（Maximum Transmission Unit）要切分成小片才能够在数据链路层中传输）：

  • Identification：长度为 2B，计数器，作为产生的 IP 包的标识，由同一个 IP 包切分而成的子 IP 包具有相同的标识，便于接收端进行重组
  • DF：长度为 1bit，含义为 Don’t Fragment，为 0 的时候表示允许切片，为 1 的时候表示不允许切片
  • MF：长度为 1bit，含义为 More Fragment，为 0 的时候表示后面不再有新的分片了，为 1 的时候表示后面还有新的分片
  • Fragment offset：长度为 13bit，表示片偏移，较长的包在分片后，某个片在原分组中的相对位置，以 8 字节为单位，即假设某个分片的偏移为 T，则实际的偏移是 8T 字节

分片举例（假设数据链层一帧的载荷长度不能超过 1420 字节，这里我们选择每一段分片的大小为 1400 字节）：

（3）第三、四、五个字：

  • TTL（Time of Live）：长度为 1 字节，表示 IP 包在网络中的生存时间，即 IP 包在网络中可以通过的路由器个数的最大值
    • IP 包每经过一个路由器称为一跳（hop），经过的路由器个数称为跳数 number of hops
    • 具体实现中，发送 IP 包的时候先给它的 TTL 赋一个初值，没经过一个路由器则 TTL 减一，减到 0 则直接丢弃，并且向源主机发一个告警包
    • TTL 的最大值为 255，Windows 系统一般为 128，UNIX 系统一般为 255，Linux 系统一般为 64，实际的跳数一般不会超过 20
  • Protocol：长度为 8bit，协议字段，表示该包中数据部分的协议类型，即上层协议类型，该字段决定了该数据报要交付到哪里
  • Header checksum：长度为 2 字节，包头校验和（注意这里只检验包头）
  • Source address：长度为 4 字节，源 IP 地址（网络地址，比如 192.168.1.0）
  • Destination address：长度为 4 字节，目的 IP 地址（同上）

（4）可选字段：以 4 字节为单位，最长为 40 字节，现实网络中很少用

包头校验和的计算：

  • 发送端：对于 IP 包头按照每 16bit 进行分组，每一组先取反码，再做累加（循环进位，高位溢出进位到末位），最后再对累加和取反，得到检验和，检验和会随着包头传输给接收端
  • 接收端：对 IP 包头同样按 16bit 分组，此时检验和恰好被分到完整的一组 16bit 中，每一组先取反码，再做循环加和，最后对累加和取反，如果为 0，则表示包头没有出错

数据发送过程中检查和填写校验和的时机：

  • 检查：路由器在接收数据之前需要检查，如果有误就直接丢弃，接收端也需要检查
  • 填写：路由器检查完包头校验和无误以后，对 TTL 做减一，然后计算新的包头校验和，再转发数据，发送端在发送之前也要填写

  • 看到 45 即可提示：IP 数据报采用 IPV4 协议，数据报首部没有变长部分，长度为 20B

子网划分和超网

子网划分

分类 IP 地址无法适应 Internet 快速发展的需要，A 类地址的主机数超过 1000 万，B 类地址的主机数也超过 6 万，需要对这些网络地址再进行细分，才能容纳下这么多设备同时连接，也就是说，需要把一个网络再划分为若干个子网（subnet）

  • 常按物理网络或单位组织机构划分子网，直观且易于管理
  • IP 地址中并不包含子网信息，子网信息的体现需要使用子网掩码（subnet mask）

要会根据完整 IP 和子网掩码求解网络 IP：

具体做法就是把二者做与运算即可，得到的网络 IP 即代表这一段局域网网段的地址

区分子网个数和一个子网内的主机个数，比如一个 C 类网络，如果它的子网掩码被进一步划分成255.255.255.224，则相当于最后八位里面，3 位划分到网络号，5 为划分到主机号，因此共有 8 个子网，每个子网网段中有 30 个主机

使用子网掩码的分组转发过程

路由器中的路由表包含了三项基本信息：

  • 目的网段地址
  • 子网掩码
  • 下一跳到达的路由器的地址（如果是与当前路由器直连的网段，则为直接交付，否则为对端路由器的接口 IP）

转发流程：

  • 从收到的分组首部提取目的 IP 地址 D
  • 先用与该路由器直接相连的各个网络的子网掩码和 D 做按位与运算，看是否和相应网络地址匹配，如果匹配，则将分组直接交付，否则进入下一步做间接交付
  • 若路由表中有目的网络地址为 D 的特定主机路由，则将分组传送给指明的下一条路由，否则执行下一步
  • 对路由表中的每一行子网掩码和 D 做按位与运算，若其结果与该行对应的网络地址匹配，则把分组传送给该行指明的下一跳路由器，否则执行下一步
  • 若路由表中有一个默认路由，将分传送给路由表中默认的路由，否则执行下一步
  • 报告转发分组出错

注意：路由表中仅包含局部路由信息，使用默认路由以确保任意分组的转发

该过程的核心操作为：把目的 IP 地址和路由表中的子网掩码做与运算，并判断是否与目的网络匹配

实际案例：主机 H1 向 H2 发送数据包后的转发过程

  • 主机 H1 根据自身设置判断目的地址是否在本地子网，即把目的地址与本地子网掩码做按位与运算，和本地网段地址比较，发现不相等，因此不在本地子网
  • 主机 H1 把数据包发送给路由器 R1，此时在同一个局域网内，主机 H1 会通过 ARP 协议查询路由器 R1 的 MAC 地址
  • 路由器 R1 收到数据包后，在路由表中逐项根据子网掩码计算匹配项，即把目的地址和每一个子网掩码做与运算，得到的结果和目的网络（网段）地址做比较，找到该目的地址对应的下一跳为接口 1
  • 路由器 R1 将数据包通过接口 1 到达子网 2，进而在子网 2 内将数据包发送给 H2，这里路由器需要通过 ARP 协议查询 H2 的 MAC 地址

上面这个转发过程经过了三个关键的步骤：

  • 主机发送数据包时，需要判断目的地址是否在本地子网网段

  • 路由器在转发数据包时，需要在路由表内查找目的网络的地址

  • 在同一个网段内，由路由器将数据包交付给接收设备，需要通过 ARP 协议将目的 IP 地址转化为目的 MAC 地址

对于一个主机而言，最重要的三个地址：

  • IP 地址
  • 子网掩码
  • 默认网关 / 网段地址 / 路由器地址

无分类编址 CIDR

CIDR（Classless Inter-Domain Routing）称为无分类域间路由

CIDR 的主要特点：

  • 消除传统 A,B,C 类地址以及划分子网的概念
  • 使用各种长度的网络前缀（network prefix）来代替分类地址中的网络号和子网号
  • IP 地址从三级编址（使用子网掩码）又回到了两级编址

IP地址::={<网络前缀>,<主机号>}\text{IP}地址::=\{<网络前缀>,<主机号>\} IP地址::={<网络前缀>,<主机号>}

CIDR 还采用斜线记法，即 CIDR 记法，在 IP 地址后面加一个斜线，后面协商网络前缀所占的位数，比如：128.14.25.7/20，表示该地址的高二十位是网络前缀

网络前缀相同的连续 IP 地址组成 CIDR 地址块

路由聚合

CIDR 编址带来的一个最大的好处在于路由聚合（route aggregation），即一个 CIDR 地址块可以代表很多地址，这样一个地址块可以看成是一个地址的聚合，称为路由聚合

路由聚合的优势：路由表中的一个项目可以表示很多个（上千个）原本传统分类地址的路由，可以减少路由表中的表项个数，并减少路由器之间交换的路由信息量

路由聚合也称为构成超网，之所以称为超网是由于 CIDR 地址块中大多包含许多个 C 类地址

CIDR 不使用子网，但仍保留地址掩码的概念，比如 /20 对应的地址掩码就是 11111111111111111111

CIDR 还有一些变式：

  • 10.0.0.0/10 可以简写为 10/10，即省略点分十进制中低位连续的 0
  • 网络前缀 + * 号的表示方法：例如 0000101000 * 表示 * 之前为网络前缀，* 为任意主机号

无分类编址的最长前缀匹配：

在使用 CIDR 时，路由表中的表项中的目的网络地址由固定长度变成了变长的网络前缀，在查找路由表时可能会得到不止一个匹配结果

因此我们考虑采取最长前缀匹配（longest prefix matching）原则：

  • 从匹配结果中选择具有最长网络前缀的路由
  • 网络前缀越长，对应作为主机 IP 的地址块就越小，因而路由就越具体（more specific）
  • 最长前缀匹配又称为最长匹配或最佳匹配

最长前缀匹配实例：

假设目的地址为 D，子网掩码为 M，子网网段地址（路由器中的项目）为 S，所谓能匹配上，即满足：

D  &  M=SD \;\& \;M = S D&M=S

而最长前缀匹配就是要在所有满足这个匹配关系的子网中，找到 M 中连续的 1 最长的那个掩码对应的子网

但是需要注意，在主机 IP 中，全零 IP 表示本机，全一 IP 表示广播，这两个特殊 IP 不可用，因此题目中所说的可分配主机地址个数应该是 8-2=6 只有 6 个

先由子网掩码可知，这个子网的网络 IP 为 30 位，对应的主机 IP 只有 2 位，再减去两个特殊地址不可用，可用的主机 IP 仅有 01 和 10 两个，而对于对应的 IP 分组，我们首先先比较前 30 位，发现前 30 位和对应子网的子网掩码按位与的结果恰好为子网的网段 IP，因此这个条消息可以发送到这个子网

按理说一条消息只能由一台主机接收，但是恰好这条消息的主机 IP 为全一，也就是 11，这恰好表示一条广播地址，因此该子网内的两台主机都能收到这个消息

网际控制报文协议 ICMP

ICMP 简介

ICMP（Internet Control Message Protocol）用于报告出错和测试等控制信息

ICMP 位于 IP 层，对应的 ICMP 报文被封装在 IP 包的数据部分，对于一个 IP 包，当包头的 Protocol 字段为 1 时，表示对应的数据部分是一条 ICMP 报文

ICMP 报文共有 9 种类型，两大类（分别是差错报告报文和询问报文）

差错报告报文：

  • Destination unreachable：终点不可达，路由器或主机无法传输报文时，需要向源主机发送此报文
  • Time exceeded：超时，当报文的 TTL 字段为 0 时，路由器将其丢弃，并向源主机发送此报文
  • Parameter problem：参数问题，路由器或主机收到的报文中头部有非法字段时，丢弃数据包，并向源主机发送此报文
  • Source quench：源点抑制，路由器或主机由于拥塞丢弃报文时，向源主机发送此报文，使其放慢发送速度
  • Redirect：重定向，路由器向主机发送此报文告知路由改变，主机下次发送数据包会给另外的路由器

询问报文：

  • Echo request / reply：回声探测，用于测试网络连通性（ping 指令会用到）
  • Timestamp request / reply：请求时间，可用于时间同步

ICMP 的应用：

（1）Ping 指令：用来测试两个主机之间的连通性，底层原理是采用 ICMP Echo request / reply 报文

  • 向目的主机发送 ICMP echo request 报文，对方收到后会回应 ICMP echo reply，根据能否收到应答判断两台主机之间是否联通

（2）Traceroute / Tracert：用来测试到另一台主机所经过的路由信息，采用 ICMP 超时报告报文

  • 逐个发出 UDP 报文，其 IP 包头中的 TTL 字段分别被设置为 1,2,3,…，直到到达目的主机，报文会在其路径上的每个路由器依次失效，由此触发该路由器向源主机发送 ICMP 超时报文，从该报文即可获得每个路由器的 IP 地址

路由算法及协议

Internet 中的设备被划分为两大类：

  • Host：主机
  • Gateway：网关

路由算法

在网络中，路由器依据路由信息（路由表）转发分组，路由信息是路由协议生成的，路由算法是路由协议的基础和核心

  • 注意：路由协议是用来生成路由信息（路由表）的，不是转发分组的，而路由算法则是用来生成路由协议的

大致的流程是：路由算法 → 路由协议 → 路由表 → 路由转发，前者不能越级决定后者

理想的路由算法需要具备的特性

  • 必须是正确的和完整的，按照此算法得到的路由必须能正确寻址
  • 在计算上应该简单，不能增加过多开销
  • 有自适应性，能根据通信量和网络拓扑的变化调整路由
  • 具有稳健性，通信量和拓扑稳定时，能快速收敛
  • 具有公平性，对所有用户公平
  • 应是最佳的，能找出最好的路由（时延最小，吞吐量最大）

不同情况下的路由需求可能各有侧重，且网络环境不断变化，因此不存在一种绝对的最佳路由算法

路由算法分为静态路由和动态路由两类

  • 静态路由选择策略
    • 非自适应路由选择
    • 简单和开销较小，但不能及时适应网络状态的变化
    • 适用于小规模且变化较少的网络，由人工设置路由
  • 动态路由选择策略
    • 自适应路由选择
    • 能较好适应网络状态的变化，但实现起来较为复杂，开销也较大
    • 适用于较大规模，变化频繁的网络，通过专门的算法和协议进行路由的计算

两类典型的动态路由算法：

  • 距离向量（distance vector）路由算法（自治系统内部）
  • 链路状态（link state）算法（自治系统内部）

Internet 采用分层次的路由，原因如下：

  • Internet 规模庞大，任何一台路由器都不可能获取和存储整个 Internet 的路由信息
  • 接入 Internet 的众多网络（管理方）不希望外界了解自己网络的内部信息

自治系统（AS，Autonomous System）：在单一的技术管理下的一组路由器，这些路由器使用一种 AS 内部的路由选择协议和共同的度量以确定分组在该 AS 内的路由，同时还使用一种 AS 之间的路由选择协议用以确定分组在 AS 之间的路由

  • 一个 AS 可使用多种内部路由选择协议和度量，但对其他 AS 表现出的是单一的和一致的路由选择策略

在自治系统背景下，Internet 路由协议可以分成两类

  • 内部网关协议 IGP，Interior Gateway Protocol：自治系统内部使用的路由选择协议，这类路由协议使用得最多，比如 RIP 和 OSPF 协议
  • 外部网关协议 EGP，External Gateway Protocol：若源站和目的站处在不同的自治系统中，当数据报传到一个自治系统的边界时，就需要一种协议将路由选择信息传递到另一个自治系统中，这样的协议就是外部网关协议 EGP
    • 目前使用最广泛的外部网关协议：BGP-4

自治系统之间的路由选择又称为：域间路由选择

自治系统内部的路由选择又称为：域内路由选择

自治系统内部的信息传递侧重效率 / 速度，自治系统之间的信息传递侧重连通性 / 可靠性，自治系统内部协议相比于外部协议而言约束更加严格，规范也更加清晰

整个互联网的架构是基于自治系统建立的

内部网关路由协议

距离向量路由与 RIP 协议

距离向量路由（distance vector routing）

  • 属于动态路由算法
  • 也称为 Bellman-Ford 或者 Ford-Fulkerson 算法（本质上用的是最短路的思想）
  • 最初用于 ARPANET，被 RIP 协议采用

距离向量路由的基本思想

  • 每个路由器维护一张表，表中给出了当前路由器到每个目的路由器的已知最佳距离和线路，并通过与相邻路由器交换距离信息来更新这张表
  • 路由器周期性地向所有相邻路由器发送它的距离表，同时它也接收每个邻居节点发来的距离表
  • 相邻路由器 XXX 发来的距离表中，XXX 到路由器 iii 的距离为 XiX_iXi​，本路由器到 XXX 的距离为 mmm，则本路由器经过 XXX 到 iii 的距离为 Xi+mX_i+mXi​+m，根据不同邻居发来的信息，计算 Xi+mX_i+mXi​+m 并取最小值，更新本路由器的路由表

距离向量路由的缺点：存在无穷计算问题

  • 对于好消息（新增链路）反应迅速，对于坏消息反应迟钝（减去链路）

以右图为例，由于在距离向量法中，每个路由器只能和它相邻的路由器交换信息，假设某一时刻 A 停止工作，但是 C 不会收到信息，C 还以为自己到 A 的距离为 2，只有 B 会收到信息，即左侧不再得到 A 的信号，则 B 会用它的右邻居 C 到 A 的距离来更新它自己到 A 的距离，也就是变成 3，下一轮同理，C 又用它的右邻居 B 到 A 的距离更新自己到 A 的距离，也就是变成 4，二者这么反复迭代，需要经过很多轮他们到 A 的距离才会都变成无穷

而实际上，当 A 停止工作的瞬间，A 到所有设备的距离都已经变成无穷了，因此相当于引入了错误信息，或者说算法在此时的收敛性是比较差的

RIP 协议

RIP 协议（Routing Information Protocol）可以看成是向量距离路由协议的一个具体实现，本质上也是向量距离路由协议的一种

  • 优点：简单，适用于小规模网络中的路由
  • 缺点：考虑过于简单，没有考虑带宽，时延，传输价格等复杂因素
  • RIP 中距离的定义：路由器跳数
  • 最长距离为 15，即允许的最长路径中最多包含 15 个路由器，距离为 16 即表示不可达
  • 直接连接距离为 0

RIP 的三个要点：

  • 仅和相邻路由器交换信息
  • 交换的信息是当前本路由器所知道的全部信息，即自己的路由表
  • 按固定的时间间隔交换路由信息，一般为 30s

本题的关键在于是 RIP 协议，有了距离为 16 就表示不可达的限制，因此 R2 和 R1 都无法到达 net1，如果是一般的距离向量路由协议，则 B 是正确的，C 因为不确定 R1 到达 net1 是否还有其他中间路由器可以引入更短的路径，因此也不保证一定对

RIP 的缺点：

  • 当网络出现故障时，需要经过较长的时间才能把此信息传送到所有路由器（距离向量路由的固有缺点）
  • 支持的网络规模有限，最大距离为 15（16 即表示不可达）
  • 路由器之间交换的路由信号是路由器中的完整路由表，随着网络规模的扩大，开销随之增加

链路状态路由与 OSPF 协议

链路状态路由（link-state routing）的基本思想：每个路由器都需要完成以下五步工作

  • 发现它的邻居节点，并学习其网络地址
    • 路由器启动时，通过发送 HELLO 包发现邻居节点
    • 两个或多个路由器相连在同一个 LAN 时，引入人工节点
  • 测量到各个邻居节点的延迟或开销（构造链路）
  • 构造一个分组，其中包含所有它刚刚知道的信息
  • 将这个分组发送给其他所有路由器
  • 计算出到每一个其他路由器的最短路径

各个路由器之间动态交换链路状态信息，每个路由器都建立一个链路状态数据库

  • 链路状态即本路由器都和哪些路由器相邻，以及该链路的度量（metric）

链路状态数据库实际上是全网的拓扑结构图，它在全网范围内是一致的（即链路状态数据库的同步）

各个路由器根据网络拓扑使用 Dijkstra（本质上如果是拓扑图的话用 DP 就可以了）算法计算从本路由器到其他节点的最佳路径，构成路由表

如果多个路由器连接在同一局域网下，这并不是严格的图结构，我们引入一个虚拟节点 N，并且设置直接连接距离 N-A，N-C，N-F 都为 0 即可

OSPF 协议

OSPF 协议（Open Shortest Path First）本质上属于链路状态路由协议，其优点为：

  • 收敛速度快，当链路状态发生变化时，节点会以洪泛（flooding）方式告知所有网络中的所有节点（除了它自己之外）
  • 适用于较大规模的网络：仅在链路状态发生变化时发送洪泛信息，不会产生很大的通信量

洪泛可以看成是一种步长为 1 的广播，其定义为：当某个节点收到分组时，会向除了输入链路外的所有其他链路发送这个分组

为了管理更大规模的网络，OSPF 将一个自治系统再次划分为若干个区域（area），每个区域都有一个 32 位的区域标识符，用点分十进制表示

划分区域的好处：利用洪泛法交换链路状态信息的范围局限于一个区域而不是整个自治系统，减少了整个网络上的通信量

一个区域内的路由器只知道本区域的完整网络拓扑，而不知道其他区域的网络拓扑

各区域会做层次结构的划分，上层的区域称为主干区域（backbone area），主干区域的作用是用来联通其他在下层的区域，主干区域的标识符规定为 0.0.0.0

  • Hello 分组：探测邻居的链路状态，确定可达性
  • Database Description 分组：同步链路数据库，传送网络的拓扑结构图
  • Link State Request 分组：链路状态发生变化时，传送新的链路状态
  • Link State Update 分组：更新链路状态
  • Link State Acknowledgement 分组：告知链路状态更新已完成

外部网关路由协议

BGP 协议

BGP 协议（Border Gateway Protocol）是不同自治系统的路由器之间交换路由信息的协议

BGP 使用场景：

  • Internet 规模很大，AS 之间的路由选择非常困难
  • AS 之间的路由选择需要考虑多种策略

因此基于上面这些困难，BGP 不寄希望于找到最佳路由，只用于寻找一条能够到达目的网络（保证连通性）且比较好的路由（不能兜圈子）

BGP 协议会在每个自治系统选定一个路由器作为发言人（即 BGP speaker）

  • 每个 AS 都至少要有一个发言人
  • 两个 BGP 发言人通过一个共享网络连接在一起
  • BGP 发言人一般是 AS 的边界路由器，但也可以不是
  • BGP 发言人与其他自治系统对应的 BGP 发言人相互交换路由信息
  • 路由信息的交换通过 TCP 协议进行，两个发言人在 TCP 连接上交换 BGP 报文以建立 BGP 会话，通过 BGP 会话交换路由信息
  • BGP 发言人除了要运行 BGP 协议外，还要运行 AS 的内部网关路由协议

信息交换与路由生成

BGP 所交换的网络可达性的信息就是要到达某个网络所要经过的一系列 AS，这些 AS 编号组成了一个路径向量（path vector）

BGP 发言人相互交换了网络可达性的信息后，就根据所采用的策略从其中找出到达各个 AS 的比较好（不是最优）的路由

比如，在下图中，AS2 的 BGP 发言人通知主干网的 BGP 发言人：要到达网络 N1，N2，N3，N4 可以经过 AS2；主干网也可以发出通知，要到达网络 N5，N6，N7 可沿着路径 (AS1，AS3)

BGP-4 使用四种报文：

  • OPEN 报文：用来与相邻的另一个 BGP 发言人建立关系
  • UPDATE 报文：用来发送某一路由的信息，以及列出要撤销的多条路由
  • KEEPALIVE 报文：用来确认打开报文和周期性地证实邻站关系
  • NOTIFICATION 报文：用来发送检测到的差错

BGP 发言人通过 OPEN 报文与另一发言人建立会话，如果对方同意会回应 KEEPALIVE 报文，双方周期性交换 KEEPALIVE 报文，以确认会话不会中断，路由信息变化时，发送 UPDATE 报文

BGP 协议特点：

  • 交换路由信息的站点数量级与自治系统个数相当，远小于这些自治系统中的网络数
  • 每个自治系统中 BGP 发言人（或边界路由器）数目很少，使得自治系统之间的路由选择不会过于复杂
  • BGP 支持 CIDR，因此 BGP 路由表包括目的网络前缀，下一跳路由器，以及到达该目的网络要经过的各个自治系统序列
  • 在 BGP 刚运行时，邻站交换完整的 BGP 路由表，以后只需要在发生变化时，更新变动部分，有利于节省网络带宽和减少处理开销

路由器

路由器简介

路由器指的是：具有多个输入 / 输出端口的专用计算机，其任务是转发分组

路由器由两部分组成：

  • 路由选择部分：按照路由选择协议工作，构建路由表
  • 分组转发部分：交换结构（switch fabric）+ 一组输入 / 端口，交换结构根据转发表进行分组的快速转发，转发表由路由表得到

转发表可以看成是路由表的子表，只有目的网段地址，子网掩码，下一跳路由器这三个字段，路由表则在此基础上还维护了更多的信息

输入的处理顺序是：物理层→数据链路层→网络层→路由器

输出的处理顺序是：路由器→网络层→数据链路层→物理层

注意：当排队的分组数量过多时，也有可能导致路由器内部缓冲区溢出，从而发生丢包

路由器性能指标：

路由器设计的关键之一是分组的高速转发，决定报文转发速率的因素有：

  • 输入 / 输出端口的处理性能
  • 交换机构的性能

线速（line speed 或 wire speed）：分组的处理速率能够达到线路上分组的传输速率

衡量路由器性能的指标：pps，packet per secnd，每秒传输的数据包数量，这里默认统一按照最小包（大小为 64B）来统计

III 的前半句是对的，后半句不对，路由器不能保证传输的 IP 分组不丢失（如果路由器的等待队列溢出，则不可避免会丢包），但是确实可以对 IP 分组头进行差错校验（checksum 字段，前面提到过 IP 分组头的差错检验发生在路由器接收分组时，重新填写 checksum 字段发生在路由器转发分组时），其余三个说法都是对的

网络的控制平面与数据平面

控制平面（control plane）：生成路由表和转发表

数据平面（data plane）：实际进行分组的快速转发

在传统网络中，路由器集成了控制平面和数据平面，缺少灵活性；而在软件定义网络（SDN，Software Defined Networking）中：

  • 控制平面和数据平面分离
  • 控制平面即控制器（controller）
  • 数据平面即路由器（SDN 把路由器称为交换机）

二者分离的优点：可以灵活，快速的控制路由 / 转发的策略

SDN 特别适用于路由策略变化频繁的网络，比如云数据中心网络

IP 组播

组播（multicast）又称为多播，用于实现一点对多点的数据传输，对于一些网络应用（比如视频会议，视频直播等），采用组播可以大大减少网络流量

  • IP 组播适用于所有人看的都是同一个视频的情况，对于回放这类视频，由于每个人进度 / 倍速都不同，则不适合使用组播

通常使用 D 类 IP 地址作为组播地址，范围是：

  • 224.0.0.0∼239.255.255.255224.0.0.0\sim239.255.255.255224.0.0.0∼239.255.255.255
  • 组播数据报：IP 包头中的目的地址为 D 类地址，协议类型为 2（IGMP 协议）

组播地址即为特定的组播组的标识符，主机通过加入组播组来接收组播数据

组播可以分为两种：

  • 在局域网中的硬件组播：将 MAC 地址中的特定地址段作为组播地址，并与 IP 组播地址形成对应关系
  • 在 Internet 中的组播
    • 路由器需要支持组播，即需要组播路由器
    • 主机通过 IGMP 协议与组播路由器通信，加入 / 退出某个组播组
    • 组播路由器之间通过组播路由协议实现组播数据报的传输

网络地址转换 NAT 和虚拟专用网 VPN

网络地址转换 NAT

专用地址（保留地址）：RFC1918 中定义了一系列专用地址（private address）

  • A 类专用地址：10.0.0.0∼10.255.255.25510.0.0.0\sim10.255.255.25510.0.0.0∼10.255.255.255，CIDR 记为 10/8
  • B 类专用地址：172.16.0.0∼172.31.255.255172.16.0.0\sim172.31.255.255172.16.0.0∼172.31.255.255，CIDR 记为 172.16/12
  • C 类专用地址：192.168.0.0∼192.168.255.255192.168.0.0\sim192.168.255.255192.168.0.0∼192.168.255.255，CIDR 记为 192.168/16

这些地址都只能用于机构的内部，Internet 中的路由器不转发目的地址为专用地址的包（路由表中不会写对应的记录）

企业 / 机构内部网络使用专用地址的优点：

  • 减少 IP 地址空间的占用，不同的内网可以对同一个专用地址复用
  • 一定程度上提高安全性（外部网络 IP 没法通过公网网关向某个内部网络 IP 发送数据，只能由内部网络 IP 主动向外部网络发送数据）

当内部网络使用专用地址时，与 Internet 的通信需要通过 NAT，即 Network Address Translation

NAT 路由器的作用：内网发送出去的数据包，把源地址从内网 IP 地址改成公网 IP 地址；外网传送回来的数据包，把目的地址从公网 IP 地址改成对应的内网 IP 地址（需要 NAT 路由器维护一张表，记录它曾经转发过的内网的 IP 数据报的专有 IP）

虚拟专用网 VPN

VPN，即 Virtual Private Network，称为虚拟专用网

多个企业 / 机构的内部网络之间互连的实现方法（比如北京和上海的两个分部各自的内网之间的连接）：

  • 租用专用线路，形成专用网，成本高昂
  • 基于公用网络，如 Internet，形成 VPN

VPN 涉及的技术包括：隧道 tunnel，加密，身份认证

基于 Internet 建立 VPN 的两种情形：

  • 内部网络通过 Internet 互连
  • 远程用户访问内部网络（remote address VPN，比如在家访问校园网）

典型的 VPN 实现技术：

  • 基于 IPSee：ESP 隧道模式，工作在网络层，适用于多种应用
  • 基于 SSL：Web 浏览器内置 SSL 支持，客户端无需安装或配置软件

适用隧道的示意图：

在使用隧道以后，路由器 R1 和 R2 之间传送的数据包，目的地址和源地址都会被改成 R1 和 R2 各自的地址，外部网络看不到内部网络中部门 A 和部门 B 的 IP 地址，因此有一定的安全性

此时，机构内部网络又称为内联网（Internet），外部网络称为外联网（Extranet），通过公共网络传输的数据内容均经过加密，外部仅能通过 IP 分组头得知是在 R1 和 R2 之间通信
