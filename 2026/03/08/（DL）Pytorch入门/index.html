Pytorch 入门

Pytorch 作为开源的 Python 机器学习库，基于 Torch 库（一个有大量机器学习算法支持的科学计算框架，有着与 Numpy 类似的张量操作），主要具有两大特征：

  • 类似于 Numpy 的张量计算，能在 GPU 或 MPS 等硬件加速器上加速
  • 基于带自动微分系统的深度神经网络

Pytorch 官网：https://pytorch.org/

张量

Tensor（张量）是 Pytorch 的核心数据结构，张量在不同学科中有不同的意义，在深度学习中张量表示一个多维数组，是标量、向量、矩阵的扩展，如一个 RGB 图像的数组就是一个三维张量（[H,W,C(R,G,B)][H,W,C(R,G,B)][H,W,C(R,G,B)]）：第一维表示像素点行坐标，第二维表示像素点列坐标，第三维表示像素点颜色通道，常见的颜色通道为 RGB 三通道，即第三维至少是一个长度为 3 的数组

如果是灰度图像，则可以用 [H,W][H,W][H,W] 两维表示，对应每个像素点填入一个 0∼2550\sim2550∼255 的灰度数值即可；如果是批量数据，则还需要新增一维 BBB 表示当前是第几条数据

张量的创建

使用torch.tensor(data)创建指定内容的张量，使用torch.Tensor(size)创建指定形状但内容未初始化的张量，后者也可以向下兼容前者，即torch.Tensor()也可以按照torch.tensor()的方式传参创建，同时tensor创建张量默认为torch.int32类型，Tensor创建张量默认为torch.float32类型

基本张量创建

torch.tensor(data) 创建指定内容的张量

torch.Tensor(size) 创建指定形状的张量

创建指定类型的张量

创建张量时对于其中元素类型的指定，有两种常见方法：

  • 使用torch.tensor()创建，传入参数dtype，可选参数有torch.int8, torch.int16, torch.int32, torch.int64, torch.float32, torch.float64等常见类型
  • 使用指定的函数构建，可选函数有torch.IntTensor(), torch.ShortTensor(), torch.ByteTensor(), torch.LongTensor(), torch.FloatTenosr(), torch.DoubleTensor()，比较特殊的是构建布尔类型的张量，可以使用torch.BoolTensor()创建

指定区间的张量创建

torch.arange(strat, end, step) 在区间内按步长创建张量

使用torch.arange()方法接收三个参数，分别表示区间的左端点（闭），右端点（开）以及取元素的步长，比如torch.arange(10, 20, 2)就表示在[10, 20)区间内，从10开始取值，每次向后跳两个元素，得到的元素序列就是[10, 12, 14, 16, 18]

torch.linspace(start, end, steps) 在区间内按元素数量创建张量

使用torch.linspace()方法接收三个参数，分别表示区间的左端点（闭），右端点（闭）以及取元素的个数，比如torch.linspace(10, 20, 5)就表示在区间[10, 20]内，从10开始取值，总共取5个元素，且最后一个元素是20，得到的元素序列就是[10.0, 12.5, 15.0, 17.5, 20.0]

计算元素的时候可以把取值个数steps转化为步长step，二者满足关系

start+(steps−1)×step=endstart + (steps-1)\times step = end start+(steps−1)×step=end

得到步长step以后即可从start开始依次得到每一个元素，torch.linspace()在创建张量时，元素类型根据是区间长度 / 元素个数能否整除决定，若可以整除则为torch.int32，不能整除则为torch.float32

torch.logspace(start, end, base, steps) 在指数区间内按指定底数创建张量

一般的规律是 stepstepstep 表示取值步长，stepsstepssteps 表示元素个数

按数值填充张量

常见的按数值填充张量的函数如下：

  • torch.zeros(size)：创建指定形状的全 0 张量
  • torch.ones(size)：创建指定形状的全 1 张量
  • torch.full(size, value)：创建指定形状和指定填充值的张量
  • torch.empty(size)：创建指定形状的未初始化张量（值是不确定的）
  • torch.zeros_like(input)：创建和给定张量形状相同的全 0 张量
  • torch.ones_like(input)：创建和给定张量形状相同的全 1 张量
  • torch.full_like(input, value)：创建和给定张量形状相同，且指定填充值的张量

如果参数只有size，创建多维张量可以直接传入若干个数（torch.zeros(2,3,4)），或者传入列表 / 元组（torch.zeros([2,3,4])），如果还有其他参数，创建多维张量必须传入列表 / 元组（torch.full([2,3,4], 10)）

特别地，使用torch.eye(n)创建 n×nn\times nn×n 的单位矩阵，torch.eye(n, m)创建 n×mn\times mn×m 的矩阵，其中取 n,mn,mn,m 中较小者作为边长构建单位阵

随机张量创建

常见的随机张量创建方法如下：

  • torch.rand(size)：创建在[0,1)上均匀分布的，指定形状的张量
  • torch.randint(low, high, size)：创建在[low, high)上均匀分布，取值都是整数，指定形状的张量
  • torch.randn(size)：创建标准正态分布的，指定形状的张量
  • torch.normal(mean, std, size)：创建自定义均值和标准差的正态分布的，指定形状的张量
  • torch.rand_like(input)：创建在[0, 1)上均匀分布的，与给定张量形状相同的张量
  • torch.randint_like(low, high, input)：创建在[low, high)上均匀分布的，与给定张量形状相同的张量
  • torch.randn_like(input)：创建标准正态分布的，与给定张量形状相同的张量

特别地，使用torch.randperm(n)可以生成 0∼n−10\sim n-10∼n−1 的随机排列

和随机种子有关的两个方法：

  • torch.random.initial_seed()：查看当前的随机种子
  • torch.manual_seed(seed)：设置随机种子seed，通常为 42

张量转换

张量元素类型转换

对于张量元素类型的转换，类比指定元素类型张量的创建，这里也有两种常见方法：

  • 通过调用Tensor.type()方法，传入参数dtype即可指定目标类型
  • 直接调用Tensor.double(), Tensor().long()等方法进行转换

Tensor 与 ndarray 的转换

Tensor → ndarray

通过Tensor.numpy()方法把Tensor转换为numpy类型，默认二者共享内存，转换之前对Tensor进行一次copy()可以避免共享内存

ndarray → Tensor

通过Tensor.from_numpy()方法把numpy转换为Tensor类型，默认二者共享内存，转换之前对numpy进行一次copy()可以避免共享内存

torch.tensor(ndarray)

直接使用torch.tensor()传入ndarray构建张量，也可以实现numpy到tensor的转换，且默认不共享内存

Tensor 与标量的转换

如果张量中仅有一个元素（这里只看元素个数，不看维度，即便是形如[[[5]]]也满足仅有一个元素），可以通过Tensor.item()方法将其提取为标量

张量数值计算

基本运算

（1）四则运算

常见四则运算实现形式如下：

  • 直接用+, -, *, /符号运算
  • 通过方法add(), sub(), mul(), div()实现，不改变原张量
  • 通过方法add_(), sub_(), mul_(), div_()实现，原地修改后写回原张量

（2）取负运算

通过直接加负号-，或者调用neg()方法、neg_()方法实现

（3）求幂运算

通过**，或者调用pow()方法、pow_()方法实现

（4）平方根运算

通过调用sqrt()方法、sqrt_()方法实现

（5）以 e 为底数的指数运算

通过调用exp()方法、exp_()方法实现

（6）以 e 为底的对数运算

通过调用log()方法、log_()方法实现

哈达玛积

哈达玛积对于两个矩阵而言，要求参与乘积的两个矩阵形状相同，运算规则是两个矩阵对应位置的元素相乘，结果仍是一个相同形状的矩阵

使用*，或者mul()方法实现两个相同形状的张量之间的对位相乘

矩阵乘法

二维张量乘法

二维张量乘法完全遵循矩阵乘法规则，可以通过以下方式实现：

  • tensor1 @ tensor2：直接使用@符号
  • tensor1.matmul(tensor2)：通过matmul()方法实现
  • tensor1.mm(tensor2)：通过mm()方法实现，该方法仅限于二维张量，对于高维张量乘法不适用

高维张量乘法

高维张量乘法，要求参与乘法的张量的后两维满足矩阵乘法的要求，即必须是[n,k]和[k,m]对应，且其余维度必须相同，高维张量的乘法可以把后两维看成整体，在其余维度上做哈达玛乘积，对应位置的两个矩阵相乘时，内部满足矩阵乘法规则

注意一种特殊情况，如果有一个张量除后两维之外，其余维度都是 1，在相乘时会先进行广播，也可以进行高维张量乘法

节省内存

张量的某些运算操作可能会为新的结果分配内存，例如X=X@Y，我们会发现运算前后使用id(X)会指向不同的位置，这是因为 python 会首先计算X@Y，为计算结果分配新的内存，再令X指向内存中的新位置

如果后续X不再使用，可以使用X[:]=X@Y减少内存开销

张量索引操作

常规索引

直接在张量的每一维中传入[R1, R2, Rk, ...]即可完成对每一维的索引，其中 RkR_kRk​ 可以是：

  • 标量 ttt：表示在第 KKK 维中取第 ttt 个元素
  • 范围 A:BA:BA:B：表示在第 KKK 维中取第 AAA 到 B−1B-1B−1 个元素，不包含最后一个元素
  • 范围+步长 A:B:CA:B:CA:B:C：表示在第 KKK 维中先取第 AAA 个元素，每次下标加 CCC 取下一个元素，直到下标范围 ≥B\ge B≥B 结束，其中如果 AAA 缺省则默认从第 0 个元素开始取，如果 BBB 缺省则默认取到最后一个元素

这部分张量的索引方式和ndarray完全相同，需要注意的是，ndarray支持步长为负数，即形如[4:1:-1]这种形式，但张量不允许，必须确保步长是正数

列表索引

列表索引即在张量的若干个维度中分别传入列表，需要注意这里传入列表的必须长度相同，各个列表对应位置的元素会组合起来作为一次选取的下标，以一个三维张量为例，我们分别传入列表[a1, a2, ..., ak], [b1, b2, ..., bk], [c1, c2, ... ck]，则表示要取张量中[a1,b2,c1], [a2,b2,c2], ..., [ak,bk,ck]位置的元素

布尔索引

布尔索引是张量索引中最灵活，也是最难的一种索引方式，其核心思想是先创建一个和原张量形状相同的布尔张量，通常记为mask，该张量对应位置的元素仅有True or False，然后进行origin[mask]操作，即可把布尔张量为True位置的原张量中的元素取出

一个简单的规律如下：

  • 用 [d1,d2,…,dk−1][d_1, d_2, \dots, d_{k-1}][d1​,d2​,…,dk−1​] 的 maskmaskmask 去索引 [d1,d2,…,dk−1,dk][d_1, d_2, \dots, dk-1, dk][d1​,d2​,…,dk−1,dk] 的张量，最后得到结果张量的形状为 [N,dk][N, d_k][N,dk​]，即所引到 NNN 行维度为 dkd_kdk​ 的张量
  • 用 [d1,d2,…,dt,…,dk][d_1, d_2, \dots, d_t, \dots, d_k][d1​,d2​,…,dt​,…,dk​] 的 maskmaskmask 去索引 [d1,d2,…,dk][d_1, d_2, \dots, d_k][d1​,d2​,…,dk​] 的张量，需要先对原张量变形，把 dtd_tdt​ 交换到最后一维，最后得到结果张量的形状为 [N,dt][N, d_t][N,dt​]，再次转置变回 [dt,N][d_t,N][dt​,N] 表示得到 NNN 列维度为 dtd_tdt​ 的张量

（1）transpose() 交换两个维度

使用transpose()交换张量的两个维度，该方法仅支持传入dim1和dim2两个维度下标，通常用于交换张量的后两个维度，也就是对后两维做矩阵转置，传入参数-1和-2即可

调整形状

（1）reshape() 调整张量形状

需要注意的是，permute()接收的参数是原张量维度的下标的一个排列，而reshape()接收的是目标张量维度的大小，我们只要保证调整前后张量中总元素个数不变即可

以[3,2,5]形状的张量为例，其总元素个数为 30，因此可以调整形状为[3,10]或者[5,6]，而调整形状为[2,10]则是非法的

（2）shape 属性查看张量形状

我们可以调用任意张量的Tensor.shape属性查看当前张量的形状，注意这个shape不需要加括号，它是张量的某个属性，类似于dtype，而不是方法

（3）view() 调整张量形状

使用view()方法调整张量形状要求当前张量所在内存必须连续，且调整形状后生成的新张量默认和原张量共享内存，该方法接收参数的格式与reshape()相同，都是接收目标张量各个维度大小所组成的列表

至于判断张量所在内存是否连续，可以调用Tensor.is_contiguous()方法，而调用Tensor.contiguous()方法可以强制把张量所在内存连续化（会开辟新的连续空间进行存储）

张量拼接操作

（1）torch.cat() 张量拼接

使用torch.cat()方法，该方法接收两个参数：由待拼接张量所组成的列表或元组，以及指定的拼接维度，要求待拼接的向量除指定的拼接维度之外，其余维度大小必须完全相同

（2）torch.stack() 张量堆叠

使用torch.stack()方法，该方法接收两个参数：由待堆叠张量所组成的列表或元组，以及指定的堆叠维度（缺省时默认按dim=0拼接），要求堆叠的向量形状完全相同

我们以形状为[3,1,5]的张量为例，演示按照各个维度堆叠后，张量的形状变化，假设两个张量分别为：

（1）按照dim=0拼接，拼接后形状为[2,3,1,5]

（2）按照dim=1拼接，拼接后形状为[3,2,1,5]

（3）按照dim=2拼接，拼接后形状为[3,1,2,5]

（4）按照dim=4拼接，拼接后形状为[3,1,5,2]

变形后形状的还原规律，可以从后往前还原，最后一维一定是单个矩阵中的列，倒数第二维一定是单个矩阵中的行，倒数第三维的矩阵编号，倒数第四维是矩阵所在的组别

自动微分模块

训练神经网络时，框架会根据设计好的模型构建一个计算图，来跟踪计算是哪些数据通过哪些操作组合起来产生输出，并通过反向传播算法来根据给定参数的损失函数的梯度调整参数（模型权重）

自动微分的关键就在于记录节点的数据与运算，开启 requires_grad=True 的张量有如下性质：

  • 原张量中的数据记录 data 属性中
  • 计算记录在张量的 grad_fn 属性中
  • 反向传播时得到的梯度值记录在 grad 属性中

计算图根据搭建方式可以分为静态图和动态图，Pytorch 为动态图机制，在计算的过程中逐步搭建计算图，同时对每个 Tensor 都存储 grad_fn（得到该张量所使用的计算类型，如 Add，Mul，MSELoss 等）供自动微分使用

如果对张量 ttt 设置 requires_grad=True，则 Pytorch 会追踪基于该张量的所有操作，并在反向传播时对其计算梯度，作为叶子节点的张量，默认reuqires_grad 参数为 True，因此所有依赖于叶子节点的节点（由叶子节点直接或间接计算得到）都会有 requires_grad=True

所有设置了 requires_grad=True 的张量，都只能通过反向传播修改其数值，即无法对其做 in-place 的修改，例如t[:, 1]=1这种修改将被视为非法修改

当计算到根节点以后，在根节点调用 backword() 方法即可反向传播，此时才会对计算图中所有节点的梯度进行计算，需要注意的是，只有标量才可以进行 backword() 反向传播，比如损失函数值，而高维张量无法反向传播

非叶子节点的梯度在反向传播之后就会被释放掉（除非设置参数 retain_grad=True，代码层面的实现即调用 T.retain_grad() 方法），而叶子节点的梯度在反向传播以后会保留 / 累积，通常在一轮计算后，需要使用 optimizer.zero_grad() 清零参数的梯度

因此即便我们设置了 requires_grad=True，调用此张量的 grad 属性，仍然会返回 None，只有当反向传播完成以后，且该张量是处在叶子节点的张量，才会得到其对应的梯度

有时我们希望将计算图中的某些张量拿出来做相关运算，但又不希望这些运算也会参与反向传播产生梯度，对我们原有的梯度产生干扰，即需要将某些计算移动到计算图之外，此时可以使用 Tensor.datch() 方法返回一个新的张量，该张量与原张量共享内存，具有相同的值，但会丢失计算图中原张量相关的计算信息（即 requires_grad 变为 False），即梯度在该张量处不会继续反向传播

对于如下图所示的计算图：

这里需要区分两个操作：

  • Tensor.clone()：创建一个和原张量数据相同，但不共享内存的新张量，不会清空梯度，对 clone 后的张量做反向传播，也会影响到原张量的梯度（直接做梯度叠加）
  • Tensor.detach()：创建一个和原张量数据相同，且共享内存的新张量，会清空梯度，对 detach 后的张量做反向传播，不会影响到原张量的梯度

由此衍生出以下两个操作：

  • Tensor.clone().detach()：对原张量做安全复制，既不共享内存，也不会影响原张量的梯度
  • Tensor.clone().detach().requires_grad(True)：在安全复制的基础上，对复制后的张量重新开启梯度，将其作为计算图的新起点

对于原始数据 X，首先进行乘方运算得到 Y，接着对 Y 进行 detach() 操作得到 U，最终 X 和 U 相乘得到 Z，这里在对 Z 做反向传播时（先默认 Z 就是标量），其中 U 这个支路不会有梯度，因此对应 UX=Z 这一步计算，我们可以直接把 U 看成常数，就相当于 X 与一个和它完全无关的常量相乘，因此 X 的梯度就是 U（δZδX=U\frac{\delta Z}{\delta X}=UδXδZ​=U），此时因为梯度传播在 U 位置处被截断，因此 Y 也不会获得梯度，Y.grad 输出为 None
